# Zymbol-Lang ची संक्षिप्त मार्गदर्शिका

**Zymbol-Lang** एक प्रतीकात्मक प्रोग्रामिंग भाषा आहे. ती कोणतेही विशिष्ट शब्द (keywords) वापरत नाही — सर्व काही प्रतीक आहे. ती प्रत्येक मानवी भाषेत समान प्रकारे कार्य करते.

- कोणतेही keywords नाहीत (`if`, `while`, `return` नाही — फक्त प्रतीके `?`, `@`, `<~`)
- पूर्ण Unicode आधार — कोणत्याही भाषेत किंवा इमोजीमध्ये ओळखकर्ते 👋
- भाषा-तटस्थ — कोड सर्व भाषांमध्ये समान आहे

---

## चल आणि स्थिरांक

```zymbol
च = १०           // चल (बदलण्यायोग्य)
पाई := ३.१४१५९   // स्थिरांक (अपरिवर्तनीय — पुन्हा नियुक्ती चुकीची आहे)
नाव = "अनू"
सक्रिय = #१     // Boolean सत्य
👋 := "नमस्कार"
```

### संयुक्त नियुक्ती

```zymbol
च = १०
च += ५    // १५
च -= ३    // १२
च *= २    // २४
च /= ३    // ८
च %= ३    // २
च ^= २    // ४
च++       // ५
च--       // ४
```

## डेटा प्रकार

| प्रकार          | उदाहरण              | `#?` चिन्ह | नोंद                                      |
|-----------------|---------------------|------------|-------------------------------------------|
| पूर्णांक        | `४२`, `-७`          | `###`      | 64-bit चिन्हासह                          |
| दशांश           | `३.१४`, `१.५e१०`    | `##.`      | वैज्ञानिक संकेत स्वीकार्य               |
| स्ट्रिंग        | `"नमस्कार"`         | `##"`      | अंतर्भाव: `"नमस्कार {नाव}"`             |
| अक्षर           | `'अ'`               | `##'`      | एक Unicode अक्षर                         |
| Boolean         | `#१`, `#०`          | `##?`      | संख्यात्मक 1 आणि 0 नाही                  |
| अ‍ॅरे (Array)   | `[१, २, ३]`         | `##]`      | सर्व घटक एकाच प्रकाराचे                 |
| टुपल            | `(अ, ब)`            | `##)`      | स्थितीनुसार                              |
| नामांकित टुपल  | `(x: १, y: २)`      | `##)`      | नाव किंवा निर्देशांकाने प्रवेश          |

## आउटपुट आणि इनपुट

```zymbol
// आउटपुट — स्वयंचलित नवीन ओळ नाही
>> "नमस्कार, जग!" ¶              // ¶ किंवा \\ स्पष्ट नवीन ओळ देतो
>> "अ=" अ " ब=" ब ¶              // अनेक मूल्ये फक्त जोडून
>> "बेरीज=" बेरीज(२, ३) ¶       // फंक्शन कॉल कोणत्याही स्थानी
>> (अ‍ॅरे$#) ¶                   // postfix ऑपरेटरना >> मध्ये कंस आवश्यक

// इनपुट
<< नाव                            // संकेताशिवाय — चलमध्ये वाचतो
<< "तुमचे नाव? " नाव             // संकेतासह
```

> `¶` आणि `\\` नवीन ओळीसाठी समतुल्य आहेत.

## स्ट्रिंग

```zymbol
// तीन योग्य पद्धती
नाव = "अनू"
वय = २५

// १. स्वल्पविराम — = किंवा := नियुक्तीत
संदेश = "नमस्कार ", नाव, "!"          // → नमस्कार अनू!
शीर्षक := "वापरकर्ता: ", नाव

// २. जोडून ठेवणे — >> आउटपुटमध्ये
>> "नमस्कार " नाव " तुम्ही " वय " वर्षांचे आहात" ¶  // → नमस्कार अनू तुम्ही २५ वर्षांचे आहात

// ३. अंतर्भाव — कोणत्याही संदर्भात
ओळख = "नमस्कार {नाव}, तुम्ही {वय} वर्षांचे आहात"  // → नमस्कार अनू, तुम्ही २५ वर्षांचे आहात
```

```zymbol
स = "नमस्कार जग"
लांबी = स$#                  // १०
उपभाग = स$[०..५]             // "नमस्क"  (शेवट वगळलेला)
आहे = स$? "जग"               // #१
भाग = "अ,ब,क,ड" / ','        // [अ, ब, क, ड]
बदल = स$~~["क":"क्"]          // बदलणे
बदल१ = स$~~["क":"क्":१]       // पहिला N
```

> `+` फक्त संख्यांसाठी. स्ट्रिंगसाठी `,`, जोडून किंवा अंतर्भाव वापरा.

## प्रवाह नियंत्रण

```zymbol
च = ७

// साधे जर
? च > ० { >> "धनात्मक" ¶ }

// जर / अन्यथा जर / अन्यथा
? च > १०० {
    >> "मोठे" ¶
} _? च > ० {
    >> "धनात्मक" ¶
} _? च == ० {
    >> "शून्य" ¶
} _ {
    >> "ऋणात्मक" ¶
}
```

`{ }` ब्लॉक्स **आवश्यक आहेत**, जरी फक्त एक ओळ असली तरी.

## जुळणी (Match)

```zymbol
// संख्यात्मक श्रेणीशी जुळणी
क्रमांक = ८५
श्रेणी = ?? क्रमांक {
    ९०..१०० : 'अ'
    ८०..८९  : 'ब'
    ७०..७९  : 'क'
    _       : 'फ'
}
>> श्रेणी ¶    // → ब

// रक्षकांसह जुळणी (मनमानी अटी)
तापमान = -५
स्थिती = ?? तापमान {
    _? तापमान < ०  : "बर्फ"
    _? तापमान < २० : "थंड"
    _? तापमान < ३५ : "उष्ण"
    _               : "अत्यंत उष्ण"
}
>> स्थिती ¶    // → बर्फ

// स्ट्रिंगशी जुळणी
रंग = "लाल"
कोड = ?? रंग {
    "लाल"   : "#FF००००"
    "हिरवा" : "#००FF००"
    _        : "#००००००"
}
>> कोड ¶
```

## लूप

```zymbol
// समावेशक श्रेणी: ०..४ वर ०,१,२,३,४ चालतो
@ न:०..४ { >> न " " }
>> ¶    // → ० १ २ ३ ४

// पायऱ्यांसह श्रेणी
@ न:१..९:२ { >> न " " }
>> ¶    // → १ ३ ५ ७ ९

// उलट श्रेणी
@ न:५..०:१ { >> न " " }
>> ¶    // → ५ ४ ३ २ १ ०

// जोपर्यंत (while)
न = १
@ न <= ६४ { न *= २ }
>> न ¶    // → १२८

// प्रत्येक घटकासाठी (for-each)
फळे = ["सफरचंद", "नाशपाती", "द्राक्षे"]
@ फ:फळे { >> फ ¶ }

// स्ट्रिंगच्या अक्षरांवर
@ अ:"नमस्कार" { >> अ "-" }
>> ¶    // → न-म-स-्-क-ा-र-

// थांबणे आणि पुढे जाणे
@ न:१..१० {
    ? न % २ == ० { @> }    // @> पुढे जा
    ? न > ७ { @! }          // @! थांबा
    >> न " "
}
>> ¶    // → १ ३ ५ ७
```

## फंक्शन्स

```zymbol
// व्याख्या आणि कॉल
बेरीज(अ, ब) { <~ अ + ब }
>> बेरीज(३, ४) ¶    // → ७

// पुनरावृत्ती (recursion)
गुणाकार(न) {
    ? न <= १ { <~ १ }
    <~ न * गुणाकार(न - १)
}
>> गुणाकार(५) ¶    // → १२०

// फंक्शनचा स्वतंत्र scope आहे — बाहेरील चलांना प्रवेश नाही
सामान्य = १००
चाचणी() {
    च = ४२    // फक्त स्थानिक
    <~ च
}
>> चाचणी() ¶    // → ४२
```

> **महत्त्वाचे**: नामांकित फंक्शन्स `नाव(params){ }` प्रथम-श्रेणीची मूल्ये नाहीत.
> युक्तिवाद म्हणून पाठविण्यासाठी गुंडाळा: `x -> बेरीज(x)`.

## लॅम्बडा आणि क्लोजर

```zymbol
// साधे लॅम्बडा (अंतर्निहित परतावा)
दुप्पट = च -> च * २
बेरीज = (अ, ब) -> अ + ब
>> दुप्पट(५) ¶      // → १०
>> बेरीज(३, ७) ¶    // → १०

// ब्लॉक लॅम्बडा (स्पष्ट परतावा)
वर्गीकृत_मूल्य = च -> {
    ? च > ० { <~ "धनात्मक" }
    _? च < ० { <~ "ऋणात्मक" }
    <~ "शून्य"
}
>> वर्गीकृत_मूल्य(५) ¶      // → धनात्मक
>> वर्गीकृत_मूल्य(०) ¶      // → शून्य
>> वर्गीकृत_मूल्य(-५) ¶     // → ऋणात्मक

// क्लोजर — लॅम्बडा बाहेरील चले पकडतो
गुणक = ३
तिप्पट = च -> च * गुणक    // 'गुणक' पकडतो
>> तिप्पट(७) ¶    // → २१

// फंक्शन फॅक्टरी
क्रिया_बनवा(न) { <~ च -> च + न }
क्रिया = क्रिया_बनवा(१०)
>> क्रिया(५) ¶    // → १५

// लॅम्बडा मूल्य म्हणून: अ‍ॅरेमध्ये संग्रहित
ऑपरेशन = [च -> च+१, च -> च*२, च -> च*च]
>> ऑपरेशन[०](५) ¶    // → ६
>> ऑपरेशन[२](५) ¶    // → २५
```

## अ‍ॅरे

अ‍ॅरे **परिवर्तनीय** असतात आणि **एकाच प्रकाराचे** घटक ठेवतात.

```zymbol
अ‍ॅरे = [१, २, ३, ४, ५]

अ‍ॅरे[०]          // १ — प्रवेश (०-आधारित)
अ‍ॅरे[-१]         // ५ — ऋणात्मक निर्देशांक (शेवटचा)
अ‍ॅरे$#           // ५ — लांबी (>> मध्ये (अ‍ॅरे$#) वापरा)

अ‍ॅरे = अ‍ॅरे$+ ६            // जोडणे → [१,२,३,४,५,६]
अ‍ॅरे२ = अ‍ॅरे$+[२] ९९       // निर्देशांक २ वर घाला
अ‍ॅरे३ = अ‍ॅरे$- ३           // पहिली घटना काढा
अ‍ॅरे४ = अ‍ॅरे$-- ३          // सर्व घटना काढा
अ‍ॅरे५ = अ‍ॅरे$-[०]          // निर्देशांकाने काढा
अ‍ॅरे६ = अ‍ॅरे$-[१..३]       // श्रेणी काढा (शेवट वगळलेला)

आहे = अ‍ॅरे$? ३            // #१ — आहे का
स्थाने = अ‍ॅरे$?? ३          // [२] — सर्व निर्देशांक
तुकडा = अ‍ॅरे$[०..३]         // [१,२,३] — तुकडा (शेवट वगळलेला)
तुकडा२ = अ‍ॅरे$[०:३]         // [१,२,३] — संख्या-आधारित वाक्यरचना

चढ = अ‍ॅरे$^+              // चढत्या क्रमाने (फक्त primitive)
उतरत = अ‍ॅरे$^-            // उतरत्या क्रमाने (फक्त primitive)

// Named/positional tuple अ‍ॅरे — $^ comparator lambda सह
डेटा = [(नाव: "कार्ला", वय: २८), (नाव: "अना", वय: २५), (नाव: "बॉब", वय: ३०)]
वयाने  = डेटा$^ (अ, ब -> अ.वय < ब.वय)
नावाने = डेटा$^ (अ, ब -> अ.नाव > ब.नाव)
>> वयाने[०].नाव ¶     // → अना
>> नावाने[०].नाव ¶    // → कार्ला

// जागी घटक अद्यतन (फक्त अ‍ॅरे)
अ‍ॅरे[१] = ९९              // नियुक्त करा
अ‍ॅरे[०] += ५              // संयुक्त: +=  -=  *=  /=  %=  ^=

// Functional अद्यतन — नवीन अ‍ॅरे परत करतो; मूळ अपरिवर्तित राहतो
अ‍ॅरे२ = अ‍ॅरे[१]$~ ९९
```

> सर्व collection ऑपरेटर **नवीन अ‍ॅरे** परत करतात. पुन्हा नियुक्त करा: `अ‍ॅरे = अ‍ॅरे$+ ४`.
> ऑपरेटर साखळी नाही — मध्यवर्ती नियुक्त्या वापरा.
> `$^+` / `$^-` **primitive अ‍ॅरे** (संख्या, स्ट्रिंग) क्रमवारी. Tuple अ‍ॅरेसाठी comparator lambda सह `$^` वापरा.

**मूल्य अर्थशास्त्र** — अ‍ॅरे दुसऱ्या चलाला नियुक्त केल्यास एक स्वतंत्र प्रत तयार होते:

```zymbol
a = [1, 2, 3]
b = a
a[0] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b अप्रभावित राहतो
```

```zymbol
// नेस्टेड अ‍ॅरे
मॅट्रिक्स = [[१,२,३],[४,५,६],[७,८,९]]
>> मॅट्रिक्स[१][२] ¶    // → ६
```

## विनाशीकरण

```zymbol
// अ‍ॅरे
अ‍ॅरे = [१०, २०, ३०, ४०, ५०]
[अ, ब, क] = अ‍ॅरे              // अ=१०  ब=२०  क=३०
[पहिला, *बाकी] = अ‍ॅरे         // पहिला=१०  बाकी=[२०,३०,४०,५०]
[च, _, ड] = [१, २, ३]         // _ टाकतो

// Positional tuple
बिंदू = (१००, २००)
(बx, बy) = बिंदू             // बx=१००  बy=२००

// Named tuple
व्यक्ती = (नाव: "अना", वय: २५, शहर: "मुंबई")
(नाव: न, वय: व) = व्यक्ती   // न="अना"  व=२५
```

## टुपल

टुपल **अपरिवर्तनीय** क्रमबद्ध कंटेनर आहेत जे **वेगवेगळ्या प्रकारांचे** मूल्ये ठेवू शकतात. अ‍ॅरेच्या विपरीत, निर्मितीनंतर घटक बदलता येत नाहीत.

```zymbol
// स्थितीय — मिश्र प्रकार स्वीकार्य
बिंदू = (१०, २०)
>> बिंदू[०] ¶    // → १०

डेटा = (४२, "नमस्कार", #१, ३.१४)
>> डेटा[२] ¶     // → #१

// नामांकित टुपल
व्यक्ती = (नाव: "ॲलिस", वय: २५)
>> व्यक्ती.नाव ¶    // → ॲलिस
>> व्यक्ती.वय ¶     // → २५
>> व्यक्ती[०] ¶     // → ॲलिस (निर्देशांकही कार्य करतो)
```

**अपरिवर्तनीयता** — टुपल घटक बदलण्याचा कोणताही प्रयत्न रनटाइम त्रुटी आहे:

```zymbol
t = (१०, २०, ३०)
// t[०] = ९९    // ❌ रनटाइम त्रुटी: टुपल अपरिवर्तनीय आहेत
// t[०] += ५    // ❌ समान त्रुटी
```

बदललेले मूल्य मिळवण्यासाठी `$~` (functional अद्यतन) वापरा — एक **नवीन** टुपल परत करतो:

```zymbol
t = (१०, २०, ३०)
t२ = t[१]$~ ९९९
>> t ¶     // → (१०, २०, ३०)   ← मूळ अपरिवर्तित
>> t२ ¶    // → (१०, ९९९, ३०)

// नामांकित टुपल — स्पष्टपणे पुनर्बांधणी करा
```

## उच्च-क्रम फंक्शन्स (HOF)

HOF ऑपरेटरना **inline लॅम्बडा** आवश्यक आहे — लॅम्बडा चल थेट कार्य करत नाही.

```zymbol
संख्या = [१, २, ३, ४, ५, ६, ७, ८, ९, १०]

// Map ($>)
दुप्पट = संख्या$> (च -> च * २)
>> दुप्पट ¶    // → [२, ४, ६, ८, १०, १२, १४, १६, १८, २०]

// Filter ($|)
सम = संख्या$| (च -> च % २ == ०)
>> सम ¶    // → [२, ४, ६, ८, १०]

// Reduce ($<) — (प्रारंभिक मूल्य, (संचयक, घटक) -> एकूण)
एकूण = संख्या$< (०, (स, च) -> स + च)
>> एकूण ¶    // → ५५
```

## अंक प्रणाली

Zymbol **69 Unicode अंक ब्लॉक्स**मध्ये संख्या दाखवू शकतो — देवनागरी, अरबी-भारतीय, थाई, क्लिंगन pIqaD, ठळक गणिती, LCD अंक आणि बरेच काही. सक्रिय मोड फक्त `>>` आउटपुटवर परिणाम करतो; अंतर्गत अंकगणित नेहमी बायनरी असते.

### प्रणाली सक्रिय करा

लक्ष्य लिपीचे `०` आणि `९` अंक `#…#` दरम्यान लिहा:

```zymbol
#०९#    // देवनागरी    (U+0966–U+096F)
#٠٩#    // अरबी-भारतीय (U+0660–U+0669)
#๐๙#    // थाई         (U+0E50–U+0E59)
#09#    // ASCII रीसेट
```

### आउटपुट आणि बुलियन

```zymbol
x = ४२
>> x ¶          // → ४२   (डिफॉल्ट ASCII)

#०९#
>> x ¶          // → ४२
>> ३.१४ ¶       // → ३.१४   (दशांश बिंदू नेहमी ASCII)
>> १ + २ ¶      // → ३

// बुलियन: # उपसर्ग नेहमी ASCII, अंक मोडनुसार
>> #१ ¶         // → #१   (देवनागरीत सत्य)
>> #० ¶         // → #०   (असत्य — ० शून्यापेक्षा वेगळे)

x = २८ > ४
>> x ¶          // → #१   (तुलना परिणाम सक्रिय मोड पाळतो)
```

### स्रोतमध्ये नेटिव्ह लिटरल

कोणत्याही समर्थित प्रणालीचे अंक रेंज, मॉड्युलो, तुलनेत वैध लिटरल आहेत:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### कोणत्याही प्रणालीत बुलियन लिटरल

कोणत्याही समर्थित ब्लॉकचा `#` + `०` किंवा `१` अंक हा वैध बुलियन लिटरल आहे:

```zymbol
#٠٩#
نشط = #١        // #१ सारखेच
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **नेहमी ASCII** आहे। `#०` (असत्य) कोणत्याही प्रणालीत `०` (शून्य पूर्णांक) पेक्षा दृश्यदृष्ट्या वेगळे आहे.
