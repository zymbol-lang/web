## டியுஐ அடிப்படைகள்

ஊடாடும் நிரல்களுக்கான முனைய பயனர் இடைமுக செயலிகள். பெரும்பாலானவை `>>| { }` தொகுதி தேவை (மாற்றுத் திரை + கச்சா முறை).

```zymbol
>>| {
    >>!                             // மாற்றுத் திரையை சுத்தம் செய்
    >>~ (௧, ௧, ௦, ௧௦) > "இயங்குகிறது"   // வரிசை ௧, நிரல் ௧, fg=௧௦ (பச்சை)
    @~ ௧௦௦௦                         // ௧ விநாடி இடைநிறுத்தம் (௧௦௦௦ ms)
    >>~ (௨, ௧) > "முடிந்தது."
}
// வெளியேறும்போது முனையம் தானாக மீட்டமைக்கப்படும்
```

```zymbol
// விசை அழுத்தம் மற்றும் முனைய அளவு
>>| {
    [வரிசைகள், நிரல்கள்] = >>?              // முனையத்தின் பரிமாணங்களைக் கேட்கவும்
    >>~ (௧, ௧) > "முனையம்: " வரிசைகள் " x " நிரல்கள்
    <<| விசை                         // தடுக்கும் விசை அழுத்தத்தைப் படிக்கவும்
    >>~ (௨, ௧) > "அழுத்தியது: " விசை
}
```

> `>>!` திரையை சுத்தம் செய்கிறது. `>>?` `[வரிசைகள், நிரல்கள்]` திருப்பித் தருகிறது. `@~ N` N மில்லி விநாடிகள் உறங்குகிறது.
> `<<|` ஒரு விசை அழுத்தத்தைப் படிக்கிறது (தடுக்கும்); `<<|?` தடுக்காமல் கணக்கெடுக்கிறது (ஏதுமில்லையென்றால் `'\0'` திருப்பித் தருகிறது).
> நிலைசார் வெளியீட்டு டப்பிள்: `(வரிசை, நிரல், BKS, fg, bg)` — எந்த இடத்தையும் காற்புள்ளியுடன் விட்டுவிடலாம் (`>>~ (,,, 196) > "சிவப்பு"`).
> BKS பிட்மாஸ்க்: `1`=தடித்த, `2`=சாய்வு, `4`=அடிக்கோடு. ANSI 256 வண்ணத் தட்டு (`0`=முனைய இயல்புநிலை).
> tree-walker மட்டும் (`>>!`, `>>?`, `@~`, `>>~` தவிர அவை `--vm` இலும் இயங்கும்).

## செயல்பாடுகள்

```zymbol
கூட்டல்(a, b) { <~ a + b }
>> கூட்டல்(௩, ௪) ¶    // → ௭

தொடர்பெருக்கம்(n) {
    ? n <= ௧ { <~ ௧ }
    <~ n * தொடர்பெருக்கம்(n - ௧)
}
>> தொடர்பெருக்கம்(௫) ¶    // → ௧௨௦
```

செயல்பாடுகள் **தனிமைப்படுத்தப்பட்ட களத்தை** கொண்டுள்ளன — அவை வெளிப்புற மாறிகளைப் படிக்க முடியாது. அழைப்பாளர் மாறிகளை மாற்ற வெளியீட்டு அளவுருக்கள் `<~` பயன்படுத்தவும்:

```zymbol
மாற்று(a<~, b<~) {
    தற்காலிக = a
    a = b
    b = தற்காலிக
}
x = ௧௦
y = ௨௦
மாற்று(x, y)
>> "x=" x " y=" y ¶    // → x=௨௦ y=௧௦
```

> பெயரிடப்பட்ட செயல்பாடுகள் **முதல்-வகுப்பு மதிப்புகள்** — நேரடியாக அனுப்பவும்: `எண்கள்$> இரட்டிப்பு`. மடக்குவதற்கு: `x -> fn(x)` செல்லுபடியாகும்.

## லாம்ப்டாக்கள் மற்றும் மூடல்கள்

```zymbol
இரட்டிப்பு = x -> x * ௨
கூட்டல் = (a, b) -> a + b
>> இரட்டிப்பு(௫) ¶    // → ௧௦
>> கூட்டல்(௩, ௭) ¶  // → ௧௦

// தொகுதி லாம்ப்டா
வகைப்படுத்து = x -> {
    ? x > ௦ { <~ "நேர்மறை" }
    _? x < ௦ { <~ "எதிர்மறை" }
    <~ "பூஜ்யம்"

// மூடல் — வெளிப்புற களத்தைப் பிடிக்கிறது
காரணி = ௩
மும்மடங்கு = x -> x * காரணி
>> மும்மடங்கு(௭) ¶    // → ௨௧

// தொழிற்சாலை
கூட்டல்_உருவாக்கி(n) { <~ x -> x + n }
பத்தின்_கூட்டல் = கூட்டல்_உருவாக்கி(௧௦)
>> பத்தின்_கூட்டல்(௫) ¶    // → ௧௫

// வரிசையில்
செயலிகள் = [x -> x+௧, x -> x*௨, x -> x*x]
>> செயலிகள்[௩](௫) ¶    // → ௨௫
```

## வரிசைகள்

வரிசைகள் **மாறக்கூடியவை** மற்றும் **ஒரே வகையான** உறுப்புகளைக் கொண்டுள்ளன.

```zymbol
arr = [௧, ௨, ௩, ௪, ௫]

x = arr[௧]      // ௧ — அணுகல் (௧-அடிப்படை: முதல் உறுப்பு)
x = arr[-௧]     // ௫ — எதிர்மறை குறியீடு (கடைசி உறுப்பு)
x = arr$#       // ௫ — நீளம் (>> இல் (arr$#) பயன்படுத்தவும்)

arr = arr$+ ௬            // சேர்க்கவும் → [௧,௨,௩,௪,௫,௬]
arr௨ = arr$+[௨] ௯௯       // நிலை ௨ இல் செருகவும் (௧-அடிப்படை)
arr௩ = arr$- ௩           // மதிப்பின் முதல் நிகழ்வை அகற்றவும்
arr௪ = arr$-- ௩          // அனைத்து நிகழ்வுகளையும் அகற்றவும்
arr௫ = arr$-[௧]          // குறியீடு ௧ இல் அகற்றவும் (முதல் உறுப்பு)
arr௬ = arr$-[௨..௩]       // வரம்பை அகற்றவும் (௧-அடிப்படை, முடிவு உள்ளடக்கியது)

உள்ளது = arr$? ௩            // #௧ — கொண்டுள்ளது
இடங்கள் = arr$?? ௩           // [௩] — மதிப்பின் அனைத்து குறியீடுகளும் (௧-அடிப்படை)
துண்டு = arr$[௧..௩]          // [௧,௨,௩] — துண்டு (௧-அடிப்படை, முடிவு உள்ளடக்கியது)
துண்டு௨ = arr$[௧:௩]          // [௧,௨,௩] — அதே, எண்ணிக்கை-அடிப்படை இலக்கணம்

ஏறுவரிசை = arr$^+             // ஏறு வரிசையில் வரிசைப்படுத்து (முதன்மைகள் மட்டும்)
இறங்குவரிசை = arr$^-            // இறங்கு வரிசையில் வரிசைப்படுத்து (முதன்மைகள் மட்டும்)

// பெயரிடப்பட்ட/நிலைசார் டப்பிள் வரிசைகள் — ஒப்பீட்டு லாம்ப்டாவுடன் $^ பயன்படுத்தவும்
தரவுத்தளம் = [(பெயர்: "கார்லா", வயது: ௨௮), (பெயர்: "அனா", வயது: ௨௫), (பெயர்: "பாப்", வயது: ௩௦)]
வயதுவாரியாக  = தரவுத்தளம்$^ (a, b -> a.வயது < b.வயது)    // வயது வாரியாக ஏறுவரிசை (<)
பெயர்வாரியாக = தரவுத்தளம்$^ (a, b -> a.பெயர் > b.பெயர்)   // பெயர் வாரியாக இறங்குவரிசை (>)
>> வயதுவாரியாக[௧].பெயர் ¶     // → அனா
>> பெயர்வாரியாக[௧].பெயர் ¶    // → கார்லா

// நேரடி உறுப்பு புதுப்பிப்பு (வரிசைகள் மட்டும்)
arr[௧] = ௯௯              // ஒதுக்குக
arr[௨] += ௫              // கூட்டு: +=  -=  *=  /=  %=  ^=

// செயல்பாட்டு புதுப்பிப்பு — ஒரு புதிய வரிசையைத் திருப்பித் தருகிறது; அசல் மாறாமல்
arr௨ = arr[௨]$~ ௯௯
```

> அனைத்து சேகரிப்பு செயலிகளும் ஒரு **புதிய வரிசையை** திருப்பித் தருகின்றன. திருப்பி ஒதுக்குக: `arr = arr$+ 4`.
> `$+` சங்கிலியாக்கப்படலாம்: `arr = arr$+ 5$+ 6$+ 7`. மற்ற செயலிகள் இடைநிலை ஒதுக்கீடுகளைப் பயன்படுத்துகின்றன.
> **குறியீட்டாக்கம் 1-அடிப்படை**: `arr[1]` முதல் உறுப்பு; `arr[0]` ஒரு இயக்க நேரப் பிழையாகும்.
> `$^+` / `$^-` **முதன்மை வரிசைகளை** (எண்கள், சரங்கள்) வரிசைப்படுத்துகின்றன. டப்பிள் வரிசைகளுக்கு ஒப்பீட்டு லாம்ப்டாவுடன் `$^` பயன்படுத்தவும் — திசை லாம்ப்டாவில் குறியிடப்பட்டுள்ளது (`<` = ஏறுவரிசை, `>` = இறங்குவரிசை).

**மதிப்பு சொற்பொருள்** — ஒரு வரிசையை மற்றொரு மாறிக்கு ஒதுக்குவது ஒரு சுயாதீன நகலை உருவாக்குகிறது:

```zymbol
a = [௧, ௨, ௩]
b = a
a[௧] = ௯௯
>> a ¶    // → [௯௯, ௨, ௩]
>> b ¶    // → [௧, ௨, ௩]   ← b பாதிக்கப்படுவதில்லை
```

```zymbol
// உள்ளமை வரிசைகள் (௧-அடிப்படை குறியீட்டாக்கம்)
அணி = [[௧,௨,௩],[௪,௫,௬],[௭,௮,௯]]
>> அணி[௨][௩] ¶    // → ௬  (வரிசை ௨, நிரல் ௩)
```

```zymbol
arr = [௧௦, ௨௦, ௩௦, ௪௦, ௫௦]
```

## டப்பிள்

டப்பிள்கள் **மாற்ற முடியாத** வரிசைப்படுத்தப்பட்ட கொள்கலன்களாகும், அவை **வெவ்வேறு வகையான** மதிப்புகளைக் கொண்டிருக்கலாம்.
வரிசைகளைப் போலல்லாமல், உருவாக்கிய பிறகு உறுப்புகளை மாற்ற முடியாது.

```zymbol
// நிலைசார் — கலப்பு வகைகள் அனுமதிக்கப்படுகின்றன
புள்ளி = (௧௦, ௨௦)
>> புள்ளி[௧] ¶    // → ௧௦

தரவு = (௪௨, "ஹலோ", #௧, ௩.௧௪)
>> தரவு[௩] ¶     // → #௧

// பெயரிடப்பட்டது
நபர் = (பெயர்: "அலிஸ்", வயது: ௨௫)
>> நபர்.பெயர் ¶    // → அலிஸ்
>> நபர்[௧] ¶      // → அலிஸ்  (குறியீடும் வேலை செய்கிறது, ௧-அடிப்படை)

// உள்ளமை
இடம் = (x: ௧௦, y: ௨௦)
p = (இடம்: இடம், குறி: "தோற்றம்")
>> p.இடம்.x ¶        // → ௧௦
```

**மாற்ற முடியாத தன்மை** — ஒரு டப்பிள் உறுப்பை மாற்றும் எந்த முயற்சியும் இயக்க நேரப் பிழையாகும்:

```zymbol
t = (௧௦, ௨௦, ௩௦)
// t[௧] = ௯௯    // ❌ இயக்க நேரப் பிழை: டப்பிள்கள் மாற்ற முடியாதவை
// t[௧] += ௫    // ❌ அதே பிழை
```

மாற்றியமைக்கப்பட்ட மதிப்பைப் பெற `$~` (செயல்பாட்டு புதுப்பிப்பு) பயன்படுத்தவும் — ஒரு **புதிய** டப்பிளைத் திருப்பித் தருகிறது:

```zymbol
t = (௧௦, ௨௦, ௩௦)
t௨ = t[௨]$~ ௯௯௯
>> t ¶     // → (௧௦, ௨௦, ௩௦)   ← அசல் மாறாமல்
>> t௨ ¶    // → (௧௦, ௯௯௯, ௩௦)

// பெயரிடப்பட்ட டப்பிள் — வெளிப்படையாக மீண்டும் உருவாக்கவும்
நபர் = (பெயர்: "அலிஸ்", வயது: ௨௫)
பெரியது  = (பெயர்: நபர்.பெயர், வயது: ௨௬)
>> நபர்.வயது ¶    // → ௨௫
>> பெரியது.வயது ¶     // → ௨௬
```

## உயர்-வரிசை செயல்பாடுகள்

```zymbol
எண்கள் = [௧, ௨, ௩, ௪, ௫, ௬, ௭, ௮, ௯, ௧௦]

இரட்டிப்பானவை  = எண்கள்$> (x -> x * ௨)                  // map  → [௨,௪,௬…௨௦]
இரட்டைப்படை    = எண்கள்$| (x -> x % ௨ == ௦)           // filter → [௨,௪,௬,௮,௧௦]
மொத்தம்    = எண்கள்$< (௦, (திரட்டி, x) -> திரட்டி + x)     // reduce → ௫௫

// இடைநிலைகள் மூலம் சங்கிலியாக்கவும்
படி௧ = எண்கள்$| (x -> x > ௩)
படி௨ = படி௧$> (x -> x * x)
>> படி௨ ¶    // → [௧௬, ௨௫, ௩௬, ௪௯, ௬௪, ௮௧, ௧௦௦]

// பெயரிடப்பட்ட செயல்பாடுகளை நேரடியாக HOF க்கு அனுப்பலாம்
இரட்டிப்பு(x) { <~ x * ௨ }
பெரியது_ஆ(x) { <~ x > ௫ }
r = எண்கள்$> இரட்டிப்பு       // ✅ நேரடிக் குறிப்பு
r = எண்கள்$| பெரியது_ஆ       // ✅ நேரடிக் குறிப்பு
```

## குழாய் செயலி

வலது பக்கம் எப்போதும் குழாய் செய்யப்பட்ட மதிப்பிற்கு `_` ஒதுக்கிடத்தை தேவைப்படுகிறது:

```zymbol
இரட்டிப்பு = x -> x * ௨
கூட்டல் = (a, b) -> a + b
அதிகரிப்பு = x -> x + ௧

வ௧ = ௫ |> இரட்டிப்பு(_)        // → ௧௦
வ௨ = ௧௦ |> கூட்டல்(_, ௫)       // → ௧௫
வ௩ = ௫ |> கூட்டல்(௨, _)        // → ௭

// சங்கிலியாக்கப்பட்டது
வ = ௫ |> இரட்டிப்பு(_) |> அதிகரிப்பு(_) |> இரட்டிப்பு(_)
>> வ ¶    // → ௨௨  (௫→௧௦→௧௧→௨௨)
```

```zymbol
>> m::கூட்டல்(௩, ௪) ¶    // → ௭  (உள் பெயர் _உள்_கூட்டல் மறைக்கப்பட்டுள்ளது)
```

> **தொகுதி விதிகள்**: `# பெயர் { }` உள்ளே `#>`, செயல்பாட்டு வரையறைகள், மற்றும் நேரடி மாறி/மாறிலி தொடக்கிகள் மட்டுமே அனுமதிக்கப்படுகின்றன. இயக்கக்கூடிய அறிக்கைகள் (`>>`, `<<`, வளையங்கள், முதலியன) E013 பிழையை எழுப்புகின்றன.

## எண் முறைகள்

Zymbol எண்களை **69 யூனிகோட் இலக்க எழுத்துருக்களில்** காட்ட முடியும் — தேவநாகரி, அரபி-இந்திய, தாய், க்ளிங்கான் pIqaD, கணித தடித்த, எல்சிடி பிரிவுகள், மற்றும் பல. செயலில் உள்ள முறை `>>` வெளியீட்டை மட்டுமே பாதிக்கிறது; உள் கணிதம் எப்போதும் இருமமாகும்.

### எழுத்துருவை செயல்படுத்துதல்

இலக்கு எழுத்துருவின் `0` மற்றும் `9` இலக்கங்களை `#…#` உள்ளே எழுதவும்:

```zymbol
#०९#    // தேவநாகரி   (U+0966–U+096F)
#٠٩#    // அரபி-இந்திய (U+0660–U+0669)
#๐๙#    // தாய்         (U+0E50–U+0E59)
#09#    // ASCII க்கு மீட்டமை
```

### வெளியீடு மற்றும் பூலியன்கள்

```zymbol
x = ௪௨
>> x ¶          // → ௪௨   (ASCII இயல்புநிலை)

#०९#
>> x ¶          // → ४२
>> ௩.௧௪ ¶       // → ३.१४   (தசமப் புள்ளி எப்போதும் ASCII)
>> ௧ + ௨ ¶      // → ३

// பூலியன்கள்: # முன்னொட்டு எப்போதும் ASCII, இலக்கம் தகவமைக்கிறது
>> #௧ ¶         // → #१   (தேவநாகரியில் உண்மை)
>> #௦ ¶         // → #०   (தவறு — ௦ முழு எண் பூஜ்ஜியத்திலிருந்து வேறுபட்டது)

x = ௨௮ > ௪
>> x ¶          // → #१   (ஒப்பீட்டு முடிவு செயலில் உள்ள முறையைப் பின்பற்றுகிறது)
```

### மூலத்தில் சொந்த இலக்க நேரடி எழுத்துக்கள்

எந்த ஆதரிக்கப்படும் எழுத்துருவின் இலக்கங்களும் செல்லுபடியாகும் நேரடி எழுத்துக்கள் — வரம்புகள், மாடுலோ, ஒப்பீடுகளில்:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### எந்த எழுத்துருவிலும் பூலியன் நேரடி எழுத்துக்கள்

`#` + எந்தத் தொகுதியிலிருந்தும் இலக்கம் `0` அல்லது `1` ஒரு செல்லுபடியாகும் பூலியன் நேரடி எழுத்து ஆகும்:

```zymbol
#٠٩#
செயலில் = #١        // #௧ க்கு சமம்
>> செயலில் ¶        // → #१
>> (#١ && #٠) ¶ // → #०
```

> `#` **எப்போதும் ASCII** ஆகும். `#0` (தவறு) எப்போதும் ஒவ்வொரு எழுத்துருவிலும் `0` (முழு எண் பூஜ்ஜியம்) என்பதிலிருந்து காட்சி ரீதியாக வேறுபட்டது.

## பதிப்பு வெளியீட்டு மாற்றங்களின் பதிவு

### v0.0.5 — டியுஐ அடிப்படைகள், ஹாட் வரையறை & சர மீளச்செய்கை _(மே 2026)_

- **உடைக்கும்** பொருத்த கிளை பிரிப்பான்: `வடிவம் : முடிவு` → `வடிவம் => முடிவு`
- **உடைக்கும்** இறக்குமதி மறுபெயர்: `<# பாதை <= மறுபெயர்` → `<# பாதை => மறுபெயர்`
- **உடைக்கும்** ஏற்றுமதி மறுபெயரிடல்: `#> { fn <= பொது }` → `#> { fn => பொது }`
- **சேர்க்கப்பட்டது** டியுஐ தொகுதி `>>| { }` — மாற்றுத் திரை + கச்சா முறை; வெளியேறும்போது சுத்தம் செய்கிறது
- **சேர்க்கப்பட்டது** நிலைசார் வெளியீடு `>>~ (வரிசை, நிரல், BKS, fg, bg) > உருப்படிகள்` — அரிய இடங்கள், ANSI 256 வண்ணம்
- **சேர்க்கப்பட்டது** விசை உள்ளீடு `<<| மாறி` (தடுக்கும்) மற்றும் `<<|? மாறி` (தடுக்காத கணக்கெடுப்பு)
- **சேர்க்கப்பட்டது** `>>!` திரையை சுத்தம் செய், `>>?` முனைய அளவைக் கேட்கவும், `@~ N` N மில்லி விநாடி உறங்கு
- **சேர்க்கப்பட்டது** ஹாட் வரையறை `°x` / `x°` — வளையங்களில் முதல் பயன்பாட்டில் மாறியை தானியங்கு தொடக்கம்
- **சேர்க்கப்பட்டது** சர மீளச்செய்கை `சரம் $* N` — ஒரு சரத்தை N முறை மீண்டும் செய்யவும்

- **உடைக்கும்** அனைத்து குறியீட்டாக்கமும் **1-அடிப்படை** ஆக மாற்றப்பட்டது — `arr[1]` முதல் உறுப்பு; `arr[0]` ஒரு இயக்க நேரப் பிழையாகும்
- **சேர்க்கப்பட்டது** பெயரிடப்பட்ட செயல்பாடுகள் **முதல்-வகுப்பு மதிப்புகள்** — நேரடியாக HOF க்கு அனுப்பவும்: `எண்கள்$> இரட்டிப்பு`
- **சேர்க்கப்பட்டது** தொகுதிகளுக்கான **தொகுதி இலக்கணம் தேவை** — `# பெயர் { ... }` — தட்டையான இலக்கணம் நீக்கப்பட்டது
- **சேர்க்கப்பட்டது** பல-பரிமாண குறியீட்டாக்கம்: `arr[i>j>k]` (வழிசெலுத்தல்), `arr[p ; q]` (தட்டையான பிரித்தெடுத்தல்)
- **சேர்க்கப்பட்டது** வகை மாற்றம் வார்ப்புகள்: `##.வெளிப்பாடு` (மிதவை), `###வெளிப்பாடு` (முழு எண் சுற்று), `##!வெளிப்பாடு` (முழு எண் துண்டிப்பு)
- **சேர்க்கப்பட்டது** சரப் பிரிப்பு: `சரம்$/ பிரிப்பான்` — `Array(சரம்)` திருப்பித் தருகிறது
- **சேர்க்கப்பட்டது** இணைப்பு உருவாக்கம்: `அடிப்படை$++ a b c` — பல உருப்படிகளைச் சேர்க்கிறது
- **சேர்க்கப்பட்டது** டைம்ஸ் வளையம்: `@ N { }` — சரியாக N முறை மீண்டும் செய்யவும்
- **சேர்க்கப்பட்டது** முத்திரையிடப்பட்ட வளைய இலக்கணம்: `@:பெயர் { }`, `@:பெயர்!`, `@:பெயர்>` — `@ @பெயர்` / `@! பெயர்` ஐ மாற்றுகிறது
- **சேர்க்கப்பட்டது** மாறி கள விதிகள்: `_பெயர்` மாறிகள் துல்லியமான தொகுதி களத்தைக் கொண்டுள்ளன; `\ மாறி` முன்னதாக அழிக்கிறது
- **சேர்க்கப்பட்டது** பொருத்த ஒப்பீட்டு வடிவங்கள்: `< 0 =>`, `> 5 =>`, `== 42 =>` போன்றவை
- **சேர்க்கப்பட்டது** தொகுதி E013 பிழை: தொகுதி உடலில் இயக்கக்கூடிய அறிக்கைகள் தடைசெய்யப்பட்டுள்ளன
- **சரி செய்யப்பட்டது** `alias.CONST` இப்போது சரியாகத் தீர்க்கிறது; `#>` செயல்பாட்டு வரையறைகளுக்குப் பிறகு வரலாம்
- **VM** முழு சமத்துவம்: 393/393 சோதனைகள் தேர்ச்சி
