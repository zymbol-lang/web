## TUI اصلي برخې

د متقابل پروګرامونو لپاره د ټرمینل کاروونکي انٹرفیس چلونکي. ډیری یې د `>>| { }` بلاک ته اړتیا لري (بدیل سکرین + خام حالت).

```zymbol
>>| {
    >>!                             // بدیل سکرین پاک کړئ
    >>~ (۱, ۱, ۰, ۱۰) > "چلیږي"   // قطار ۱, ستنه ۱, fg=۱۰ (شین)
    @~ ۱۰۰۰                         // ۱ ثانیه ودریږئ (۱۰۰۰ ms)
    >>~ (۲, ۱) > "پای ته ورسیده."
}
// د وتلو په وخت کې ټرمینل په اتوماتيک ډول بیرته راګرځول کیږي
```

```zymbol
// کیلي فشار او د ټرمینل اندازه
>>| {
    [قطارونه, ستنې] = >>?              // د ټرمینل ابعاد پوښتنه وکړئ
    >>~ (۱, ۱) > "ټرمینل: " قطارونه " x " ستنې
    <<| کیلي                         // بلاک کولو کیلي فشار ولولئ
    >>~ (۲, ۱) > "تاسو فشار کړ: " کیلي
}
```

> `>>!` سکرین پاکوي. `>>?` `[قطارونه, ستنې]` راګرځوي. `@~ N` N میلی ثانیه ویده کیږي.
> `<<|` یو کیلي فشار لوستل (بلاک کول); `<<|?` پرته له بلاک کولو پوښتنه کوي (که هیڅ نه وي `'\0'` راګرځوي).
> موقعیتي محصول ټوپل: `(قطار, ستنه, BKS, fg, bg)` — هر سلاټ د کوما سره پریښودل کیدی شي (`>>~ (,,, 196) > "سور"`).
> BKS بټماسک: `1`=بولډ, `2`=ایټالیک, `4`=لاندې کرښه. ANSI 256 رنګ پیلټ (`0`=ټرمینل ډیفالټ).
> یوازې tree-walker (د `>>!`, `>>?`, `@~`, `>>~` پرته کوم چې په `--vm` کې هم کار کوي).

## صف

صفونه **د بدلون وړ** دي او د **ورته ډول** عناصر لري.

```zymbol
arr = [۱, ۲, ۳, ۴, ۵]

x = arr[۱]      // ۱ — لاسرسی (۱-بنسټ: لومړی عنصر)
x = arr[-۱]     // ۵ — منفي شاخص (وروستی عنصر)
x = arr$#       // ۵ — اوږدوالی (په >> کې (arr$#) وکاروئ)

arr = arr$+ ۶            // اضافه کړئ → [۱,۲,۳,۴,۵,۶]
arr۲ = arr$+[۲] ۹۹       // په موقعیت ۲ کې داخل کړئ (۱-بنسټ)
arr۳ = arr$- ۳           // د ارزښت لومړۍ پېښه لرې کړئ
arr۴ = arr$-- ۳          // ټولې پېښې لرې کړئ
arr۵ = arr$-[۱]          // په شاخص ۱ کې لرې کړئ (لومړی عنصر)
arr۶ = arr$-[۲..۳]       // سیمه لرې کړئ (۱-بنسټ, پای شامل دی)

لري = arr$? ۳            // #۱ — لري
ځایونه = arr$?? ۳           // [۳] — د ارزښت ټول شاخصونه (۱-بنسټ)
ټوټه = arr$[۱..۳]          // [۱,۲,۳] — ټوټه (۱-بنسټ, پای شامل دی)
ټوټه۲ = arr$[۱:۳]          // [۱,۲,۳] — ورته, د شمېرنې پر بنسټ ترکیب

پورته = arr$^+             // پورته ترتیب کړئ (یوازې اصلي ډولونه)
ښکته = arr$^-            // ښکته ترتیب کړئ (یوازې اصلي ډولونه)

// نومول شوي/موقعیتي ټوپل صفونه — د پرتله کونکي لمبدا سره $^ وکاروئ
ډیټابیس = [(نوم: "کارلا", عمر: ۲۸), (نوم: "انا", عمر: ۲۵), (نوم: "بوب", عمر: ۳۰)]

د_نوم_له_مخې = ډیټابیس$^ (a, b -> a.نوم > b.نوم)   // د نوم له مخې ښکته (>)
>> د_عمر_له_مخې[۱].نوم ¶     // → انا
>> د_نوم_له_مخې[۱].نوم ¶    // → کارلا

// مستقیم عنصر تازه کول (یوازې صف)
arr[۱] = ۹۹              // وټاکئ
arr[۲] += ۵              // مرکب: +=  -=  *=  /=  %=  ^=

// فعالیتي تازه کول — یو نوی صف راګرځوي؛ اصلي بدلیږي نه

> ټول راټولونکي چلونکي یو **نوی صف** راګرځوي. بیرته وټاکئ: `arr = arr$+ 4`.
> `$+` سلسله کیدی شي: `arr = arr$+ 5$+ 6$+ 7`. نور چلونکي منځنۍ ټاکنې کاروي.
> **شاخصونه 1-بنسټ دی**: `arr[1]` لومړی عنصر دی؛ `arr[0]` د چلولو وخت تېروتنه ده.
> `$^+` / `$^-` **اصلي صفونه** (شمېرې, تارونه) ترتیبوي. د ټوپل صفونو لپاره، د پرتله کونکي لمبدا سره `$^` وکاروئ — لوری په لمبدا کې کوډ شوی دی (`<` = پورته, `>` = ښکته).

// نومول شوی
کس = (نوم: "ایلیس", عمر: ۲۵)
>> کس.نوم ¶    // → ایلیس
>> کس[۱] ¶      // → ایلیس  (شاخص هم کار کوي, ۱-بنسټ)

// نیسټیډ
ځای = (x: ۱۰, y: ۲۰)
p = (ځای: ځای, نښه: "اصلي")
>> p.ځای.x ¶        // → ۱۰
```

**د بدلون وړ نه والی** — د ټوپل عنصر د بدلولو هر ډول هڅه د چلولو وخت تېروتنه ده:

```zymbol
t = (۱۰, ۲۰, ۳۰)
// t[۱] = ۹۹    // ❌ د چلولو وخت تېروتنه: ټوپلونه د بدلون وړ نه دي
// t[۱] += ۵    // ❌ ورته تېروتنه
```

د بدل شوي ارزښت د ترلاسه کولو لپاره `$~` (فعالیتي تازه کول) وکاروئ — یو **نوی** ټوپل راګرځوي:

```zymbol
t = (۱۰, ۲۰, ۳۰)
t۲ = t[۲]$~ ۹۹۹
>> t ¶     // → (۱۰, ۲۰, ۳۰)   ← اصلي نه دی بدل شوی
>> t۲ ¶    // → (۱۰, ۹۹۹, ۳۰)

// نومول شوی ټوپل — په واضح ډول بیا جوړ کړئ
کس = (نوم: "ایلیس", عمر: ۲۵)
لوی  = (نوم: کس.نوم, عمر: ۲۶)
>> کس.عمر ¶    // → ۲۵
>> لوی.عمر ¶     // → ۲۶
```

## لوړ-درجه دندې

```zymbol
شمېرې = [۱, ۲, ۳, ۴, ۵, ۶, ۷, ۸, ۹, ۱۰]

دوه_چنده شوي  = شمېرې$> (x -> x * ۲)                  // map  → [۲,۴,۶…۲۰]
جفت    = شمېرې$| (x -> x % ۲ == ۰)           // filter → [۲,۴,۶,۸,۱۰]
ټول    = شمېرې$< (۰, (راټولوونکی, x) -> راټولوونکی + x)     // reduce → ۵۵

// د منځنیو له لارې سلسله
ګام۱ = شمېرې$| (x -> x > ۳)
ګام۲ = ګام۱$> (x -> x * x)
>> ګام۲ ¶    // → [۱۶, ۲۵, ۳۶, ۴۹, ۶۴, ۸۱, ۱۰۰]

// نومول شوي دندې مستقیم HOF ته استول کیدی شي

لوی دی(x) { <~ x > ۵ }

r = شمېرې$| لوی دی       // ✅ مستقیم حواله
```

## عددي حالتونه

Zymbol کولی شي شمېرې په **69 یونیکوډ عدد لیکدودونو** کې وښیې — دیوناګري, عربي-هندي, تای, کلینګون pIqaD, ریاضيکي بولډ, LCD برخې, او نور. فعال حالت یوازې د `>>` محصول اغیزه کوي؛ داخلي شمېرپوهنه تل بائنری ده.

### د لیکدود فعالول

د هدف لیکدود `0` او `9` عددونه د `#…#` دننه ولیکئ:

```zymbol
#०९#    // دیوناګري   (U+0966–U+096F)
#٠٩#    // عربي-هندي (U+0660–U+0669)
#๐๙#    // تای         (U+0E50–U+0E59)
#09#    // ASCII ته بیا تنظیم کړئ
```

### محصول او بولینونه

```zymbol
x = ۴۲
>> x ¶          // → ۴۲   (ASCII ډیفالټ)

#०९#
>> x ¶          // → ४२
>> ۳.۱۴ ¶       // → ३.१४   (لسیزه نقطه تل ASCII ده)
>> ۱ + ۲ ¶      // → ३

// بولینونه: # مخډنډ تل ASCII دی, عدد تطابق کوي
>> #۱ ¶         // → #१   (په دیوناګري کې ریښتیا)
>> #۰ ¶         // → #०   (دروغ — د ۰ بشپړ عدد صفر څخه جلا دی)

x = ۲۸ > ۴
>> x ¶          // → #१   (د پرتله کولو پایله فعال حالت تعقیبوي)
```

### په سرچینه کې اصلي عدد لغويونه

د هر ملاتړ شوي لیکدود عددونه اعتبار لري — په سیمو، ماډلو، پرتله کولو کې:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### په هر لیکدود کې بولین لغويونه

`#` + د هر بلاک څخه عدد `0` یا `1` یو اعتبار لرونکی بولین لغوي دی:

```zymbol
#٠٩#
فعال = #١        // د #۱ سره ورته
>> فعال ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **تل ASCII** دی. `#0` (دروغ) تل په هر لیکدود کې د `0` (بشپړ عدد صفر) څخه په لید کې توپیر لري.
