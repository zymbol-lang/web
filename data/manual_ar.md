# دليل Zymbol-Lang

> **تمت المراجعة للإصدار v0.0.5 — 2026-05-14**

**Zymbol-Lang** هي لغة برمجة رمزية. لا كلمات مفتاحية — كل شيء رمز. تعمل بشكل متطابق في أي لغة بشرية.

- لا `if`، ولا `while`، ولا `return` — فقط `?`، `@`، `<~`
- يونيكود كامل — معرفات بأي لغة أو إيموجي
- محايدة للغة البشر — الكود هو نفسه في كل مكان

**إصدار المُفسِّر**: v0.0.5 | **تغطية الاختبارات**: 436/436 (تكافؤ TW ↔ VM)

## المتغيرات والثوابت

```zymbol
س = ١٠              // متغير قابل للتغيير
π := ٣.١٤١٥٩        // ثابت — إعادة التعيين خطأ في وقت التشغيل
اسم = "أليس"
نشط = #١            // منطقي صحيح
👋 := "مرحباً"
```

```zymbol
س = ١٠    // ١٠
س += ٥    // ١٥
س -= ٣    // ١٢
س *= ٢    // ٢٤
س /= ٣    // ٨
س %= ٣    // ٢
س ^= ٢    // ٤
س++        // ٥
س--        // ٤
```

`°` (علامة الدرجة، U+00B0) تُهيِّئ المتغير تلقائياً إلى قيمته المحايدة عند أول استخدام:

```zymbol
أعداد = [٣, ١, ٤, ١, ٥]
@ ن:أعداد {
    °المجموع += ن    // تهيئة تلقائية إلى ٠ فوق الحلقة؛ تبقى بعد @
}
>> المجموع ¶         // → ١٤
```

> `°س` (بادئة) ترسو فوق الحلقة — النتيجة متاحة بعد `@`.
> `س°` (لاحقة) ترسو داخل الحلقة — تموت عند انتهاء الحلقة.
> فقط tree-walker.

## بدائيات TUI

معاملات واجهة الطرفية للبرامج التفاعلية. معظمها يتطلب كتلة `>>| { }` (شاشة بديلة + وضع خام).

```zymbol
>>| {
    >>!                             // مسح الشاشة البديلة
    >>~ (١, ١, ٠, ١٠) > "جاري"     // صف ١, عمود ١, fg=١٠ (أخضر)
    @~ ١٠٠٠                         // توقف ثانية واحدة (١٠٠٠ مللي ثانية)
    >>~ (٢, ١) > "تم."
}
// يتم استعادة الطرفية تلقائياً عند الخروج
```

```zymbol
// ضغطة المفتاح وحجم الطرفية
>>| {
    [الصفوف, الأعمدة] = >>?              // استعلام أبعاد الطرفية
    >>~ (١, ١) > "الطرفية: " الصفوف " × " الأعمدة
    <<| مفتاح                         // قراءة ضغطة مفتاح مانعة
    >>~ (٢, ١) > "ضغطت: " مفتاح
}
```

> `>>!` تمسح الشاشة. `>>?` تعيد `[الصفوف, الأعمدة]`. `@~ ن` توقف ن مللي ثانية.
> `<<|` تقرأ ضغطة مفتاح واحدة (مانعة); `<<|?` تفحص بدون منع (تعيد `'\0'` إذا لم يكن هناك مفتاح).
> صفة المخرج المحدد الموضع: `(الصف, العمود, BKS, fg, bg)` — يمكن حذف أي خانة بفاصلة (`>>~ (,,, 196) > "أحمر"`).
> BKS قناع بت: `1`=عريض، `2`=مائل، `4`=تسطير. لوحة ANSI 256 لوناً (`0`=إفتراضي الطرفية).
> فقط tree-walker (باستثناء `>>!`، `>>?`، `@~`، `>>~` والتي تعمل أيضاً في `--vm`).

## الحلقات

```zymbol
@ i:٠..٤  { >> i " " }        // نطاق شامل:  ٠ ١ ٢ ٣ ٤
@ i:١..٩:٢ { >> i " " }       // بخطوة:         ١ ٣ ٥ ٧ ٩
@ i:٥..٠:١ { >> i " " }       // عكسي:           ٥ ٤ ٣ ٢ ١ ٠

ن = ١
@ ن <= ٦٤ { ن *= ٢ }
>> ن ¶                        // → ١٢٨  (while)

فواكه = ["تفاح", "إجاص", "عنب"]
@ ف:فواكه { >> ف ¶ }         // لكل عنصر في المصفوفة

@ ح:"مرحباً" { >> ح "-" }
>> ¶                          // → م-ر-ح-ب-ا-  (لكل حرف في السلسلة)

@ i:١..١٠ {
    ? i % ٢ == ٠ { @> }       // @> continue
    ? i > ٧ { @! }             // @! break
    >> i " "
}
>> ¶                          // → ١ ٣ ٥ ٧

// حلقة لا نهائية
i = ٠
@ {
    i++
    ? i >= ٥ { @! }
    >> i " "
}
>> ¶                          // → ١ ٢ ٣ ٤

// حلقة موسومة (break متداخل)
العداد = ٠
@:خارجي {
    العداد++
    ? العداد >= ٣ { @:خارجي! }
}
>> العداد ¶                    // → ٣
```

## الدوال

```zymbol
جمع(أ, ب) { <~ أ + ب }
>> جمع(٣, ٤) ¶    // → ٧

عاملي(ن) {
    ? ن <= ١ { <~ ١ }
    <~ ن * عاملي(ن - ١)
}
>> عاملي(٥) ¶    // → ١٢٠
```

الدوال لها **نطاق معزول** — لا يمكنها قراءة المتغيرات الخارجية. استخدم معاملات المخرجات `<~` لتعديل متغيرات المتصل:

```zymbol
تبديل(أ<~, ب<~) {
    مؤقت = أ
    أ = ب
    ب = مؤقت
}
س = ١٠
ص = ٢٠
تبديل(س, ص)
>> "س=" س " ص=" ص ¶    // → س=٢٠ ص=١٠
```

> الدوال المسماة هي **قيم من الدرجة الأولى** — يمكن تمريرها مباشرة: `nums$> ضعف`. الالتفاف: `س -> د(س)` صحيح أيضاً.

## المصفوفات

المصفوفات **قابلة للتغيير** وتحتوي على عناصر من **نفس النوع**.

```zymbol
مصفوفة = [١, ٢, ٣, ٤, ٥]

س = مصفوفة[١]      // ١ — وصول (١-based: أول عنصر)
س = مصفوفة[-١]     // ٥ — فهرس سالب (آخر عنصر)
س = مصفوفة$#       // ٥ — الطول (استخدم (مصفوفة$#) في >>)

مصفوفة = مصفوفة$+ ٦            // إلحاق → [١,٢,٣,٤,٥,٦]
مصفوفة٢ = مصفوفة$+[٢] ٩٩       // إدراج في الموضع ٢ (١-based)
مصفوفة٣ = مصفوفة$- ٣           // إزالة أول ظهور للقيمة
مصفوفة٤ = مصفوفة$-- ٣          // إزالة كل الظهورات
مصفوفة٥ = مصفوفة$-[١]          // إزالة في الفهرس ١ (أول عنصر)
مصفوفة٦ = مصفوفة$-[٢..٣]       // إزالة نطاق (١-based, النهاية شاملة)

يوجد = مصفوفة$? ٣            // #١ — يحتوي
المواقع = مصفوفة$?? ٣           // [٣] — كل فهارس القيمة (١-based)
شريحة = مصفوفة$[١..٣]          // [١,٢,٣] — شريحة (١-based, النهاية شاملة)
شريحة٢ = مصفوفة$[١:٣]          // [١,٢,٣] — نفس الشيء, بناء جملة مبني على العدد

تصاعدي = مصفوفة$^+             // مرتب تصاعدياً  (البدائيات فقط)
تنازلي = مصفوفة$^-            // مرتب تنازلياً (البدائيات فقط)

// مصفوفات الصفوف المسماة/الموضعية — استخدم $^ مع لامبدا مقارنة
قاعدة = [(الاسم: "كارلا", العمر: ٢٨), (الاسم: "آنا", العمر: ٢٥), (الاسم: "بوب", العمر: ٣٠)]
حسب_العمر  = قاعدة$^ (أ, ب -> أ.العمر < ب.العمر)    // تصاعدي بالعمر  (<)
حسب_الاسم = قاعدة$^ (أ, ب -> أ.الاسم > ب.الاسم)   // تنازلي بالاسم (>)
>> حسب_العمر[١].الاسم ¶     // → آنا
>> حسب_الاسم[١].الاسم ¶    // → كارلا

// تحديث عنصر مباشر (المصفوفات فقط)
مصفوفة[١] = ٩٩              // تعيين
مصفوفة[٢] += ٥              // مركب: +=  -=  *=  /=  %=  ^=

// تحديث وظيفي — يعيد مصفوفة جديدة؛ الأصلي لم يتغير
مصفوفة٢ = مصفوفة[٢]$~ ٩٩
```

> جميع معاملات المجموعة تعيد **مصفوفة جديدة**. عيّن العودة: `مصفوفة = مصفوفة$+ 4`.
> `$+` يمكن تسلسله: `مصفوفة = مصفوفة$+ 5$+ 6$+ 7`. المعاملات الأخرى تستخدم تعيينات وسيطة.
> **الفهرسة 1-based**: `مصفوفة[1]` هو أول عنصر؛ `مصفوفة[0]` هو خطأ في وقت التشغيل.
> `$^+` / `$^-` ترتب **مصفوفات البدائيات** (أرقام، سلاسل نصية). لمصفوفات الصفوف استخدم `$^` مع لامبدا مقارنة — الاتجاه مشفر في اللامبدا (`<` = تصاعدي، `>` = تنازلي).

**دلالات القيمة** — تعيين مصفوفة إلى متغير آخر ينشئ نسخة مستقلة:

```zymbol
أ = [١, ٢, ٣]
ب = أ
أ[١] = ٩٩
>> أ ¶    // → [٩٩, ٢, ٣]
>> ب ¶    // → [١, ٢, ٣]   ← ب غير متأثر
```

```zymbol
// مصفوفات متداخلة (فهرسة ١-based)
مصفوفة = [[١,٢,٣],[٤,٥,٦],[٧,٨,٩]]
>> مصفوفة[٢][٣] ¶    // → ٦  (صف ٢, عمود ٣)
```

## الصفوف

الصفوف هي حاويات مرتبة **غير قابلة للتغيير** يمكنها الاحتفاظ بقيم من **أنواع مختلفة**.
على عكس المصفوفات، لا يمكن تغيير العناصر بعد الإنشاء.

```zymbol
// موضعي — أنواع مختلطة مسموحة
نقطة = (١٠, ٢٠)
>> نقطة[١] ¶    // → ١٠

بيانات = (٤٢, "مرحباً", #١, ٣.١٤)
>> بيانات[٣] ¶     // → #١

// مسمى
شخص = (الاسم: "أليس", العمر: ٢٥)
>> شخص.الاسم ¶    // → أليس
>> شخص[١] ¶      // → أليس  (الفهرس يعمل أيضاً, ١-based)

// متداخل
موضع = (س: ١٠, ص: ٢٠)
م = (موضع: موضع, وسم: "الأصل")
>> م.موضع.س ¶        // → ١٠
```

**عدم قابلية التغيير** — أي محاولة لتعديل عنصر صف هي خطأ في وقت التشغيل:

```zymbol
ت = (١٠, ٢٠, ٣٠)
// ت[١] = ٩٩    // ❌ خطأ في وقت التشغيل: الصفوف غير قابلة للتغيير
// ت[١] += ٥    // ❌ نفس الخطأ
```

لاستنتاج قيمة معدلة استخدم `$~` (تحديث وظيفي) — يعيد صفاً **جديداً**:

```zymbol
ت = (١٠, ٢٠, ٣٠)
ت٢ = ت[٢]$~ ٩٩٩
>> ت ¶     // → (١٠, ٢٠, ٣٠)   ← الأصلي لم يتغير
>> ت٢ ¶    // → (١٠, ٩٩٩, ٣٠)

// صف مسمى — إعادة بناء صريحة
شخص = (الاسم: "أليس", العمر: ٢٥)
أكبر  = (الاسم: شخص.الاسم, العمر: ٢٦)
>> شخص.العمر ¶    // → ٢٥
>> أكبر.العمر ¶     // → ٢٦
```

## دوال الرتبة الأعلى

```zymbol
أعداد = [١, ٢, ٣, ٤, ٥, ٦, ٧, ٨, ٩, ١٠]

مضاعفة  = أعداد$> (س -> س * ٢)                  // map  → [٢,٤,٦…٢٠]
أزواج    = أعداد$| (س -> س % ٢ == ٠)           // filter → [٢,٤,٦,٨,١٠]
المجموع    = أعداد$< (٠, (المجمع, س) -> المجمع + س)     // reduce → ٥٥

// تسلسل عبر وسائط
خطوة١ = أعداد$| (س -> س > ٣)
خطوة٢ = خطوة١$> (س -> س * س)
>> خطوة٢ ¶    // → [١٦, ٢٥, ٣٦, ٤٩, ٦٤, ٨١, ١٠٠]

// الدوال المسماة يمكن تمريرها مباشرة إلى HOF
مضاعفة(س) { <~ س * ٢ }
كبير(س) { <~ س > ٥ }
ن = أعداد$> مضاعفة       // ✅ مرجع مباشر
ن = أعداد$| كبير       // ✅ مرجع مباشر
```

## معامل الأنبوب

الجانب الأيمن يتطلب دائماً `_` كعنصر نائب للقيمة التي تمر عبر الأنبوب:

```zymbol
ضعف = س -> س * ٢
جمع = (أ, ب) -> أ + ب
زيادة = س -> س + ١

ن١ = ٥ |> ضعف(_)        // → ١٠
ن٢ = ١٠ |> جمع(_, ٥)    // → ١٥
ن٣ = ٥ |> جمع(٢, _)     // → ٧

// تسلسل
ن = ٥ |> ضعف(_) |> زيادة(_) |> ضعف(_)
>> ن ¶    // → ٢٢  (٥→١٠→١١→٢٢)
```

## الأنماط العددية

يمكن لـ Zymbol عرض الأرقام في **69 نظاماً من أنظمة أرقام يونيكود** — الديفاناغاري، العربية الهندية، التايلاندية، Klingon pIqaD، الرياضية العريضة، أجزاء LCD، والمزيد. الوضع النشط يؤثر فقط على مخرج `>>`؛ الحساب الداخلي دائماً ثنائي.

### تفعيل نظام كتابة

اكتب الرقم `0` والرقم `9` من نظام الكتابة المستهدف محاطين بـ `#…#`:

```zymbol
#०९#    // ديفاناغاري   (U+0966–U+096F)
#٠٩#    // عربية هندية (U+0660–U+0669)
#๐๙#    // تايلاندية     (U+0E50–U+0E59)
#09#    // إعادة تعيين إلى ASCII
```

### المخرجات والمنطقيان

```zymbol
س = ٤٢
>> س ¶          // → ٤٢   (ASCII افتراضي)

#०९#
>> س ¶          // → ४२
>> ٣.١٤ ¶       // → ३.१४   (النقطة العشرية دائماً ASCII)
>> ١ + ٢ ¶      // → ३

// المنطقيان: البادئة # دائماً ASCII, الرقم يتكيف
>> #١ ¶         // → #१   (صحيح في ديفاناغاري)
>> #٠ ¶         // → #०   (خطأ — متميز عن ०  الصفر الصحيح)

س = ٢٨ > ٤
>> س ¶          // → #१   (نتيجة المقارنة تتبع الوضع النشط)
```

### الحرف الرقمية الأصلية في المصدر

أرقام أي نظام كتابة مدعوم هي حرف صحيحة — في النطاقات، المعامل، المقارنات:

```zymbol
#٠٩#

@ i:١..١٥ {
    ? i % ١٥ == ٠ { >> "FizzBuzz" ¶ }
    _? i % ٣  == ٠ { >> "Fizz" ¶ }
    _? i % ٥  == ٠ { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### الحرف المنطقية في أي نظام كتابة

`#` + رقم `0` أو `1` من أي كتلة هو حرف منطقي صحيح:

```zymbol
#٠٩#
نشط = #١        // نفس #١
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` هو **دائماً ASCII**. `#0` (خطأ) دائماً متميز بصرياً عن `0` (الصفر الصحيح) في كل نظام كتابة.

## معاملات البيانات

```zymbol
// تحويلات الأنواع
ع = ##.٤٢         // → ٤٢.٠  (إلى عشري)
ص = ###٣.٧        // → ٤     (إلى صحيح, تقريب)
م = ##!٣.٧        // → ٣     (إلى صحيح, اقتطاع)

// تحليل سلسلة إلى رقم
ق١ = #|"٤٢"|      // → ٤٢  (صحيح)
ق٢ = #|"٣.١٤"|    // → ٣.١٤  (عشري)
ق٣ = #|"abc"|     // → "abc"  (آمن, بدون خطأ)

// تقريب / اقتطاع
ط = ٣.١٤١٥٩٢٦٥
مدور٢ = #.٢|ط|      // → ٣.١٤  (تقريب إلى منزلتين عشريتين)
مدور٤ = #.٤|ط|      // → ٣.١٤١٦
مقتطع٢ = #!٢|ط|      // → ٣.١٤  (اقتطاع)

// تنسيق الأرقام
منسق = #,|١٢٣٤٥٦٧|  // → ١,٢٣٤,٥٦٧  (مفصول بفواصل)
علمي = #^|١٢٣٤٥.٦٧٨|    // → ١.٢٣٤٥٦٧٨e٤  (علمي)

// الحرف الأساسية
أ = 0x41         // → 'A'  (ست عشري → حرف)
ب = 0b01000001   // → 'A'  (ثنائي → حرف)
ت = 0o101        // → 'A'  (ثماني → حرف)

// تحويل الأساس للمخرج
ست_عشري = 0x|٢٥٥|    // → "0x00FF"
ثنائي = 0b|٦٥|     // → "0b1000001"
ثماني = 0o|٨|      // → "0o10"
عشري = 0d|٢٥٥|    // → "0d0255"
```

## مرجع الرموز

| الرمز | العملية | الرمز | العملية |
|-------|---------|-------|---------|
| `=` | متغير | `$#` | الطول |
| `:=` | ثابت | `$+` | إلحاق (قابل للتسلسل) |
| `>>` | مخرج | `$+[i]` | إدراج في الفهرس (1-based) |
| `<<` | مدخل | `$-` | إزالة الأول بالقيمة |
| `¶` / `\\` | سطر جديد | `$--` | إزالة الكل بالقيمة |
| `?` | إذا | `$-[i]` | إزالة في الفهرس (1-based) |
| `_?` | وإلا إذا | `$-[i..j]` | إزالة نطاق (1-based) |
| `_` | وإلا / wildcard | `$?` | يحتوي |
| `??` | تطابق | `$??` | إيجاد كل الفهارس (1-based) |
| `@` | حلقة | `$[s..e]` | شريحة (1-based) |
| `@ N { }` | حلقة N مرة | `$>` | map |
| `@!` | break | `$\|` | filter |
| `@>` | continue | `$<` | reduce |
| `@:الاسم { }` | حلقة موسومة | `$/ محدد` | تقسيم سلسلة |
| `@:الاسم!` | break للوسم | `$++ أ ب ج` | بناء تسلسلي |
| `@:الاسم>` | continue للوسم | `arr[i>j>k]` | فهرس تنقل |
| `->` | لامبدا | `arr[i] = val` | تحديث عنصر (المصفوفات فقط) |
| `arr[i] += val` | تحديث مركب | `arr[i]$~` | تحديث وظيفي (نسخة جديدة) |
| `$^+` | ترتيب تصاعدي (بدائيات) | `$^-` | ترتيب تنازلي (بدائيات) |
| `$^` | ترتيب بمقارنة (الصفوف) | `<~` | إرجاع |
| `\|>` | أنبوب | `!?` | حاول |
| `:!` | التقط | `:>` | أخيراً |
| `#1` | صحيح | `#0` | خطأ |
| `$!` | هل هو خطأ | `$!!` | نشر الخطأ |
| `<#` | استيراد | `#>` | تصدير |
| `#` | إعلان وحدة | `::` | استدعاء وحدة |
| `.` | وصول إلى حقل | `#?` | بيانات وصف النوع |
| `#\|..\|` | تحليل رقم | `##.` | تحويل إلى عشري |
| `###` | تحويل إلى صحيح (تقريب) | `##!` | تحويل إلى صحيح (اقتطاع) |
| `#.N\|..\|` | تقريب | `#!N\|..\|` | اقتطاع |
| `#,\|..\|` | تنسيق فاصلة | `#^\|..\|` | علمي |
| `#d0d9#` | تبديل النمط العددي | `#09#` | إعادة تعيين إلى ASCII |
| `<\ ..\>` | تنفيذ صدفة | `>\<` | وسائط CLI |
| `\ متغير` | تدمير متغير صريح | `°س` / `س°` | تعريف ساخن (تهيئة تلقائية) |

| `<<\|` | ضغطة مفتاح مانعة | `<<\|?` | فحص ضغطة مفتاح غير مانع |
