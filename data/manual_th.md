# คู่มือ Zymbol-Lang ฉบับย่อ

**Zymbol-Lang** คือภาษาโปรแกรมเชิงสัญลักษณ์ ไม่มีคีย์เวิร์ดใดๆ — ทุกอย่างคือสัญลักษณ์ ทำงานเหมือนกันทุกประการไม่ว่าจะใช้ภาษาธรรมชาติใดก็ตาม

- ไม่มีคีย์เวิร์ด (`if`, `while`, `return` ไม่มีอยู่ — มีแต่สัญลักษณ์ `?`, `@`, `<~`)
- รองรับ Unicode อย่างสมบูรณ์ — ตัวอักษรในทุกภาษาและอีโมจิใช้เป็นตัวระบุได้ 👋
- ไม่ขึ้นอยู่กับภาษาธรรมชาติ — โค้ดเหมือนกันในทุกภาษา

## ตัวแปรและค่าคงที่

```zymbol
ต = ๑๐              // ตัวแปร (เปลี่ยนได้)
พาย := ๓.๑๔๑๕๙      // ค่าคงที่ (เปลี่ยนไม่ได้ — กำหนดใหม่จะเกิดข้อผิดพลาด)
ชื่อ = "อนา"
ใช้งาน = #๑         // จริง (บูลีน)
👋 := "สวัสดี"
```

```zymbol
x = ๑๐
x += ๕    // ๑๕
x -= ๓    // ๑๒
x *= ๒    // ๒๔
x /= ๓    // ๘
x %= ๓    // ๒
x ^= ๒    // ๔
x++       // ๕
x--       // ๔
```

## ชนิดข้อมูล

| ชนิด              | ตัวอย่าง            | สัญลักษณ์ `#?` | หมายเหตุ                              |
|-------------------|---------------------|----------------|---------------------------------------|
| จำนวนเต็ม         | `๔๒`, `-๗`          | `###`          | จำนวนเต็มมีเครื่องหมาย 64 บิต        |
| ทศนิยม            | `๓.๑๔`, `๑.๕e๑๐`    | `##.`          | รองรับสัญกรณ์วิทยาศาสตร์             |
| สตริง             | `"สวัสดี"`          | `##"`          | แทรกค่า: `"สวัสดี {ชื่อ}"`           |
| ตัวอักษร          | `'ก'`               | `##'`          | อักขระ Unicode หนึ่งตัว               |
| บูลีน             | `#๑`, `#๐`          | `##?`          | ไม่ใช่ตัวเลข 1 และ 0                 |
| อาร์เรย์          | `[๑, ๒, ๓]`         | `##]`          | ทุกสมาชิกต้องเป็นชนิดเดียวกัน        |
| ทูเพิล            | `(ก, ข)`            | `##)`          | เข้าถึงตามตำแหน่ง                     |
| ทูเพิลมีชื่อ      | `(x: ๑, y: ๒)`      | `##)`          | เข้าถึงด้วยชื่อหรือดัชนีก็ได้        |

```zymbol
// ตรวจสอบชนิดข้อมูล — คืนค่า (ชนิด, จำนวนหลัก, ค่า)
เมตา = ๔๒#?
>> เมตา ¶         // → (###, ๒, ๔๒)
ชนิด = เมตา[๐]
>> ชนิด ¶         // → ###
```

## การแสดงผลและรับข้อมูล

```zymbol
>> "สวัสดี" ¶                       // ¶ หรือ \\ เพื่อขึ้นบรรทัดใหม่อย่างชัดเจน
>> "ก=" ก " ข=" ข ¶                 // ค่าหลายตัวด้วยการวางเคียงกัน
>> (รายการ$#) ¶                     // ตัวดำเนินการ postfix ต้องใส่วงเล็บ

<< ชื่อ                             // ไม่มี prompt — อ่านเข้าตัวแปร
<< "ชื่อของคุณคือ? " ชื่อ           // มี prompt
```

> `¶` และ `\\` เทียบเท่ากันในฐานะการขึ้นบรรทัดใหม่

## สตริง

```zymbol
// สามรูปแบบการต่อสตริง
ชื่อ = "อนา"
อายุ = ๒๕

ข้อความ = "สวัสดี ", ชื่อ, "!"              // จุลภาค — ในการกำหนดค่า
>> "สวัสดี " ชื่อ " อายุ " อายุ " ปี" ¶    // วางเคียงกัน — ในการแสดงผล
คำอธิบาย = "สวัสดี {ชื่อ} อายุ {อายุ} ปี"  // การแทรกค่า — ใช้ได้ทุกที่
```

```zymbol
ส = "สวัสดีโลก"
ยาว = ส$#                  // length
ย่อย = ส$[๐..๕]             // slice (ไม่รวมท้าย)
มี = ส$? "โลก"              // #๑
แยก = "ก,ข,ค,ง" / ','      // [ก, ข, ค, ง]
แทน = ส$~~["ด":"ดี"]         // แทนที่
แทน๑ = ส$~~["ด":"ดี":๑]     // N แรก
```

> `+` สำหรับตัวเลขเท่านั้น สำหรับสตริง ให้ใช้ `,`, การต่อกัน หรือการแทรก

## การควบคุมการทำงาน

```zymbol
ต = ๗

? ต > ๐ { >> "บวก" ¶ }

? ต > ๑๐๐ {
    >> "ใหญ่" ¶
} _? ต > ๐ {
    >> "บวก" ¶
} _? ต == ๐ {
    >> "ศูนย์" ¶
} _ {
    >> "ลบ" ¶
}
```

> บล็อก `{ }` **จำเป็นเสมอ แม้จะมีแค่บรรทัดเดียว**

## Match

```zymbol
// Match ด้วยช่วง
คะแนน = ๘๕
เกรด = ?? คะแนน {
    ๙๐..๑๐๐ : 'A'
    ๘๐..๘๙  : 'B'
    ๗๐..๗๙  : 'C'
    _       : 'F'
}
>> เกรด ¶    // → B

// Match ด้วยสตริง
สี = "แดง"
รหัส = ?? สี {
    "แดง"   : "#FF๐๐๐๐"
    "เขียว" : "#๐๐FF๐๐"
    _        : "#๐๐๐๐๐๐"
}

// Match ด้วยเงื่อนไข guard
อุณหภูมิ = -๕
สภาพ = ?? อุณหภูมิ {
    _? อุณหภูมิ < ๐  : "น้ำแข็ง"
    _? อุณหภูมิ < ๒๐ : "เย็น"
    _? อุณหภูมิ < ๓๕ : "อุ่น"
    _                : "ร้อน"
}
>> สภาพ ¶    // → น้ำแข็ง

// รูปแบบคำสั่ง (block arms)
?? น {
    ๐       : { >> "ศูนย์" ¶ }
    _? น < ๐: { >> "ลบ" ¶ }
    _       : { >> "บวก" ¶ }
}
```

## การวนซ้ำ

```zymbol
@ ช:๐..๔  { >> ช " " }        // ช่วงรวมสองปลาย:  ๐ ๑ ๒ ๓ ๔
@ ช:๑..๙:๒ { >> ช " " }       // พร้อมก้าว:        ๑ ๓ ๕ ๗ ๙
@ ช:๕..๐:๑ { >> ช " " }       // ย้อนกลับ:         ๕ ๔ ๓ ๒ ๑ ๐

น = ๑
@ น <= ๖๔ { น *= ๒ }
>> น ¶                        // → ๑๒๘  (while)

ผลไม้ = ["แอปเปิ้ล", "ส้ม", "มะม่วง"]
@ ฟ:ผลไม้ { >> ฟ ¶ }         // for-each อาร์เรย์

@ ค:"สวัสดี" { >> ค "-" }
>> ¶                          // → ส-ว-ั-ส-ด-ี-  (for-each สตริง)

@ ช:๑..๑๐ {
    ? ช % ๒ == ๐ { @> }       // @> continue
    ? ช > ๗ { @! }             // @! break
    >> ช " "
}
>> ¶                          // → ๑ ๓ ๕ ๗

// ลูปไม่สิ้นสุด
ช = ๐
@ {
    ช++
    ? ช >= ๕ { @! }
    >> ช " "
}
>> ¶                          // → ๑ ๒ ๓ ๔

// ลูปมีชื่อ (nested break)
นับ = ๐
@ @นอก {
    นับ++
    ? นับ >= ๓ { @! นอก }
}
>> นับ ¶                    // → ๓
```

## ฟังก์ชัน

```zymbol
บวก(ก, ข) { <~ ก + ข }
>> บวก(๓, ๔) ¶    // → ๗

แฟกทอเรียล(น) {
    ? น <= ๑ { <~ ๑ }
    <~ น * แฟกทอเรียล(น - ๑)
}
>> แฟกทอเรียล(๕) ¶    // → ๑๒๐
```

ฟังก์ชันมี **ขอบเขตแยกต่างหาก** — ไม่สามารถอ่านตัวแปรภายนอกได้ ใช้พารามิเตอร์เอาต์พุต `<~` เพื่อแก้ไขตัวแปรของผู้เรียก:

```zymbol
สลับ(ก<~, ข<~) {
    ชั่วคราว = ก
    ก = ข
    ข = ชั่วคราว
}
x = ๑๐
y = ๒๐
สลับ(x, y)
>> "x=" x " y=" y ¶    // → x=๒๐ y=๑๐
```

> ฟังก์ชันมีชื่อไม่ใช่ first-class ในการส่งเป็นอาร์กิวเมนต์ ต้องห่อ: `x -> ชื่อฟังก์ชัน(x)`

## แลมบ์ดา และ โคลเชอร์

```zymbol
คูณสอง = ต -> ต * ๒
ผลรวม = (ก, ข) -> ก + ข
>> คูณสอง(๕) ¶    // → ๑๐
>> ผลรวม(๓, ๗) ¶  // → ๑๐

// บล็อก แลมบ์ดา
จำแนกค่า = ต -> {
    ? ต > ๐ { <~ "บวก" }
    _? ต < ๐ { <~ "ลบ" }
    <~ "ศูนย์"
}

// โคลเชอร์ — จับตัวแปรจาก scope ภายนอก
ตัวคูณ = ๓
คูณสาม = ต -> ต * ตัวคูณ
>> คูณสาม(๗) ¶    // → ๒๑

// Function factory
สร้างตัวบวก(น) { <~ ต -> ต + น }
บวกสิบ = สร้างตัวบวก(๑๐)
>> บวกสิบ(๕) ¶    // → ๑๕

// เก็บในอาร์เรย์
การดำเนินการ = [ต -> ต+๑, ต -> ต*๒, ต -> ต*ต]
>> การดำเนินการ[๒](๕) ¶    // → ๒๕
```

## อาร์เรย์

อาร์เรย์เป็น**แบบที่เปลี่ยนแปลงได้** และเก็บสมาชิกที่มี**ชนิดเดียวกัน**

```zymbol
รายการ = [๑, ๒, ๓, ๔, ๕]

รายการ[๐]          // ๑ — เข้าถึง (นับจาก ๐)
รายการ[-๑]         // ๕ — ดัชนีลบ (สุดท้าย)
รายการ$#           // ๕ — ความยาว

รายการ = รายการ$+ ๖            // ต่อท้าย → [๑,๒,๓,๔,๕,๖]
รายการ๒ = รายการ$+[๒] ๙๙       // แทรกที่ดัชนี ๒
รายการ๓ = รายการ$- ๓           // ลบค่าแรกที่ตรง
รายการ๔ = รายการ$-- ๓          // ลบทุกค่าที่ตรง
รายการ๕ = รายการ$-[๐]          // ลบตามดัชนี
รายการ๖ = รายการ$-[๑..๓]       // ลบช่วง (ไม่รวมท้าย)

มี = รายการ$? ๓            // #๑
ตำแหน่ง = รายการ$?? ๓      // [๒] — ดัชนีทั้งหมด
ส่วน = รายการ$[๐..๓]       // [๑,๒,๓]
ส่วน๒ = รายการ$[๐:๓]       // [๑,๒,๓] — ไวยากรณ์นับ

น้อยไปมาก = รายการ$^+     // เรียงน้อยไปมาก (primitives)
มากไปน้อย = รายการ$^-     // เรียงมากไปน้อย (primitives)

// Named tuple arrays
ข้อมูล = [(ชื่อ: "Carla", อายุ: ๒๘), (ชื่อ: "Ana", อายุ: ๒๕), (ชื่อ: "Bob", อายุ: ๓๐)]
ตามอายุ  = ข้อมูล$^ (ก, ข -> ก.อายุ < ข.อายุ)
ตามชื่อ = ข้อมูล$^ (ก, ข -> ก.ชื่อ > ข.ชื่อ)
>> ตามอายุ[๐].ชื่อ ¶     // → Ana
>> ตามชื่อ[๐].ชื่อ ¶    // → Carla

// อัปเดตสมาชิกโดยตรง (อาร์เรย์เท่านั้น)
รายการ[๑] = ๙๙              // กำหนดค่า
รายการ[๐] += ๕              // ผสม: +=  -=  *=  /=  %=  ^=

// อัปเดตแบบ functional — คืนอาร์เรย์ใหม่ อาร์เรย์เดิมไม่เปลี่ยน
รายการ๒ = รายการ[๑]$~ ๙๙
```

> ตัวดำเนินการคอลเลกชันทั้งหมดคืน**รายการใหม่** กำหนดค่าใหม่: `รายการ = รายการ$+ ๔`
> ไม่สามารถเชื่อมโซ่ได้ — ใช้การกำหนดค่าระหว่างกลาง
> `$^+` / `$^-` สำหรับ **primitive arrays** สำหรับ tuple arrays ใช้ `$^` พร้อม comparator lambda

**ความหมายค่า** — การกำหนดอาร์เรย์ให้กับตัวแปรอื่นจะสร้างสำเนาที่เป็นอิสระ:

```zymbol
a = [๑, ๒, ๓]
b = a
a[๐] = ๙๙
>> a ¶    // → [๙๙, ๒, ๓]
>> b ¶    // → [๑, ๒, ๓]   ← b ไม่ได้รับผลกระทบ
```

```zymbol
// Nested arrays
matrix = [[๑,๒,๓],[๔,๕,๖],[๗,๘,๙]]
>> matrix[๑][๒] ¶    // → ๖
```

## การแยกโครงสร้าง

```zymbol
// อาร์เรย์
รายการ = [๑๐, ๒๐, ๓๐, ๔๐, ๕๐]
[ก, ข, ค] = รายการ              // ก=๑๐  ข=๒๐  ค=๓๐
[แรก, *ที่เหลือ] = รายการ         // แรก=๑๐  ที่เหลือ=[๒๐,๓๐,๔๐,๕๐]
[x, _, z] = [๑, ๒, ๓]            // _ ทิ้งค่า

// Positional tuple
จุด = (๑๐๐, ๒๐๐)
(px, py) = จุด                   // px=๑๐๐  py=๒๐๐

// Named tuple
คน = (ชื่อ: "Ana", อายุ: ๒๕, เมือง: "กรุงเทพ")
(ชื่อ: น, อายุ: อ) = คน           // น="Ana"  อ=๒๕
```

## ทูเพิล

ทูเพิลเป็นคอนเทนเนอร์ที่มีลำดับและ**ไม่สามารถเปลี่ยนแปลงได้** ซึ่งสามารถเก็บค่าที่มี**ชนิดต่างกัน** ได้ ต่างจากอาร์เรย์ ไม่สามารถเปลี่ยนสมาชิกหลังจากสร้างแล้ว

```zymbol
// Positional
จุด = (๑๐, ๒๐)
>> จุด[๐] ¶    // → ๑๐

ข้อมูล = (๔๒, "hello", #๑, ๓.๑๔)
>> ข้อมูล[๒] ¶     // → #๑

// Named
บุคคล = (ชื่อ: "อลิซ", อายุ: ๒๕)
>> บุคคล.ชื่อ ¶    // → อลิซ
>> บุคคล[๐] ¶      // → อลิซ  (ดัชนีก็ใช้ได้)

// Nested
ตำแหน่ง = (x: ๑๐, y: ๒๐)
จุด๒ = (ตำแหน่ง: ตำแหน่ง, ป้าย: "จุดกำเนิด")
>> จุด๒.ตำแหน่ง.x ¶        // → ๑๐
```

**ความไม่เปลี่ยนแปลง** — ความพยายามใดๆ ที่จะแก้ไขสมาชิกของทูเพิลจะทำให้เกิดข้อผิดพลาดขณะทำงาน:

```zymbol
t = (๑๐, ๒๐, ๓๐)
// t[๐] = ๙๙    // ❌ ข้อผิดพลาดขณะทำงาน: ทูเพิลไม่สามารถเปลี่ยนแปลงได้
// t[๐] += ๕    // ❌ ข้อผิดพลาดเดียวกัน
```

ในการสร้างค่าที่แก้ไขแล้ว ให้ใช้ `$~` (อัปเดตแบบ functional) — คืน**ทูเพิลใหม่**:

```zymbol
t = (๑๐, ๒๐, ๓๐)
t2 = t[๑]$~ ๙๙๙
>> t ¶     // → (๑๐, ๒๐, ๓๐)   ← ต้นฉบับไม่เปลี่ยนแปลง
>> t2 ¶    // → (๑๐, ๙๙๙, ๓๐)

// Named tuple — สร้างใหม่อย่างชัดเจน
บุคคล = (ชื่อ: "อลิซ", อายุ: ๒๕)
อายุมากกว่า = (ชื่อ: บุคคล.ชื่อ, อายุ: ๒๖)
>> บุคคล.อายุ ¶       // → ๒๕
>> อายุมากกว่า.อายุ ¶  // → ๒๖
```

## ฟังก์ชันอันดับสูง

> ตัวดำเนินการ HOF ต้องใช้ **แลมบ์ดา อินไลน์** — ไม่สามารถส่งตัวแปร แลมบ์ดา โดยตรงได้

```zymbol
ตัวเลข = [๑, ๒, ๓, ๔, ๕, ๖, ๗, ๘, ๙, ๑๐]

คู่สอง  = ตัวเลข$> (ต -> ต * ๒)                // map  → [๒,๔,๖…๒๐]
จำนวนคู่ = ตัวเลข$| (ต -> ต % ๒ == ๐)           // filter → [๒,๔,๖,๘,๑๐]
ผลรวม    = ตัวเลข$< (๐, (สะสม, ต) -> สะสม + ต)  // reduce → ๕๕

// เชื่อมโซ่ผ่านตัวกลาง
ขั้น๑ = ตัวเลข$| (ต -> ต > ๓)
ขั้น๒ = ขั้น๑$> (ต -> ต * ต)
>> ขั้น๒ ¶    // → [๑๖, ๒๕, ๓๖, ๔๙, ๖๔, ๘๑, ๑๐๐]

// ฟังก์ชันมีชื่อใน HOF — ต้องห่อด้วย lambda
คูณสอง๒(ต) { <~ ต * ๒ }
ผล = ตัวเลข$> (ต -> คูณสอง๒(ต))    // ✅
```

## ระบบตัวเลข

Zymbol สามารถแสดงตัวเลขใน **69 ชุดอักขระ Unicode** — ไทย, เทวนาครี, อาหรับ-อินดิก, คลิงออน pIqaD, ตัวหนาทางคณิตศาสตร์, ตัวเลข LCD และอื่นๆ โหมดที่ใช้งานอยู่มีผลเฉพาะกับเอาต์พุต `>>` เท่านั้น การคำนวณภายในเป็นเลขฐานสองเสมอ

### เปิดใช้งานระบบ

เขียนตัวเลข `๐` และ `๙` ของสคริปต์เป้าหมายระหว่าง `#…#`:

```zymbol
#๐๙#    // ไทย        (U+0E50–U+0E59)
#๐๙#    // Thai       (U+0E50–U+0E59)
#٠٩#    // อาหรับ-อินดิก (U+0660–U+0669)
#09#    // รีเซ็ต ASCII
```

### เอาต์พุตและบูลีน

```zymbol
x = ๔๒
>> x ¶          // → ๔๒   (ASCII ค่าเริ่มต้น)

#๐๙#
>> x ¶          // → ๔๒
>> ๓.๑๔ ¶       // → ๓.๑๔   (จุดทศนิยมเป็น ASCII เสมอ)
>> ๑ + ๒ ¶      // → ๓

>> #๑ ¶         // → #๑   (จริงในภาษาไทย)
>> #๐ ¶         // → #๐   (เท็จ — ต่างจาก ๐ ศูนย์)

x = ๒๘ > ๔
>> x ¶          // → #๑   (ผลการเปรียบเทียบตามโหมดที่ใช้งาน)
```

### ตัวอักษรพื้นเมืองในซอร์ส

ตัวเลขของระบบที่รองรับใดๆ เป็นตัวอักษรที่ถูกต้องในช่วง ตัวดำเนินการโมดูโล การเปรียบเทียบ:

```zymbol
#๐๙#

@ ช:๑..๒๐ {
    ? ช % ๑๕ == ๐ { >> "FizzBuzz" ¶ }
    _? ช % ๓  == ๐ { >> "Fizz" ¶ }
    _? ช % ๕  == ๐ { >> "Buzz" ¶ }
    _ { >> ช ¶ }
}
```

### ตัวอักษรบูลีนในระบบใดก็ได้

`#` + ตัวเลข `๐` หรือ `๑` จากบล็อกที่รองรับใดๆ คือตัวอักษรบูลีนที่ถูกต้อง:

```zymbol
#٠٩#
نشط = #١        // เหมือน #๑
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **เป็น ASCII เสมอ** `#๐` (เท็จ) แตกต่างจาก `๐` (จำนวนเต็มศูนย์) ในทุกระบบ
