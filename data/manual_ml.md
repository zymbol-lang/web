# Zymbol-Lang മാനുവൽ

**Zymbol-Lang** ഒരു ചിഹ്ന പ്രോഗ്രാമിംഗ് ഭാഷയാണ്. കീവേഡുകൾ ഇല്ല — എല്ലാം ഒരു ചിഹ്നമാണ്. ഏത് മനുഷ്യ ഭാഷയിലും ഒരേ രീതിയിൽ പ്രവർത്തിക്കുന്നു.

- `if`, `while`, `return` ഇല്ല — കേവലം ചിഹ്നങ്ങൾ `?`, `@`, `<~`
- പൂർണ Unicode — ഏത് ഭാഷയിലും അല്ലെങ്കിൽ ഇമോജിയിലും ഐഡന്റിഫയറുകൾ 👋
- മനുഷ്യ-ഭാഷ നിഷ്പക്ഷം — കോഡ് എല്ലാ ഭാഷകളിലും ഒരേ ആണ്

## ചരങ്ങളും സ്ഥിരങ്ങളും
(Variables & Constants)

```zymbol
ഇ = ൧൦              // ചരം (മാറ്റാവുന്നത്)
PI := ൩.൧൪൧൫൯       // സ്ഥിരം — പുനർനിയോഗം ഒരു റൺടൈം പിഴവാണ്
പേര് = "അലീസ്"
സക്രിയം = #൧       // ബൂലിയൻ ശരി
👋 := "ഹലോ"
```

```zymbol
ഇ = ൧൦
ഇ += ൫    // ൧൫
ഇ -= ൩    // ൧൨
ഇ *= ൨    // ൨൪
ഇ /= ൩    // ൮
ഇ %= ൩    // ൨
ഇ ^= ൨    // ൪
ഇ++       // ൫
ഇ--       // ൪
```

## ഡാറ്റ തരങ്ങൾ
(Data Types)

| തരം | ലിറ്ററൽ | `#?` ടാഗ് | കുറിപ്പ് |
|-----|---------|-----------|---------|
| Int | `൪൨`, `-൭` | `###` | 64-bit signed |
| Float | `൩.൧൪`, `൧.൫e൧൦` | `##.` | ശാസ്ത്രീയ നൊട്ടേഷൻ ശരിയാണ് |
| String | `"വാചകം"` | `##"` | ഇന്റർപോളേഷൻ: `"ഹലോ {പേര്}"` |
| Char | `'A'` | `##'` | ഒറ്റ Unicode അക്ഷരം |
| Bool | `#൧`, `#൦` | `##?` | സംഖ്യ അല്ല — `#൧ ≠ ൧` |
| Array | `[൧, ൨, ൩]` | `##]` | ഒരേ തരം ഘടകങ്ങൾ |
| Tuple | `(a, b)` | `##)` | സ്ഥാനം അടിസ്ഥാനം |
| Named Tuple | `(x: ൧, y: ൨)` | `##)` | നാമകരണ ഫീൽഡുകൾ |

```zymbol
// തരം പരിശോധന — (തരം, അക്കങ്ങൾ, മൂല്യം) തിരിച്ചു നൽകുന്നു
meta = ൪൨#?
>> meta ¶         // → (###, ൨, ൪൨)
t = meta[൦]
>> t ¶            // → ###
```

## ഔട്ട്പുട്ടും ഇൻപുട്ടും
(Output & Input)

```zymbol
>> "ഹലോ" ¶                          // ¶ അല്ലെങ്കിൽ \\ പ്രത്യക്ഷ പുതിയ വരി
>> "a=" ഇ " b=" ന ¶                  // ജക്സ്റ്റപോസിഷൻ — ഒന്നിലധികം മൂല്യങ്ങൾ
>> (ശ്രേണി$#) ¶                      // പോസ്റ്റ്ഫിക്സ് ഓപ്പറേറ്ററുകൾക്ക് >> ൽ ( ) വേണം

<< പേര്                              // ചരത്തിലേക്ക് വായിക്കുക (പ്രോംപ്റ്റ് ഇല്ല)
<< "പേര് നൽകൂ: " പേര്               // പ്രോംപ്റ്റ് സഹിതം
```

> `¶` (സ്പാനിഷ് കീബോർഡിൽ AltGr+R) ഉം `\\` ഉം തുല്യ പുതിയ വരികളാണ്.

## ഓപ്പറേറ്ററുകൾ
(Operators)

```zymbol
// ഗണിതം — അസൈൻമെന്റ് ഉപയോഗിക്കുക; >> ൽ ചില ഓപ്പറേറ്ററുകൾക്ക് പ്രത്യേകതകൾ ഉണ്ട്
ക = ൧൦
ഖ = ൩
r൧ = ക + ഖ    // ൧൩     r൨ = ക - ഖ    // ൭
r൩ = ക * ഖ    // ൩൦     r൪ = ക / ഖ    // ൩  (പൂർണസംഖ്യ ഭാഗഹാരം)
r൫ = ക % ഖ    // ൧      r൬ = ക ^ ഖ    // ൧൦൦൦  (ഘാതം)

// താരതമ്യം
ക == ഖ    // #൦    ക <> ഖ    // #൧    ക < ഖ    // #൦
ക <= ഖ    // #൦   ക > ഖ     // #൧    ക >= ഖ   // #൧

// ലോജിക്കൽ
#൧ && #൦    // #൦
#൧ || #൦    // #൧
!#൧         // #൦
```

## സ്ട്രിംഗുകൾ
(Strings)

```zymbol
// മൂന്ന് സംയോജന രൂപങ്ങൾ
പേര് = "അലീസ്"
ന = ൪൨

സന്ദേശം = "ഹലോ ", പേര്, "!"            // കോമ — അസൈൻമെന്റിൽ
>> "ഹലോ " പേര് " നിങ്ങൾക്ക് " ന ¶      // ജക്സ്റ്റപോസിഷൻ — >> ൽ
വിവരണം = "ഹലോ {പേര്}, നിങ്ങൾക്ക് {ന}"  // ഇന്റർപോളേഷൻ — എവിടെയും
```

```zymbol
s = "ഹലോ വേൾഡ്"
len = s$#                  // നീളം
sub = s$[൦..൪]             // "ഹലോ"  (അവസാനം ഒഴിവാക്കും)
has = s$? "വേൾഡ്"          // #൧
parts = "a,b,c,d" / ','    // [a, b, c, d]
rep = s$~~["l":"L"]        // മാറ്റിസ്ഥാപനം
rep൧ = s$~~["l":"L":൧]     // ആദ്യ N മാറ്റിസ്ഥാപനങ്ങൾ
```

> `+` സംഖ്യകൾക്ക് മാത്രം. സ്ട്രിംഗിന് `,`, ജക്സ്റ്റപോസിഷൻ, അല്ലെങ്കിൽ ഇന്റർപോളേഷൻ ഉപയോഗിക്കുക.

## നിയന്ത്രണ പ്രവാഹം
(Control Flow)

```zymbol
ഇ = ൭

? ഇ > ൦ { >> "ധനാത്മകം" ¶ }

? ഇ > ൧൦൦ {
    >> "വലുത്" ¶
} _? ഇ > ൦ {
    >> "ധനാത്മകം" ¶
} _? ഇ == ൦ {
    >> "പൂജ്യം" ¶
} _ {
    >> "ഋണാത്മകം" ¶
}
```

> `{ }` ബ്രേസുകൾ ഒറ്റ പ്രസ്താവനയ്ക്കും **നിർബന്ധമാണ്**.

## മാച്ച്
(Match)

```zymbol
// റേഞ്ചുകൾ
സ്കോർ = ൮൫
ഗ്രേഡ് = ?? സ്കോർ {
    ൯൦..൧൦൦ : 'A'
    ൮൦..൮൯  : 'B'
    ൭൦..൭൯  : 'C'
    _       : 'F'
}
>> ഗ്രേഡ് ¶    // → B

// സ്ട്രിംഗ്
നിറം = "ചുവപ്പ്"
കോഡ് = ?? നിറം {
    "ചുവപ്പ്"  : "#FF൦൦൦൦"
    "പച്ച"     : "#൦൦FF൦൦"
    _          : "#൦൦൦൦൦൦"
}

// ഗാർഡുകൾ
താപനില = -൫
അവസ്ഥ = ?? താപനില {
    _? താപനില < ൦  : "ഐസ്"
    _? താപനില < ൨൦ : "തണുപ്പ്"
    _? താപനില < ൩൫ : "ചൂട്"
    _              : "വളരെ ചൂട്"
}
>> അവസ്ഥ ¶    // → ഐസ്

// പ്രസ്താവന രൂപം (ബ്ലോക്ക് ശാഖകൾ)
?? ന {
    ൦       : { >> "പൂജ്യം" ¶ }
    _? ന < ൦: { >> "ഋണാത്മകം" ¶ }
    _       : { >> "ധനാത്മകം" ¶ }
}
```

## ലൂപ്പുകൾ
(Loops)

```zymbol
@ ഇ:൦..൪  { >> ഇ " " }        // ഉൾക്കൊള്ളുന്ന റേഞ്ച്:  ൦ ൧ ൨ ൩ ൪
@ ഇ:൧..൯:൨ { >> ഇ " " }       // ഘട്ടം സഹിതം:           ൧ ൩ ൫ ൭ ൯
@ ഇ:൫..൦:൧ { >> ഇ " " }       // വിപരീതം:               ൫ ൪ ൩ ൨ ൧ ൦

ന = ൧
@ ന <= ൬൪ { ന *= ൨ }
>> ന ¶                        // → ൧൨൮  (while)

ഫലങ്ങൾ = ["ആപ്പിൾ", "പിയർ", "മുന്തിരി"]
@ ഫ:ഫലങ്ങൾ { >> ഫ ¶ }         // ശ്രേണിയിൽ for-each

@ ക:"ഹലോ" { >> ക "-" }
>> ¶                          // → ഹ-ല-ോ-  (സ്ട്രിംഗ് അക്ഷരങ്ങളിലൂടെ)

@ ഇ:൧..൧൦ {
    ? ഇ % ൨ == ൦ { @> }       // @> തുടരുക
    ? ഇ > ൭ { @! }             // @! നിർത്തുക
    >> ഇ " "
}
>> ¶                          // → ൧ ൩ ൫ ൭

// അനന്ത ലൂപ്പ്
i = ൦
@ {
    i++
    ? i >= ൫ { @! }
    >> i " "
}
>> ¶                          // → ൧ ൨ ൩ ൪

// ലേബൽ ചെയ്ത ലൂപ്പ് (നെസ്റ്റഡ് ബ്രേക്ക്)
count = ൦
@ @outer {
    count++
    ? count >= ൩ { @! outer }
}
>> count ¶                    // → ൩
```

## ഫങ്ഷനുകൾ
(Functions)

```zymbol
കൂട്ടുക(ക, ഖ) { <~ ക + ഖ }
>> കൂട്ടുക(൩, ൪) ¶    // → ൭

ഘടകം(ന) {
    ? ന <= ൧ { <~ ൧ }
    <~ ന * ഘടകം(ന - ൧)
}
>> ഘടകം(൫) ¶    // → ൧൨൦
```

ഫങ്ഷനുകൾക്ക് **ഒറ്റപ്പെട്ട സ്കോപ്പ്** ഉണ്ട് — ബാഹ്യ ചരങ്ങൾ വായിക്കാൻ കഴിയില്ല. കോളർ ചരങ്ങൾ മാറ്റാൻ ഔട്ട്പുട്ട് പാരാമീറ്ററുകൾ `<~` ഉപയോഗിക്കുക:

```zymbol
swap(ക<~, ഖ<~) {
    tmp = ക
    ക = ഖ
    ഖ = tmp
}
ഇ = ൧൦
യ = ൨൦
swap(ഇ, യ)
>> "ഇ=" ഇ " യ=" യ ¶    // → ഇ=൨൦ യ=൧൦
```

> നാമകരണ ഫങ്ഷനുകൾ ഫസ്റ്റ്-ക്ലാസ് അല്ല. ആർഗ്യുമെന്റ് ആയി കൈമാറാൻ പൊതിഞ്ഞുവെക്കുക: `x -> കൂട്ടുക(x)`.

## ലാംഡകളും ക്ലോഷറുകളും
(Lambdas & Closures)

```zymbol
ഇരട്ടി = ഇ -> ഇ * ൨
തുക = (ക, ഖ) -> ക + ഖ
>> ഇരട്ടി(൫) ¶    // → ൧൦
>> തുക(൩, ൭) ¶    // → ൧൦

// ബ്ലോക്ക് ലാംഡ
തരംതിരിക്കൽ = ഇ -> {
    ? ഇ > ൦ { <~ "ധനാത്മകം" }
    _? ഇ < ൦ { <~ "ഋണാത്മകം" }
    <~ "പൂജ്യം"
}

// ക്ലോഷർ — ബാഹ്യ സ്കോപ്പ് പിടിക്കുന്നു
ഘടകം = ൩
മൂന്നിരട്ടി = ഇ -> ഇ * ഘടകം
>> മൂന്നിരട്ടി(൭) ¶    // → ൨൧

// ഫാക്ടറി
ആഡർ_ഉണ്ടാക്കുക(ന) { <~ ഇ -> ഇ + ന }
പത്ത്_കൂട്ടുക = ആഡർ_ഉണ്ടാക്കുക(൧൦)
>> പത്ത്_കൂട്ടുക(൫) ¶    // → ൧൫

// ശ്രേണിയിൽ
ക്രിയകൾ = [ഇ -> ഇ+൧, ഇ -> ഇ*൨, ഇ -> ഇ*ഇ]
>> ക്രിയകൾ[൨](൫) ¶    // → ൨൫
```

## ശ്രേണികൾ
(Arrays)

Arrays are **mutable** and hold elements of the **same type**.

```zymbol
ശ്രേണി = [൧, ൨, ൩, ൪, ൫]

ശ്രേണി[൦]          // ൧ — ആക്സസ് (൦ മുതൽ ഇൻഡക്സ്)
ശ്രേണി[-൧]         // ൫ — നെഗറ്റീവ് ഇൻഡക്സ് (അവസാനം)
ശ്രേണി$#           // ൫ — നീളം (>> ൽ (ശ്രേണി$#) ഉപയോഗിക്കുക)

ശ്രേണി = ശ്രേണി$+ ൬            // ചേർക്കുക → [൧,൨,൩,൪,൫,൬]
arr൨ = ശ്രേണി$+[൨] ൯൯           // ഇൻഡക്സ് ൨ ൽ തിരുകുക
arr൩ = ശ്രേണി$- ൩               // മൂല്യത്തിന്റെ ആദ്യ ഉദാഹരണം നീക്കംചെയ്യുക
arr൪ = ശ്രേണി$-- ൩              // എല്ലാ ഉദാഹരണങ്ങളും നീക്കംചെയ്യുക
arr൫ = ശ്രേണി$-[൦]              // ഇൻഡക്സിൽ നീക്കംചെയ്യുക
arr൬ = ശ്രേണി$-[൧..൩]           // റേഞ്ച് നീക്കംചെയ്യുക (അവസാനം ഒഴിവാക്കും)

has = ശ്രേണി$? ൩                // #൧ — ഉൾക്കൊള്ളുന്നു
pos = ശ്രേണി$?? ൩               // [൨] — മൂല്യത്തിന്റെ എല്ലാ ഇൻഡക്സുകളും
sl = ശ്രേണി$[൦..൩]              // [൧,൨,൩] — സ്ലൈസ് (അവസാനം ഒഴിവാക്കും)
sl൨ = ശ്രേണി$[൦:൩]              // [൧,൨,൩] — എണ്ണം അടിസ്ഥാനമാക്കിയ സിന്റാക്സ്

asc = ശ്രേണി$^+                 // ആരോഹണ ക്രമം (primitives മാത്രം)
desc = ശ്രേണി$^-                // അവരോഹണ ക്രമം (primitives മാത്രം)

// നാമകരണ/സ്ഥാനം ടുപ്പിൾ ശ്രേണികൾ — $^ comparator lambda സഹിതം
db = [(name: "കാർള", age: ൨൮), (name: "ആന", age: ൨൫), (name: "ബോബ്", age: ൩൦)]
by_age  = db$^ (ക, ഖ -> ക.age < ഖ.age)    // പ്രായം അനുസരിച്ച് ആരോഹണം (<)
by_name = db$^ (ക, ഖ -> ക.name > ഖ.name)  // പേര് അനുസരിച്ച് അവരോഹണം (>)
>> by_age[൦].name ¶     // → ആന
>> by_name[൦].name ¶    // → കാർള

// Direct element update (arrays only)
ശ്രേണി[൧] = ൯൯              // അസൈൻ ചെയ്യുക
ശ്രേണി[൦] += ൫              // compound: +=  -=  *=  /=  %=  ^=

// Functional update — returns a new array; original unchanged
arr൨ = ശ്രേണി[൧]$~ ൯൯
```

> എല്ലാ ശേഖരണ ഓപ്പറേറ്ററുകളും **പുതിയ ശ്രേണി** തിരിച്ചു നൽകുന്നു. തിരിച്ച് അസൈൻ ചെയ്യുക: `ശ്രേണി = ശ്രേണി$+ ൪`.
> ഓപ്പറേറ്ററുകൾ ചെയ്ൻ ചെയ്യാൻ കഴിയില്ല — ഇടക്ക് അസൈൻമെന്റ് ഉപയോഗിക്കുക.
> `$^+` / `$^-` **primitive ശ്രേണികൾ** (സംഖ്യകൾ, സ്ട്രിംഗ്) sort ചെയ്യുന്നു. ടുപ്പിൾ ശ്രേണികൾക്ക് `$^` ഒരു comparator lambda സഹിതം ഉപയോഗിക്കുക — ദിശ lambda ൽ encode ചെയ്തിരിക്കുന്നു (`<` = ആരോഹണം, `>` = അവരോഹണം).

**Value semantics** — assigning an array to another variable creates an independent copy:

```zymbol
a = [1, 2, 3]
b = a
a[0] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b is unaffected
```

```zymbol
// നെസ്റ്റഡ് ശ്രേണികൾ
matrix = [[൧,൨,൩],[൪,൫,൬],[൭,൮,൯]]
>> matrix[൧][൨] ¶    // → ൬
```

## ഡിസ്ട്രക്ചറിംഗ്
(Destructuring)

```zymbol
// ശ്രേണി
arr = [൧൦, ൨൦, ൩൦, ൪൦, ൫൦]
[ക, ഖ, ഗ] = arr              // ക=൧൦  ഖ=൨൦  ഗ=൩൦
[first, *rest] = arr         // first=൧൦  rest=[൨൦,൩൦,൪൦,൫൦]
[x, _, z] = [൧, ൨, ൩]        // _ ഒഴിവാക്കുന്നു

// സ്ഥാനം ടുപ്പിൾ
point = (൧൦൦, ൨൦൦)
(px, py) = point             // px=൧൦൦  py=൨൦൦

// നാമകരണ ടുപ്പിൾ
വ്യക്തി = (name: "ആന", age: ൨൫, city: "കൊച്ചി")
(name: ന, age: ക) = വ്യക്തി   // ന="ആന"  ക=൨൫
```

## ടപ്പിളുകൾ
(Tuples)

Tuples are **immutable** ordered containers that can hold values of **different types**. Unlike arrays, elements cannot be changed after creation.

```zymbol
// സ്ഥാനം
point = (൧൦, ൨൦)
>> point[൦] ¶    // → ൧൦

data = (42, "hello", #1, 3.14)
>> data[2] ¶     // → #1

// നാമകരണ
വ്യക്തി = (name: "അലീസ്", age: ൨൫)
>> വ്യക്തി.name ¶    // → അലീസ്
>> വ്യക്തി[൦] ¶      // → അലീസ്  (ഇൻഡക്സും പ്രവർത്തിക്കുന്നു)

// നെസ്റ്റഡ്
pos = (x: ൧൦, y: ൨൦)
p = (pos: pos, label: "ഉത്ഭവം")
>> p.pos.x ¶        // → ൧൦
```

**Immutability** — any attempt to modify a tuple element is a runtime error:

```zymbol
t = (10, 20, 30)
// t[0] = 99    // ❌ runtime error: tuples are immutable
// t[0] += 5    // ❌ same error
```

To derive a modified value use `$~` (functional update) — returns a **new** tuple:

```zymbol
t = (10, 20, 30)
t2 = t[1]$~ 999
>> t ¶     // → (10, 20, 30)   ← original unchanged
>> t2 ¶    // → (10, 999, 30)

// Named tuple — rebuild explicitly
വ്യക്തി = (name: "Alice", age: 25)
മൂപ്പൻ = (name: വ്യക്തി.name, age: 26)
>> വ്യക്തി.age ¶    // → 25
>> മൂപ്പൻ.age ¶     // → 26
```

## ഉയർന്ന-ക്രമ ഫങ്ഷനുകൾ
(Higher-Order Functions)

> HOF ഓപ്പറേറ്ററുകൾക്ക് **ഇൻലൈൻ ലാംഡ** വേണം — lambda ചരങ്ങൾ നേരിട്ട് കൈമാറിയാൽ പ്രവർത്തിക്കില്ല.

```zymbol
സംഖ്യകൾ = [൧, ൨, ൩, ൪, ൫, ൬, ൭, ൮, ൯, ൧൦]

doubled  = സംഖ്യകൾ$> (ഇ -> ഇ * ൨)                // map  → [൨,൪,൬…൨൦]
evens    = സംഖ്യകൾ$| (ഇ -> ഇ % ൨ == ൦)           // filter → [൨,൪,൬,൮,൧൦]
total    = സംഖ്യകൾ$< (൦, (acc, ഇ) -> acc + ഇ)     // reduce → ൫൫

// ഇടക്ക് വഴി ചെയ്ൻ
step൧ = സംഖ്യകൾ$| (ഇ -> ഇ > ൩)
step൨ = step൧$> (ഇ -> ഇ * ഇ)
>> step൨ ¶    // → [൧൬, ൨൫, ൩൬, ൪൯, ൬൪, ൮൧, ൧൦൦]

// HOF ൽ നാമകരണ ഫങ്ഷനുകൾ — lambda ൽ പൊതിഞ്ഞുവെക്കുക
ഇരട്ടി(ഇ) { <~ ഇ * ൨ }
r = സംഖ്യകൾ$> (ഇ -> ഇരട്ടി(ഇ))    // ✅
```

## പൈപ്പ് ഓപ്പറേറ്റർ
(Pipe Operator)

RHS ന് pipe ചെയ്ത മൂല്യത്തിനായി `_` placeholder ആവശ്യമാണ്:

```zymbol
ഇരട്ടി = ഇ -> ഇ * ൨
കൂട്ടുക = (ക, ഖ) -> ക + ഖ
inc = ഇ -> ഇ + ൧

൫ |> ഇരട്ടി(_)        // → ൧൦
൧൦ |> കൂട്ടുക(_, ൫)   // → ൧൫
൫ |> കൂട്ടുക(൨, _)    // → ൭

// ചെയ്ൻ
r = ൫ |> ഇരട്ടി(_) |> inc(_) |> ഇരട്ടി(_)
>> r ¶    // → ൨൨  (൫→൧൦→൧൧→൨൨)
```

## സംഖ്യാ സമ്പ്രദായങ്ങൾ

Zymbol **69 Unicode അക്ക ബ്ലോക്കുകളിൽ** സംഖ്യകൾ പ്രദർശിപ്പിക്കാൻ കഴിയും — മലയാളം, ദേവനാഗരി, അറബിക്-ഇൻഡിക്, തായ്, ക്ലിംഗൺ pIqaD, തുടങ്ങിയവ. സക്രിയ മോഡ് `>>` ഔട്ട്‌പുട്ടിൽ മാത്രം പ്രഭാവം ചെലുത്തുന്നു; ആഭ്യന്തര ഗണന എല്ലായ്‌പ്പോഴും ദ്വൈത.

### ഒരു സിസ്റ്റം സക്രിയമാക്കുക

ആവശ്യമുള്ള ലിപിയിലെ `൦`, `൯` അക്കങ്ങൾ `#…#` ഇടയ്ക്ക് എഴുതുക:

```zymbol
#൦൯#    // ML script (൦..൯)
#٠٩#    // Arabic-Indic  (U+0660–U+0669)
#09#    // reset to ASCII
```

### ഔട്ട്‌പുട്ടും ബൂലിയനും

```zymbol
x = ൪൨
>> x ¶          // → ൪൨   (ASCII ഡിഫോൾട്ട്)

#൦൯#
>> x ¶          // → ൪൨
>> ൩.൧൪ ¶       // → ൩.൧൨൩൪   (ദശാംശ ബിന്ദു എല്ലായ്‌പ്പോഴും ASCII)
>> ൧ + ൨ ¶      // → ൩

>> #൧ ¶         // → #൧   (സത്യം)
>> #൦ ¶         // → #൦   (തെറ്റ് — ൦ ൽ നിന്ന് വ്യത്യസ്തം)

x = ൨൮ > ൪
>> x ¶          // → #൧   (താരതമ്യ ഫലം)
```

### ഉറവിടത്തിൽ നേറ്റിവ് ലിറ്ററലുകൾ

പിന്തുണയ്ക്കുന്ന ഏതെങ്കിലും സിസ്റ്റത്തിലെ അക്കങ്ങൾ ശ്രേണികൾ, മൊഡ്യൂളോ, താരതമ്യങ്ങളിൽ സാധുതയുള്ള ലിറ്ററലുകൾ:

```zymbol
#൦൯#

@ ഇ:൧..൧൫ {
    ? ഇ % ൧൫ == ൦ { >> "FizzBuzz" ¶ }
    _? ഇ % ൩  == ൦ { >> "Fizz" ¶ }
    _? ഇ % ൫  == ൦ { >> "Buzz" ¶ }
    _ { >> ഇ ¶ }
}
```

### ഏത് സിസ്റ്റത്തിലും ബൂലിയൻ ലിറ്ററലുകൾ

ഏതെങ്കിലും പിന്തുണയ്ക്കുന്ന ബ്ലോക്കിലെ `#` + `൦` അല്ലെങ്കിൽ `൧` അക്കം ഒരു സാധുതയുള്ള ബൂലിയൻ ലിറ്ററൽ:

```zymbol
#٠٩#
نشط = #١        // same as #൧
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **എല്ലായ്‌പ്പോഴും ASCII**. `#൦` (തെറ്റ്) ഏത് സിസ്റ്റത്തിലും `൦` (പൂജ്യം പൂർണ്ണ സംഖ്യ) ൽ നിന്ന് ദൃശ്യപരമായി വ്യത്യസ്തം.

(Data Operators)

```zymbol

v൧ = #|"൪൨"|      // → ൪൨  (Int)

r൨ = #.൨|pi|      // → ൩.൧൪  (൨ ദശാംശ സ്ഥാനത്ത് വൃത്താകൃതി)
r൪ = #.൪|pi|      // → ൩.൧൪൧൬

// സംഖ്യ ഫോർമാറ്റിംഗ്

sci = #^|൧൨൩൪൫.൬൭൮|    // → ൧.൨൩൪൫൬൭൮e൪  (ശാസ്ത്രീയ)

// ബേസ് ലിറ്ററലുകൾ

c = 0o101        // → 'A'  (octal)

bin = 0b|൬൫|     // → "0b1000001"
oct = 0o|൮|      // → "0o10"
dec = 0d|൨൫൫|    // → "0d0255"
```

### v0.0.3 — Unicode സംഖ്യാ സമ്പ്രദായങ്ങളും LSP മെച്ചപ്പെടുത്തലുകളും _(ഏപ്രിൽ 2026)_

- **ചേർത്തു** 69 Unicode അക്ക ബ്ലോക്കുകൾ മോഡ്-സ്വിച്ച് ടോക്കൺ `#d0d9#`-ഉം
- **ചേർത്തു** ഏത് സിസ്റ്റത്തിലും ബൂലിയൻ ലിറ്ററലുകൾ — `#൧` / `#൦`, `#١` / `#٠`, മുതലായവ
- **ചേർത്തു** ക്ലിംഗൺ pIqaD അക്കങ്ങൾ (CSUR PUA U+F8F0–U+F8F9)
- **ചേർത്തു** VM Opcode `SetNumeralMode` — ട്രീ-വാകറുമായി പൂർണ്ണ സമത
- **ചേർത്തു** REPL എക്കോ, വേരിയബിൾ ഡിസ്‌പ്ലേ എന്നിവയിൽ ആക്ടീവ് അക്ക മോഡ് ആദരിക്കുന്നു
- **മാറ്റി** ബൂലിയൻ `>>` ഔട്ട്‌പുട്ടിൽ എല്ലാ മോഡുകളിലും `#` പ്രിഫിക്സ് (`#൦` / `#൧`) ഉൾപ്പെടുത്തി

### v0.0.2_01 — ഓപ്പറേറ്റർ പേരുമാറ്റം _(30 മാർച്ച് 2026)_

- **മാറ്റി** `c|..|` → `#,|..|` ഉം `e|..|` → `#^|..|` ഉം — `#` പ്രിഫിക്സ് കുടുംബവുമായി പൊരുത്തം
- **ചേർത്തു** ഒഴിച്ചുകൊടുക്കൽ ഉപനാമം: മൊഡ്യൂൾ അംഗങ്ങളെ വ്യത്യസ്ത പേരിൽ വീണ്ടും-ഒഴിച്ചുകൊടുക്കൽ

### v0.0.2 — ശേഖരണ പുനർരൂപകൽപ്പന ഇൻസ്‌റ്റാളറുകൾ _(24 മാർച്ച് 2026)_

- **ചേർത്തു** അറേകൾക്കും സ്ട്രിംഗുകൾക്കും ഏകീകൃത `$` ഓപ്പറേറ്റർ കുടുംബം (`$#`, `$+`, `$?`, `$-`, `$[..]`)
- **ചേർത്തു** അറേകൾ, ടപ്പിൾ, നേമ്ഡ് ടപ്പിൾ എന്നിവ ഡിസ്ട്രക്ചർ ചെയ്യൽ
