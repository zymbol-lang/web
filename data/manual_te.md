# Zymbol-Lang యొక్క సంక్షిప్త మార్గదర్శి

**Zymbol-Lang** ఒక సంకేతాత్మక ప్రోగ్రామింగ్ భాష. ఇందులో ఏ కీవర్డ్ లేదు — అన్నీ సంకేతాలే. ఇది ఏ మానవ భాషలోనైనా ఒకే విధంగా పనిచేస్తుంది.

- కీవర్డ్‌లు లేవు (`if`, `while`, `return` అనేవి లేవు — కేవలం సంకేతాలు `?`, `@`, `<~`)
- పూర్తి Unicode — ఏ భాషలో లేదా ఇమోజీలో అయినా గుర్తింపుదారులు 👋
- మానవ-భాష తటస్థ — కోడ్ అన్ని భాషల్లో ఒకేలా ఉంటుంది

## డేటా రకాలు

| రకం | లిటరల్ | `#?` ట్యాగ్ | గమనికలు |
|-----|--------|------------|---------|
| పూర్ణాంకం (Int) | `౪౨`, `-౭` | `###` | 64-bit సంతకం చేయబడింది |
| దశాంశం (Float) | `౩.౧౪`, `౧.౫e౧౦` | `##.` | శాస్త్రీయ సంకేతనం అనుమతిస్తుంది |
| స్ట్రింగ్ (String) | `"text"` | `##"` | ఇంటర్‌పోలేషన్: `"హలో {పేరు}"` |
| అక్షరం (Char) | `'A'` | `##'` | ఒక Unicode అక్షరం |
| బూలియన్ (Bool) | `#౧`, `#౦` | `##?` | సంఖ్యాపరంగా కాదు — `#౧ ≠ ౧` |
| జాబితా (Array) | `[౧, ౨, ౩]` | `##]` | సజాతి మూలకాలు |
| టుపుల్ (Tuple) | `(a, b)` | `##)` | స్థాన ఆధారిత |
| నామాంకిత టుపుల్ | `(x: ౧, y: ౨)` | `##)` | నామాంకిత ఫీల్డ్‌లు |

```zymbol
// రకం అంతర్దర్శనం — (రకం, అంకెలు, విలువ) అందిస్తుంది
meta = ౪౨#?
>> meta ¶         // → (###, ౨, ౪౨)
t = meta[౦]
>> t ¶            // → ###
```

## అవుట్‌పుట్ మరియు ఇన్‌పుట్

```zymbol
>> "నమస్కారం" ¶                      // ¶ లేదా \\ స్పష్టమైన కొత్త వరుస
>> "a=" a " b=" b ¶                  // juxtaposition — అనేక విలువలు
>> (arr$#) ¶                         // పోస్ట్‌ఫిక్స్ ఆపరేటర్‌కు >> లో ( ) అవసరం

<< పేరు                               // ప్రాంప్ట్ లేకుండా చదువుతుంది
<< "మీ పేరు: " పేరు                  // ప్రాంప్ట్‌తో
```

> `¶` (స్పానిష్ కీబోర్డ్‌లో AltGr+R) మరియు `\\` సమానమైన కొత్త వరుసలు.

## ఆపరేటర్లు

```zymbol
// గణితం — నియామకాలు ఉపయోగించండి; కొన్ని ఆపరేటర్లు >> లో నేరుగా విచిత్రంగా ఉంటాయి
a = ౧౦
b = ౩
r౧ = a + b    // ౧౩     r౨ = a - b    // ౭
r౩ = a * b    // ౩౦     r౪ = a / b    // ౩  (పూర్ణాంక భాగహారం)
r౫ = a % b    // ౧      r౬ = a ^ b    // ౧౦౦౦  (ఘాతం)

// పోలిక
a == b    // #౦    a <> b    // #౧    a < b    // #౦
a <= b    // #౦   a > b     // #౧    a >= b   // #౧

// తార్కికం
#౧ && #౦    // #౦
#౧ || #౦    // #౧
!#౧         // #౦
```

## స్ట్రింగ్‌లు

```zymbol
// మూడు అనుసంధాన రూపాలు
పేరు = "ఆలిస్"
న = ౪౨

msg = "హలో ", పేరు, "!"              // కామా — నియామకాల్లో
>> "హలో " పేరు " మీకు " న ¶         // juxtaposition — >> లో
desc = "హలో {పేరు}, మీకు {న} ఉంది"  // ఇంటర్‌పోలేషన్ — ఎక్కడైనా
```

```zymbol
s = "Hello World"
len = s$#                  // ౧౧
sub = s$[౦..౫]             // "Hello"  (చివర మినహాయించబడుతుంది)
has = s$? "World"          // #౧
parts = "a,b,c,d" / ','    // [a, b, c, d]
rep = s$~~["l":"L"]        // "HeLLo WorLd"
rep౧ = s$~~["l":"L":౧]     // "HeLlo World"  (మొదటి N మాత్రమే)
```

> `+` సంఖ్యలకు మాత్రమే. స్ట్రింగ్‌లకు `,`, juxtaposition, లేదా ఇంటర్‌పోలేషన్ ఉపయోగించండి.

## ప్రవాహ నియంత్రణ

```zymbol
x = ౭

? x > ౦ { >> "ధనాత్మక" ¶ }

? x > ౧౦౦ {
    >> "పెద్దది" ¶
} _? x > ౦ {
    >> "ధనాత్మక" ¶
} _? x == ౦ {
    >> "సున్నా" ¶
} _ {
    >> "ఋణాత్మక" ¶
}
```

> `{ }` బ్లాక్‌లు **తప్పనిసరి** ఒక్క స్టేట్‌మెంట్‌కు కూడా.

## లూప్‌లు

```zymbol
@ i:౦..౪  { >> i " " }        // పరిధి సమ్మిళితం:  ౦ ౧ ౨ ౩ ౪
@ i:౧..౯:౨ { >> i " " }       // స్టెప్‌తో:         ౧ ౩ ౫ ౭ ౯
@ i:౫..౦:౧ { >> i " " }       // రివర్స్:           ౫ ౪ ౩ ౨ ౧ ౦

న = ౧
@ న <= ౬౪ { న *= ౨ }
>> న ¶                        // → ౧౨౮  (while)

పళ్ళు = ["ఆపిల్", "నాశపండు", "ద్రాక్ష"]
@ f:పళ్ళు { >> f ¶ }          // జాబితాపై for-each

@ c:"hello" { >> c "-" }
>> ¶                          // → h-e-l-l-o-  (స్ట్రింగ్ అక్షరాలపై)

@ i:౧..౧౦ {
    ? i % ౨ == ౦ { @> }       // @> కొనసాగించు
    ? i > ౭ { @! }             // @! ఆపు
    >> i " "
}
>> ¶                          // → ౧ ౩ ౫ ౭

// అనంత లూప్
i = ౦
@ {
    i++
    ? i >= ౫ { @! }
    >> i " "
}
>> ¶                          // → ౧ ౨ ౩ ౪

// లేబుల్ చేసిన లూప్ (నెస్టెడ్ బ్రేక్)
count = ౦
@ @outer {
    count++
    ? count >= ౩ { @! outer }
}
>> count ¶                    // → ౩
```

## ఫంక్షన్లు

```zymbol
కూడిక(a, b) { <~ a + b }
>> కూడిక(౩, ౪) ¶    // → ౭

గుణకం(న) {
    ? న <= ౧ { <~ ౧ }
    <~ న * గుణకం(న - ౧)
}
>> గుణకం(౫) ¶    // → ౧౨౦
```

ఫంక్షన్లకు **వేరే స్కోప్** ఉంటుంది — అవి బాహ్య వేరియబుల్‌లు చదవలేవు. అవుట్‌పుట్ పారామీటర్‌లు `<~` ఉపయోగించి కాలర్ వేరియబుల్‌లు మార్చవచ్చు:

```zymbol
swap(a<~, b<~) {
    tmp = a
    a = b
    b = tmp
}
x = ౧౦
y = ౨౦
swap(x, y)
>> "x=" x " y=" y ¶    // → x=౨౦ y=౧౦
```

> నామాంకిత ఫంక్షన్‌లు ప్రథమ-తరగతి కాదు. ఆర్గ్యుమెంట్‌గా పాస్ చేయడానికి చుట్టండి: `x -> కూడిక(x)`.

## లాంబ్డా మరియు క్లోజర్లు

```zymbol
double = x -> x * ౨
sum = (a, b) -> a + b
>> double(౫) ¶    // → ౧౦
>> sum(౩, ౭) ¶    // → ౧౦

// బ్లాక్ లాంబ్డా
classify = x -> {
    ? x > ౦ { <~ "ధనాత్మక" }
    _? x < ౦ { <~ "ఋణాత్మక" }
    <~ "సున్నా"
}

// క్లోజర్ — బాహ్య స్కోప్ క్యాప్చర్ చేస్తుంది
factor = ౩
triple = x -> x * factor
>> triple(౭) ¶    // → ౨౧

// ఫ్యాక్టరీ
make_adder(న) { <~ x -> x + న }

>> add౧౦(౫) ¶    // → ౧౫

// జాబితాల్లో
ops = [x -> x+౧, x -> x*౨, x -> x*x]
>> ops[౨](౫) ¶    // → ౨౫
```

## జాబితా

జాబితాలు **మార్చగలిగేవి** (mutable) మరియు **ఒకే రకమైన** అంశాలను కలిగి ఉంటాయి.

```zymbol
arr = [౧, ౨, ౩, ౪, ౫]

arr[౦]          // ౧ — యాక్సెస్ (౦-ఆధారిత)
arr[-౧]         // ౫ — ఋణాత్మక సూచిక (చివరిది)
arr$#           // ౫ — పొడవు (>> లో (arr$#) ఉపయోగించండి)

arr = arr$+ ౬            // జోడించు → [౧,౨,౩,౪,౫,౬]
arr౨ = arr$+[౨] ౯౯       // సూచిక ౨ వద్ద చొప్పించు
arr౩ = arr$- ౩           // విలువ యొక్క మొదటి సంభవాన్ని తీసివేయి
arr౪ = arr$-- ౩          // అన్ని సంభవాలు తీసివేయి
arr౫ = arr$-[౦]          // సూచిక వద్ద తీసివేయి
arr౬ = arr$-[౧..౩]       // పరిధి తీసివేయి (చివర మినహాయించబడుతుంది)

has = arr$? ౩            // #౧ — కలిగి ఉంది
pos = arr$?? ౩           // [౨] — విలువ యొక్క అన్ని సూచికలు
sl = arr$[౦..౩]          // [౧,౨,౩] — స్లైస్ (చివర మినహాయించబడుతుంది)
sl౨ = arr$[౦:౩]          // [౧,౨,౩] — గణన-ఆధారిత సింటాక్స్

asc = arr$^+             // ఆరోహణ క్రమంలో వరుస  (ప్రిమిటివ్‌లు మాత్రమే)
desc = arr$^-            // అవరోహణ క్రమంలో వరుస (ప్రిమిటివ్‌లు మాత్రమే)

// నామాంకిత/స్థాన టుపుల్ జాబితాలు — కంపారేటర్ లాంబ్డాతో $^ ఉపయోగించండి
db = [(name: "Carla", age: ౨౮), (name: "Ana", age: ౨౫), (name: "Bob", age: ౩౦)]
by_age  = db$^ (a, b -> a.age < b.age)    // వయసు ప్రకారం ఆరోహణ  (<)
by_name = db$^ (a, b -> a.name > b.name)  // పేరు ప్రకారం అవరోహణ (>)
>> by_age[౦].name ¶     // → Ana
>> by_name[౦].name ¶    // → Carla

// నేరుగా అంశ నవీకరణ (జాబితాలు మాత్రమే)
arr[౧] = ౯౯              // నియమించు
arr[౦] += ౫              // compound: +=  -=  *=  /=  %=  ^=

// ఫంక్షనల్ నవీకరణ — కొత్త జాబితా అందిస్తుంది; అసలు మారదు
arr౨ = arr[౧]$~ ౯౯
```

> అన్ని సేకరణ ఆపరేటర్లు **కొత్త జాబితాను** అందిస్తాయి. తిరిగి నియమించండి: `arr = arr$+ ౪`.
> ఆపరేటర్లను చైన్ చేయలేరు — మధ్యంతర నియామకాలు ఉపయోగించండి.
> `$^+` / `$^-` **ప్రిమిటివ్ జాబితాలు** (సంఖ్యలు, స్ట్రింగ్‌లు) వరుసపెట్టుతాయి. టుపుల్ జాబితాలకు కంపారేటర్ లాంబ్డాతో `$^` ఉపయోగించండి — దిశ లాంబ్డాలో ఎన్కోడ్ చేయబడుతుంది (`<` = ఆరోహణ, `>` = అవరోహణ).

**విలువ సెమాంటిక్స్** — జాబితాను మరొక వేరియబుల్‌కు నియమించడం స్వతంత్ర కాపీని సృష్టిస్తుంది:

```zymbol
a = [౧, ౨, ౩]
b = a
a[౦] = ౯౯
>> a ¶    // → [౯౯, ౨, ౩]
>> b ¶    // → [౧, ౨, ౩]   ← b మారలేదు
```

```zymbol
// నెస్టెడ్ జాబితాలు
matrix = [[౧,౨,౩],[౪,౫,౬],[౭,౮,౯]]
>> matrix[౧][౨] ¶    // → ౬
```

## విభజన

```zymbol
// జాబితా
arr = [౧౦, ౨౦, ౩౦, ౪౦, ౫౦]
[a, b, c] = arr              // a=౧౦  b=౨౦  c=౩౦
[first, *rest] = arr         // first=౧౦  rest=[౨౦,౩౦,౪౦,౫౦]
[x, _, z] = [౧, ౨, ౩]        // _ విస్మరిస్తుంది

// స్థాన టుపుల్
point = (౧౦౦, ౨౦౦)
(px, py) = point             // px=౧౦౦  py=౨౦౦

// నామాంకిత టుపుల్
వ్యక్తి = (name: "Ana", age: ౨౫, city: "Madrid")
(name: న, age: వ) = వ్యక్తి  // న="Ana"  వ=౨౫
```

## టుపుల్

టుపుల్‌లు **మార్చలేని** (immutable) క్రమబద్ధ కంటైనర్లు, అవి **వేర్వేరు రకాల** విలువలను కలిగి ఉండవచ్చు. జాబితాల వలె కాకుండా, సృష్టించిన తర్వాత అంశాలను మార్చలేరు.

```zymbol
// స్థాన
point = (౧౦, ౨౦)
>> point[౦] ¶    // → ౧౦

డేటా = (42, "నమస్కారం", #1, 3.14)
>> డేటా[౨] ¶     // → #1

// నామాంకిత
వ్యక్తి = (name: "Alice", age: ౨౫)
>> వ్యక్తి.name ¶    // → Alice
>> వ్యక్తి[౦] ¶      // → Alice  (సూచిక కూడా పనిచేస్తుంది)

// నెస్టెడ్
pos = (x: ౧౦, y: ౨౦)
p = (pos: pos, label: "origin")
>> p.pos.x ¶        // → ౧౦
```

**మార్చలేమి (Inmutabilidad)** — టుపుల్ అంశాన్ని మార్చే ప్రయత్నం రన్‌టైమ్ లోపం:

```zymbol
t = (౧౦, ౨౦, ౩౦)
// t[౦] = ౯౯    // ❌ రన్‌టైమ్ లోపం: టుపుల్‌లు మార్చలేనివి
// t[౦] += ౫    // ❌ అదే లోపం
```

మార్చిన విలువను పొందేందుకు `$~` (ఫంక్షనల్ నవీకరణ) ఉపయోగించండి — **కొత్త టుపుల్** అందిస్తుంది:

```zymbol
t = (౧౦, ౨౦, ౩౦)
t2 = t[౧]$~ 999
>> t ¶     // → (౧౦, ౨౦, ౩౦)   ← అసలు మారలేదు
>> t2 ¶    // → (౧౦, 999, ౩౦)

// నామాంకిత టుపుల్ — స్పష్టంగా పునర్నిర్మించు
వ్యక్తి = (name: "Alice", age: ౨౫)
వ్యక్తి2  = (name: వ్యక్తి.name, age: ౨౬)
>> వ్యక్తి.age ¶    // → ౨౫
>> వ్యక్తి2.age ¶   // → ౨౬
```

## ఉన్నత-క్రమ ఫంక్షన్లు

> HOF ఆపరేటర్‌లకు **ఇన్‌లైన్ లాంబ్డా** అవసరం — లాంబ్డా వేరియబుల్‌లు నేరుగా పాస్ చేయడం పనిచేయదు.

```zymbol
సంఖ్యలు = [౧, ౨, ౩, ౪, ౫, ౬, ౭, ౮, ౯, ౧౦]

doubled  = సంఖ్యలు$> (x -> x * ౨)                // map  → [౨,౪,౬…౨౦]
evens    = సంఖ్యలు$| (x -> x % ౨ == ౦)           // filter → [౨,౪,౬,౮,౧౦]
total    = సంఖ్యలు$< (౦, (acc, x) -> acc + x)     // reduce → ౫౫

// మధ్యంతరాల ద్వారా చైన్
step౧ = సంఖ్యలు$| (x -> x > ౩)
step౨ = step౧$> (x -> x * x)
>> step౨ ¶    // → [౧౬, ౨౫, ౩౬, ౪౯, ౬౪, ౮౧, ౧౦౦]

// HOF లో నామాంకిత ఫంక్షన్లు — లాంబ్డాలో చుట్టండి
double(x) { <~ x * ౨ }
r = సంఖ్యలు$> (x -> double(x))    // ✅
```

## పైప్ ఆపరేటర్

RHS ఎల్లప్పుడూ పైప్ చేయబడిన విలువకు ప్లేస్‌హోల్డర్‌గా `_` అవసరం:

```zymbol
double = x -> x * ౨
add = (a, b) -> a + b
inc = x -> x + ౧

౫ |> double(_)        // → ౧౦
౧౦ |> add(_, ౫)       // → ౧౫
౫ |> add(౨, _)        // → ౭

// చైన్ చేసినది
r = ౫ |> double(_) |> inc(_) |> double(_)
>> r ¶    // → ౨౨  (౫→౧౦→౧౧→౨౨)
```

## సంఖ్యా పద్ధతులు

Zymbol **69 Unicode అంక నిలయాల్లో** సంఖ్యలు ప్రదర్శించగలదు — తెలుగు, దేవనాగరి, అరబిక్-ఇండిక్, థాయ్, క్లింగాన్ pIqaD మరియు మరిన్ని. క్రియాశీల మోడ్ `>>` అవుట్‌పుట్‌పై మాత్రమే ప్రభావం చూపుతుంది; అంతర్గత గణన ఎల్లప్పుడూ ద్విమాన.

### ఒక వ్యవస్థను సక్రియం చేయండి

కావలసిన లిపి యొక్క `౦` మరియు `౯` అంకెలను `#…#` మధ్య రాయండి:

```zymbol
#౦౯#    // TE script (౦..౯)
#٠٩#    // Arabic-Indic  (U+0660–U+0669)
#09#    // reset to ASCII
```

### అవుట్‌పుట్ మరియు బూలియన్

```zymbol
x = ౪౨
>> x ¶          // → ౪౨   (ASCII డిఫాల్ట్)

#౦౯#
>> x ¶          // → ౪౨
>> ౩.౧౪ ¶       // → ౩.౧౨౩౪   (దశాంశ బిందువు ఎల్లప్పుడూ ASCII)
>> ౧ + ౨ ¶      // → ౩

>> #౧ ¶         // → #౧   (నిజం)
>> #౦ ¶         // → #౦   (తప్పు — ౦ నుండి వేరు)

x = ౨౮ > ౪
>> x ¶          // → #౧   (పోలిక ఫలితం)
```

### సోర్స్‌లో స్థానిక లిటరల్స్

ఏదైనా మద్దతు ఇచ్చే వ్యవస్థ యొక్క అంకెలు పరిధులు, మాడ్యులో, పోలికలలో చెల్లుబాటు అయ్యే లిటరల్స్:

```zymbol
#౦౯#

@ i:౧..౧౫ {
    ? i % ౧౫ == ౦ { >> "FizzBuzz" ¶ }
    _? i % ౩  == ౦ { >> "Fizz" ¶ }
    _? i % ౫  == ౦ { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### ఏ వ్యవస్థలోనైనా బూలియన్ లిటరల్స్

ఏదైనా మద్దతు ఇచ్చే బ్లాక్ యొక్క `#` + `౦` లేదా `౧` అంకె ఒక చెల్లుబాటు అయ్యే బూలియన్ లిటరల్:

```zymbol
#٠٩#
نشط = #١        // same as #౧
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **ఎల్లప్పుడూ ASCII**. `#౦` (తప్పు) ఏ వ్యవస్థలోనైనా `౦` (సున్నా పూర్ణాంకం) నుండి దృశ్యమానంగా భిన్నంగా ఉంటుంది.

```zymbol
// సంఖ్య ఫార్మాటింగ్
fmt = #,|౧౨౩౪౫౬౭|      // → ౧,౨౩౪,౫౬౭  (కామా వేరుచేయబడింది)
sci = #^|౧౨౩౪౫.౬౭౮|    // → ౧.౨౩౪౫౬౭౮e౪  (శాస్త్రీయ)

// బేస్ లిటరల్స్
a = 0x41         // → 'A'  (హెక్స్)

hex = 0x|౨౫౫|    // → "0x00FF"
bin = 0b|౬౫|     // → "0b1000001"
oct = 0o|౮|      // → "0o10"
dec = 0d|౨౫౫|    // → "0d0255"
```
