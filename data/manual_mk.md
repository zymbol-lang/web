# Прирачник на Zymbol-Lang

> **Прегледано за v0.0.5 — 2026-05-12**

**Zymbol-Lang** е симболички програмски јазик. Без клучни зборови — сè е симбол. Работи идентично на секој човечки јазик.

- Без `if`, `while`, `return` — само `?`, `@`, `<~`
- Целосен Unicode — идентификатори на кој било јазик или емоџи
- Независен од човечкиот јазик — кодот е ист насекаде

**Верзија на интерпретерот**: v0.0.5 | **Покриеност со тестови**: 436/436 (TW ↔ VM паритет)

## Променливи и Константи

```zymbol
x = 10              // променлива
PI := 3.14159       // константа — повторно доделување е грешка при извршување
ime = "Алиса"
aktivno = #1        // логичка вистина
👋 := "Здраво"
```

```zymbol
x = 10    // 10
x += 5    // 15
x -= 3    // 12
x *= 2    // 24
x /= 3    // 8
x %= 3    // 2
x ^= 2    // 4
x++        // 5
x--        // 4
```

`°` (знак за степен, U+00B0) автоматски иницијализира променлива на нејзината неутрална вредност при прва употреба:

```zymbol
broevi = [3, 1, 4, 1, 5]
@ n:broevi {
    °vkupno += n    // авто-иницијализација на 0 над јамката; достапна по @
}
>> vkupno ¶         // → 14
```

> `°x` (префикс) сидри над јамката — резултатот е достапен по `@`.
> `x°` (суфикс) сидри внатре во јамката — исчезнува кога јамката завршува.
> Само tree-walker.

## Типови Податоци

| Тип | Литерал | Таг `#?` | Забелешки |
|-----|---------|----------|---------|
| Цел | `42`, `-7` | `###` | 64-битен со предзнак |
| Децимален | `3.14`, `1.5e10` | `##.` | Научна нотација OK |
| Низ | `"текст"` | `##"` | Интерполација: `"Здраво {ime}"` |
| Знак | `'A'` | `##'` | Единечен Unicode знак |
| Логички | `#1`, `#0` | `##?` | НЕ е нумерички — `#1 ≠ 1` |
| Низа | `[1, 2, 3]` | `##]` | Хомогени елементи |
| Торка | `(a, b)` | `##)` | Позициска |
| Именувана Торка | `(x: 1, y: 2)` | `##)` | Именувани полиња |
| Функција | именувана референца | `##()` | Прв ред; прикажува `<funct/N>` |
| Ламбда | `x -> x * 2` | `##->` | Прв ред; прикажува `<lambd/N>` |

```zymbol
// Интроспекција на тип — враќа (тип, цифри, вредност)
meta = 42#?
>> meta ¶         // → (###, 2, 42)
t = meta[1]
>> t ¶            // → ###
```

## TUI Примитиви

Терминални UI оператори за интерактивни програми. Повеќето бараат блок `>>| { }` (алтернативен екран + суров режим).

```zymbol
>>| {
    >>!                             // бришење на алтернативниот екран
    >>~ (1, 1, 0, 10) > "Работи"   // ред 1, колона 1, fg=10 (зелена)
    @~ 1000                         // пауза 1 секунда (1000 мс)
    >>~ (2, 1) > "Готово."
}
// терминалот автоматски обновен при излез
```

```zymbol
// Притисок на копче и димензии на терминалот
>>| {
    [redovi, koloni] = >>?              // барање на димензиите на терминалот
    >>~ (1, 1) > "Терминал: " redovi " x " koloni
    <<| kopce                           // блокирачко читање на копче
    >>~ (2, 1) > "Притиснато: " kopce
}
```

> `>>!` го брише екранот. `>>?` враќа `[redovi, koloni]`. `@~ N` спие N милисекунди.
> `<<|` чита едно притискање на копче (блокирачко); `<<|?` проверува без блокирање (враќа `'\0'` ако нема).
> Торка на позициониран излез: `(red, kolona, SIM, fg, bg)` — кој било слот може да се изостави со запирка (`>>~ (,,, 196) > "црвена"`).
> Маска SIM: `1`=Задебелено, `2`=Курзив, `4`=Подвлечено. ANSI 256-бојна палета (`0`=стандарден терминал).
> Само tree-walker (освен `>>!`, `>>?`, `@~`, `>>~` кои работат и со `--vm`).

## Јамки

```zymbol
@ i:0..4  { >> i " " }        // опсег вклучен:  0 1 2 3 4
@ i:1..9:2 { >> i " " }       // со чекор:        1 3 5 7 9
@ i:5..0:1 { >> i " " }       // обратно:         5 4 3 2 1 0

n = 1
@ n <= 64 { n *= 2 }
>> n ¶                        // → 128  (while)

ovoshje = ["јаболко", "круша", "грозје"]
@ v:ovoshje { >> v ¶ }        // for-each низа

@ c:"zdravo" { >> c "-" }
>> ¶                          // → z-d-r-a-v-o-  (for-each низ)

@ i:1..10 {
    ? i % 2 == 0 { @> }       // @> продолжи
    ? i > 7 { @! }             // @! прекини
    >> i " "
}
>> ¶                          // → 1 3 5 7

// Бесконечна јамка
i = 0
@ {
    i++
    ? i >= 5 { @! }
    >> i " "
}
>> ¶                          // → 1 2 3 4

// Означена јамка (вгнездено прекинување)
brojac = 0
@:nadvoresnа {
    brojac++
    ? brojac >= 3 { @:nadvoresnа! }
}
>> brojac ¶                   // → 3
```

## Функции

```zymbol
soberi(a, b) { <~ a + b }
>> soberi(3, 4) ¶    // → 7

faktorijel(n) {
    ? n <= 1 { <~ 1 }
    <~ n * faktorijel(n - 1)
}
>> faktorijel(5) ¶    // → 120
```

Функциите имаат **изолиран опсег** — не можат да читаат надворешни променливи. Користете излезни параметри `<~` за менување на променливи на повикувачот:

```zymbol
zameni(a<~, b<~) {
    tmp = a
    a = b
    b = tmp
}
x = 10
y = 20
zameni(x, y)
>> "x=" x " y=" y ¶    // → x=20 y=10
```

> Именуваните функции се **вредности од прв ред** — проследете директно: `broevi$> udvoи`. За обвиткување: `x -> fn(x)` е исто така валидно.

## Низи

Низите се **менливи** и содржат елементи од **ист тип**.

```zymbol
niza = [1, 2, 3, 4, 5]

x = niza[1]      // 1 — пристап (1-базно: прв елемент)
x = niza[-1]     // 5 — негативен индекс (последен елемент)
x = niza$#       // 5 — должина (користете (niza$#) во >>)

niza = niza$+ 6            // додавање → [1,2,3,4,5,6]
niza2 = niza$+[2] 99       // вметнување на позиција 2 (1-базно)
niza3 = niza$- 3           // отстранување на прво јавување на вредност
niza4 = niza$-- 3          // отстранување на сите јавувања
niza5 = niza$-[1]          // отстранување на индекс 1 (прв елемент)
niza6 = niza$-[2..3]       // отстранување на опсег (1-базно, крај вклучен)

ima = niza$? 3             // #1 — содржи
poz = niza$?? 3            // [3] — сите индекси на вредноста (1-базно)
isecok = niza$[1..3]       // [1,2,3] — исечок (1-базно, крај вклучен)
isecok2 = niza$[1:3]       // [1,2,3] — исто, синтакса со број

rastечki = niza$^+         // сортирани растечки (само примитиви)
opadnuvачki = niza$^-      // сортирани опаѓачки (само примитиви)

// Низи на именувани/позициски торки — користете $^ со ламбда компаратор
bp = [(ime: "Карла", god: 28), (ime: "Ана", god: 25), (ime: "Боб", god: 30)]
po_god  = bp$^ (a, b -> a.god < b.god)    // растечки по години  (<)
po_ime  = bp$^ (a, b -> a.ime > b.ime)    // опаѓачки по ime (>)
>> po_god[1].ime ¶     // → Ана
>> po_ime[1].ime ¶     // → Карла

// Директно ажурирање на елемент (само низи)
niza[1] = 99              // доделување
niza[2] += 5              // составено: +=  -=  *=  /=  %=  ^=

// Функционално ажурирање — враќа нова низа; оригиналот непроменет
niza2 = niza[2]$~ 99
```

> Сите оператори на колекции враќаат **нова низа**. Доделете назад: `niza = niza$+ 4`.
> `$+` може да се поврзи во синџир: `niza = niza$+ 5$+ 6$+ 7`. Другите оператори користат меѓупрератки.
> **Индексирањето е 1-базно**: `niza[1]` е прв елемент; `niza[0]` е грешка при извршување.
> `$^+` / `$^-` сортираат **примитивни низи** (броеви, низови). За низи на торки користете `$^` со ламбда компаратор — насоката е кодирана во ламбдата (`<` = растечко, `>` = опаѓачко).

**Семантика на вредности** — доделување на низа на друга променлива создава независна копија:

```zymbol
a = [1, 2, 3]
b = a
a[1] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b не е под влијание
```

```zymbol
// Вгнездени низи (1-базно индексирање)
matrica = [[1,2,3],[4,5,6],[7,8,9]]
>> matrica[2][3] ¶    // → 6  (ред 2, колона 3)
```

## Торки

Торките се **непроменливи** уредени контејнери кои можат да содржат вредности од **различни типови**.
За разлика од низите, елементите не можат да се менуваат по создавањето.

```zymbol
// Позициски — дозволени мешани типови
tocka = (10, 20)
>> tocka[1] ¶    // → 10

podatoci = (42, "zdravo", #1, 3.14)
>> podatoci[3] ¶ // → #1

// Именувани
osoba = (ime: "Алиса", god: 25)
>> osoba.ime ¶    // → Алиса
>> osoba[1] ¶     // → Алиса  (индексот исто работи, 1-базно)

// Вгнездени
poz = (x: 10, y: 20)
p = (poz: poz, etiketa: "почеток")
>> p.poz.x ¶      // → 10
```

**Непроменливост** — секој обид за менување на елемент на торка е грешка при извршување:

```zymbol
t = (10, 20, 30)
// t[1] = 99    // ❌ грешка при извршување: торките се непроменливи
// t[1] += 5    // ❌ иста грешка
```

За изведување на изменета вредност користете `$~` (функционално ажурирање) — враќа **нова** торка:

```zymbol
t = (10, 20, 30)
t2 = t[2]$~ 999
>> t ¶     // → (10, 20, 30)   ← оригиналот непроменет
>> t2 ¶    // → (10, 999, 30)

// Именувана торка — експлицитна реконструкција
osoba = (ime: "Алиса", god: 25)
postar = (ime: osoba.ime, god: 26)
>> osoba.god ¶    // → 25
>> postar.god ¶   // → 26
```

## Оператор Цевка

Десната страна секогаш бара `_` како заменик за препратената вредност:

```zymbol
udvoi = x -> x * 2
soberi = (a, b) -> a + b
zgolemi = x -> x + 1

r1 = 5 |> udvoi(_)          // → 10
r2 = 10 |> soberi(_, 5)     // → 15
r3 = 5 |> soberi(2, _)      // → 7

// Поврзано во синџир
r = 5 |> udvoi(_) |> zgolemi(_) |> udvoi(_)
>> r ¶    // → 22  (5→10→11→22)
```

## Нумерички Режими

Zymbol може да прикажува броеви во **69 Unicode скрипти за цифри** — Деванагари, Арапско-Индиски, Тајландски, Клингонски pIqaD, Математички задебелен, LCD сегменти и повеќе. Активниот режим влијае само на `>>` излезот; внатрешната аритметика е секогаш бинарна.

### Активирање на скрипта

Напишете ги цифрите `0` и `9` на целната скрипта затворени во `#…#`:

```zymbol
#०९#    // Деванагари   (U+0966–U+096F)
#٠٩#    // Арапско-Индиски (U+0660–U+0669)
#๐๙#    // Тајландски   (U+0E50–U+0E59)
#09#    // ресет на ASCII
```

### Излез и логички вредности

```zymbol
x = 42
>> x ¶          // → 42   (ASCII стандардно)

#०९#
>> x ¶          // → ४२
>> 3.14 ¶       // → ३.१४   (децималната точка е секогаш ASCII)
>> 1 + 2 ¶      // → ३

// Логички: префиксот # е секогаш ASCII, цифрата се прилагодува
>> #1 ¶         // → #१   (вистина во Деванагари)
>> #0 ¶         // → #०   (лага — различно од ०  цел број нула)

x = 28 > 4
>> x ¶          // → #१   (резултатот од споредба го следи активниот режим)
```

### Нативни цифровни литерали во изворниот код

Цифрите на која било поддржана скрипта се валидни литерали — во опсези, модуло, споредби:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### Логички литерали во која било скрипта

`#` + цифра `0` или `1` од кој било блок е валиден логички литерал:

```zymbol
#٠٩#
aktivno = #١        // исто како #1
>> aktivno ¶        // → #١
>> (#١ && #٠) ¶     // → #٠
```

> `#` е **секогаш ASCII**. `#0` (лага) е секогаш визуелно различно од `0` (цел број нула) во секоја скрипта.

## Оператори за Податоци

```zymbol
// Конверзии на тип
f = ##.42         // → 42.0  (во Децимален)
i = ###3.7        // → 4     (во Цел, заокружување)
t = ##!3.7        // → 3     (во Цел, кратење)

// Парсирање на низ во број
v1 = #|"42"|      // → 42  (Цел)
v2 = #|"3.14"|    // → 3.14  (Децимален)
v3 = #|"abc"|     // → "abc"  (безбедно неуспевање, без грешка)

// Заокружување / кратење
pi = 3.14159265
v2 = #.2|pi|      // → 3.14  (заокружи на 2 децимални места)
v4 = #.4|pi|      // → 3.1416
t2 = #!2|pi|      // → 3.14  (кратење)

// Форматирање на броеви
fmt = #,|1234567|  // → 1,234,567  (одвоено со запирки)
nau = #^|12345.678|    // → 1.2345678e4  (научна нотација)

// Литерали за основа
a = 0x41         // → 'A'  (хексадецимален)
b = 0b01000001   // → 'A'  (бинарен)
c = 0o101        // → 'A'  (октален)

// Излез на конверзија на основа
hex = 0x|255|    // → "0x00FF"
bin = 0b|65|     // → "0b1000001"
oct = 0o|8|      // → "0o10"
dec = 0d|255|    // → "0d0255"
```

## Интеграција со Shell

```zymbol
datum = <\ date +%Y-%m-%d \>     // зафаќа stdout (вклучувајќи го завршниот \n)
>> "Денес: " datum

fajl = "podatoci.txt"
sodrzhina = <\ cat {fajl} \>     // интерполација во командите

izlez = </"./podskript.zy"/>     // извршува друга Zymbol скрипта, зафаќа излез
>> izlez
```

> `><` ги зафаќа CLI аргументите како низа на низови (само tree-walker).

## Потполн Пример: FizzBuzz

```zymbol
klasificiraj(broj) {
    ? broj % 15 == 0 { <~ "FizzBuzz" }
    _? broj % 3  == 0 { <~ "Fizz" }
    _? broj % 5  == 0 { <~ "Buzz" }
    _ { <~ broj }
}

@ i:1..20 { >> klasificiraj(i) ¶ }
```

## Референца на Симболи

| Симбол | Операција | Симбол | Операција |
|--------|-----------|--------|-----------|
| `=` | променлива | `$#` | должина |
| `:=` | константа | `$+` | додавање (синџирливо) |
| `>>` | излез | `$+[i]` | вметнување на индекс (1-базно) |
| `<<` | влез | `$-` | отстранување на прво по вредност |
| `¶` / `\\` | нов ред | `$--` | отстранување на сите по вредност |
| `?` | ако | `$-[i]` | отстранување на индекс (1-базно) |
| `_?` | инаку-ако | `$-[i..j]` | отстранување на опсег (1-базно) |
| `_` | инаку / шаблон | `$?` | содржи |
| `??` | совпаѓање | `$??` | наоѓање на сите индекси (1-базно) |
| `@` | јамка | `$[s..e]` | исечок (1-базно) |
| `@ N { }` | јамка N пати | `$>` | map |
| `@!` | прекинување | `$\|` | filter |
| `@>` | продолжи | `$<` | reduce |
| `@:ime { }` | означена јамка | `$/ razd` | делење на низ |
| `@:ime!` | прекини означена | `$++ a b c` | конкатенација |
| `@:ime>` | продолжи означена | `niza[i>j>k]` | навигациски индекс |
| `->` | ламбда | `niza[i] = vred` | ажурирање на елемент (само низи) |
| `niza[i] += vred` | составено ажур. | `niza[i]$~` | функционално ажур. (нова копија) |
| `$^+` | сортирање растечко (прим.) | `$^-` | сортирање опаѓачко (прим.) |
| `$^` | сортирање со компаратор (торки) | `<~` | враќање |
| `\|>` | цевка | `!?` | обид |
| `:!` | зафаќање | `:>` | на крајот |
| `#1` | вистина | `#0` | лага |
| `$!` | е грешка | `$!!` | пропагирање на грешка |
| `<#` | увоз | `#>` | извоз |
| `#` | декларација на модул | `::` | повик на модул |
| `.` | пристап до поле | `#?` | метаподатоци за тип |
| `#\|..\|` | парсирање на број | `##.` | конверзија во Децимален |
| `###` | конверзија во Цел (заокр.) | `##!` | конверзија во Цел (крат.) |
| `#.N\|..\|` | заокружување | `#!N\|..\|` | кратење |
| `#,\|..\|` | формат со запирки | `#^\|..\|` | научна нотација |
| `#d0d9#` | менување на цифровен режим | `#09#` | ресет на ASCII |
| `<\ ..\>` | извршување на shell | `>\<` | CLI аргументи |
| `\ prom` | експлицитно уништување | `°x` / `x°` | врела дефиниција (авто-инит) |
| `>>|` | TUI блок (алт. екран) | `>>~` | позициониран излез |
| `>>!` | бришење на екранот | `>>?` | барање на димензии на терминал |
| `<<\|` | блокирачко копче | `<<\|?` | неблокирачко копче |
| `@~ N` | спиење N милисекунди | `$*` | повторување на низ N пати |

## Евиденција на Промени

### v0.0.5 — TUI Примитиви, Врела Дефиниција и Повторување на Низ _(мај 2026)_

- **Промена** Одделувач на гранка за совпаѓање: `шаблон : резултат` → `шаблон => резултат`
- **Промена** Псевдоним за увоз: `<# пат <= псевдоним` → `<# пат => псевдоним`
- **Промена** Преименување при извоз: `#> { fn <= pub }` → `#> { fn => pub }`
- **Додадено** TUI блок `>>| { }` — алтернативен екран + суров режим; чисти при излез
- **Додадено** Позициониран излез `>>~ (ред, кол, СИМ, fg, bg) > елементи` — ретки слотови, 256-бојна ANSI
- **Додадено** Влез на копче `<<| пром` (блокирачко) и `<<|? пром` (неблокирачко испитување)
- **Додадено** `>>!` бришење, `>>?` барање на димензии, `@~ N` спиење N милисекунди
- **Додадено** Врела дефиниција `°x` / `x°` — авто-иницијализација при прва употреба во јамки
- **Додадено** Повторување на низ `niz $* N` — го повторува низот N пати
- **VM** Паритет: 436/436 тестови поминуваат

### v0.0.4 — 1-Базно Индексирање, Функции Прв Ред и Блокови на Модули _(април 2026)_

- **Промена** Целото индексирање е префрлено на **1-базно** — `niza[1]` е прв елемент; `niza[0]` е грешка
- **Додадено** Именуваните функции се **вредности од прв ред** — проследете директно на HOF: `broevi$> udvoi`
- **Додадено** Задолжителна **блоковска синтакса** за модули: `# ime { ... }` — рамната синтакса е отстранета
- **Додадено** Повеќедимензионално индексирање: `niza[i>j>k]` (навигација), `niza[p ; q]` (рамно извлекување)
- **Додадено** Конверзии на тип: `##.изр` (Децимален), `###изр` (Цел заокр.), `##!изр` (Цел крат.)
- **Додадено** Делење на низ: `niz$/ razd` — враќа `Array(String)`
- **Додадено** Конкатенација: `osnova$++ a b c` — додава повеќе елементи
- **Додадено** Јамка N пати: `@ N { }` — повторува точно N пати
- **Додадено** Синтакса за означена јамка: `@:ime { }`, `@:ime!`, `@:ime>` — ги заменува `@ @ime` / `@! ime`
- **Додадено** Правила за опсег на променливи: `_ime` има точен блоковски опсег; `\ prom` уништува рано
- **Додадено** Шаблони за споредба во совпаѓање: `< 0 :`, `> 5 :`, `== 42 :` итн.
- **Додадено** Грешка E013 на модулот: извршните наредби во телото на модулот се забранети
- **Поправено** `take_variable` повеќе не ги оштетува константите на модулот при враќање
- **Поправено** `псевдоним.CONST` сега правилно се разрешува; `#>` може да се појави по дефинициите на функции
