# Zymbol-Lang 手册

> **已为 v0.0.5 修订 — 2026-05-14**

**Zymbol-Lang** 是一种符号编程语言。没有关键字 — 一切都是符号。在任何人类语言中都能相同地工作。

- 没有 `if`、`while`、`return` — 只有 `?`、`@`、`<~`
- 完整 Unicode — 任何语言或表情符号中的标识符
- 人类语言无关 — 代码在任何地方都是相同的

**解释器版本**: v0.0.5 | **测试覆盖率**: 436/436 (TW ↔ VM 奇偶校验)

## 变量和常量

```zymbol
x = 10              // 可变变量
π := 3.14159        // 常量 — 重新赋值是运行时错误
名称 = "爱丽丝"
活跃 = #1         // 布尔值 true
👋 := "你好"
```

```zymbol
x = 10    // 10
x += 5    // 15
x -= 3    // 12
x *= 2    // 24
x /= 3    // 8
x %= 3    // 2
x ^= 2    // 4
x++        // 5
x--        // 4
```

`°` (度符号, U+00B0) 在首次使用时自动将变量初始化为其中性值:

```zymbol
数字 = [3, 1, 4, 1, 5]
@ n:数字 {
    °总和 += n    // 在循环上方自动初始化为 0；在 @ 之后存活
}
>> 总和 ¶         // → 14
```

> `°x` (前缀) 锚定在循环上方 — 结果在 `@` 之后可访问。
> `x°` (后缀) 锚定在循环内部 — 当循环结束时消亡。
> 仅限 tree-walker。

## 数据类型

| 类型 | 字面量 | `#?` 标签 | 注释 |
|------|---------|----------|---------|
| 整数 | `42`, `-7` | `###` | 64 位有符号 |
| 浮点数 | `3.14`, `1.5e10` | `##.` | 允许科学记数法 |
| 字符串 | `"文本"` | `##"` | 插值: `"你好 {名称}"` |
| 字符 | `'A'` | `##'` | 单个 Unicode 字符 |
| 布尔值 | `#1`, `#0` | `##?` | 非数字 — `#1 ≠ 1` |
| 数组 | `[1, 2, 3]` | `##]` | 同质元素 |
| 元组 | `(a, b)` | `##)` | 位置性 |
| 命名元组 | `(x: 1, y: 2)` | `##)` | 命名字段 |
| 函数 | 命名函数引用 | `##()` | 一等公民；显示 `<funct/N>` |
| Lambda | `x -> x * 2` | `##->` | 一等公民；显示 `<lambd/N>` |

```zymbol
// 类型内省 — 返回 (类型，数字，值)
元数据 = 42#?
>> 元数据 ¶         // → (###, 2, 42)
t = 元数据[1]
>> t ¶            // → ###
```

## TUI 原语

用于交互程序的终端用户界面运算符。大多数需要 `>>| { }` 块 (备用屏幕 + 原始模式)。

```zymbol
>>| {
    >>!                             // 清除备用屏幕
    >>~ (1, 1, 0, 10) > "正在运行"   // 行 1, 列 1, fg=10 (绿色)
    @~ 1000                         // 暂停 1 秒 (1000 毫秒)
    >>~ (2, 1) > "已完成。"
}
// 退出时终端自动恢复
```

```zymbol
// 按键和终端大小
>>| {
    [行, 列] = >>?              // 查询终端尺寸
    >>~ (1, 1) > "终端: " 行 " x " 列
    <<| 键                         // 读取阻塞按键
    >>~ (2, 1) > "按下了: " 键
}
```

> `>>!` 清除屏幕。`>>?` 返回 `[行, 列]`。`@~ N` 休眠 N 毫秒。
> `<<|` 读取一次按键 (阻塞); `<<|?` 非阻塞轮询 (如果没有则返回 `'\0'`)。
> 位置输出元组: `(行, 列, BKS, fg, bg)` — 任何槽位可以用逗号省略 (`>>~ (,,, 196) > "红色"`)。
> BKS 位掩码: `1`=粗体, `2`=斜体, `4`=下划线。ANSI 256 色调色板 (`0`=终端默认)。
> 仅限 tree-walker (除了 `>>!`, `>>?`, `@~`, `>>~` 也适用于 `--vm`)。

## 字符串

```zymbol
// 两种连接形式
名称 = "爱丽丝"
n = 42

>> "你好 " 名称 " 你有 " n " 个" ¶       // 并列 — 在 >> 中
```

```zymbol
s = "你好世界"
```

## 控制流

```zymbol

}
```

## 循环

```zymbol
@ i:0..4  { >> i " " }        // 范围包含:  0 1 2 3 4
@ i:1..9:2 { >> i " " }       // 带步长:         1 3 5 7 9
@ i:5..0:1 { >> i " " }       // 反向:           5 4 3 2 1 0

n = 1
@ n <= 64 { n *= 2 }
>> n ¶                        // → 128  (while)

水果 = ["苹果", "梨", "葡萄"]
@ f:水果 { >> f ¶ }         // 数组的每个元素

@ c:"你好" { >> c "-" }
>> ¶                          // → 你-好-  (字符串的每个字符)

@ i:1..10 {
    ? i % 2 == 0 { @> }       // @> 继续
    ? i > 7 { @! }             // @! 中断
    >> i " "
}
>> ¶                          // → 1 3 5 7

// 无限循环
i = 0
@ {
    i++
    ? i >= 5 { @! }
    >> i " "
}
>> ¶                          // → 1 2 3 4

// 带标签的循环 (嵌套中断)
计数 = 0
@:外部 {
    计数++
    ? 计数 >= 3 { @:外部! }
}
>> 计数 ¶                    // → 3
```

## 函数

```zymbol
加法(a, b) { <~ a + b }
>> 加法(3, 4) ¶    // → 7

阶乘(n) {
    ? n <= 1 { <~ 1 }
    <~ n * 阶乘(n - 1)
}
>> 阶乘(5) ¶    // → 120
```

函数具有**隔离的作用域** — 它们无法读取外部变量。使用输出参数 `<~` 来修改调用者的变量:

```zymbol
交换(a<~, b<~) {
    临时 = a
    a = b
    b = 临时
}
x = 10
y = 20
交换(x, y)
>> "x=" x " y=" y ¶    // → x=20 y=10
```

> 命名函数是**一等值** — 直接传递: `数字$> 加倍`。要包装: `x -> fn(x)` 也是有效的。

## 数组

数组是**可变的**，包含**相同类型**的元素。

```zymbol
arr = [1, 2, 3, 4, 5]

x = arr[1]      // 1 — 访问 (1-based: 第一个元素)
x = arr[-1]     // 5 — 负索引 (最后一个元素)
x = arr$#       // 5 — 长度 (在 >> 中使用 (arr$#))

arr = arr$+ 6            // 追加 → [1,2,3,4,5,6]
arr2 = arr$+[2] 99       // 在位置 2 插入 (1-based)
arr3 = arr$- 3           // 移除值的第一次出现
arr4 = arr$-- 3          // 移除所有出现
arr5 = arr$-[1]          // 在索引 1 处移除 (第一个元素)
arr6 = arr$-[2..3]       // 移除范围 (1-based, 包含结束)

包含 = arr$? 3            // #1 — 包含
位置 = arr$?? 3           // [3] — 值的所有索引 (1-based)
切片 = arr$[1..3]          // [1,2,3] — 切片 (1-based, 包含结束)
切片2 = arr$[1:3]          // [1,2,3] — 相同，基于计数的语法

升序 = arr$^+             // 升序排序 (仅原始类型)
降序 = arr$^-            // 降序排序 (仅原始类型)

// 命名/位置元组数组 — 使用带有比较器 lambda 的 $^
数据库 = [(名称: "卡拉", 年龄: 28), (名称: "安娜", 年龄: 25), (名称: "鲍勃", 年龄: 30)]
按年龄  = 数据库$^ (a, b -> a.年龄 < b.年龄)    // 按年龄升序 (<)
按名称 = 数据库$^ (a, b -> a.名称 > b.名称)   // 按名称降序 (>)
>> 按年龄[1].名称 ¶     // → 安娜
>> 按名称[1].名称 ¶    // → 卡拉

// 直接元素更新 (仅数组)
arr[1] = 99              // 赋值
arr[2] += 5              // 复合: +=  -=  *=  /=  %=  ^=

// 函数式更新 — 返回一个新数组；原始数组不变
arr2 = arr[2]$~ 99
```

> 所有集合运算符返回一个**新数组**。赋值回去: `arr = arr$+ 4`。
> `$+` 可以链式调用: `arr = arr$+ 5$+ 6$+ 7`。其他运算符使用中间赋值。
> **索引 1-based**: `arr[1]` 是第一个元素；`arr[0]` 是运行时错误。
> `$^+` / `$^-` 排序**原始类型数组** (数字，字符串)。对于元组数组，使用带有比较器 lambda 的 `$^` — 方向在 lambda 中编码 (`<` = 升序, `>` = 降序)。

**值语义** — 将数组赋值给另一个变量会创建一个独立的副本:

```zymbol
a = [1, 2, 3]
b = a
a[1] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b 不受影响
```

```zymbol
// 嵌套数组 (1-based 索引)
矩阵 = [[1,2,3],[4,5,6],[7,8,9]]
>> 矩阵[2][3] ¶    // → 6  (行 2, 列 3)
```

## 元组

元组是**不可变的**有序容器，可以包含**不同类型**的值。
与数组不同，创建后元素无法更改。

```zymbol
// 位置性 — 允许混合类型
点 = (10, 20)
>> 点[1] ¶    // → 10

数据 = (42, "你好", #1, 3.14)
>> 数据[3] ¶     // → #1

// 命名
人 = (名称: "爱丽丝", 年龄: 25)
>> 人.名称 ¶    // → 爱丽丝
>> 人[1] ¶      // → 爱丽丝  (索引也有效, 1-based)

// 嵌套
位置 = (x: 10, y: 20)
p = (位置: 位置, 标签: "原点")
>> p.位置.x ¶        // → 10
```

**不可变性** — 任何修改元组元素的尝试都是运行时错误:

```zymbol
t = (10, 20, 30)
// t[1] = 99    // ❌ 运行时错误: 元组是不可变的
// t[1] += 5    // ❌ 同样的错误
```

要获取修改后的值，使用 `$~` (函数式更新) — 返回一个**新**元组:

```zymbol
t = (10, 20, 30)
t2 = t[2]$~ 999
>> t ¶     // → (10, 20, 30)   ← 原始不变
>> t2 ¶    // → (10, 999, 30)

// 命名元组 — 显式重建
人 = (名称: "爱丽丝", 年龄: 25)
年长的  = (名称: 人.名称, 年龄: 26)
>> 人.年龄 ¶    // → 25
>> 年长的.年龄 ¶     // → 26
```

## 模块

```zymbol
// lib/calc.zy — 模块体用花括号括起来
# calc {
    #> { 加法, get_PI }

    _π := 3.14159
    加法(a, b) { <~ a + b }
    get_PI() { <~ _π }
```

```zymbol
// main.zy

>> π ¶               // → 3.14159
```

```zymbol
// 用不同的公共名称导出
# mylib {

}
```

## 数字模式

Zymbol 可以用 **69 种 Unicode 数字脚本**显示数字 — 天城文、阿拉伯-印度文、泰文、克林贡语 pIqaD、数学粗体、LCD 段等。活动模式仅影响 `>>` 输出；内部算术始终是二进制的。

### 激活一个脚本

将目标脚本的 `0` 和 `9` 数字写在 `#…#` 内:

```zymbol
#०९#    // 天城文   (U+0966–U+096F)
#٠٩#    // 阿拉伯-印度文 (U+0660–U+0669)
#๐๙#    // 泰文         (U+0E50–U+0E59)
#09#    // 重置为 ASCII
```

### 输出和布尔值

```zymbol
x = 42
>> x ¶          // → 42   (ASCII 默认)

#०९#
>> x ¶          // → ४२
>> 3.14 ¶       // → ३.१४   (小数点始终是 ASCII)
>> 1 + 2 ¶      // → ३

// 布尔值: # 前缀始终是 ASCII，数字会自适应
>> #1 ¶         // → #१   (天城文中为 true)
>> #0 ¶         // → #०   (false — 与 ० 整数零不同)

x = 28 > 4
>> x ¶          // → #१   (比较结果遵循活动模式)
```

### 源代码中的原生数字字面量

任何受支持的脚本的数字都是有效的字面量 — 在范围、模数、比较中:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### 任何脚本中的布尔字面量

`#` + 任何块中的数字 `0` 或 `1` 是有效的布尔字面量:

```zymbol
#٠٩#
活跃 = #١        // 等同于 #1
>> 活跃 ¶        // → #१
>> (#१ && #०) ¶ // → #०
```

> `#` **始终是 ASCII**。`#0` (false) 在每个脚本中总是在视觉上与 `0` (整数零) 不同。

## 数据运算符

```zymbol
// 类型转换
f = ##.42         // → 42.0  (转换为浮点数)
i = ###3.7        // → 4     (转换为整数，四舍五入)
t = ##!3.7        // → 3     (转换为整数，截断)

// 将字符串解析为数字
v1 = #|"42"|      // → 42  (整数)
v2 = #|"3.14"|    // → 3.14  (浮点数)
v3 = #|"abc"|     // → "abc"  (安全，无错误)

// 四舍五入 / 截断
π = 3.14159265
四舍五入2 = #.2|π|      // → 3.14  (四舍五入到 2 位小数)
四舍五入4 = #.4|π|      // → 3.1416
截断2 = #!2|π|      // → 3.14  (截断)

// 数字格式化
格式化 = #,|1234567|  // → 1,234,567  (逗号分隔)
科学记数法 = #^|12345.678|    // → 1.2345678e4  (科学记数法)

// 进制字面量
a = 0x41         // → 'A'  (十六进制)
b = 0b01000001   // → 'A'  (二进制)
c = 0o101        // → 'A'  (八进制)

// 进制转换输出
十六进制 = 0x|255|    // → "0x00FF"
二进制 = 0b|65|     // → "0b1000001"
八进制 = 0o|8|      // → "0o10"
十进制 = 0d|255|    // → "0d0255"
```

## Shell 集成

```zymbol
日期 = <\ date +%Y-%m-%d \>     // 捕获 stdout (包括末尾的 \n)
>> "今天: " 日期

文件 = "data.txt"
内容 = <\ cat {文件} \>      // 命令中的插值

输出 = </"./subscript.zy"/>   // 执行另一个 Zymbol 脚本，捕获输出
>> 输出
```

> `><` 将 CLI 参数捕获为字符串数组 (仅限 tree-walker)。

## 完整示例: FizzBuzz

```zymbol
分类(数字) {
    ? 数字 % 15 == 0 { <~ "FizzBuzz" }
    _? 数字 % 3  == 0 { <~ "Fizz" }
    _? 数字 % 5  == 0 { <~ "Buzz" }
    _ { <~ 数字 }
}

@ i:1..20 { >> 分类(i) ¶ }
```

### v0.0.5 — TUI 原语、热定义和字符串重复 _(2026年5月)_

- **破坏性更改** 匹配臂分隔符: `模式 : 结果` → `模式 => 结果`
- **破坏性更改** 导入别名: `<# 路径 <= 别名` → `<# 路径 => 别名`
- **破坏性更改** 导出重命名: `#> { fn <= 公共 }` → `#> { fn => 公共 }`
- **新增** TUI 块 `>>| { }` — 备用屏幕 + 原始模式；退出时清理
- **新增** 位置输出 `>>~ (行, 列, BKS, fg, bg) > 项` — 稀疏槽位，ANSI 256 色
- **新增** 按键输入 `<<| 变量` (阻塞) 和 `<<|? 变量` (非阻塞轮询)

- **VM** 奇偶校验: 436/436 测试通过

### v0.0.4 — 1-based 索引、一等函数和块模块 _(2026年4月)_

- **破坏性更改** 所有索引改为 **1-based** — `arr[1]` 是第一个元素；`arr[0]` 是运行时错误
- **新增** 命名函数是**一等值** — 直接传递给 HOF: `数字$> 加倍`
- **新增** 模块**必须使用块语法** — `# 名称 { ... }` — 移除了扁平语法
- **新增** 多维索引: `arr[i>j>k]` (导航), `arr[p ; q]` (扁平提取)
- **新增** 类型转换: `##.表达式` (浮点数), `###表达式` (整数四舍五入), `##!表达式` (整数截断)
- **新增** 字符串分割: `字符串$/ 分隔符` — 返回 `Array(字符串)`
- **新增** 连接构建: `基$++ a b c` — 追加多个项
- **新增** 次数循环: `@ N { }` — 精确重复 N 次
- **新增** 带标签的循环语法: `@:名称 { }`, `@:名称!`, `@:名称>` — 替代 `@ @名称` / `@! 名称`
- **新增** 变量作用域规则: `_名称` 变量具有精确的块作用域；`\ 变量` 提前销毁
- **新增** 匹配比较模式: `< 0 =>`, `> 5 =>`, `== 42 =>` 等
