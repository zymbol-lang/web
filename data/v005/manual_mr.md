## व्हेरिएबल्स आणि स्थिरांक


```zymbol

x -= ३    // १२
x *= २    // २४
x /= ३    // ८
```

`°` (अंश चिन्ह, U+00B0) पहिल्या वापरात व्हेरिएबलला त्याच्या तटस्थ मूल्यावर स्वयंचलितपणे प्रारंभ करते:

```zymbol
संख्या = [३, १, ४, १, ५]
@ n:संख्या {
    °एकूण += n    // लूपच्या वर स्वयंचलितपणे ० वर प्रारंभ; @ नंतर टिकून राहते
}
>> एकूण ¶         // → १४
```

> `°x` (उपसर्ग) लूपच्या वर अँकर करतो — परिणाम `@` नंतर प्रवेशयोग्य आहे.
> `x°` (प्रत्यय) लूपच्या आत अँकर करतो — जेव्हा लूप संपतो तेव्हा मरतो.

## टीयूआय प्रिमिटिव्ह

इंटरएक्टिव्ह प्रोग्रामसाठी टर्मिनल यूजर इंटरफेस ऑपरेटर. बहुतेकांना `>>| { }` ब्लॉक आवश्यक आहे (वैकल्पिक स्क्रीन + कच्चा मोड).

```zymbol
>>| {
    >>!                             // वैकल्पिक स्क्रीन साफ करा
    >>~ (१, १, ०, १०) > "चालू आहे"   // ओळ १, स्तंभ १, fg=१० (हिरवा)
    @~ १०००                         // १ सेकंद विराम (१००० ms)
    >>~ (२, १) > "पूर्ण झाले."
}
// बाहेर पडताना टर्मिनल स्वयंचलितपणे पुनर्संचयित होते
```

```zymbol
// की प्रेस आणि टर्मिनल आकार
>>| {
    [ओळी, स्तंभ] = >>?              // टर्मिनलचे परिमाण विचारा
    >>~ (१, १) > "टर्मिनल: " ओळी " x " स्तंभ
    <<| की                         // ब्लॉकिंग की प्रेस वाचा
    >>~ (२, १) > "दाबले: " की
}
```

> `>>!` स्क्रीन साफ करते. `>>?` `[ओळी, स्तंभ]` परत करते. `@~ N` N मिलिसेकंद झोपते.
> `<<|` एक की प्रेस वाचते (ब्लॉकिंग); `<<|?` ब्लॉक न करता पोल करते (काहीही नसल्यास `'\0'` परत करते).
> स्थानिक आउटपुट टपल: `(ओळ, स्तंभ, BKS, fg, bg)` — कोणताही स्लॉट स्वल्पविरामाने वगळला जाऊ शकतो (`>>~ (,,, 196) > "लाल"`).
> BKS बिटमास्क: `1`=बोल्ड, `2`=इटालिक, `4`=अधोरेखित. ANSI 256 रंग पॅलेट (`0`=टर्मिनल डीफॉल्ट).
> फक्त tree-walker (`>>!`, `>>?`, `@~`, `>>~` वगळता जे `--vm` मध्येही चालतात).

## अरे

अरे **बदलता येण्याजोगे** असतात आणि **समान प्रकारचे** घटक धारण करतात.

```zymbol
arr = [१, २, ३, ४, ५]

x = arr[१]      // १ — प्रवेश (१-आधारित: पहिला घटक)
x = arr[-१]     // ५ — नकारात्मक निर्देशांक (शेवटचा घटक)
x = arr$#       // ५ — लांबी (>> मध्ये (arr$#) वापरा)

arr = arr$+ ६            // जोडा → [१,२,३,४,५,६]
arr२ = arr$+[२] ९९       // स्थान २ वर घाला (१-आधारित)
arr३ = arr$- ३           // मूल्याची पहिली घटना काढा
arr४ = arr$-- ३          // सर्व घटना काढा
arr५ = arr$-[१]          // निर्देशांक १ वर काढा (पहिला घटक)
arr६ = arr$-[२..३]       // श्रेणी काढा (१-आधारित, शेवट समाविष्ट)

आहे = arr$? ३            // #१ — समाविष्ट आहे
स्थाने = arr$?? ३           // [३] — मूल्याचे सर्व निर्देशांक (१-आधारित)
स्लाइस = arr$[१..३]          // [१,२,३] — स्लाइस (१-आधारित, शेवट समाविष्ट)
स्लाइस२ = arr$[१:३]          // [१,२,३] — समान, मोजणी-आधारित सिंटॅक्स

चढता = arr$^+             // चढत्या क्रमाने लावा (फक्त आदिम)
उतरता = arr$^-            // उतरत्या क्रमाने लावा (फक्त आदिम)

// नामांकित/स्थानिक टपल अरे — तुलनाकारी लॅम्ब्डासह $^ वापरा
डेटाबेस = [(नाव: "कार्ला", वय: २८), (नाव: "अना", वय: २५), (नाव: "बॉब", वय: ३०)]
वयानुसार  = डेटाबेस$^ (a, b -> a.वय < b.वय)    // वयानुसार चढता (<)
नावानुसार = डेटाबेस$^ (a, b -> a.नाव > b.नाव)   // नावानुसार उतरता (>)
>> वयानुसार[१].नाव ¶     // → अना
>> नावानुसार[१].नाव ¶    // → कार्ला

// थेट घटक अद्यतन (फक्त अरे)
arr[१] = ९९              // असाइन
arr[२] += ५              // संयुक्त: +=  -=  *=  /=  %=  ^=

// कार्यात्मक अद्यतन — एक नवीन अरे परत करते; मूळ अपरिवर्तित
arr२ = arr[२]$~ ९९
```

> सर्व संग्रह ऑपरेटर एक **नवीन अरे** परत करतात. परत असाइन करा: `arr = arr$+ 4`.
> `$+` साखळीकृत केले जाऊ शकते: `arr = arr$+ 5$+ 6$+ 7`. इतर ऑपरेटर मध्यवर्ती असाइनमेंट वापरतात.
> **निर्देशांकन 1-आधारित**: `arr[1]` पहिला घटक आहे; `arr[0]` एक रनटाइम त्रुटी आहे.
> `$^+` / `$^-` **आदिम अरे** (संख्या, स्ट्रिंग) लावतात. टपल अरेसाठी तुलनाकारी लॅम्ब्डासह `$^` वापरा — दिशा लॅम्ब्डामध्ये एन्कोड केलेली आहे (`<` = चढता, `>` = उतरता).

**मूल्य शब्दार्थ** — अरेला दुसर्या व्हेरिएबलमध्ये असाइन केल्याने एक स्वतंत्र प्रत तयार होते:

```zymbol
a = [१, २, ३]
b = a
a[१] = ९९
>> a ¶    // → [९९, २, ३]
>> b ¶    // → [१, २, ३]   ← b प्रभावित होत नाही
```

```zymbol
// नेस्टेड अरे (१-आधारित निर्देशांकन)
मॅट्रिक्स = [[१,२,३],[४,५,६],[७,८,९]]
>> मॅट्रिक्स[२][३] ¶    // → ६  (ओळ २, स्तंभ ३)
```

## डिस्ट्रक्चरिंग

```zymbol
// अरे
arr = [१०, २०, ३०, ४०, ५०]
[a, b, c] = arr              // a=१०  b=२०  c=३०
[पहिला, *उर्वरित] = arr         // पहिला=१०  उर्वरित=[२०,३०,४०,५०]
[x, _, z] = [१, २, ३]        // _ टाकून देतो

// स्थानिक टपल
बिंदू = (१००, २००)
(px, py) = बिंदू             // px=१००  py=२००

// नामांकित टपल
व्यक्ती = (नाव: "अना", वय: २५, शहर: "माद्रिद")
(नाव: n, वय: v) = व्यक्ती   // n="अना"  v=२५
```

## टपल

टपल हे **अपरिवर्तनीय** क्रमबद्ध कंटेनर आहेत जे **वेगवेगळ्या प्रकारची** मूल्ये धारण करू शकतात.
अरेच्या विपरीत, निर्मितीनंतर घटक बदलता येत नाहीत.

```zymbol
// स्थानिक — मिश्र प्रकार अनुमत
बिंदू = (१०, २०)
>> बिंदू[१] ¶    // → १०

डेटा = (४२, "हॅलो", #१, ३.१४)
>> डेटा[३] ¶     // → #१

// नामांकित
व्यक्ती = (नाव: "अॅलिस", वय: २५)
>> व्यक्ती.नाव ¶    // → अॅलिस
>> व्यक्ती[१] ¶      // → अॅलिस  (निर्देशांक देखील कार्य करतो, १-आधारित)

// नेस्टेड
स्थान = (x: १०, y: २०)
p = (स्थान: स्थान, लेबल: "मूळ")
>> p.स्थान.x ¶        // → १०
```

**अपरिवर्तनीयता** — टपल घटक बदलण्याचा कोणताही प्रयत्न रनटाइम त्रुटी आहे:

```zymbol
t = (१०, २०, ३०)
// t[१] = ९९    // ❌ रनटाइम त्रुटी: टपल अपरिवर्तनीय आहेत
// t[१] += ५    // ❌ समान त्रुटी
```

बदललेले मूल्य मिळविण्यासाठी `$~` (कार्यात्मक अद्यतन) वापरा — एक **नवीन** टपल परत करते:

```zymbol
t = (१०, २०, ३०)
t२ = t[२]$~ ९९९
>> t ¶     // → (१०, २०, ३०)   ← मूळ अपरिवर्तित
>> t२ ¶    // → (१०, ९९९, ३०)

// नामांकित टपल — स्पष्टपणे पुनर्रचना करा
व्यक्ती = (नाव: "अॅलिस", वय: २५)
मोठे  = (नाव: व्यक्ती.नाव, वय: २६)
>> व्यक्ती.वय ¶    // → २५
>> मोठे.वय ¶     // → २६
```

## उच्च-क्रम फंक्शन्स

```zymbol
संख्या = [१, २, ३, ४, ५, ६, ७, ८, ९, १०]

दुप्पट_केलेले  = संख्या$> (x -> x * २)                  // map  → [२,४,६…२०]
सम    = संख्या$| (x -> x % २ == ०)           // filter → [२,४,६,८,१०]
एकूण    = संख्या$< (०, (संचयक, x) -> संचयक + x)     // reduce → ५५

// मध्यवर्तींद्वारे साखळी
पाऊल१ = संख्या$| (x -> x > ३)
पाऊल२ = पाऊल१$> (x -> x * x)
>> पाऊल२ ¶    // → [१६, २५, ३६, ४९, ६४, ८१, १००]

// नामांकित फंक्शन्स थेट HOF मध्ये पास केली जाऊ शकतात
दुप्पट(x) { <~ x * २ }
मोठे_आहे(x) { <~ x > ५ }
r = संख्या$> दुप्पट       // ✅ थेट संदर्भ
r = संख्या$| मोठे_आहे       // ✅ थेट संदर्भ
```

## पाइप ऑपरेटर

उजवीकडे नेहमी पाइप केलेल्या मूल्यासाठी `_` प्लेसहोल्डर आवश्यक असतो:

```zymbol
दुप्पट = x -> x * २
बेरीज = (a, b) -> a + b
वाढ = x -> x + १

प१ = ५ |> दुप्पट(_)        // → १०
प२ = १० |> बेरीज(_, ५)       // → १५
प३ = ५ |> बेरीज(२, _)        // → ७

// साखळीबद्ध
प = ५ |> दुप्पट(_) |> वाढ(_) |> दुप्पट(_)
>> प ¶    // → २२  (५→१०→११→२२)
```

## अंकीय मोड

Zymbol संख्या **69 युनिकोड अंक लिपींमध्ये** प्रदर्शित करू शकते — देवनागरी, अरबी-भारतीय, थाई, क्लिंगॉन pIqaD, गणितीय बोल्ड, एलसीडी सेगमेंट, आणि बरेच काही. सक्रिय मोड फक्त `>>` आउटपुटवर परिणाम करतो; अंतर्गत अंकगणित नेहमी बायनरी असते.

### एक लिपी सक्रिय करणे

लक्ष्य लिपीचे `0` आणि `9` अंक `#…#` च्या आत लिहा:

```zymbol
#०९#    // देवनागरी   (U+0966–U+096F)
#٠٩#    // अरबी-भारतीय (U+0660–U+0669)
#๐๙#    // थाई         (U+0E50–U+0E59)
#09#    // ASCII वर रीसेट करा
```

### आउटपुट आणि बुलियन

```zymbol
x = ४२
>> x ¶          // → ४२   (ASCII डीफॉल्ट)

#०९#
>> x ¶          // → ४२
>> ३.१४ ¶       // → ३.१४   (दशांश बिंदू नेहमी ASCII)
>> १ + २ ¶      // → ३

// बुलियन: # उपसर्ग नेहमी ASCII, अंक अनुकूलित होतो
>> #१ ¶         // → #१   (देवनागरीमध्ये सत्य)
>> #० ¶         // → #०   (असत्य — ० पूर्णांक शून्यापेक्षा वेगळे)

x = २८ > ४
>> x ¶          // → #१   (तुलना परिणाम सक्रिय मोडचे अनुसरण करतो)
```

### स्त्रोतामध्ये मूळ अंक लिटरल्स

कोणत्याही समर्थित लिपीचे अंक वैध लिटरल्स आहेत — श्रेणी, मोड्युलो, तुलनांमध्ये:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### कोणत्याही लिपीमध्ये बुलियन लिटरल्स

`#` + कोणत्याही ब्लॉकमधील अंक `0` किंवा `1` हे वैध बुलियन लिटरल आहे:

```zymbol
#٠٩#
सक्रिय = #١        // #१ सारखेच
>> सक्रिय ¶        // → #१
>> (#१ && #०) ¶ // → #०
```

> `#` **नेहमी ASCII** आहे. `#0` (असत्य) नेहमी प्रत्येक लिपीमध्ये `0` (पूर्णांक शून्य) पासून दृश्यदृष्ट्या वेगळे असते.

## डेटा ऑपरेटर

```zymbol
// प्रकार रूपांतरण कास्ट
f = ##.४२         // → ४२.०  (फ्लोटिंगमध्ये)
i = ###३.७        // → ४     (पूर्णांकात, गोल)
t = ##!३.७        // → ३     (पूर्णांकात, ट्रंकेट)

// स्ट्रिंगला संख्येत पार्स करा
v१ = #|"४२"|      // → ४२  (पूर्णांक)
v२ = #|"३.१४"|    // → ३.१४  (फ्लोटिंग)
v३ = #|"abc"|     // → "abc"  (सुरक्षित, त्रुटी नाही)

// गोल / ट्रंकेट
π = ३.१४१५९२६५
गोल२ = #.२|π|      // → ३.१४  (२ दशांश स्थानांवर गोल)
गोल४ = #.४|π|      // → ३.१४१६
ट्रंक२ = #!२|π|      // → ३.१४  (ट्रंकेट)

// संख्या स्वरूपन
स्वरूप = #,|१२३४५६७|  // → १,२३४,५६७  (स्वल्पविराम-विभक्त)
वैज्ञानिक = #^|१२३४५.६७८|    // → १.२३४५६७८e४  (वैज्ञानिक)

// बेस लिटरल्स
a = 0x41         // → 'A'  (हेक्स)
b = 0b01000001   // → 'A'  (बायनरी)
c = 0o101        // → 'A'  (ऑक्टल)

// बेस रूपांतरण आउटपुट
हेक्स = 0x|२५५|    // → "0x00FF"
बायनरी = 0b|६५|     // → "0b1000001"
ऑक्टल = 0o|८|      // → "0o10"
दशांश = 0d|२५५|    // → "0d0255"
```

| चिन्ह | कार्य | चिन्ह | कार्य |
|--------|-----------|--------|-----------|
| `=` | व्हेरिएबल | `$#` | लांबी |

| `@ N { }` | N वेळा लूप | `$>` | map |
| `@!` | तोडा | `$\|` | filter |
| `@>` | सुरू ठेवा | `$<` | reduce |
| `@:नाव { }` | लेबलयुक्त लूप | `$/ सीमांकक` | स्ट्रिंग विभाजित करा |
| `@:नाव!` | लेबल तोडा | `$++ a b c` | कॉन्कॅट बिल्ड |

| `->` | लॅम्ब्डा | `arr[i] = मूल्य` | घटक अद्यतनित करा (फक्त अरे) |

| `$^+` | चढता लावा (आदिम) | `$^-` | उतरता लावा (आदिम) |
| `$^` | तुलनाकारीने लावा (टपल) | `<~` | परत करा |
| `\|>` | पाइप | `!?` | प्रयत्न करा |
| `:!` | पकडा | `:>` | शेवटी |
| `#1` | सत्य | `#0` | असत्य |
| `$!` | त्रुटी आहे का | `$!!` | त्रुटी प्रसारित करा |
| `<#` | आयात करा | `#>` | निर्यात करा |
| `#` | मॉड्यूल घोषित करा | `::` | मॉड्यूल कॉल करा |
| `.` | फील्ड प्रवेश | `#?` | प्रकार मेटाडेटा |
| `#\|..\|` | संख्या पार्स करा | `##.` | फ्लोटिंगमध्ये रूपांतरित करा |
| `###` | पूर्णांकात रूपांतरित करा (गोल) | `##!` | पूर्णांकात रूपांतरित करा (ट्रंकेट) |
| `#.N\|..\|` | गोल | `#!N\|..\|` | ट्रंकेट |
| `#,\|..\|` | स्वल्पविराम स्वरूप | `#^\|..\|` | वैज्ञानिक |
| `#d0d9#` | अंकीय मोड स्विच करा | `#09#` | ASCII वर रीसेट करा |
| `<\ ..\>` | शेल कार्यान्वित करा | `>\<` | CLI आर्ग्युमेंट्स |
| `\ व्हेरिएबल` | स्पष्टपणे व्हेरिएबल नष्ट करा | `°x` / `x°` | हॉट व्याख्या (ऑटो-प्रारंभ) |
| `>>|` | टीयूआय ब्लॉक (वैकल्पिक स्क्रीन) | `>>~` | स्थानिक आउटपुट |
| `>>!` | स्क्रीन साफ करा | `>>?` | टर्मिनल आकार विचारा |
| `<<\|` | ब्लॉकिंग की प्रेस | `<<\|?` | नॉन-ब्लॉकिंग की प्रेस पोल |
| `@~ N` | N मिलिसेकंद झोपा | `$*` | स्ट्रिंग N वेळा पुनरावृत्ती करा |

## रिलीज चेंजलॉग

### v0.0.5 — टीयूआय प्रिमिटिव्ह, हॉट व्याख्या आणि स्ट्रिंग पुनरावृत्ती _(मे २०२६)_

- **ब्रेकिंग** मॅच आर्म सेपरेटर: `पॅटर्न : परिणाम` → `पॅटर्न => परिणाम`
- **ब्रेकिंग** आयात उपनाव: `<# पाथ <= उपनाव` → `<# पाथ => उपनाव`
- **ब्रेकिंग** निर्यात पुनर्नामकरण: `#> { fn <= पब }` → `#> { fn => पब }`
- **जोडले** टीयूआय ब्लॉक `>>| { }` — वैकल्पिक स्क्रीन + कच्चा मोड; बाहेर पडताना साफ करते
- **जोडले** स्थानिक आउटपुट `>>~ (ओळ, स्तंभ, BKS, fg, bg) > आयटम` — विरळ स्लॉट, ANSI 256 रंग
- **जोडले** की इनपुट `<<| व्हेरिएबल` (ब्लॉकिंग) आणि `<<|? व्हेरिएबल` (नॉन-ब्लॉकिंग पोल)
- **जोडले** `>>!` स्क्रीन साफ करा, `>>?` टर्मिनल आकार विचारा, `@~ N` N मिलिसेकंद झोपा
- **जोडले** हॉट व्याख्या `°x` / `x°` — लूपमध्ये पहिल्या वापरात व्हेरिएबल स्वयंचलित प्रारंभ
- **जोडले** स्ट्रिंग पुनरावृत्ती `स्ट्रिंग $* N` — एक स्ट्रिंग N वेळा पुनरावृत्ती करा
- **व्हीएम** समानता: 436/436 टेस्ट उत्तीर्ण

### v0.0.4 — 1-आधारित निर्देशांकन, प्रथम-श्रेणी फंक्शन्स आणि ब्लॉक मॉड्यूल्स _(एप्रिल २०२६)_

- **ब्रेकिंग** सर्व निर्देशांकन **1-आधारित** वर बदलले — `arr[1]` पहिला घटक आहे; `arr[0]` एक रनटाइम त्रुटी आहे
- **जोडले** नामांकित फंक्शन्स **प्रथम-श्रेणी मूल्ये** आहेत — थेट HOF मध्ये पास करा: `संख्या$> दुप्पट`
- **जोडले** मॉड्यूल्ससाठी **ब्लॉक सिंटॅक्स आवश्यक** — `# नाव { ... }` — फ्लॅट सिंटॅक्स काढून टाकला
- **जोडले** बहु-आयामी निर्देशांकन: `arr[i>j>k]` (नेव्हिगेशन), `arr[p ; q]` (फ्लॅट निष्कर्षण)
- **जोडले** प्रकार रूपांतरण कास्ट: `##.एक्सप्रेशन` (फ्लोटिंग), `###एक्सप्रेशन` (पूर्णांक गोल), `##!एक्सप्रेशन` (पूर्णांक ट्रंकेट)
- **जोडले** स्ट्रिंग विभाजन: `स्ट्रिंग$/ सीमांकक` — `Array(स्ट्रिंग)` परत करते
- **जोडले** कॉन्कॅट बिल्ड: `बेस$++ a b c` — एकाधिक आयटम जोडते
- **जोडले** टाइम्स लूप: `@ N { }` — अगदी N वेळा पुनरावृत्ती करा
- **जोडले** लेबलयुक्त लूप सिंटॅक्स: `@:नाव { }`, `@:नाव!`, `@:नाव>` — `@ @नाव` / `@! नाव` च्या जागी
- **जोडले** व्हेरिएबल स्कोप नियम: `_नाव` व्हेरिएबल्सचा अचूक ब्लॉक स्कोप असतो; `\ व्हेरिएबल` लवकर नष्ट करते
- **जोडले** मॅच तुलना पॅटर्न: `< 0 =>`, `> 5 =>`, `== 42 =>` इ.
- **जोडले** मॉड्यूल E013 त्रुटी: मॉड्यूल बॉडीमध्ये कार्यान्वित करण्यायोग्य विधाने प्रतिबंधित आहेत
- **दुरुस्त केले** `alias.CONST` आता योग्यरित्या सोडवते; `#>` फंक्शन व्याख्यांनंतर येऊ शकते
- **व्हीएम** पूर्ण समानता: 393/393 टेस्ट उत्तीर्ण
