# راهنمای Zymbol-Lang

> **بازبینی شده برای نسخه v0.0.5 — ۲۰۲۶-۰۵-۱۴**

**Zymbol-Lang** یک زبان برنامه‌نویسی نمادین است. بدون کلمات کلیدی — همه چیز نماد است. در هر زبان انسانی به یک شکل کار می‌کند.

- بدون `if`، `while`، `return` — فقط `?`، `@`، `<~`
- یونیکد کامل — شناسه‌ها در هر زبان یا ایموجی
- مستقل از زبان انسان — کد در همه جا یکسان است

**نسخه مفسر**: v0.0.5 | **پوشش آزمون**: ۴۳۶/۴۳۶ (برابری TW ↔ VM)

## اولیه‌های TUI

عملگرهای رابط ترمینال برای برنامه‌های تعاملی. بیشتر آنها نیاز به بلوک `>>| { }` (صفحه جایگزین + حالت خام) دارند.

```zymbol
>>| {
    >>!                             // پاک کردن صفحه جایگزین
    >>~ (۱, ۱, ۰, ۱۰) > "در حال اجرا"  // ردیف ۱, ستون ۱, fg=۱۰ (سبز)
    @~ ۱۰۰۰                         // توقف ۱ ثانیه (۱۰۰۰ میلی‌ثانیه)
    >>~ (۲, ۱) > "انجام شد."
}
// ترمینال به طور خودکار در خروج بازیابی می‌شود
```

```zymbol
// فشار کلید و اندازه ترمینال
>>| {
    [rows, cols] = >>?              // پرس و جو ابعاد ترمینال
    >>~ (۱, ۱) > "ترمینال: " rows " در " cols
    <<| key                         // خواندن فشار کلید مسدودکننده
    >>~ (۲, ۱) > "فشردید: " key
}
```

> `>>!` صفحه را پاک می‌کند. `>>?` `[rows, cols]` را برمی‌گرداند. `@~ N` N میلی‌ثانیه می‌خوابد.
> `<<|` یک فشار کلید را می‌خواند (مسدودکننده); `<<|?` بدون مسدود کردن نظرسنجی می‌کند (در صورت نبودن `'\0'` برمی‌گرداند).
> خروجی موقعیت‌دار تاپل: `(row, col, BKS, fg, bg)` — هر شکافی ممکن است با کاما حذف شود (`>>~ (,,, 196) > "red"`).
> BKS ماسک بیتی: `1`=پررنگ، `2`=کج، `4`=زیرخط. پالت ANSI ۲۵۶ رنگ (`0`=پیش‌فرض ترمینال).
> فقط tree-walker (به جز `>>!`، `>>?`، `@~`، `>>~` که در `--vm` نیز کار می‌کنند).

## توابع

```zymbol
add(a, b) { <~ a + b }
>> add(۳, ۴) ¶    // → ۷

factorial(n) {
    ? n <= ۱ { <~ ۱ }
    <~ n * factorial(n - ۱)
}
>> factorial(۵) ¶    // → ۱۲۰
```

```zymbol
    b = tmp
```

## آرایه‌ها

آرایه‌ها **قابل تغییر** هستند و عناصری از **نوع یکسان** دارند.

```zymbol
arr = [۱, ۲, ۳, ۴, ۵]

x = arr[۱]      // ۱ — دسترسی (۱-پایه: اولین عنصر)
x = arr[-۱]     // ۵ — اندیس منفی (آخرین عنصر)
x = arr$#       // ۵ — طول (از (arr$#) در >> استفاده کنید)

arr = arr$+ ۶            // افزودن → [۱,۲,۳,۴,۵,۶]
arr۲ = arr$+[۲] ۹۹       // درج در موقعیت ۲ (۱-پایه)
arr۳ = arr$- ۳           // حذف اولین وقوع مقدار
arr۴ = arr$-- ۳          // حذف همه وقوع‌ها
arr۵ = arr$-[۱]          // حذف در اندیس ۱ (اولین عنصر)
arr۶ = arr$-[۲..۳]       // حذف محدوده (۱-پایه, پایان شامل)

has = arr$? ۳            // #۱ — شامل است
pos = arr$?? ۳           // [۳] — همه اندیس‌های مقدار (۱-پایه)
sl = arr$[۱..۳]          // [۱,۲,۳] — برش (۱-پایه, پایان شامل)
sl۲ = arr$[۱:۳]          // [۱,۲,۳] — یکسان, نحو مبتنی بر تعداد

asc = arr$^+             // مرتب شده صعودی  (فقط اولیه‌ها)
desc = arr$^-            // مرتب شده نزولی (فقط اولیه‌ها)

// آرایه‌های تاپل نام‌دار/موقعیتی — از $^ با لامبدای مقایسه‌کننده استفاده کنید
db = [(name: "کارلا", age: ۲۸), (name: "آنا", age: ۲۵), (name: "باب", age: ۳۰)]
by_age  = db$^ (a, b -> a.age < b.age)    // صعودی بر اساس سن  (<)
by_name = db$^ (a, b -> a.name > b.name)  // نزولی بر اساس نام (>)
>> by_age[۱].name ¶     // → آنا
>> by_name[۱].name ¶    // → کارلا

// به‌روزرسانی مستقیم عنصر (فقط آرایه‌ها)
arr[۱] = ۹۹              // تخصیص
arr[۲] += ۵              // مرکب: +=  -=  *=  /=  %=  ^=

// به‌روزرسانی تابعی — یک آرایه جدید برمی‌گرداند؛ اصلی بدون تغییر
arr۲ = arr[۲]$~ ۹۹
```

> همه عملگرهای مجموعه یک **آرایه جدید** برمی‌گردانند. تخصیص برگشتی: `arr = arr$+ 4`.
> `$+` می‌تواند زنجیره شود: `arr = arr$+ 5$+ 6$+ 7`. سایر عملگرها از تخصیص‌های میانی استفاده می‌کنند.
> **اندیس‌گذاری ۱-پایه**: `arr[1]` اولین عنصر است؛ `arr[0]` خطای زمان اجراست.
> `$^+` / `$^-` آرایه‌های **اولیه** (اعداد، رشته‌ها) را مرتب می‌کنند. برای آرایه‌های تاپل از `$^` با لامبدای مقایسه‌کننده استفاده کنید — جهت در لامبدا رمزگذاری شده است (`<` = صعودی، `>` = نزولی).

**معناشناسی مقدار** — تخصیص یک آرایه به متغیر دیگر یک کپی مستقل ایجاد می‌کند:

```zymbol
a = [۱, ۲, ۳]
b = a
a[۱] = ۹۹
>> a ¶    // → [۹۹, ۲, ۳]
>> b ¶    // → [۱, ۲, ۳]   ← b تحت تأثیر قرار نمی‌گیرد
```

```zymbol

matrix = [[۱,۲,۳],[۴,۵,۶],[۷,۸,۹]]
```

## تاپل‌ها

تاپل‌ها ظروف مرتب **غیرقابل تغییر** هستند که می‌توانند مقادیری از **انواع مختلف** داشته باشند.
بر خلاف آرایه‌ها، عناصر پس از ایجاد قابل تغییر نیستند.

```zymbol
// موقعیتی — انواع مختلط مجاز
point = (۱۰, ۲۰)
>> point[۱] ¶    // → ۱۰

data = (۴۲, "سلام", #۱, ۳.۱۴)
>> data[۳] ¶     // → #۱

// نام‌دار
person = (name: "آلیس", age: ۲۵)
>> person.name ¶    // → آلیس
>> person[۱] ¶      // → آلیس  (اندیس نیز کار می‌کند, ۱-پایه)

// تو در تو
pos = (x: ۱۰, y: ۲۰)
p = (pos: pos, label: "مبدأ")
>> p.pos.x ¶        // → ۱۰
```

**غیرقابل تغییر** — هر تلاش برای تغییر عنصر تاپل خطای زمان اجراست:

```zymbol
t = (۱۰, ۲۰, ۳۰)
// t[۱] = ۹۹    // ❌ خطای زمان اجرا: تاپل‌ها غیرقابل تغییر هستند
// t[۱] += ۵    // ❌ همان خطا
```

برای استخراج مقدار تغییر یافته از `$~` (به‌روزرسانی تابعی) استفاده کنید — یک تاپل **جدید** برمی‌گرداند:

```zymbol
t = (۱۰, ۲۰, ۳۰)
t۲ = t[۲]$~ ۹۹۹
>> t ¶     // → (۱۰, ۲۰, ۳۰)   ← اصلی بدون تغییر
>> t۲ ¶    // → (۱۰, ۹۹۹, ۳۰)

// تاپل نام‌دار — بازسازی صریح
person = (name: "آلیس", age: ۲۵)
older  = (name: person.name, age: ۲۶)
>> person.age ¶    // → ۲۵
>> older.age ¶     // → ۲۶
```

```zymbol
step۱ = nums$| (x -> x > ۳)
step۲ = step۱$> (x -> x * x)
>> step۲ ¶    // → [۱۶, ۲۵, ۳۶, ۴۹, ۶۴, ۸۱, ۱۰۰]

// توابع نام‌دار می‌توانند مستقیماً به HOF ارسال شوند
double(x) { <~ x * ۲ }
is_big(x) { <~ x > ۵ }
r = nums$> double       // ✅ ارجاع مستقیم
r = nums$| is_big       // ✅ ارجاع مستقیم
```

```zymbol
r۳ = ۵ |> add(۲, _)        // → ۷

// زنجیره شده

>> r ¶    // → ۲۲  (۵→۱۰→۱۱→۲۲)
```

## حالت‌های عددی

Zymbol می‌تواند اعداد را در **۶۹ سیستم رقمی یونیکد** نمایش دهد — دواناگاری، عربی-هندی، تایلندی، Klingon pIqaD، ریاضی پررنگ، بخش‌های LCD، و بیشتر. حالت فعال فقط بر خروجی `>>` تأثیر می‌گذارد؛ محاسبات داخلی همیشه دودویی است.

### فعال کردن یک خط

رقم `0` و `9` خط مورد نظر را درون `#…#` بنویسید:

```zymbol
#०९#    // دواناگاری   (U+0966–U+096F)
#٠٩#    // عربی-هندی (U+0660–U+0669)
#๐๙#    // تایلندی     (U+0E50–U+0E59)
#09#    // بازنشانی به ASCII
```

### خروجی و بولی‌ها

```zymbol
x = ۴۲
>> x ¶          // → ۴۲   (ASCII پیش‌فرض)

#०९#
>> x ¶          // → ४२
>> ۳.۱۴ ¶       // → ३.१४   (نقطه اعشار همیشه ASCII)
>> ۱ + ۲ ¶      // → ३

// بولی‌ها: پیشوند # همیشه ASCII, رقم تطبیق می‌یابد
>> #۱ ¶         // → #१   (درست در دواناگاری)
>> #۰ ¶         // → #०   (نادرست — متمایز از ०  عدد صحیح صفر)

x = ۲۸ > ۴
>> x ¶          // → #१   (نتیجه مقایسه از حالت فعال پیروی می‌کند)
```

### Literalهای رقمی بومی در کد منبع

ارقام هر خط پشتیبانی شده literalهای معتبری هستند — در محدوده‌ها، مدول، مقایسه‌ها:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### literalهای بولی در هر خط

`#` + رقم `0` یا `1` از هر بلوک یک literal بولی معتبر است:

```zymbol
#٠٩#
active = #١        // همان #۱
>> active ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **همیشه ASCII** است. `#0` (نادرست) همیشه از نظر بصری از `0` (عدد صحیح صفر) در هر خط متمایز است.
