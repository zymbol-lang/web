> **v0.0.5 માટે સમીક્ષિત — 2026-05-14**

`°` (ડિગ્રી ચિહ્ન, U+00B0) પ્રથમ ઉપયોગમાં ચલને તેના તટસ્થ મૂલ્યમાં સ્વચાલિત રીતે પ્રારંભ કરે છે:

```zymbol

@ n:સંખ્યાઓ {
    °કુલ += n    // લૂપની ઉપર સ્વચાલિત રીતે ૦ પર પ્રારંભ; @ પછી ટકી રહે છે
```

> `°x` (ઉપસર્ગ) લૂપની ઉપર એન્કર કરે છે — પરિણામ `@` પછી ઍક્સેસ કરી શકાય છે.

## TUI પ્રિમિટિવ્સ

ઇન્ટરેક્ટિવ પ્રોગ્રામ્સ માટે ટર્મિનલ યુઝર ઇન્ટરફેસ ઓપરેટર્સ. મોટાભાગના `>>| { }` બ્લોકની જરૂર છે (વૈકલ્પિક સ્ક્રીન + કાચો મોડ).

```zymbol
>>| {
    >>!                             // વૈકલ્પિક સ્ક્રીન સાફ કરો
    >>~ (૧, ૧, ૦, ૧૦) > "ચાલી રહ્યું છે"  // પંક્તિ ૧, સ્તંભ ૧, fg=૧૦ (લીલો)
    @~ ૧૦૦૦                         // ૧ સેકન્ડ વિરામ (૧૦૦૦ ms)
    >>~ (૨, ૧) > "પૂર્ણ થયું."
}
// બહાર નીકળતી વખતે ટર્મિનલ સ્વચાલિત રીતે પુનઃસ્થાપિત થાય છે
```

```zymbol
// કી દબાવવી અને ટર્મિનલ માપ
>>| {
    [પંક્તિઓ, સ્તંભો] = >>?              // ટર્મિનલના પરિમાણો પૂછો
    >>~ (૧, ૧) > "ટર્મિનલ: " પંક્તિઓ " x " સ્તંભો
    <<| કી                         // બ્લૉકિંગ કી પ્રેસ વાંચો
    >>~ (૨, ૧) > "દબાવ્યું: " કી
}
```

> `>>!` સ્ક્રીન સાફ કરે છે. `>>?` `[પંક્તિઓ, સ્તંભો]` પરત કરે છે. `@~ N` N મિલિસેકંડ ઊંઘે છે.
> `<<|` એક કી પ્રેસ વાંચે છે (બ્લૉકિંગ); `<<|?` બ્લૉક કર્યા વિના પોલ કરે છે (કોઈ ન હોય તો `'\0'` પરત કરે છે).
> સ્થાનીય આઉટપુટ ટપલ: `(પંક્તિ, સ્તંભ, BKS, fg, bg)` — કોઈપણ સ્લોટ અલ્પવિરામ સાથે છોડી શકાય છે (`>>~ (,,, 196) > "લાલ"`).
> BKS બિટમાસ્ક: `1`=બોલ્ડ, `2`=ત્રાંસુ, `4`=નીચે લીટી. ANSI 256 રંગ પેલેટ (`0`=ટર્મિનલ ડિફૉલ્ટ).
> માત્ર tree-walker (`>>!`, `>>?`, `@~`, `>>~` સિવાય જે `--vm` માં પણ ચાલે છે).

## ફંક્શન્સ

```zymbol
સરવાળો(a, b) { <~ a + b }
>> સરવાળો(૩, ૪) ¶    // → ૭

અવયવી(n) {
    ? n <= ૧ { <~ ૧ }
    <~ n * અવયવી(n - ૧)
}
>> અવયવી(૫) ¶    // → ૧૨૦
```

ફંક્શન્સમાં **અલગ સ્કોપ** હોય છે — તેઓ બાહ્ય ચલો વાંચી શકતા નથી. કૉલર ચલો બદલવા માટે આઉટપુટ પરિમાણો `<~` નો ઉપયોગ કરો:

```zymbol
અદલાબદલી(a<~, b<~) {
    અસ્થાયી = a
    a = b
    b = અસ્થાયી
}
x = ૧૦
y = ૨૦
અદલાબદલી(x, y)
>> "x=" x " y=" y ¶    // → x=૨૦ y=૧૦
```

> નામાંકિત ફંક્શન્સ **પ્રથમ-વર્ગના મૂલ્યો** છે — સીધા પાસ કરો: `સંખ્યાઓ$> બમણું`. લપેટવા માટે: `x -> fn(x)` પણ માન્ય છે.

## લેમ્બડા અને ક્લોઝર

```zymbol
બમણું = x -> x * ૨
સરવાળો = (a, b) -> a + b
>> બમણું(૫) ¶    // → ૧૦
>> સરવાળો(૩, ૭) ¶  // → ૧૦

// બ્લોક લેમ્બડા
વર્ગીકરણ = x -> {
    ? x > ૦ { <~ "ધન" }
    _? x < ૦ { <~ "ઋણ" }
    <~ "શૂન્ય"
}

// ક્લોઝર — બાહ્ય સ્કોપ કેપ્ચર કરે છે
અવયવ = ૩
ત્રિગુણ = x -> x * અવયવ
>> ત્રિગુણ(૭) ¶    // → ૨૧

// ફેક્ટરી
સરવાળો_બનાવનાર(n) { <~ x -> x + n }
દશ_સરવાળો = સરવાળો_બનાવનાર(૧૦)
>> દશ_સરવાળો(૫) ¶    // → ૧૫

// ઍરેમાં
ઓપરેટર્સ = [x -> x+૧, x -> x*૨, x -> x*x]
>> ઓપરેટર્સ[૩](૫) ¶    // → ૨૫
```

## ઍરે

ઍરે **બદલી શકાય તેવા** છે અને **સમાન પ્રકારના** તત્વો ધરાવે છે.

```zymbol
arr = [૧, ૨, ૩, ૪, ૫]

x = arr[૧]      // ૧ — ઍક્સેસ (૧-આધારિત: પ્રથમ તત્વ)
x = arr[-૧]     // ૫ — નકારાત્મક સૂચકાંક (અંતિમ તત્વ)
x = arr$#       // ૫ — લંબાઈ (>> માં (arr$#) નો ઉપયોગ કરો)

arr = arr$+ ૬            // ઉમેરો → [૧,૨,૩,૪,૫,૬]
arr૨ = arr$+[૨] ૯૯       // સ્થિતિ ૨ પર દાખલ કરો (૧-આધારિત)
arr૩ = arr$- ૩           // મૂલ્યની પ્રથમ ઘટના દૂર કરો
arr૪ = arr$-- ૩          // બધી ઘટનાઓ દૂર કરો
arr૫ = arr$-[૧]          // સૂચકાંક ૧ પર દૂર કરો (પ્રથમ તત્વ)
arr૬ = arr$-[૨..૩]       // રેન્જ દૂર કરો (૧-આધારિત, અંત સમાવિષ્ટ)

છે = arr$? ૩            // #૧ — સમાવેશ કરે છે
સ્થાનો = arr$?? ૩           // [૩] — મૂલ્યના તમામ સૂચકાંકો (૧-આધારિત)
સ્લાઇસ = arr$[૧..૩]          // [૧,૨,૩] — સ્લાઇસ (૧-આધારિત, અંત સમાવિષ્ટ)
સ્લાઇસ૨ = arr$[૧:૩]          // [૧,૨,૩] — સમાન, ગણતરી-આધારિત સિન્ટેક્સ

ઊર્ધ્વ = arr$^+             // ઊર્ધ્વક્રમે સજાવો (માત્ર આદિમ)
નિમ્ન = arr$^-            // નિમ્નક્રમે સજાવો (માત્ર આદિમ)

// નામાંકિત/સ્થાનીય ટપલ ઍરે — તુલનાકારી લેમ્બડા સાથે $^ નો ઉપયોગ કરો
ડેટાબેઝ = [(નામ: "કાર્લા", ઉંમર: ૨૮), (નામ: "અના", ઉંમર: ૨૫), (નામ: "બોબ", ઉંમર: ૩૦)]
ઉંમર_પ્રમાણે  = ડેટાબેઝ$^ (a, b -> a.ઉંમર < b.ઉંમર)    // ઉંમર પ્રમાણે ઊર્ધ્વ (<)
નામ_પ્રમાણે = ડેટાબેઝ$^ (a, b -> a.નામ > b.નામ)   // નામ પ્રમાણે નિમ્ન (>)
>> ઉંમર_પ્રમાણે[૧].નામ ¶     // → અના
>> નામ_પ્રમાણે[૧].નામ ¶    // → કાર્લા

// સીધું તત્વ અપડેટ (માત્ર ઍરે)
arr[૧] = ૯૯              // અસાઇન
arr[૨] += ૫              // સંયુક્ત: +=  -=  *=  /=  %=  ^=

// કાર્યાત્મક અપડેટ — એક નવો ઍરે પરત કરે છે; મૂળ અપરિવર્તિત
arr૨ = arr[૨]$~ ૯૯
```

> બધા કલેક્શન ઓપરેટરો એક **નવો ઍરે** પરત કરે છે. પાછળ અસાઇન કરો: `arr = arr$+ 4`.
> `$+` ને સાંકળી શકાય છે: `arr = arr$+ 5$+ 6$+ 7`. અન્ય ઓપરેટરો મધ્યવર્તી અસાઇનમેન્ટનો ઉપયોગ કરે છે.
> **સૂચકાંકીકરણ 1-આધારિત**: `arr[1]` પ્રથમ તત્વ છે; `arr[0]` એક રનટાઇમ ભૂલ છે.
> `$^+` / `$^-` **આદિમ ઍરે** (સંખ્યાઓ, શબ્દમાળાઓ) સજાવે છે. ટપલ ઍરે માટે તુલનાકારી લેમ્બડા સાથે `$^` નો ઉપયોગ કરો — દિશા લેમ્બડામાં એન્કોડેડ છે (`<` = ઊર્ધ્વ, `>` = નિમ્ન).

**મૂલ્ય સિમેન્ટિક્સ** — ઍરેને બીજા ચલમાં અસાઇન કરવાથી સ્વતંત્ર નકલ બને છે:

```zymbol
a = [૧, ૨, ૩]
b = a
a[૧] = ૯૯
>> a ¶    // → [૯૯, ૨, ૩]
>> b ¶    // → [૧, ૨, ૩]   ← b અપ્રભાવિત
```

```zymbol
// નેસ્ટેડ ઍરે (૧-આધારિત સૂચકાંકીકરણ)
મેટ્રિક્સ = [[૧,૨,૩],[૪,૫,૬],[૭,૮,૯]]
>> મેટ્રિક્સ[૨][૩] ¶    // → ૬  (પંક્તિ ૨, સ્તંભ ૩)
```

## ટપલ

ટપલ એ **અપરિવર્તનીય** ક્રમબદ્ધ કન્ટેનર છે જે **વિવિધ પ્રકારના** મૂલ્યો ધરાવી શકે છે.
ઍરેથી વિપરીત, બનાવ્યા પછી તત્વો બદલી શકાતા નથી.

```zymbol
// સ્થાનીય — મિશ્ર પ્રકારો માન્ય
બિંદુ = (૧૦, ૨૦)
>> બિંદુ[૧] ¶    // → ૧૦

ડેટા = (૪૨, "હેલ્લો", #૧, ૩.૧૪)
>> ડેટા[૩] ¶     // → #૧

// નામાંકિત
વ્યક્તિ = (નામ: "એલિસ", ઉંમર: ૨૫)
>> વ્યક્તિ.નામ ¶    // → એલિસ
>> વ્યક્તિ[૧] ¶      // → એલિસ  (સૂચકાંક પણ કામ કરે છે, ૧-આધારિત)

// નેસ્ટેડ
સ્થાન = (x: ૧૦, y: ૨૦)
p = (સ્થાન: સ્થાન, લેબલ: "મૂળ")
>> p.સ્થાન.x ¶        // → ૧૦
```

**અપરિવર્તનીયતા** — ટપલ તત્વને બદલવાનો કોઈપણ પ્રયાસ રનટાઇમ ભૂલ છે:

```zymbol
t = (૧૦, ૨૦, ૩૦)
// t[૧] = ૯૯    // ❌ રનટાઇમ ભૂલ: ટપલ અપરિવર્તનીય છે
// t[૧] += ૫    // ❌ સમાન ભૂલ
```

બદલાયેલ મૂલ્ય મેળવવા માટે `$~` (કાર્યાત્મક અપડેટ) નો ઉપયોગ કરો — એક **નવો** ટપલ પરત કરે છે:

```zymbol
t = (૧૦, ૨૦, ૩૦)
t૨ = t[૨]$~ ૯૯૯
>> t ¶     // → (૧૦, ૨૦, ૩૦)   ← મૂળ અપરિવર્તિત
>> t૨ ¶    // → (૧૦, ૯૯૯, ૩૦)

// નામાંકિત ટપલ — સ્પષ્ટપણે પુનઃનિર્માણ કરો
વ્યક્તિ = (નામ: "એલિસ", ઉંમર: ૨૫)
મોટું  = (નામ: વ્યક્તિ.નામ, ઉંમર: ૨૬)
>> વ્યક્તિ.ઉંમર ¶    // → ૨૫
>> મોટું.ઉંમર ¶     // → ૨૬
```

## ઉચ્ચ-ક્રમ ફંક્શન્સ

```zymbol
સંખ્યાઓ = [૧, ૨, ૩, ૪, ૫, ૬, ૭, ૮, ૯, ૧૦]

બમણાં  = સંખ્યાઓ$> (x -> x * ૨)                  // map  → [૨,૪,૬…૨૦]
બેકી    = સંખ્યાઓ$| (x -> x % ૨ == ૦)           // filter → [૨,૪,૬,૮,૧૦]
કુલ    = સંખ્યાઓ$< (૦, (સંચયક, x) -> સંચયક + x)     // reduce → ૫૫

// મધ્યવર્તીઓ દ્વારા સાંકળો
પગલું૧ = સંખ્યાઓ$| (x -> x > ૩)
પગલું૨ = પગલું૧$> (x -> x * x)
>> પગલું૨ ¶    // → [૧૬, ૨૫, ૩૬, ૪૯, ૬૪, ૮૧, ૧૦૦]

// નામાંકિત ફંક્શન્સ સીધા HOF માં પાસ કરી શકાય છે
બમણું(x) { <~ x * ૨ }
મોટું_છે(x) { <~ x > ૫ }
r = સંખ્યાઓ$> બમણું       // ✅ સીધો સંદર્ભ
r = સંખ્યાઓ$| મોટું_છે       // ✅ સીધો સંદર્ભ
```

## પાઇપ ઓપરેટર

જમણી બાજુએ હંમેશા પાઇપ કરેલ મૂલ્ય માટે `_` પ્લેસહોલ્ડરની જરૂર પડે છે:

```zymbol
બમણું = x -> x * ૨
સરવાળો = (a, b) -> a + b
વધારો = x -> x + ૧

પ૧ = ૫ |> બમણું(_)        // → ૧૦
પ૨ = ૧૦ |> સરવાળો(_, ૫)       // → ૧૫
પ૩ = ૫ |> સરવાળો(૨, _)        // → ૭

// સાંકળેલું
પ = ૫ |> બમણું(_) |> વધારો(_) |> બમણું(_)
>> પ ¶    // → ૨૨  (૫→૧૦→૧૧→૨૨)
```

## અંકાત્મક મોડ્સ

Zymbol સંખ્યાઓને **69 યુનિકોડ અંક લિપિઓ**માં પ્રદર્શિત કરી શકે છે — દેવનાગરી, અરબી-ભારતીય, થાઇ, ક્લિન્ગોન pIqaD, ગાણિતિક બોલ્ડ, LCD સેગમેન્ટ્સ, અને વધુ. સક્રિય મોડ ફક્ત `>>` આઉટપુટને અસર કરે છે; આંતરિક અંકગણિત હંમેશા દ્વિઆધારી છે.

### લિપિ સક્રિય કરવી

લક્ષ્ય લિપિના `0` અને `9` અંકો `#…#` ની અંદર લખો:

```zymbol
#०९#    // દેવનાગરી   (U+0966–U+096F)
#٠٩#    // અરબી-ભારતીય (U+0660–U+0669)
#๐๙#    // થાઇ         (U+0E50–U+0E59)
#09#    // ASCII પર રીસેટ કરો
```

### આઉટપુટ અને બુલિયન

```zymbol
x = ૪૨
>> x ¶          // → ૪૨   (ASCII મૂળભૂત)

#०९#
>> x ¶          // → ४२
>> ૩.૧૪ ¶       // → ३.१४   (દશાંશ બિંદુ હંમેશા ASCII)
>> ૧ + ૨ ¶      // → ३

// બુલિયન: # ઉપસર્ગ હંમેશા ASCII, અંક અનુકૂલિત થાય છે
>> #૧ ¶         // → #१   (દેવનાગરીમાં સાચું)
>> #૦ ¶         // → #०   (ખોટું — ૦ પૂર્ણાંક શૂન્યથી અલગ)

x = ૨૮ > ૪
>> x ¶          // → #१   (સરખામણી પરિણામ સક્રિય મોડને અનુસરે છે)
```

### સ્ત્રોતમાં મૂળ અંક લિટરલ્સ

કોઈપણ સમર્થિત લિપિના અંકો માન્ય લિટરલ્સ છે — રેન્જ, મોડ્યુલો, સરખામણીઓમાં:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### કોઈપણ લિપિમાં બુલિયન લિટરલ્સ

`#` + કોઈપણ બ્લોકમાંથી અંક `0` અથવા `1` એક માન્ય બુલિયન લિટરલ છે:

```zymbol
#٠٩#
સક્રિય = #١        // #૧ ની સમકક્ષ
>> સક્રિય ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **હંમેશા ASCII** છે. `#0` (ખોટું) હંમેશા દરેક લિપિમાં `0` (પૂર્ણાંક શૂન્ય) થી દૃષ્ટિની રીતે અલગ છે.

```zymbol
// પ્રકાર રૂપાંતરણ કાસ્ટ
f = ##.૪૨         // → ૪૨.૦  (ફ્લોટિંગમાં)

// શબ્દમાળાને સંખ્યામાં પાર્સ કરો
v૧ = #|"૪૨"|      // → ૪૨  (પૂર્ણાંક)
v૨ = #|"૩.૧૪"|    // → ૩.૧૪  (ફ્લોટિંગ)
v૩ = #|"abc"|     // → "abc"  (સલામત, કોઈ ભૂલ નહીં)

// રાઉન્ડ / ટ્રંકેટ
π = ૩.૧૪૧૫૯૨૬૫
રાઉન્ડ૨ = #.૨|π|      // → ૩.૧૪  (૨ દશાંશ સ્થાનો પર રાઉન્ડ)
રાઉન્ડ૪ = #.૪|π|      // → ૩.૧૪૧૬
ટ્રંક૨ = #!૨|π|      // → ૩.૧૪  (ટ્રંકેટ)

// સંખ્યા ફોર્મેટિંગ
ફોર્મેટ = #,|૧૨૩૪૫૬૭|  // → ૧,૨૩૪,૫૬૭  (અલ્પવિરામ-અલગ)
વૈજ્ઞાનિક = #^|૧૨૩૪૫.૬૭૮|    // → ૧.૨૩૪૫૬૭૮e૪  (વૈજ્ઞાનિક)

// બેઝ લિટરલ્સ
a = 0x41         // → 'A'  (હેક્સ)
b = 0b01000001   // → 'A'  (બાઇનરી)
c = 0o101        // → 'A'  (ઓક્ટલ)

// બેઝ રૂપાંતરણ આઉટપુટ
હેક્સ = 0x|૨૫૫|    // → "0x00FF"
બાઇનરી = 0b|૬૫|     // → "0b1000001"
ઓક્ટલ = 0o|૮|      // → "0o10"
દશાંશ = 0d|૨૫૫|    // → "0d0255"
```

### v0.0.5 — TUI પ્રિમિટિવ્સ, હોટ વ્યાખ્યા અને શબ્દમાળા પુનરાવર્તન _(મે २०२६)_

- **બ્રેકિંગ** મેચ આર્મ સેપરેટર: `પેટર્ન : પરિણામ` → `પેટર્ન => પરિણામ`
- **બ્રેકિંગ** આયાત ઉપનામ: `<# પાથ <= ઉપનામ` → `<# પાથ => ઉપનામ`
- **બ્રેકિંગ** નિકાસ નામ બદલો: `#> { fn <= પબ }` → `#> { fn => પબ }`
- **ઉમેર્યું** TUI બ્લોક `>>| { }` — વૈકલ્પિક સ્ક્રીન + કાચો મોડ; બહાર નીકળતી વખતે સાફ કરે છે
- **ઉમેર્યું** સ્થાનીય આઉટપુટ `>>~ (પંક્તિ, સ્તંભ, BKS, fg, bg) > આઇટમ્સ` — છૂટાછવાયા સ્લોટ્સ, ANSI 256 રંગ
- **ઉમેર્યું** કી ઇનપુટ `<<| ચલ` (બ્લૉકિંગ) અને `<<|? ચલ` (નોન-બ્લૉકિંગ પોલ)
- **ઉમેર્યું** `>>!` સ્ક્રીન સાફ કરો, `>>?` ટર્મિનલ માપ પૂછો, `@~ N` N મિલિસેકંડ ઊંઘો
- **ઉમેર્યું** હોટ વ્યાખ્યા `°x` / `x°` — લૂપ્સમાં પ્રથમ ઉપયોગ પર ચલ સ્વચાલિત પ્રારંભ
- **ઉમેર્યું** શબ્દમાળા પુનરાવર્તન `શબ્દમાળા $* N` — એક શબ્દમાળાને N વાર પુનરાવર્તિત કરો
- **VM** સમાનતા: 436/436 ટેસ્ટ પાસ

### v0.0.4 — 1-આધારિત સૂચકાંકીકરણ, પ્રથમ-વર્ગ ફંક્શન્સ અને બ્લોક મોડ્યુલો _(એપ્રિલ २०२६)_

- **બ્રેકિંગ** તમામ સૂચકાંકીકરણ **1-આધારિત** માં બદલાયું — `arr[1]` પ્રથમ તત્વ છે; `arr[0]` એક રનટાઇમ ભૂલ છે
- **ઉમેર્યું** નામાંકિત ફંક્શન્સ **પ્રથમ-વર્ગના મૂલ્યો** છે — સીધા HOF માં પાસ કરો: `સંખ્યાઓ$> બમણું`
- **ઉમેર્યું** મોડ્યુલો માટે **બ્લોક સિન્ટેક્સ જરૂરી** — `# નામ { ... }` — ફ્લેટ સિન્ટેક્સ દૂર કર્યું
- **ઉમેર્યું** બહુ-પરિમાણીય સૂચકાંકીકરણ: `arr[i>j>k]` (નેવિગેશન), `arr[p ; q]` (ફ્લેટ નિષ્કર્ષણ)
- **ઉમેર્યું** પ્રકાર રૂપાંતરણ કાસ્ટ: `##.એક્સપ્રેશન` (ફ્લોટિંગ), `###એક્સપ્રેશન` (પૂર્ણાંક રાઉન્ડ), `##!એક્સપ્રેશન` (પૂર્ણાંક ટ્રંકેટ)
- **ઉમેર્યું** શબ્દમાળા વિભાજન: `શબ્દમાળા$/ સીમા` — `Array(શબ્દમાળા)` પરત કરે છે
- **ઉમેર્યું** કોન્કેટ બિલ્ડ: `આધાર$++ a b c` — બહુવિધ આઇટમ્સ ઉમેરે છે
- **ઉમેર્યું** ટાઇમ્સ લૂપ: `@ N { }` — બરાબર N વાર પુનરાવર્તન કરો
- **ઉમેર્યું** લેબલવાળો લૂપ સિન્ટેક્સ: `@:નામ { }`, `@:નામ!`, `@:નામ>` — `@ @નામ` / `@! નામ` ને બદલે છે

- **ઉમેર્યું** કોઈપણ લિપિમાં બુલિયન લિટરલ્સ — `#१` / `#०`, `#١` / `#٠`, વગેરે
- **ઉમેર્યું** ક્લિન્ગોન pIqaD અંકો (CSUR PUA U+F8F0–U+F8F9)
- **ઉમેર્યું** VM ઓપકોડ `SetNumeralMode` — tree-walker સાથે સંપૂર્ણ સમાનતા
- **બદલ્યું** બુલિયન `>>` આઉટપુટ હવે તમામ મોડ્સમાં `#` ઉપસર્ગ (`#0` / `#1`) સમાવે છે

### v0.0.2_01 — ઓપરેટર નામ બદલ્યું _(૩૦ માર્ચ २०२६)_

- **બદલ્યું** `c|..|` → `#,|..|` અને `e|..|` → `#^|..|` — `#` ઉપસર્ગ પરિવાર સાથે સુસંગત
- **ઉમેર્યું** નિકાસ ઉપનામ: જુદા નામ હેઠળ મોડ્યુલ સભ્યોને ફરીથી નિકાસ કરો

### v0.0.2 — કલેક્શન API પુનઃડિઝાઇન અને ઇન્સ્ટોલર્સ _(૨૪ માર્ચ २०२६)_

- **ઉમેર્યું** ઍરે અને શબ્દમાળાઓ માટે એકીકૃત `$` ઓપરેટર પરિવાર (`$#`, `$+`, `$?`, `$-`, `$[..]`)
- **ઉમેર્યું** ઍરે, ટપલ અને નામાંકિત ટપલ માટે ડિસ્ટ્રક્ચરિંગ અસાઇનમેન્ટ
- **ઉમેર્યું** નકારાત્મક સૂચકાંકો (`arr[-1]` = અંતિમ તત્વ)
- **ઉમેર્યું** મૂળ ઇન્સ્ટોલર્સ — Linux (deb/rpm/pkg/musl), macOS (Intel + Apple Silicon), Windows (MSI, winget)
