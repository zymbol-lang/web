# คู่มือ Zymbol-Lang

> **ตรวจสอบสำหรับ v0.0.5 — 2026-05-14**

**Zymbol-Lang** เป็นภาษาโปรแกรมเชิงสัญลักษณ์ ไม่มีคำสำคัญ — ทุกอย่างคือสัญลักษณ์ ทำงานเหมือนกันในทุกภาษาของมนุษย์

- ไม่มี `if`, `while`, `return` — มีเพียง `?`, `@`, `<~`
- ยูนิโคดเต็มรูปแบบ — ตัวระบุในภาษาใดๆ หรืออีโมจิ
- ไม่ขึ้นกับภาษามนุษย์ — โค้ดเหมือนกันทุกที่

## พื้นฐาน TUI

ตัวดำเนินการอินเทอร์เฟซผู้ใช้เทอร์มินัลสำหรับโปรแกรมโต้ตอบ ส่วนใหญ่ต้องใช้บล็อก `>>| { }` (หน้าจอสำรอง + โหมดดิบ)

```zymbol
>>| {
    >>!                             // ล้างหน้าจอสำรอง
    >>~ (๑, ๑, ๐, ๑๐) > "กำลังทำงาน"   // แถว ๑, คอลัมน์ ๑, fg=๑๐ (เขียว)
    @~ ๑๐๐๐                         // หยุด ๑ วินาที (๑๐๐๐ ms)
    >>~ (๒, ๑) > "เสร็จสมบูรณ์"
}
// เทอร์มินัลถูกเรียกคืนโดยอัตโนมัติเมื่อออก
```

```zymbol
// การกดปุ่มและขนาดเทอร์มินัล
>>| {
    [แถว, คอลัมน์] = >>?              // สอบถามขนาดเทอร์มินัล
    >>~ (๑, ๑) > "เทอร์มินัล: " แถว " x " คอลัมน์
    <<| ปุ่ม                         // อ่านการกดปุ่มแบบบล็อก
    >>~ (๒, ๑) > "กด: " ปุ่ม
}
```

> `>>!` ล้างหน้าจอ `>>?` คืนค่า `[แถว, คอลัมน์]` `@~ N` หลับ N มิลลิวินาที
> `<<|` อ่านการกดปุ่มเดียว (บล็อก); `<<|?` ตรวจสอบโดยไม่บล็อก (คืนค่า `'\0'` หากไม่มี)
> ทัปเปิลเอาต์พุตตำแหน่ง: `(แถว, คอลัมน์, BKS, fg, bg)` — สามารถละเว้นช่องใดๆ ด้วยจุลภาคได้ (`>>~ (,,, 196) > "แดง"`)
> BKS บิตมาสก์: `1`=ตัวหนา, `2`=ตัวเอียง, `4`=ขีดเส้นใต้ จานสี ANSI 256 สี (`0`=ค่าเริ่มต้นเทอร์มินัล)
> เฉพาะ tree-walker (ยกเว้น `>>!`, `>>?`, `@~`, `>>~` ซึ่งทำงานใน `--vm` ด้วย)

```zymbol
arr๒ = arr[๒]$~ ๙๙
```

> ตัวดำเนินการสะสมทั้งหมดคืนค่า **อาร์เรย์ใหม่** กำหนดกลับ: `arr = arr$+ 4`
> `$+` สามารถเชื่อมโยงได้: `arr = arr$+ 5$+ 6$+ 7` ตัวดำเนินการอื่นใช้การกำหนดระหว่างกลาง
> **การจัดทำดัชนี 1-based**: `arr[1]` เป็นองค์ประกอบแรก; `arr[0]` เป็นข้อผิดพลาดขณะรันไทม์
> `$^+` / `$^-` จัดเรียง **อาร์เรย์พื้นฐาน** (ตัวเลข, สตริง) สำหรับอาร์เรย์ทัปเปิล ใช้ `$^` กับแลมบ์ดาเปรียบเทียบ — ทิศทางถูกเข้ารหัสในแลมบ์ดา (`<` = น้อยไปมาก, `>` = มากไปน้อย)

**ความหมายค่า** — การกำหนดอาร์เรย์ให้ตัวแปรอื่นสร้างสำเนาอิสระ:

```zymbol
a = [๑, ๒, ๓]
b = a
a[๑] = ๙๙
>> a ¶    // → [๙๙, ๒, ๓]
>> b ¶    // → [๑, ๒, ๓]   ← b ไม่ได้รับผลกระทบ
```

```zymbol
// อาร์เรย์ซ้อน (การจัดทำดัชนี ๑-based)
เมทริกซ์ = [[๑,๒,๓],[๔,๕,๖],[๗,๘,๙]]
>> เมทริกซ์[๒][๓] ¶    // → ๖  (แถว ๒, คอลัมน์ ๓)
```

## ทัปเปิล

ทัปเปิลเป็นคอนเทนเนอร์เรียงลำดับ **เปลี่ยนแปลงไม่ได้** ที่สามารถเก็บค่าหลาย **ชนิดต่างกัน**
ต่างจากอาร์เรย์ องค์ประกอบไม่สามารถเปลี่ยนแปลงได้หลังสร้าง

```zymbol
// เชิงตำแหน่ง — อนุญาตชนิดผสม
จุด = (๑๐, ๒๐)
>> จุด[๑] ¶    // → ๑๐

ข้อมูล = (๔๒, "สวัสดี", #๑, ๓.๑๔)
>> ข้อมูล[๓] ¶     // → #๑

// มีชื่อ
บุคคล = (ชื่อ: "อลิซ", อายุ: ๒๕)
>> บุคคล.ชื่อ ¶    // → อลิซ
>> บุคคล[๑] ¶      // → อลิซ  (ดัชนีก็ใช้ได้, ๑-based)

// ซ้อน
ตำแหน่ง = (x: ๑๐, y: ๒๐)
p = (ตำแหน่ง: ตำแหน่ง, ป้ายกำกับ: "ต้นทาง")
>> p.ตำแหน่ง.x ¶        // → ๑๐
```

**ความไม่เปลี่ยนแปลงได้** — ความพยายามใดๆ ที่จะแก้ไของค์ประกอบทัปเปิลเป็นข้อผิดพลาดขณะรันไทม์:

```zymbol
t = (๑๐, ๒๐, ๓๐)
// t[๑] = ๙๙    // ❌ ข้อผิดพลาดขณะรันไทม์: ทัปเปิลเปลี่ยนแปลงไม่ได้
// t[๑] += ๕    // ❌ ข้อผิดพลาดเดียวกัน
```

เพื่อให้ได้ค่าที่แก้ไขแล้ว ใช้ `$~` (การอัปเดตเชิงฟังก์ชัน) — คืนค่า **ทัปเปิลใหม่**:

```zymbol
t = (๑๐, ๒๐, ๓๐)
t๒ = t[๒]$~ ๙๙๙
>> t ¶     // → (๑๐, ๒๐, ๓๐)   ← ต้นฉบับไม่เปลี่ยนแปลง
>> t๒ ¶    // → (๑๐, ๙๙๙, ๓๐)

// ทัปเปิลที่มีชื่อ — สร้างใหม่โดยชัดเจน
บุคคล = (ชื่อ: "อลิซ", อายุ: ๒๕)
แก่กว่า  = (ชื่อ: บุคคล.ชื่อ, อายุ: ๒๖)
>> บุคคล.อายุ ¶    // → ๒๕
>> แก่กว่า.อายุ ¶     // → ๒๖
```

## ฟังก์ชันลำดับสูง

```zymbol
ตัวเลข = [๑, ๒, ๓, ๔, ๕, ๖, ๗, ๘, ๙, ๑๐]

สองเท่า  = ตัวเลข$> (x -> x * ๒)                  // map  → [๒,๔,๖…๒๐]
คู่    = ตัวเลข$| (x -> x % ๒ == ๐)           // filter → [๒,๔,๖,๘,๑๐]

ขั้น๑ = ตัวเลข$| (x -> x > ๓)
ขั้น๒ = ขั้น๑$> (x -> x * x)
>> ขั้น๒ ¶    // → [๑๖, ๒๕, ๓๖, ๔๙, ๖๔, ๘๑, ๑๐๐]

// ฟังก์ชันที่มีชื่อสามารถส่งตรงไปยัง HOF ได้

ใหญ่(x) { <~ x > ๕ }
r = ตัวเลข$> สองเท่า       // ✅ การอ้างอิงโดยตรง
r = ตัวเลข$| ใหญ่       // ✅ การอ้างอิงโดยตรง
```

## โหมดตัวเลข

Zymbol สามารถแสดงตัวเลขใน **69 ชุดหลักยูนิโคด** — เทวนาครี, อารบิก-อินดิก, ไทย, Klingon pIqaD, ตัวหนาทางคณิตศาสตร์, ส่วน LCD, และอื่นๆ โหมดที่ใช้งานมีผลกับเอาต์พุต `>>` เท่านั้น; เลขคณิตภายในเป็นฐานสองเสมอ

### การเปิดใช้งานชุดอักษร

เขียนหลัก `0` และ `9` ของชุดอักษรเป้าหมายภายใน `#…#`:

```zymbol
#०९#    // เทวนาครี   (U+0966–U+096F)
#٠٩#    // อารบิก-อินดิก (U+0660–U+0669)
#๐๙#    // ไทย         (U+0E50–U+0E59)
#09#    // รีเซ็ตเป็น ASCII
```

### เอาต์พุตและบูลีน

```zymbol
x = ๔๒
>> x ¶          // → ๔๒   (ASCII ค่าเริ่มต้น)

#๐๙#
>> x ¶          // → ๔๒
>> ๓.๑๔ ¶       // → ๓.๑๔   (จุดทศนิยมเป็น ASCII เสมอ)
>> ๑ + ๒ ¶      // → ๓

// บูลีน: คำนำหน้า # เป็น ASCII เสมอ หลักจะปรับตาม
>> #๑ ¶         // → #๑   (จริงในเทวนาครี)
>> #๐ ¶         // → #๐   (เท็จ — แตกต่างจาก ๐ จำนวนเต็มศูนย์)

x = ๒๘ > ๔
>> x ¶          // → #๑   (ผลการเปรียบเทียบเป็นไปตามโหมดที่ใช้งาน)
```

### ลิเทอรัลหลักดั้งเดิมในซอร์ส

หลักของชุดอักษรที่รองรับใดๆ เป็นลิเทอรัลที่ถูกต้อง — ในช่วง, มอดุโล, การเปรียบเทียบ:

```zymbol
#๐๙#

@ i:๑..๑๕ {
    ? i % ๑๕ == ๐ { >> "FizzBuzz" ¶ }
    _? i % ๓  == ๐ { >> "Fizz" ¶ }
    _? i % ๕  == ๐ { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### ลิเทอรัลบูลีนในชุดอักษรใดๆ

`#` + หลัก `0` หรือ `1` จากบล็อกใดๆ เป็นลิเทอรัลบูลีนที่ถูกต้อง:

```zymbol
#๐๙#
ทำงาน = #๑        // เหมือนกับ #๑
>> ทำงาน ¶        // → #๑
>> (#๑ && #๐) ¶ // → #๐
```

> `#` **เป็น ASCII เสมอ** `#0` (เท็จ) แยกความแตกต่างทางการมองเห็นจาก `0` (จำนวนเต็มศูนย์) ได้เสมอในทุกชุดอักษร

| `\|>` | ไปป์ | `!?` | ลอง |

## บันทึกการเปลี่ยนแปลงรุ่น

### v0.0.5 — พื้นฐาน TUI, นิยามร้อน และการทำซ้ำสตริง _(พฤษภาคม 2026)_

- **การเปลี่ยนแปลงที่ทำลาย** ตัวคั่นแขนจับคู่: `รูปแบบ : ผลลัพธ์` → `รูปแบบ => ผลลัพธ์`
- **การเปลี่ยนแปลงที่ทำลาย** นามแฝงการนำเข้า: `<# พาธ <= นามแฝง` → `<# พาธ => นามแฝง`
- **การเปลี่ยนแปลงที่ทำลาย** การเปลี่ยนชื่อส่งออก: `#> { fn <= สาธารณะ }` → `#> { fn => สาธารณะ }`
- **เพิ่ม** บล็อก TUI `>>| { }` — หน้าจอสำรอง + โหมดดิบ; ล้างเมื่อออก
- **เพิ่ม** เอาต์พุตตำแหน่ง `>>~ (แถว, คอลัมน์, BKS, fg, bg) > รายการ` — ช่องว่างกระจัดกระจาย, สี ANSI 256
- **เพิ่ม** อินพุตปุ่ม `<<| ตัวแปร` (บล็อก) และ `<<|? ตัวแปร` (ตรวจสอบแบบไม่บล็อก)
- **เพิ่ม** `>>!` ล้างหน้าจอ, `>>?` สอบถามขนาดเทอร์มินัล, `@~ N` หลับ N มิลลิวินาที
- **เพิ่ม** นิยามร้อน `°x` / `x°` — เริ่มต้นตัวแปรอัตโนมัติเมื่อใช้ครั้งแรกในลูป
- **เพิ่ม** การทำซ้ำสตริง `สตริง $* N` — ทำซ้ำสตริง N ครั้ง
- **VM** ความเท่าเทียม: 436/436 การทดสอบผ่าน

### v0.0.4 — การจัดทำดัชนี 1-based, ฟังก์ชันระดับหนึ่ง และโมดูลบล็อก _(เมษายน 2026)_

- **การเปลี่ยนแปลงที่ทำลาย** การจัดทำดัชนีทั้งหมดเปลี่ยนเป็น **1-based** — `arr[1]` เป็นองค์ประกอบแรก; `arr[0]` เป็นข้อผิดพลาดขณะรันไทม์
- **เพิ่ม** ฟังก์ชันที่มีชื่อเป็น **ค่าระดับหนึ่ง** — ส่งตรงไปยัง HOF: `ตัวเลข$> สองเท่า`
- **เพิ่ม** **ไวยากรณ์บล็อกจำเป็น** สำหรับโมดูล: `# ชื่อ { ... }` — เอาไวยากรณ์แบบราบออก
- **เพิ่ม** การจัดทำดัชนีหลายมิติ: `arr[i>j>k]` (การนำทาง), `arr[p ; q]` (การแยกแบบราบ)
- **เพิ่ม** การแปลงชนิด: `##.นิพจน์` (ทศนิยม), `###นิพจน์` (จำนวนเต็มปัดเศษ), `##!นิพจน์` (จำนวนเต็มตัด)
- **เพิ่ม** การแยกสตริง: `สตริง$/ ตัวคั่น` — คืนค่า `Array(สตริง)`
- **เพิ่ม** การสร้างแบบต่อ: `ฐาน$++ a b c` — เพิ่มหลายรายการ
- **เพิ่ม** ลูปตามจำนวนครั้ง: `@ N { }` — ทำซ้ำ N ครั้งพอดี
- **เพิ่ม** ไวยากรณ์ลูปที่มีป้ายกำกับ: `@:ชื่อ { }`, `@:ชื่อ!`, `@:ชื่อ>` — แทนที่ `@ @ชื่อ` / `@! ชื่อ`
- **เพิ่ม** กฎขอบเขตตัวแปร: ตัวแปร `_ชื่อ` มีขอบเขตบล็อกที่แน่นอน; `\ ตัวแปร` ทำลายก่อนกำหนด
- **เพิ่ม** รูปแบบการเปรียบเทียบการจับคู่: `< 0 =>`, `> 5 =>`, `== 42 =>` ฯลฯ
- **เพิ่ม** ข้อผิดพลาดโมดูล E013: ห้ามใช้คำสั่งที่ปฏิบัติการได้ในเนื้อความโมดูล
- **แก้ไข** `alias.CONST` แก้ไขได้ถูกต้องแล้ว; `#>` สามารถมาหลังนิยามฟังก์ชันได้
- **VM** ความเท่าเทียมสมบูรณ์: 393/393 การทดสอบผ่าน
