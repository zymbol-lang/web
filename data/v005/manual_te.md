# Zymbol-Lang మాన్యువల్

> **v0.0.5 కోసం సమీక్షించబడింది — 2026-05-14**

**Zymbol-Lang** ఒక ప్రతీకాత్మక ప్రోగ్రామింగ్ భాష. కీవర్డ్లు లేవు — ప్రతిదీ ప్రతీక. ఏ మానవ భాషలోనైనా ఒకేలా పనిచేస్తుంది.

- `if`, `while`, `return` లేవు — కేవలం `?`, `@`, `<~` మాత్రమే
- పూర్తి యూనికోడ్ — ఏ భాషలోనైనా లేదా ఎమోజీలోనైనా గుర్తింపుదార్లు
- మానవ భాషా తటస్థం — కోడ్ ప్రతిచోటా ఒకేలా ఉంటుంది

**ఇంటర్ప్రెటర్ వెర్షన్**: v0.0.5 | **టెస్ట్ కవరేజ్**: 436/436 (TW ↔ VM సమానత్వం)

`°` (డిగ్రీ గుర్తు, U+00B0) మొదటి ఉపయోగంలో వేరియబుల్ను దాని తటస్థ విలువకు స్వయంచాలకంగా ప్రారంభిస్తుంది:

```zymbol
సంఖ్యలు = [౩, ౧, ౪, ౧, ౫]
@ n:సంఖ్యలు {
    °మొత్తం += n    // లూప్ పైన స్వయంచాలకంగా ౦ కి ప్రారంభించు; @ తర్వాత మనుగడలో ఉంటుంది
}
>> మొత్తం ¶         // → ౧౪
```

> `°x` (ఉపసర్గ) లూప్ పైన లంగరు వేస్తుంది — ఫలితం `@` తర్వాత అందుబాటులో ఉంటుంది.
> `x°` (ప్రత్యయం) లూప్ లోపల లంగరు వేస్తుంది — లూప్ ముగిసినప్పుడు చనిపోతుంది.

## డేటా రకాలు

| రకం | సాహిత్యం | `#?` ట్యాగ్ | గమనికలు |
|------|---------|----------|---------|
| పూర్ణాంకం | `42`, `-7` | `###` | 64-బిట్ గుర్తుతో |
| తేలియాడే | `3.14`, `1.5e10` | `##.` | శాస్త్రీయ సంజ్ఞామానం అనుమతించబడుతుంది |
| తీగ | `"టెక్స్ట్"` | `##"` | ఇంటర్పోలేషన్: `"హలో {పేరు}"` |
| అక్షరం | `'అ'` | `##'` | ఒకే యూనికోడ్ అక్షరం |
| బూలియన్ | `#1`, `#0` | `##?` | సంఖ్యాత్మకం కాదు — `#1 ≠ 1` |
| శ్రేణి | `[1, 2, 3]` | `##]` | సజాతీయ మూలకాలు |
| టపుల్ | `(a, b)` | `##)` | స్థానిక |
| నామకరణం చేసిన టపుల్ | `(x: 1, y: 2)` | `##)` | నామకరణం చేసిన ఫీల్డ్లు |

## అవుట్పుట్ మరియు ఇన్పుట్

```zymbol
>> "హలో" ¶                       // ¶ లేదా \\ స్పష్టమైన కొత్త పంక్తి కోసం
```

## టియుఐ ప్రిమిటివ్లు

ఇంటరాక్టివ్ ప్రోగ్రామ్ల కోసం టెర్మినల్ యూజర్ ఇంటర్ఫేస్ ఆపరేటర్లు. చాలావరకు `>>| { }` బ్లాక్ అవసరం (ప్రత్యామ్నాయ స్క్రీన్ + ముడి మోడ్).

```zymbol
>>| {
    >>!                             // ప్రత్యామ్నాయ స్క్రీన్ను క్లియర్ చేయండి
    >>~ (౧, ౧, ౦, ౧౦) > "నడుస్తోంది"   // వరుస ౧, నిలువు వరుస ౧, fg=౧౦ (ఆకుపచ్చ)
    @~ ౧౦౦౦                         // ౧ సెకను విరామం (౧౦౦౦ ms)
    >>~ (౨, ౧) > "పూర్తయింది."
}
// నిష్క్రమించినప్పుడు టెర్మినల్ స్వయంచాలకంగా పునరుద్ధరించబడుతుంది
```

```zymbol
// కీ ప్రెస్ మరియు టెర్మినల్ పరిమాణం
>>| {
    [వరుసలు, నిలువు_వరుసలు] = >>?              // టెర్మినల్ కొలతలను ప్రశ్నించండి
    >>~ (౧, ౧) > "టెర్మినల్: " వరుసలు " x " నిలువు_వరుసలు
    <<| కీ                         // బ్లాకింగ్ కీ ప్రెస్ చదవండి
    >>~ (౨, ౧) > "నొక్కారు: " కీ
}
```

> `>>!` స్క్రీన్ను క్లియర్ చేస్తుంది. `>>?` `[వరుసలు, నిలువు_వరుసలు]` తిరిగి ఇస్తుంది. `@~ N` N మిల్లీసెకన్లు నిద్రిస్తుంది.
> `<<|` ఒక కీ ప్రెస్ను చదువుతుంది (బ్లాకింగ్); `<<|?` బ్లాక్ చేయకుండా పోల్ చేస్తుంది (ఏదీ లేకపోతే `'\0'` తిరిగి ఇస్తుంది).
> స్థానిక అవుట్పుట్ టపుల్: `(వరుస, నిలువు_వరుస, BKS, fg, bg)` — ఏదైనా స్లాట్ను కామాతో వదిలివేయవచ్చు (`>>~ (,,, 196) > "ఎరుపు"`).
> BKS బిట్మాస్క్: `1`=బోల్డ్, `2`=వాలుగా, `4`=అండర్లైన్. ANSI 256 రంగుల ప్యాలెట్ (`0`=టెర్మినల్ డిఫాల్ట్).
> tree-walker మాత్రమే (`>>!`, `>>?`, `@~`, `>>~` తప్ప అవి `--vm` లో కూడా నడుస్తాయి).

```zymbol
s = "హలో ప్రపంచం"
పొడవు = s$#                  // ౧౧
ఉప = s$[౧..౫]             // "హలో"  (౧-ఆధారిత, చివరి చేర్చబడింది)
```

## లూప్లు

```zymbol

>> n ¶                        // → ౧౨౮  (while)
```

## ఫంక్షన్లు

```zymbol
కూడిక(a, b) { <~ a + b }
>> కూడిక(౩, ౪) ¶    // → ౭

కారణాంకం(n) {
    ? n <= ౧ { <~ ౧ }
    <~ n * కారణాంకం(n - ౧)
}
>> కారణాంకం(౫) ¶    // → ౧౨౦
```

ఫంక్షన్లు **వేరుచేయబడిన స్కోప్** కలిగి ఉంటాయి — అవి బాహ్య వేరియబుల్లను చదవలేవు. కాలర్ వేరియబుల్లను మార్చడానికి అవుట్పుట్ పారామీటర్లు `<~` ఉపయోగించండి:

```zymbol
మార్పిడి(a<~, b<~) {
    తాత్కాలిక = a
    a = b
    b = తాత్కాలిక
}
x = ౧౦
y = ౨౦
మార్పిడి(x, y)
>> "x=" x " y=" y ¶    // → x=౨౦ y=౧౦
```

> నామకరణం చేసిన ఫంక్షన్లు **మొదటి-తరగతి విలువలు** — నేరుగా పంపండి: `సంఖ్యలు$> రెట్టింపు`. చుట్టడానికి: `x -> fn(x)` కూడా చెల్లుతుంది.

## లాంబ్డాలు మరియు క్లోజర్లు

```zymbol
రెట్టింపు = x -> x * ౨
కూడిక = (a, b) -> a + b
>> రెట్టింపు(౫) ¶    // → ౧౦
>> కూడిక(౩, ౭) ¶  // → ౧౦

// బ్లాక్ లాంబ్డా
వర్గీకరణ = x -> {
    ? x > ౦ { <~ "ధనాత్మకం" }
    _? x < ౦ { <~ "ఋణాత్మకం" }
    <~ "సున్నా"
}

// క్లోజర్ — బాహ్య స్కోప్ను సంగ్రహిస్తుంది
కారకం = ౩
మూడు_రెట్లు = x -> x * కారకం
>> మూడు_రెట్లు(౭) ¶    // → ౨౧

// ఫ్యాక్టరీ
కూడిక_నిర్మాత(n) { <~ x -> x + n }
పది_కూడిక = కూడిక_నిర్మాత(౧౦)
>> పది_కూడిక(౫) ¶    // → ౧౫

// శ్రేణిలో
ఆపరేటర్లు = [x -> x+౧, x -> x*౨, x -> x*x]
>> ఆపరేటర్లు[౩](౫) ¶    // → ౨౫
```

## శ్రేణి

శ్రేణులు **మార్చగలిగేవి** మరియు **ఒకే రకమైన** మూలకాలను కలిగి ఉంటాయి.

```zymbol
arr = [౧, ౨, ౩, ౪, ౫]

x = arr[౧]      // ౧ — ప్రాప్యత (౧-ఆధారిత: మొదటి మూలకం)
x = arr[-౧]     // ౫ — ప్రతికూల సూచిక (చివరి మూలకం)
x = arr$#       // ౫ — పొడవు (>> లో (arr$#) ఉపయోగించండి)

arr = arr$+ ౬            // జోడించు → [౧,౨,౩,౪,౫,౬]
arr౨ = arr$+[౨] ౯౯       // స్థానం ౨ వద్ద చొప్పించు (౧-ఆధారిత)
arr౩ = arr$- ౩           // విలువ యొక్క మొదటి సంఘటనను తీసివేయి
arr౪ = arr$-- ౩          // అన్ని సంఘటనలను తీసివేయి
arr౫ = arr$-[౧]          // సూచిక ౧ వద్ద తీసివేయి (మొదటి మూలకం)
arr౬ = arr$-[౨..౩]       // పరిధిని తీసివేయి (౧-ఆధారిత, చివరి చేర్చబడింది)

ఉంది = arr$? ౩            // #౧ — కలిగి ఉంటుంది
స్థానాలు = arr$?? ౩           // [౩] — విలువ యొక్క అన్ని సూచికలు (౧-ఆధారిత)
ముక్క = arr$[౧..౩]          // [౧,౨,౩] — ముక్క (౧-ఆధారిత, చివరి చేర్చబడింది)
ముక్క౨ = arr$[౧:౩]          // [౧,౨,౩] — అదే, గణన-ఆధారిత వాక్యనిర్మాణం

ఆరోహణ = arr$^+             // ఆరోహణ క్రమంలో క్రమీకరించు (ప్రాథమికాలు మాత్రమే)
అవరోహణ = arr$^-            // అవరోహణ క్రమంలో క్రమీకరించు (ప్రాథమికాలు మాత్రమే)

// నామకరణం/స్థానిక టపుల్ శ్రేణి — పోలిక లాంబ్డాతో $^ ఉపయోగించండి
డేటాబేస్ = [(పేరు: "కార్లా", వయస్సు: ౨౮), (పేరు: "అనా", వయస్సు: ౨౫), (పేరు: "బాబ్", వయస్సు: ౩౦)]
వయస్సు_ప్రకారం  = డేటాబేస్$^ (a, b -> a.వయస్సు < b.వయస్సు)    // వయస్సు ప్రకారం ఆరోహణ (<)
పేరు_ప్రకారం = డేటాబేస్$^ (a, b -> a.పేరు > b.పేరు)   // పేరు ప్రకారం అవరోహణ (>)
>> వయస్సు_ప్రకారం[౧].పేరు ¶     // → అనా
>> పేరు_ప్రకారం[౧].పేరు ¶    // → కార్లా

// ప్రత్యక్ష మూలకం నవీకరణ (శ్రేణులు మాత్రమే)
arr[౧] = ౯౯              // అసైన్
arr[౨] += ౫              // సమ్మేళనం: +=  -=  *=  /=  %=  ^=

// క్రియాత్మక నవీకరణ — కొత్త శ్రేణిని తిరిగి ఇస్తుంది; అసలు మారదు
arr౨ = arr[౨]$~ ౯౯
```

> అన్ని సేకరణ ఆపరేటర్లు **కొత్త శ్రేణిని** తిరిగి ఇస్తాయి. తిరిగి అసైన్ చేయండి: `arr = arr$+ 4`.
> `$+` గొలుసు కట్టవచ్చు: `arr = arr$+ 5$+ 6$+ 7`. ఇతర ఆపరేటర్లు మధ్యంతర అసైన్మెంట్లను ఉపయోగిస్తాయి.
> **సూచికీకరణ 1-ఆధారిత**: `arr[1]` మొదటి మూలకం; `arr[0]` ఒక రన్టైమ్ ఎర్రర్.
> `$^+` / `$^-` **ప్రాథమిక శ్రేణులను** (సంఖ్యలు, తీగలు) క్రమీకరిస్తాయి. టపుల్ శ్రేణుల కోసం పోలిక లాంబ్డాతో `$^` ఉపయోగించండి — దిశ లాంబ్డాలో ఎన్కోడ్ చేయబడింది (`<` = ఆరోహణ, `>` = అవరోహణ).

**విలువ అర్థశాస్త్రం** — శ్రేణిని మరొక వేరియబుల్కు అసైన్ చేయడం స్వతంత్ర కాపీని సృష్టిస్తుంది:

```zymbol
a = [౧, ౨, ౩]
b = a
a[౧] = ౯౯
>> a ¶    // → [౯౯, ౨, ౩]
>> b ¶    // → [౧, ౨, ౩]   ← b ప్రభావితం కాదు
```

```zymbol
// గూడుకట్టిన శ్రేణులు (౧-ఆధారిత సూచికీకరణ)
మాత్రిక = [[౧,౨,౩],[౪,౫,౬],[౭,౮,౯]]
>> మాత్రిక[౨][౩] ¶    // → ౬  (వరుస ౨, నిలువు వరుస ౩)
```

## నిర్మాణ విచ్ఛేదన

```zymbol
// శ్రేణి
arr = [౧౦, ౨౦, ౩౦, ౪౦, ౫౦]
[a, b, c] = arr              // a=౧౦  b=౨౦  c=౩౦
[మొదటిది, *మిగిలినవి] = arr         // మొదటిది=౧౦  మిగిలినవి=[౨౦,౩౦,౪౦,౫౦]
[x, _, z] = [౧, ౨, ౩]        // _ తోసిపుచ్చుతుంది

// స్థానిక టపుల్
బిందువు = (౧౦౦, ౨౦౦)
(px, py) = బిందువు             // px=౧౦౦  py=౨౦౦

// నామకరణం చేసిన టపుల్
వ్యక్తి = (పేరు: "అనా", వయస్సు: ౨౫, నగరం: "మాడ్రిడ్")
(పేరు: n, వయస్సు: v) = వ్యక్తి   // n="అనా"  v=౨౫
```

## టపుల్

టపుల్లు **మార్చలేని** క్రమబద్ధమైన కంటైనర్లు, ఇవి **విభిన్న రకాల** విలువలను కలిగి ఉంటాయి.
శ్రేణుల వలె కాకుండా, సృష్టించిన తర్వాత మూలకాలను మార్చలేరు.

```zymbol
// స్థానిక — మిశ్రమ రకాలు అనుమతించబడతాయి
బిందువు = (౧౦, ౨౦)
>> బిందువు[౧] ¶    // → ౧౦

డేటా = (౪౨, "హలో", #౧, ౩.౧౪)
>> డేటా[౩] ¶     // → #౧

// నామకరణం చేసినది
వ్యక్తి = (పేరు: "అలిస్", వయస్సు: ౨౫)
>> వ్యక్తి.పేరు ¶    // → అలిస్
>> వ్యక్తి[౧] ¶      // → అలిస్  (సూచిక కూడా పనిచేస్తుంది, ౧-ఆధారిత)

// గూడుకట్టినది
స్థానం = (x: ౧౦, y: ౨౦)
p = (స్థానం: స్థానం, లేబుల్: "మూలం")
>> p.స్థానం.x ¶        // → ౧౦
```

**మార్పులేనితనం** — టపుల్ మూలకాన్ని మార్చడానికి చేసే ఏ ప్రయత్నమైనా రన్టైమ్ ఎర్రర్:

```zymbol
t = (౧౦, ౨౦, ౩౦)
// t[౧] = ౯౯    // ❌ రన్టైమ్ ఎర్రర్: టపుల్లు మార్చలేనివి
// t[౧] += ౫    // ❌ అదే ఎర్రర్
```

మార్పు చేసిన విలువను పొందడానికి `$~` (క్రియాత్మక నవీకరణ) ఉపయోగించండి — **కొత్త** టపుల్ను తిరిగి ఇస్తుంది:

```zymbol
t = (౧౦, ౨౦, ౩౦)
t౨ = t[౨]$~ ౯౯౯
>> t ¶     // → (౧౦, ౨౦, ౩౦)   ← అసలు మారదు
>> t౨ ¶    // → (౧౦, ౯౯౯, ౩౦)

// నామకరణం చేసిన టపుల్ — స్పష్టంగా పునర్నిర్మించండి
వ్యక్తి = (పేరు: "అలిస్", వయస్సు: ౨౫)
పెద్దది  = (పేరు: వ్యక్తి.పేరు, వయస్సు: ౨౬)
>> వ్యక్తి.వయస్సు ¶    // → ౨౫
>> పెద్దది.వయస్సు ¶     // → ౨౬
```

## ఉన్నత-క్రమ ఫంక్షన్లు

```zymbol
సంఖ్యలు = [౧, ౨, ౩, ౪, ౫, ౬, ౭, ౮, ౯, ౧౦]

రెట్టింపు_చేసినవి  = సంఖ్యలు$> (x -> x * ౨)                  // map  → [౨,౪,౬…౨౦]
సరి    = సంఖ్యలు$| (x -> x % ౨ == ౦)           // filter → [౨,౪,౬,౮,౧౦]
మొత్తం    = సంఖ్యలు$< (౦, (సంచాయకం, x) -> సంచాయకం + x)     // reduce → ౫౫

// మధ్యవర్తుల ద్వారా గొలుసు కట్టండి
దశ౧ = సంఖ్యలు$| (x -> x > ౩)
దశ౨ = దశ౧$> (x -> x * x)
>> దశ౨ ¶    // → [౧౬, ౨౫, ౩౬, ౪౯, ౬౪, ౮౧, ౧౦౦]

// నామకరణం చేసిన ఫంక్షన్లను నేరుగా HOF కు పంపవచ్చు
రెట్టింపు(x) { <~ x * ౨ }
పెద్దది_అవునా(x) { <~ x > ౫ }
r = సంఖ్యలు$> రెట్టింపు       // ✅ ప్రత్యక్ష సూచన
r = సంఖ్యలు$| పెద్దది_అవునా       // ✅ ప్రత్యక్ష సూచన
```

## పైప్ ఆపరేటర్

కుడి వైపు ఎల్లప్పుడూ పైప్ చేసిన విలువ కోసం `_` ప్లేస్హోల్డర్ అవసరం:

```zymbol
రెట్టింపు = x -> x * ౨
కూడిక = (a, b) -> a + b
పెంపు = x -> x + ౧

ఫ౧ = ౫ |> రెట్టింపు(_)        // → ౧౦
ఫ౨ = ౧౦ |> కూడిక(_, ౫)       // → ౧౫
ఫ౩ = ౫ |> కూడిక(౨, _)        // → ౭

// గొలుసు కట్టినది
ఫ = ౫ |> రెట్టింపు(_) |> పెంపు(_) |> రెట్టింపు(_)
>> ఫ ¶    // → ౨౨  (౫→౧౦→౧౧→౨౨)
```

```zymbol
<# ./mylib => ఎం

>> ఎం::కూడిక(౩, ౪) ¶    // → ౭  (అంతర్గత పేరు _అంతర్గత_కూడిక దాచబడింది)
```

> **మాడ్యూల్ నియమాలు**: `# పేరు { }` లోపల `#>`, ఫంక్షన్ నిర్వచనాలు, మరియు సాహిత్య వేరియబుల్/స్థిరాంక ప్రారంభకాలు మాత్రమే అనుమతించబడతాయి. అమలు చేయదగిన స్టేట్మెంట్లు (`>>`, `<<`, లూప్లు, మొదలైనవి) E013 లోపాన్ని పెంచుతాయి.

## సంఖ్యా విధానాలు

Zymbol సంఖ్యలను **69 యూనికోడ్ అంకెల లిపుల్లో** ప్రదర్శించగలదు — దేవనాగరి, అరబిక్-ఇండిక్, థాయ్, క్లింగాన్ pIqaD, గణిత బోల్డ్, LCD సెగ్మెంట్లు, మరియు మరిన్ని. చురుకైన విధానం `>>` అవుట్పుట్ను మాత్రమే ప్రభావితం చేస్తుంది; అంతర్గత అంకగణితం ఎల్లప్పుడూ బైనరీ.

### ఒక లిపిని సక్రియం చేయడం

లక్ష్య లిపి యొక్క `0` మరియు `9` అంకెలను `#…#` లోపల వ్రాయండి:

```zymbol
#०९#    // దేవనాగరి   (U+0966–U+096F)
#٠٩#    // అరబిక్-ఇండిక్ (U+0660–U+0669)
#๐๙#    // థాయ్         (U+0E50–U+0E59)
#09#    // ASCII కు రీసెట్ చేయండి
```

### అవుట్పుట్ మరియు బూలియన్లు

```zymbol
x = ౪౨
>> x ¶          // → ౪౨   (ASCII డిఫాల్ట్)

#०९#
>> x ¶          // → ४२
>> ౩.౧౪ ¶       // → ३.१४   (దశాంశ బిందువు ఎల్లప్పుడూ ASCII)
>> ౧ + ౨ ¶      // → ३

// బూలియన్లు: # ఉపసర్గ ఎల్లప్పుడూ ASCII, అంకె అనుగుణంగా మారుతుంది
>> #౧ ¶         // → #१   (దేవనాగరిలో నిజం)
>> #౦ ¶         // → #०   (తప్పు — ౦ పూర్ణాంక సున్నాకి భిన్నంగా ఉంటుంది)

x = ౨౮ > ౪
>> x ¶          // → #१   (పోలిక ఫలితం చురుకైన విధానాన్ని అనుసరిస్తుంది)
```

### మూలంలో స్థానిక అంకెల సాహిత్యాలు

ఏదైనా మద్దతు ఇవ్వబడిన లిపి యొక్క అంకెలు చెల్లుబాటు అయ్యే సాహిత్యాలు — పరిధుల్లో, మాడ్యులోలో, పోలికల్లో:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### ఏ లిపిలోనైనా బూలియన్ సాహిత్యాలు

`#` + ఏదైనా బ్లాక్ నుండి అంకె `0` లేదా `1` చెల్లుబాటు అయ్యే బూలియన్ సాహిత్యం:

```zymbol
#٠٩#
చురుకుగా = #١        // #౧ కు సమానం
>> చురుకుగా ¶        // → #१
>> (#١ && #٠) ¶ // → #०
```

> `#` **ఎల్లప్పుడూ ASCII**. `#0` (తప్పు) ప్రతి లిపిలోనూ `0` (పూర్ణాంక సున్నా) నుండి దృశ్యపరంగా భిన్నంగా ఉంటుంది.

## డేటా ఆపరేటర్లు

```zymbol
// రకం మార్పిడి అచ్చులు
f = ##.౪౨         // → ౪౨.౦  (తేలియాడేదానికి)
i = ###౩.౭        // → ౪     (పూర్ణాంకానికి, చుట్టుముట్టు)
t = ##!౩.౭        // → ౩     (పూర్ణాంకానికి, తుంచివేత)

// తీగను సంఖ్యగా పార్స్ చేయండి
v౧ = #|"౪౨"|      // → ౪౨  (పూర్ణాంకం)
v౨ = #|"౩.౧౪"|    // → ౩.౧౪  (తేలియాడేది)
v౩ = #|"abc"|     // → "abc"  (సురక్షితం, లోపం లేదు)

// చుట్టుముట్టు / తుంచివేత
π = ౩.౧౪౧౫౯౨౬౫
చుట్టుముట్టు౨ = #.౨|π|      // → ౩.౧౪  (౨ దశాంశ స్థానాలకు చుట్టుముట్టు)
చుట్టుముట్టు౪ = #.౪|π|      // → ౩.౧౪౧౬
తుంచివేత౨ = #!౨|π|      // → ౩.౧౪  (తుంచివేత)

// సంఖ్య ఫార్మాటింగ్
ఫార్మాట్ = #,|౧౨౩౪౫౬౭|  // → ౧,౨౩౪,౫౬౭  (కామా-వేరు చేయబడింది)
శాస్త్రీయ = #^|౧౨౩౪౫.౬౭౮|    // → ౧.౨౩౪౫౬౭౮e౪  (శాస్త్రీయ)

// ఆధార సాహిత్యాలు
a = 0x41         // → 'A'  (హెక్సాడెసిమల్)
b = 0b01000001   // → 'A'  (బైనరీ)
c = 0o101        // → 'A'  (ఆక్టల్)

// ఆధార మార్పిడి అవుట్పుట్
హెక్స్ = 0x|౨౫౫|    // → "0x00FF"
బైనరీ = 0b|౬౫|     // → "0b1000001"
ఆక్టల్ = 0o|౮|      // → "0o10"
దశాంశ = 0d|౨౫౫|    // → "0d0255"
```

## షెల్ ఇంటిగ్రేషన్

```zymbol
తేదీ = <\ date +%Y-%m-%d \>     // stdout ను సంగ్రహిస్తుంది (చివరిలో \n చేర్చబడింది)
```

## విడుదల మార్పుల చిట్టా

### v0.0.5 — టియుఐ ప్రిమిటివ్లు, హాట్ నిర్వచనం & తీగ పునరావృతం _(మే 2026)_

- **బ్రేకింగ్** సరిపోలిక చేయి విభజన: `నమూనా : ఫలితం` → `నమూనా => ఫలితం`
- **బ్రేకింగ్** దిగుమతి మారుపేరు: `<# మార్గం <= మారుపేరు` → `<# మార్గం => మారుపేరు`
- **బ్రేకింగ్** ఎగుమతి పేరు మార్పు: `#> { fn <= పబ్ }` → `#> { fn => పబ్ }`
- **జోడించబడింది** టియుఐ బ్లాక్ `>>| { }` — ప్రత్యామ్నాయ స్క్రీన్ + ముడి మోడ్; నిష్క్రమించినప్పుడు క్లియర్ చేస్తుంది
- **జోడించబడింది** స్థానిక అవుట్పుట్ `>>~ (వరుస, నిలువు_వరుస, BKS, fg, bg) > అంశాలు` — విరళ స్లాట్లు, ANSI 256 రంగు
- **జోడించబడింది** కీ ఇన్పుట్ `<<| వేరియబుల్` (బ్లాకింగ్) మరియు `<<|? వేరియబుల్` (నాన్-బ్లాకింగ్ పోల్)
- **జోడించబడింది** `>>!` స్క్రీన్ను క్లియర్ చేయి, `>>?` టెర్మినల్ పరిమాణాన్ని ప్రశ్నించు, `@~ N` N మిల్లీసెకన్లు నిద్రించు
- **జోడించబడింది** హాట్ నిర్వచనం `°x` / `x°` — లూప్లలో మొదటి ఉపయోగంలో వేరియబుల్ ఆటో ప్రారంభం
- **జోడించబడింది** తీగ పునరావృతం `తీగ $* N` — ఒక తీగను N సార్లు పునరావృతం చేయి

### v0.0.4 — 1-ఆధారిత సూచికీకరణ, మొదటి-తరగతి ఫంక్షన్లు & బ్లాక్ మాడ్యూల్స్ _(ఏప్రిల్ 2026)_

- **బ్రేకింగ్** అన్ని సూచికీకరణ **1-ఆధారిత** గా మార్చబడింది — `arr[1]` మొదటి మూలకం; `arr[0]` ఒక రన్టైమ్ ఎర్రర్
- **జోడించబడింది** నామకరణం చేసిన ఫంక్షన్లు **మొదటి-తరగతి విలువలు** — నేరుగా HOF కు పంపండి: `సంఖ్యలు$> రెట్టింపు`
- **జోడించబడింది** మాడ్యూల్స్ కోసం **బ్లాక్ వాక్యనిర్మాణం అవసరం** — `# పేరు { ... }` — ఫ్లాట్ వాక్యనిర్మాణం తీసివేయబడింది
- **జోడించబడింది** బహుళ-డైమెన్షనల్ సూచికీకరణ: `arr[i>j>k]` (నావిగేషన్), `arr[p ; q]` (ఫ్లాట్ ఎక్స్ట్రాక్షన్)
- **జోడించబడింది** రకం మార్పిడి అచ్చులు: `##.ఎక్స్ప్రెషన్` (తేలియాడేది), `###ఎక్స్ప్రెషన్` (పూర్ణాంకం చుట్టుముట్టు), `##!ఎక్స్ప్రెషన్` (పూర్ణాంకం తుంచివేత)
- **జోడించబడింది** తీగ విభజన: `తీగ$/ విభజన గుర్తు` — `Array(తీగ)` ను తిరిగి ఇస్తుంది
- **జోడించబడింది** కాంకాట్ బిల్డ్: `ఆధారం$++ a b c` — బహుళ అంశాలను జోడిస్తుంది
- **జోడించబడింది** టైమ్స్ లూప్: `@ N { }` — సరిగ్గా N సార్లు పునరావృతం చేయండి
- **జోడించబడింది** లేబుల్ చేసిన లూప్ వాక్యనిర్మాణం: `@:పేరు { }`, `@:పేరు!`, `@:పేరు>` — `@ @పేరు` / `@! పేరు` ను భర్తీ చేస్తుంది
- **జోడించబడింది** వేరియబుల్ స్కోప్ నియమాలు: `_పేరు` వేరియబుల్లు ఖచ్చితమైన బ్లాక్ స్కోప్ కలిగి ఉంటాయి; `\ వేరియబుల్` ముందుగా నాశనం చేస్తుంది
- **జోడించబడింది** సరిపోలిక పోలిక నమూనాలు: `< 0 =>`, `> 5 =>`, `== 42 =>` మొదలైనవి
- **జోడించబడింది** మాడ్యూల్ E013 ఎర్రర్: మాడ్యూల్ బాడీలో అమలు చేయదగిన స్టేట్మెంట్లు నిషేధించబడ్డాయి
- **సరిచేయబడింది** `alias.CONST` ఇప్పుడు సరిగ్గా పరిష్కరిస్తుంది; `#>` ఫంక్షన్ నిర్వచనాల తర్వాత రావచ్చు
- **VM** పూర్తి సమానత్వం: 393/393 పరీక్షలు ఉత్తీర్ణమయ్యాయి
