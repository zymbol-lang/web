`°` (डिग्री चिह्न, U+00B0) पहले उपयोग पर एक चर को उसके तटस्थ मान में स्वचालित रूप से आरंभ करता है:

```zymbol
संख्याएँ = [३, १, ४, १, ५]
@ n:संख्याएँ {
    °कुल += n    // लूप के ऊपर स्वचालित रूप से ० पर आरंभ; @ के बाद बना रहता है
}
>> कुल ¶         // → १४
```

> `°x` (उपसर्ग) लूप के ऊपर लंगर डालता है — परिणाम `@` के बाद पहुँच योग्य है।
> `x°` (प्रत्यय) लूप के अंदर लंगर डालता है — जब लूप समाप्त होता है तो मर जाता है।

## टीयूआई प्रिमिटिव

इंटरैक्टिव प्रोग्रामों के लिए टर्मिनल यूज़र इंटरफेस ऑपरेटर। अधिकांश को `>>| { }` ब्लॉक की आवश्यकता होती है (वैकल्पिक स्क्रीन + कच्चा मोड)।

```zymbol
>>| {
    >>!                             // वैकल्पिक स्क्रीन साफ़ करें
    >>~ (१, १, ०, १०) > "चल रहा है"   // पंक्ति १, स्तंभ १, fg=१० (हरा)
    @~ १०००                         // १ सेकंड रुकें (१००० ms)
    >>~ (२, १) > "पूर्ण हुआ।"
}
// बाहर निकलने पर टर्मिनल स्वचालित रूप से पुनर्स्थापित हो जाता है
```

```zymbol
// कुंजी दबाना और टर्मिनल आकार
>>| {
    [पंक्तियाँ, स्तंभ] = >>?              // टर्मिनल के आयाम पूछें
    >>~ (१, १) > "टर्मिनल: " पंक्तियाँ " x " स्तंभ
    <<| कुंजी                         // ब्लॉकिंग कुंजी प्रेस पढ़ें
    >>~ (२, १) > "दबाया: " कुंजी
}
```

> `>>!` स्क्रीन साफ़ करता है। `>>?` `[पंक्तियाँ, स्तंभ]` लौटाता है। `@~ N` N मिलीसेकंड सोता है।
> `<<|` एक कुंजी प्रेस पढ़ता है (ब्लॉकिंग); `<<|?` बिना ब्लॉक किए पोल करता है (यदि कोई नहीं तो `'\0'` लौटाता है)।
> स्थानीय आउटपुट टपल: `(पंक्ति, स्तंभ, BKS, fg, bg)` — कोई भी स्लॉट अल्पविराम से छोड़ा जा सकता है (`>>~ (,,, 196) > "लाल"`)।
> BKS बिटमास्क: `1`=बोल्ड, `2`=इटैलिक, `4`=अंडरलाइन। ANSI 256 रंग पैलेट (`0`=टर्मिनल डिफ़ॉल्ट)।
> केवल tree-walker (`>>!`, `>>?`, `@~`, `>>~` को छोड़कर जो `--vm` में भी चलते हैं)।

## ऐरे

ऐरे **परिवर्तनशील** होते हैं और **समान प्रकार के** तत्व रखते हैं।

```zymbol
arr = [१, २, ३, ४, ५]

x = arr[१]      // १ — पहुँच (१-आधारित: पहला तत्व)
x = arr[-१]     // ५ — ऋणात्मक सूचकांक (अंतिम तत्व)
x = arr$#       // ५ — लंबाई (>> में (arr$#) का उपयोग करें)

arr = arr$+ ६            // जोड़ें → [१,२,३,४,५,६]
arr२ = arr$+[२] ९९       // स्थिति २ पर डालें (१-आधारित)
arr३ = arr$- ३           // मान की पहली घटना हटाएँ
arr४ = arr$-- ३          // सभी घटनाएँ हटाएँ
arr५ = arr$-[१]          // सूचकांक १ पर हटाएँ (पहला तत्व)
arr६ = arr$-[२..३]       // रेंज हटाएँ (१-आधारित, अंत सम्मिलित)

है = arr$? ३            // #१ — सम्मिलित है
स्थान = arr$?? ३           // [३] — मान के सभी सूचकांक (१-आधारित)
स्लाइस = arr$[१..३]          // [१,२,३] — स्लाइस (१-आधारित, अंत सम्मिलित)
स्लाइस२ = arr$[१:३]          // [१,२,३] — समान, गणना-आधारित सिंटैक्स

आरोही = arr$^+             // आरोही क्रम में सजाएँ (केवल आदिम)
अवरोही = arr$^-            // अवरोही क्रम में सजाएँ (केवल आदिम)

// नामांकित/स्थानीय टपल ऐरे — तुलनाकारी लैम्ब्डा के साथ $^ का उपयोग करें
डेटाबेस = [(नाम: "कार्ला", आयु: २८), (नाम: "अना", आयु: २५), (नाम: "बॉब", आयु: ३०)]
आयु_अनुसार  = डेटाबेस$^ (a, b -> a.आयु < b.आयु)    // आयु के अनुसार आरोही (<)
नाम_अनुसार = डेटाबेस$^ (a, b -> a.नाम > b.नाम)   // नाम के अनुसार अवरोही (>)
>> आयु_अनुसार[१].नाम ¶     // → अना
>> नाम_अनुसार[१].नाम ¶    // → कार्ला

// सीधा तत्व अद्यतन (केवल ऐरे)
arr[१] = ९९              // असाइन
arr[२] += ५              // यौगिक: +=  -=  *=  /=  %=  ^=

// कार्यात्मक अद्यतन — एक नया ऐरे लौटाता है; मूल अपरिवर्तित
arr२ = arr[२]$~ ९९
```

> सभी कलेक्शन ऑपरेटर एक **नया ऐरे** लौटाते हैं। वापस असाइन करें: `arr = arr$+ 4`।
> `$+` को श्रृंखलाबद्ध किया जा सकता है: `arr = arr$+ 5$+ 6$+ 7`। अन्य ऑपरेटर मध्यवर्ती असाइनमेंट का उपयोग करते हैं।
> **सूचकांकन 1-आधारित**: `arr[1]` पहला तत्व है; `arr[0]` एक रनटाइम त्रुटि है।
> `$^+` / `$^-` **आदिम ऐरे** (संख्याएँ, स्ट्रिंग) सजाते हैं। टपल ऐरे के लिए तुलनाकारी लैम्ब्डा के साथ `$^` का उपयोग करें — दिशा लैम्ब्डा में एन्कोडेड है (`<` = आरोही, `>` = अवरोही)।

**मान शब्दार्थ** — ऐरे को दूसरे चर में असाइन करने से एक स्वतंत्र प्रतिलिपि बनती है:

```zymbol
a = [१, २, ३]
b = a
a[१] = ९९
>> a ¶    // → [९९, २, ३]
>> b ¶    // → [१, २, ३]   ← b अप्रभावित
```

```zymbol
// नेस्टेड ऐरे (१-आधारित सूचकांकन)
मैट्रिक्स = [[१,२,३],[४,५,६],[७,८,९]]
>> मैट्रिक्स[२][३] ¶    // → ६  (पंक्ति २, स्तंभ ३)
```

## डिस्ट्रक्चरिंग

```zymbol
// ऐरे
arr = [१०, २०, ३०, ४०, ५०]
[a, b, c] = arr              // a=१०  b=२०  c=३०
[प्रथम, *शेष] = arr         // प्रथम=१०  शेष=[२०,३०,४०,५०]
[x, _, z] = [१, २, ३]        // _ त्यागता है

// स्थानीय टपल
बिंदु = (१००, २००)
(px, py) = बिंदु             // px=१००  py=२००

// नामांकित टपल
व्यक्ति = (नाम: "अना", आयु: २५, शहर: "मैड्रिड")
(नाम: n, आयु: a) = व्यक्ति   // n="अना"  a=२५
```

## टपल

टपल **अपरिवर्तनीय** क्रमबद्ध कंटेनर हैं जो **विभिन्न प्रकार के** मान धारण कर सकते हैं।
ऐरे के विपरीत, निर्माण के बाद तत्वों को बदला नहीं जा सकता।

```zymbol
// स्थानीय — मिश्रित प्रकार अनुमत
बिंदु = (१०, २०)
>> बिंदु[१] ¶    // → १०

डेटा = (४२, "हैलो", #१, ३.१४)
>> डेटा[३] ¶     // → #१

// नामांकित
व्यक्ति = (नाम: "ऐलिस", आयु: २५)
>> व्यक्ति.नाम ¶    // → ऐलिस
>> व्यक्ति[१] ¶      // → ऐलिस  (सूचकांक भी काम करता है, १-आधारित)

// नेस्टेड
स्थिति = (x: १०, y: २०)
p = (स्थिति: स्थिति, लेबल: "मूल")
>> p.स्थिति.x ¶        // → १०
```

**अपरिवर्तनीयता** — टपल तत्व को बदलने का कोई भी प्रयास रनटाइम त्रुटि है:

```zymbol
t = (१०, २०, ३०)
// t[१] = ९९    // ❌ रनटाइम त्रुटि: टपल अपरिवर्तनीय हैं
// t[१] += ५    // ❌ समान त्रुटि
```

परिवर्तित मान प्राप्त करने के लिए `$~` (कार्यात्मक अद्यतन) का उपयोग करें — एक **नया** टपल लौटाता है:

```zymbol
t = (१०, २०, ३०)
t२ = t[२]$~ ९९९
>> t ¶     // → (१०, २०, ३०)   ← मूल अपरिवर्तित
>> t२ ¶    // → (१०, ९९९, ३०)

// नामांकित टपल — स्पष्ट रूप से पुनर्निर्माण करें
व्यक्ति = (नाम: "ऐलिस", आयु: २५)
बड़ा  = (नाम: व्यक्ति.नाम, आयु: २६)
>> व्यक्ति.आयु ¶    // → २५
>> बड़ा.आयु ¶     // → २६
```

## उच्च-क्रम फंक्शन

```zymbol
संख्याएँ = [१, २, ३, ४, ५, ६, ७, ८, ९, १०]

दोगुने  = संख्याएँ$> (x -> x * २)                  // map  → [२,४,६…२०]
सम    = संख्याएँ$| (x -> x % २ == ०)           // filter → [२,४,६,८,१०]
कुल    = संख्याएँ$< (०, (संचायक, x) -> संचायक + x)     // reduce → ५५

// मध्यवर्ती के माध्यम से श्रृंखलाबद्ध करें
चरण१ = संख्याएँ$| (x -> x > ३)
चरण२ = चरण१$> (x -> x * x)
>> चरण२ ¶    // → [१६, २५, ३६, ४९, ६४, ८१, १००]

// नामांकित फंक्शन सीधे HOF में पास किए जा सकते हैं
दोगुना(x) { <~ x * २ }
बड़ा_है(x) { <~ x > ५ }
r = संख्याएँ$> दोगुना       // ✅ सीधा संदर्भ
r = संख्याएँ$| बड़ा_है       // ✅ सीधा संदर्भ
```

## पाइप ऑपरेटर

दाएँ पक्ष को हमेशा पाइप किए गए मान के लिए `_` प्लेसहोल्डर की आवश्यकता होती है:

```zymbol
दोगुना = x -> x * २
योग = (a, b) -> a + b
वृद्धि = x -> x + १

प१ = ५ |> दोगुना(_)        // → १०
प२ = १० |> योग(_, ५)       // → १५
प३ = ५ |> योग(२, _)        // → ७

// श्रृंखलाबद्ध
प = ५ |> दोगुना(_) |> वृद्धि(_) |> दोगुना(_)
>> प ¶    // → २२  (५→१०→११→२२)
```

## अंकीय मोड

Zymbol संख्याओं को **69 यूनिकोड अंक लिपियों** में प्रदर्शित कर सकता है — देवनागरी, अरबी-भारतीय, थाई, क्लिंगॉन pIqaD, गणितीय बोल्ड, LCD सेगमेंट, और बहुत कुछ। सक्रिय मोड केवल `>>` आउटपुट को प्रभावित करता है; आंतरिक अंकगणित हमेशा द्विआधारी होता है।

### एक लिपि सक्रिय करना

लक्ष्य लिपि के `0` और `9` अंक `#…#` के अंदर लिखें:

```zymbol
#०९#    // देवनागरी   (U+0966–U+096F)
#٠٩#    // अरबी-भारतीय (U+0660–U+0669)
#๐๙#    // थाई         (U+0E50–U+0E59)
#09#    // ASCII पर रीसेट करें
```

### आउटपुट और बूलियन

```zymbol
x = ४२
>> x ¶          // → ४२   (ASCII डिफ़ॉल्ट)

#०९#
>> x ¶          // → ४२
>> ३.१४ ¶       // → ३.१४   (दशमलव बिंदु हमेशा ASCII)
>> १ + २ ¶      // → ३

// बूलियन: # उपसर्ग हमेशा ASCII, अंक अनुकूलित होता है
>> #१ ¶         // → #१   (देवनागरी में सत्य)
>> #० ¶         // → #०   (असत्य — ० पूर्णांक शून्य से भिन्न)

x = २८ > ४
>> x ¶          // → #१   (तुलना परिणाम सक्रिय मोड का अनुसरण करता है)
```

### स्रोत में मूल अंक लिटरल

कोई भी समर्थित लिपि के अंक मान्य लिटरल हैं — रेंज, मोडुलो, तुलनाओं में:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### किसी भी लिपि में बूलियन लिटरल

`#` + किसी भी ब्लॉक से अंक `0` या `1` एक मान्य बूलियन लिटरल है:

```zymbol
#٠٩#
सक्रिय = #१        // #१ के समान
>> सक्रिय ¶        // → #१
>> (#१ && #०) ¶ // → #०
```

> `#` **हमेशा ASCII** है। `#0` (असत्य) हमेशा प्रत्येक लिपि में `0` (पूर्णांक शून्य) से दृश्य रूप से भिन्न होता है।

## डेटा ऑपरेटर

```zymbol
// प्रकार रूपांतरण कास्ट
f = ##.४२         // → ४२.०  (फ्लोटिंग में)
i = ###३.७        // → ४     (पूर्णांक में, राउंड)
t = ##!३.७        // → ३     (पूर्णांक में, ट्रंकेट)

// स्ट्रिंग को संख्या में पार्स करें
v१ = #|"४२"|      // → ४२  (पूर्णांक)
v२ = #|"३.१४"|    // → ३.१४  (फ्लोटिंग)
v३ = #|"abc"|     // → "abc"  (सुरक्षित, कोई त्रुटि नहीं)

// राउंड / ट्रंकेट
π = ३.१४१५९२६५
राउंड२ = #.२|π|      // → ३.१४  (२ दशमलव स्थानों पर राउंड)
राउंड४ = #.४|π|      // → ३.१४१६
ट्रंक२ = #!२|π|      // → ३.१४  (ट्रंकेट)

// संख्या फ़ॉर्मेटिंग
फ़ॉर्मेट = #,|१२३४५६७|  // → १,२३४,५६७  (अल्पविराम-अलग)
वैज्ञानिक = #^|१२३४५.६७८|    // → १.२३४५६७८e४  (वैज्ञानिक)

// आधार लिटरल
a = 0x41         // → 'A'  (हेक्स)
b = 0b01000001   // → 'A'  (बाइनरी)
c = 0o101        // → 'A'  (ऑक्टल)

// आधार रूपांतरण आउटपुट
हेक्स = 0x|२५५|    // → "0x00FF"
बाइनरी = 0b|६५|     // → "0b1000001"
ऑक्टल = 0o|८|      // → "0o10"
दशमलव = 0d|२५५|    // → "0d0255"
```

## प्रतीक संदर्भ

| प्रतीक | संचालन | प्रतीक | संचालन |
|--------|-----------|--------|-----------|
| `=` | चर | `$#` | लंबाई |
| `:=` | स्थिरांक | `$+` | जोड़ें (श्रृंखलायोग्य) |
| `>>` | आउटपुट | `$+[i]` | सूचकांक पर डालें (1-आधारित) |
| `<<` | इनपुट | `$-` | मान द्वारा पहला हटाएँ |
| `¶` / `\\` | नई पंक्ति | `$--` | मान द्वारा सभी हटाएँ |
| `?` | यदि | `$-[i]` | सूचकांक पर हटाएँ (1-आधारित) |
| `_?` | अन्यथा-यदि | `$-[i..j]` | रेंज हटाएँ (1-आधारित) |
| `_` | अन्यथा / वाइल्डकार्ड | `$?` | सम्मिलित है |
| `??` | मैच | `$??` | सभी सूचकांक खोजें (1-आधारित) |
| `@` | लूप | `$[s..e]` | स्लाइस (1-आधारित) |
| `@ N { }` | N बार लूप | `$>` | map |
| `@!` | तोड़ें | `$\|` | filter |
| `@>` | जारी रखें | `$<` | reduce |
| `@:नाम { }` | लेबलयुक्त लूप | `$/ सीमांकक` | स्ट्रिंग विभाजित करें |
| `@:नाम!` | लेबल तोड़ें | `$++ a b c` | कॉन्कैट बिल्ड |
| `@:नाम>` | लेबल जारी रखें | `arr[i>j>k]` | नेविगेशन सूचकांक |
| `->` | लैम्ब्डा | `arr[i] = मान` | तत्व अद्यतन करें (केवल ऐरे) |
| `arr[i] += मान` | यौगिक अद्यतन | `arr[i]$~` | कार्यात्मक अद्यतन (नई प्रतिलिपि) |
| `$^+` | आरोही सजाएँ (आदिम) | `$^-` | अवरोही सजाएँ (आदिम) |
| `$^` | तुलनाकारी के साथ सजाएँ (टपल) | `<~` | लौटाएँ |
| `\|>` | पाइप | `!?` | प्रयास करें |
| `:!` | पकड़ें | `:>` | अंत में |
| `#1` | सत्य | `#0` | असत्य |
| `$!` | क्या त्रुटि है | `$!!` | त्रुटि प्रसारित करें |
| `<#` | आयात करें | `#>` | निर्यात करें |
| `#` | मॉड्यूल घोषित करें | `::` | मॉड्यूल कॉल करें |
| `.` | फ़ील्ड पहुँच | `#?` | प्रकार मेटाडेटा |
| `#\|..\|` | संख्या पार्स करें | `##.` | फ्लोटिंग में रूपांतरित करें |
| `###` | पूर्णांक में रूपांतरित करें (राउंड) | `##!` | पूर्णांक में रूपांतरित करें (ट्रंकेट) |
| `#.N\|..\|` | राउंड | `#!N\|..\|` | ट्रंकेट |
| `#,\|..\|` | अल्पविराम प्रारूप | `#^\|..\|` | वैज्ञानिक |
| `#d0d9#` | अंकीय मोड स्विच करें | `#09#` | ASCII पर रीसेट करें |
| `<\ ..\>` | शेल निष्पादित करें | `>\<` | CLI तर्क |
| `\ चर` | स्पष्ट रूप से चर नष्ट करें | `°x` / `x°` | हॉट परिभाषा (ऑटो-आरंभ) |
| `>>|` | टीयूआई ब्लॉक (वैकल्पिक स्क्रीन) | `>>~` | स्थानीय आउटपुट |
| `>>!` | स्क्रीन साफ़ करें | `>>?` | टर्मिनल आकार पूछें |
| `<<\|` | ब्लॉकिंग कुंजी प्रेस | `<<\|?` | नॉन-ब्लॉकिंग कुंजी प्रेस पोल |
| `@~ N` | N मिलीसेकंड सोएँ | `$*` | स्ट्रिंग N बार दोहराएँ |

## रिलीज़ चेंजलॉग

### v0.0.5 — टीयूआई प्रिमिटिव, हॉट परिभाषा और स्ट्रिंग दोहराना _(मई २०२६)_

- **ब्रेकिंग** मैच आर्म सेपरेटर: `पैटर्न : परिणाम` → `पैटर्न => परिणाम`
- **ब्रेकिंग** आयात उपनाम: `<# पथ <= उपनाम` → `<# पथ => उपनाम`
- **ब्रेकिंग** निर्यात नाम बदलें: `#> { fn <= पब }` → `#> { fn => पब }`
- **जोड़ा गया** टीयूआई ब्लॉक `>>| { }` — वैकल्पिक स्क्रीन + कच्चा मोड; बाहर निकलने पर साफ़ करता है
- **जोड़ा गया** स्थानीय आउटपुट `>>~ (पंक्ति, स्तंभ, BKS, fg, bg) > आइटम` — विरल स्लॉट, ANSI 256 रंग
- **जोड़ा गया** कुंजी इनपुट `<<| चर` (ब्लॉकिंग) और `<<|? चर` (नॉन-ब्लॉकिंग पोल)
- **जोड़ा गया** `>>!` स्क्रीन साफ़ करें, `>>?` टर्मिनल आकार पूछें, `@~ N` N मिलीसेकंड सोएँ
- **जोड़ा गया** हॉट परिभाषा `°x` / `x°` — लूप में पहले उपयोग पर चर स्वचालित आरंभ
- **जोड़ा गया** स्ट्रिंग दोहराना `स्ट्रिंग $* N` — एक स्ट्रिंग को N बार दोहराएँ
- **वीएम** समानता: 436/436 टेस्ट पास

### v0.0.4 — 1-आधारित सूचकांकन, प्रथम-श्रेणी फंक्शन और ब्लॉक मॉड्यूल _(अप्रैल २०२६)_

- **ब्रेकिंग** सभी सूचकांकन **1-आधारित** में बदल गया — `arr[1]` पहला तत्व है; `arr[0]` एक रनटाइम त्रुटि है
- **जोड़ा गया** नामांकित फंक्शन **प्रथम-श्रेणी मान** हैं — सीधे HOF में पास करें: `संख्याएँ$> दोगुना`
- **जोड़ा गया** मॉड्यूल के लिए **ब्लॉक सिंटैक्स आवश्यक** — `# नाम { ... }` — फ्लैट सिंटैक्स हटा दिया गया
- **जोड़ा गया** बहु-आयामी सूचकांकन: `arr[i>j>k]` (नेविगेशन), `arr[p ; q]` (फ्लैट निष्कर्षण)
- **जोड़ा गया** प्रकार रूपांतरण कास्ट: `##.एक्सप्रेशन` (फ्लोटिंग), `###एक्सप्रेशन` (पूर्णांक राउंड), `##!एक्सप्रेशन` (पूर्णांक ट्रंकेट)
- **जोड़ा गया** स्ट्रिंग विभाजन: `स्ट्रिंग$/ सीमांकक` — `Array(स्ट्रिंग)` लौटाता है
- **जोड़ा गया** कॉन्कैट बिल्ड: `आधार$++ a b c` — एकाधिक आइटम जोड़ता है
- **जोड़ा गया** टाइम्स लूप: `@ N { }` — बिल्कुल N बार दोहराएँ
- **जोड़ा गया** लेबलयुक्त लूप सिंटैक्स: `@:नाम { }`, `@:नाम!`, `@:नाम>` — `@ @नाम` / `@! नाम` को प्रतिस्थापित करता है
- **जोड़ा गया** चर स्कोप नियम: `_नाम` चरों का सटीक ब्लॉक स्कोप होता है; `\ चर` जल्दी नष्ट करता है
- **जोड़ा गया** मैच तुलना पैटर्न: `< 0 =>`, `> 5 =>`, `== 42 =>` आदि
- **जोड़ा गया** मॉड्यूल E013 त्रुटि: मॉड्यूल बॉडी में निष्पादन योग्य स्टेटमेंट निषिद्ध हैं
- **सुधारा गया** `alias.CONST` अब सही ढंग से हल करता है; `#>` फंक्शन परिभाषाओं के बाद आ सकता है
- **वीएम** पूर्ण समानता: 393/393 टेस्ट पास
