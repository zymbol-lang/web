# Zymbol-Lang ની સંક્ષિપ્ત માર્ગદર્શિકા

**Zymbol-Lang** એ એક પ્રતીકાત્મક પ્રોગ્રામિંગ ભાષા છે. તે કોઈ keywords વાપરતી નથી — બધું જ પ્રતીક છે. તે દરેક માનવ ભાષામાં એ જ રીતે કામ કરે છે.

- Keywords નહીં (`if`, `while`, `return` અસ્તિત્વ ધરાવતા નથી — ફક્ત પ્રતીકો `?`, `@`, `<~`)
- સંપૂર્ણ Unicode — ઓળખો (identifiers) કોઈ પણ ભાષામાં અથવા emoji 👋
- માનવ-ભાષા-નિષ્પક્ષ — કોડ દરેક ભાષામાં સમાન છે

## ચલ અને સ્થિરાંક

```zymbol
ચ = ૧૦           // ચલ (mutable)
પાઈ := ૩.૧૪૧૫૯   // સ્થિરાંક (immutable — ફરીથી સોંપ્યે error)
નામ = "અનુ"
સક્રિય = #૧     // boolean true
👋 := "નમસ્તે"
```

## ડેટા પ્રકારો

| પ્રકાર        | ઉદાહરણ              | `#?` પ્રતીક | નોંધ                                  |
|---------------|---------------------|-------------|---------------------------------------|
| Integer       | `૪૨`, `-૭`          | `###`       | 64-bit signed                         |
| Float         | `૩.૧૪`, `૧.૫e૧૦`    | `##.`       | Scientific notation ઠીક              |
| String        | `"નમસ્તે"`          | `##"`       | Interpolation: `"નમસ્તે {નામ}"`       |
| Char          | `'અ'`               | `##'`       | એક Unicode character                  |
| Boolean       | `#૧`, `#૦`          | `##?`       | numeric 1 અને 0 નહીં                 |
| Array         | `[૧, ૨, ૩]`         | `##]`       | બધા elements સમાન પ્રકારના હોવા જોઈએ |
| ટ્યુપલ         | `(a, b)`            | `##)`       | સ્થાન-આધારિત                         |
| નામ-ભ ટ્યુપલ   | `(x: ૧, y: ૨)`      | `##)`       | નામ અથવા index દ્વારા access          |

## આઉટપુટ અને ઇનપુટ

```zymbol
// આઉટપુટ — newline આપોઆપ ઉમેરાતી નથી
>> "નમસ્તે" ¶                      // ¶ અથવા \\ = explicit newline
>> "a=" ક " b=" ખ ¶                // juxtaposition દ્વારા અનેક મૂલ્ય
>> "સરવાળો=" સરવાળો(૨, ૩) ¶       // ફંક્શન call કોઈ પણ સ્થાને
>> (સૂચિ$#) ¶                      // postfix operators ને parentheses ફરજિયાત

// ઇનપુટ
<< નામ                             // prompt વગર — variable માં વાંચે
<< "તમારું નામ? " નામ              // prompt સાથે
```

> `¶` અથવા `\\` newline તરીકે સમકક્ષ છે.

## સ્ટ્રિંગ

```zymbol
// ત્રણ માન્ય પ્રકાર
નામ = "અનુ"
ઉ = ૨૫

// ૧. Comma — = અથવા := સોંપણીમાં
સંદેશ = "નમસ્તે ", નામ, "!"        // → નમસ્તે અનુ!
શીર્ષક := "વપરાશકર્તા: ", નામ

// ૨. Juxtaposition — >> આઉટપુટમાં
>> "નમસ્તે " નામ " ઉંમર " ઉ ¶     // → નમસ્તે અનુ ઉંમર ૨૫

// ૩. Interpolation — કોઈ પણ સ્થાને
વર્ણ = "નમસ્તે {નામ}, ઉંમર {ઉ}"   // → નમસ્તે અનુ, ઉંમર ૨૫
```

```zymbol
સ = "નમસ્તે દુનિયા"
લં = સ$#                  // ૧૪
ઉ = સ$[૦..૫]              // "નમસ્ત"  (end exclusive)
છ = સ$? "દુનિ"            // #૧
ભ = "ક,ખ,ગ,ઘ" / ','       // [ક, ખ, ગ, ઘ]
બ = સ$~~["ત":"ટ"]          // replace
બ૧ = સ$~~["ત":"ટ":૧]       // first N
```

> `+` ફક્ત સંખ્યાઓ માટે. Strings માટે `,`, juxtaposition અથવા interpolation વાપરો.

```zymbol
ચ = ૭
```

## લૂપ

```zymbol
// Inclusive range: ૦..૪ → ૦,૧,૨,૩,૪
@ ઇ:૦..૪ { >> ઇ " " }
>> ¶    // → ૦ ૧ ૨ ૩ ૪

// Step સાથે range
@ ઇ:૧..૯:૨ { >> ઇ " " }
>> ¶    // → ૧ ૩ ૫ ૭ ૯

// ઊલટો range
@ ઇ:૫..૦:૧ { >> ઇ " " }
>> ¶    // → ૫ ૪ ૩ ૨ ૧ ૦

// While
ન = ૧
@ ન <= ૬૪ { ન *= ૨ }
>> ન ¶    // → ૧૨૮

// Array ઉપર for-each
ફળ = ["સફરજન", "નાસપાતી", "દ્રાક્ષ"]
@ ફ:ફળ { >> ફ ¶ }

// String characters ઉપર
@ ક:"નમસ્તે" { >> ક "-" }
>> ¶    // → ન-મ-સ-્-ત-ે-

// Break અને Continue
@ ઇ:૧..૧૦ {
    ? ઇ % ૨ == ૦ { @> }    // @> continue
    ? ઇ > ૭ { @! }          // @! break
    >> ઇ " "
}
>> ¶    // → ૧ ૩ ૫ ૭
```

## ફંક્શન

```zymbol
// ઘોષણા અને call
સરવાળો(અ, બ) { <~ અ + બ }
>> સરવાળો(૩, ૪) ¶    // → ૭

// Recursion
ગુણાકાર(ન) {
    ? ન <= ૧ { <~ ૧ }
    <~ ન * ગુણાકાર(ન - ૧)
}
>> ગુણાકાર(૫) ¶    // → ૧૨૦

// ફંક્શન isolated scope ધરાવે — outer variables access કરી શકે નહીં
_વૈશ્વિક = ૧૦૦
પરીક્ષણ() {
    ચ = ૪૨    // ફક્ત local
    <~ ચ
}
>> પરીક્ષણ() ¶    // → ૪૨
```

> **મહત્ત્વ**: Named functions `નામ(params){ }` first-class values નથી.
> argument તરીકે પાસ કરવા, wrapper વાપરો: `x -> સરવાળો(x)`.

## લૅમ્બ્ડા અને ક્લોઝ઼ર

```zymbol
// સરળ લૅમ્બ્ડા (implicit return)
બેગણું = ચ -> ચ * ૨
સરવાળો = (ક, ખ) -> ક + ખ
>> બેગણું(૫) ¶    // → ૧૦
>> સરવાળો(૩, ૭) ¶  // → ૧૦

// Block લૅમ્બ્ડા (explicit return)
વર્ગીકૃત_મૂલ્ય = ચ -> {
    ? ચ > ૦ { <~ "ધનાત્મક" }
    _? ચ < ૦ { <~ "ઋણાત્મક" }
    <~ "શૂન્ય"
}
>> વર્ગીકૃત_મૂલ્ય(૫) ¶     // → ધનાત્મક
>> વર્ગીકૃત_મૂલ્ય(૦) ¶     // → શૂન્ય
>> વર્ગીકૃત_મૂલ્ય(-૫) ¶    // → ઋણાત્મક

// ક્લોઝ઼ર — લૅમ્બ્ડા outer scope variables capture કરે
ગુણક = ૩
ત્રણગણું = ચ -> ચ * ગુણક    // 'ગુણક' capture કરે
>> ત્રણગણું(૭) ¶    // → ૨૧

// Function factory
ક્રિયા(ન) { <~ ચ -> ચ + ન }
જોડ૧૦ = ક્રિયા(૧૦)
>> જોડ૧૦(૫) ¶    // → ૧૫

// લૅમ્બ્ડાs as values: arrays માં store
ક્રિયા = [ચ -> ચ+૧, ચ -> ચ*૨, ચ -> ચ*ચ]
>> ક્રિયા[૦](૫) ¶    // → ૬
>> ક્રિયા[૨](૫) ¶    // → ૨૫
```

## સૂચિ

સૂચિઓ **mutable** છે અને **એ જ પ્રકારના** elements ધરાવે છે.

```zymbol
સૂચિ = [૧, ૨, ૩, ૪, ૫]

સૂચિ[૦]          // ૧ — access (૦-based)
સૂચિ[-૧]         // ૫ — negative index (last)
સૂચિ$#           // ૫ — length (use (સૂચિ$#) in >>)

સૂચિ = સૂચિ$+ ૬            // append → [૧,૨,૩,૪,૫,૬]
સૂચિ૨ = સૂચિ$+[૨] ૯૯       // insert at index ૨
સૂચિ૩ = સૂચિ$- ૩           // remove first occurrence
સૂચિ૪ = સૂચિ$-- ૩          // remove all occurrences
સૂચિ૫ = સૂચિ$-[૦]          // remove at index
સૂચિ૬ = સૂચિ$-[૧..૩]       // remove range (end exclusive)

છ = સૂચિ$? ૩            // #૧ — contains
સ્થ = સૂચિ$?? ૩          // [૨] — all indices
ભ = સૂચિ$[૦..૩]          // [૧,૨,૩] — slice
ભ૨ = સૂચિ$[૦:૩]          // [૧,૨,૩] — count syntax

ચ = સૂચિ$^+              // sorted ascending (primitives)
ઉ = સૂચિ$^-              // sorted descending (primitives)

// Named tuple arrays — $^ with comparator
ડ = [(નામ: "Carla", ઉ: ૨૮), (નામ: "Ana", ઉ: ૨૫), (નામ: "Bob", ઉ: ૩૦)]
ઉ_ક્રમ  = ડ$^ (ક, ખ -> ક.ઉ < ખ.ઉ)
ન_ક્રમ = ડ$^ (ક, ખ -> ક.નામ > ખ.નામ)
>> ઉ_ક્રમ[૦].નામ ¶     // → Ana
>> ન_ક્રમ[૦].નામ ¶    // → Carla

// સીધું element update (ફક્ત સૂચિ)
સૂચિ[૧] = ૯૯              // સોંપો
સૂચિ[૦] += ૫              // compound: +=  -=  *=  /=  %=  ^=

// Functional update — નવી સૂચિ return; મૂળ અપરિવર્તિત
સૂચિ૨ = સૂચિ[૧]$~ ૯૯
```

> બધા collection operators **નવી સૂચિ** return કરે. પાછી સોંપો: `સૂચિ = સૂચિ$+ ૪`.
> Chaining નહીં: intermediate assignments વાપરો.
> `$^+` / `$^-` **primitive arrays** (numbers, strings) sort. Tuple arrays માટે comparator lambda સાથે `$^` વાપરો.

**Value semantics** — સૂચિ બીજા variable ને સોંપ્યે independent copy બને:

```zymbol
a = [1, 2, 3]
b = a
a[0] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b અપ્રભાવિત
```

```zymbol
// Nested arrays
matrix = [[૧,૨,૩],[૪,૫,૬],[૭,૮,૯]]
>> matrix[૧][૨] ¶    // → ૬
```

## ટ્યુપલ

ટ્યુપલ **immutable** ક્રમાંકિત containers છે જે **અલગ-અલગ પ્રકારના** values ધારણ કરી શકે. Arrays થી વિપરીત, creation પછી elements બદલી શકાય નહીં.

```zymbol
// Positional — mixed types
બિ = (૧૦, ૨૦)
>> બિ[૦] ¶    // → ૧૦

ડ = (૪૨, "નમસ્તે", #૧, ૩.૧૪)
>> ડ[૨] ¶     // → #૧

// Named tuple
વ્યક્તિ = (નામ: "એલિસ", ઉંમર: ૨૫)
>> વ્યક્તિ.નામ ¶    // → એલિસ
>> વ્યક્તિ.ઉંમર ¶   // → ૨૫
>> વ્યક્તિ[૦] ¶     // → એલિસ (index પણ કામ કરે)
```

**Immutability** — tuple element બદલવાનો કોઈ પ્રયાસ runtime error:

```zymbol
t = (૧૦, ૨૦, ૩૦)
// t[૦] = ૯૯    // ❌ runtime error: tuples are immutable
// t[૦] += ૫    // ❌ same error
```

બદલાયેલ value મેળવવા `$~` (functional update) વાપરો — **નવું** tuple return:

```zymbol
t = (૧૦, ૨૦, ૩૦)
t૨ = t[૧]$~ ૯૯૯
>> t ¶     // → (૧૦, ૨૦, ૩૦)   ← original unchanged
>> t૨ ¶    // → (૧૦, ૯૯૯, ૩૦)

// Named tuple — explicitly rebuild
વ્યક્તિ = (નામ: "એલિસ", ઉંમર: ૨૫)
મોટ  = (નામ: વ્યક્તિ.નામ, ઉંમર: ૨૬)
>> વ્યક્તિ.ઉંમર ¶    // → ૨૫
>> મોટ.ઉંમર ¶        // → ૨૬
```

## ઉચ્ચ-ક્રમ ફંક્શન

HOF operators ને **ઇનલાઇન લૅમ્બ્ડા** ફરજિયાત — લૅમ્બ્ડા variable directly નહીં.

```zymbol
સંખ્યાઓ = [૧, ૨, ૩, ૪, ૫, ૬, ૭, ૮, ૯, ૧૦]

// Map ($>)
બેગણાં = સંખ્યાઓ$> (ચ -> ચ * ૨)
>> બેગણાં ¶    // → [૨, ૪, ૬, ૮, ૧૦, ૧૨, ૧૪, ૧૬, ૧૮, ૨૦]

// Filter ($|)
જોડ = સંખ્યાઓ$| (ચ -> ચ % ૨ == ૦)
>> જોડ ¶    // → [૨, ૪, ૬, ૮, ૧૦]

// Reduce ($<) — (initial, (acc, elem) -> expr)
કુલ = સંખ્યાઓ$< (૦, (સ, ચ) -> સ + ચ)
```
