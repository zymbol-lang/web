# راهنمای فشرده Zymbol-Lang

**Zymbol-Lang** یک زبان برنامه‌نویسی نمادین است. بدون کلمات کلیدی — فقط نمادها. در هر زبان انسانی یکسان کار می‌کند.

- بدون کلمات کلیدی (`if`، `while`، `return` وجود ندارند — فقط نمادها `?`، `@`، `<~`)
- پشتیبانی از Unicode — شناسه‌ها به هر زبانی از جمله ایموجی 👋
- طراحی زبان‌مستقل — کد در همه زبان‌ها یکسان است

```zymbol
م = ۱۰           // متغیر (قابل تغییر)
```

## خروجی و ورودی

```zymbol
// خروجی — بدون سطر جدید خودکار
>> "سلام، دنیا!" ¶                  // ¶ یا \\ برای سطر جدید صریح
>> "الف=" الف " ب=" ب ¶              // چند مقدار با juxtaposition
>> "جمع=" جمع(۲, ۳) ¶                // فراخوانی تابع در هر موقعیت
>> (آرایه$#) ¶                       // عملگر پسوندی نیاز به پرانتز دارد

// ورودی
<< نام                               // بدون prompt — مقدار در متغیر
<< "نام خود را وارد کنید: " نام      // با prompt
```

> `¶` یا `\\` برای سطر جدید الزامی است.

## رشته‌ها

سه روش معتبر — هر کدام برای کاربرد خاص:

```zymbol
نام = "آلیس"
ن = ۲۵

// ۱. عملگر کاما — در انتساب (= یا :=)
پیام = "سلام، ", نام, "!"          // → سلام، آلیس!
عنوان := "کاربر: ", نام

// ۲. juxtaposition — در >>
>> "سلام، " نام " سن: " ن ¶        // → سلام، آلیس سن: ۲۵

// ۳. درج — در هر جایی
متن = "سلام، {نام}، سن: {ن}"       // → سلام، آلیس، سن: ۲۵
```

```zymbol
// عملگرهای رشته
س = "دنیا"
>> (س$#) ¶                         // → ۴  (طول — پرانتز لازم در >>)
برش = س$[۰..۳]                     // "دنی"  (برش)
س۲ = س$~~["دنیا":"کیهان"]          // "کیهان"  (جایگزینی)
س۳ = س$~~["ی":۲]                   // اول ۲ تکرار "ی" را جایگزین می‌کند
```

> **توجه**: `+` فقط برای اعداد است. الحاق رشته با `+` هشدار تولید می‌کند.

## کنترل جریان

```zymbol
م = ۷

// شرط ساده
? م > ۰ { >> "مثبت" ¶ }

// if / else if / else
? م > ۱۰۰ {
    >> "خیلی بزرگ" ¶
} _? م > ۰ {
    >> "مثبت" ¶
} _? م == ۰ {
    >> "صفر" ¶
} _ {
    >> "منفی" ¶
}
```

بلوک `{ }` **همیشه الزامی** است، حتی برای یک خط.

## حلقه‌ها

```zymbol
// محدوده شامل: ۰..۴ شامل ۰،۱،۲،۳،۴
@ ن:۰..۴ { >> ن " " }
>> ¶    // → ۰ ۱ ۲ ۳ ۴

// محدوده با گام
@ ن:۱..۹:۲ { >> ن " " }
>> ¶    // → ۱ ۳ ۵ ۷ ۹

// محدوده معکوس
@ ن:۵..۰:۱ { >> ن " " }
>> ¶    // → ۵ ۴ ۳ ۲ ۱ ۰

// while
ن = ۱
@ ن <= ۶۴ { ن *= ۲ }
>> ن ¶    // → ۱۲۸

// for-each روی آرایه
میوه‌ها = ["سیب", "گلابی", "انگور"]
@ م:میوه‌ها { >> م ¶ }

// تکرار روی کاراکترهای رشته
@ ک:"سلام" { >> ک "-" }
>> ¶    // → س-ل-ا-م-

// break و continue
@ ن:۱..۱۰ {
    ? ن % ۲ == ۰ { @> }    // @> = continue
    ? ن > ۷ { @! }          // @! = break
    >> ن " "
}
>> ¶    // → ۱ ۳ ۵ ۷
```

## توابع

```zymbol
// تعریف و فراخوانی
جمع(الف، ب) { <~ الف + ب }
>> جمع(۳, ۴) ¶    // → ۷

// بازگشتی
فاکتوریل(ن) {
    ? ن <= ۱ { <~ ۱ }
    <~ ن * فاکتوریل(ن - ۱)
}
>> فاکتوریل(۵) ¶    // → ۱۲۰
```

توابع **scope ایزوله** دارند — به متغیرهای بیرونی دسترسی ندارند. از پارامترهای خروجی `<~` برای تغییر متغیرهای فراخواننده استفاده کنید:

```zymbol
تعویض(الف<~, ب<~) {
    موقت = الف
    الف = ب
    ب = موقت
}
م = ۱۰
ن = ۲۰
تعویض(م, ن)
>> "م=" م " ن=" ن ¶    // → م=۲۰ ن=۱۰
```

> توابع نامدار مقادیر درجه اول نیستند. برای ارسال به HOF: `x -> تابع(x)`.

## لامبدا و بستارها

```zymbol
// لامبدا ساده (بازگشت ضمنی)
دوبرابر = م -> م * ۲
جمع_ل = (الف، ب) -> الف + ب
>> دوبرابر(۵) ¶    // → ۱۰
>> جمع_ل(۳, ۷) ¶   // → ۱۰

// لامبدا بلوکی (بازگشت صریح)
دسته‌بندی = م -> {
    ? م > ۰ { <~ "مثبت" }
    _? م < ۰ { <~ "منفی" }
    <~ "صفر"
}
>> دسته‌بندی(۵) ¶     // → مثبت
>> دسته‌بندی(۰) ¶     // → صفر
>> دسته‌بندی(-۵) ¶    // → منفی

// بستار — لامبدا متغیر بیرونی را ضبط می‌کند
ضریب = ۳
سه‌برابر = م -> م * ضریب    // 'ضریب' ضبط می‌شود
>> سه‌برابر(۷) ¶    // → ۲۱

// تابع که لامبدا برمی‌گرداند
سازنده(ن) { <~ م -> م + ن }
ده‌تا = سازنده(۱۰)
>> ده‌تا(۵) ¶    // → ۱۵

// آرایه از لامبداها
عملیات = [م -> م+۱, م -> م*۲, م -> م*م]
>> عملیات[۰](۵) ¶    // → ۶
>> عملیات[۲](۵) ¶    // → ۲۵
```

## آرایه‌ها

آرایه‌ها **قابل تغییر** هستند و عناصر با **نوع یکسان** نگه می‌دارند.

```zymbol
arr = [۱, ۲, ۳, ۴, ۵]

arr[۰]          // ۱ — دسترسی (ایندکس از صفر)
arr[-۱]         // ۵ — ایندکس منفی (آخرین)
arr$#           // ۵ — طول (پرانتز لازم در >>)

arr = arr$+ ۶            // افزودن → [۱,۲,۳,۴,۵,۶]
arr۲ = arr$+[۲] ۹۹       // درج در ایندکس ۲
arr۳ = arr$- ۳           // حذف اولین تکرار مقدار
arr۴ = arr$-- ۳          // حذف همه تکرارها
arr۵ = arr$-[۰]          // حذف در ایندکس
arr۶ = arr$-[۱..۳]       // حذف محدوده (انتها منحصر)

دارد = arr$? ۳           // #۱ — وجود دارد
مواضع = arr$?? ۳         // [۲] — همه ایندکس‌های مقدار
برش = arr$[۰..۳]         // [۱,۲,۳] — برش (انتها منحصر)
برش۲ = arr$[۰:۳]         // [۱,۲,۳] — با تعداد

صعودی = arr$^+           // مرتب‌سازی صعودی (فقط اولیه‌ها)
نزولی = arr$^-           // مرتب‌سازی نزولی (فقط اولیه‌ها)

// آرایه تاپل‌ها — از $^ با لامبدا مقایسه استفاده کنید
db = [(name: "کارلا", age: ۲۸), (name: "آنا", age: ۲۵), (name: "باب", age: ۳۰)]
بر_اساس_سن = db$^ (الف، ب -> الف.age < ب.age)
>> بر_اساس_سن[۰].name ¶    // → آنا

// به‌روزرسانی مستقیم عنصر (فقط آرایه‌ها)
arr[۱] = ۹۹              // انتساب
arr[۰] += ۵              // مرکب: +=  -=  *=  /=  %=  ^=

// به‌روزرسانی تابعی — آرایه جدید برمی‌گرداند؛ اصل تغییر نمی‌کند
arr2 = arr[۱]$~ ۹۹
```

> همه عملگرهای مجموعه **آرایه جدید** برمی‌گردانند. نتیجه را ذخیره کنید: `arr = arr$+ ۴`.
> زنجیره‌سازی مجاز نیست — از انتساب‌های میانی استفاده کنید.
> `$^+` / `$^-` **آرایه‌های اولیه** را مرتب می‌کنند. برای تاپل‌ها از `$^` با لامبدا مقایسه استفاده کنید.

**معناشناسی مقدار** — انتساب یک آرایه به متغیر دیگر یک کپی مستقل ایجاد می‌کند:

```zymbol
a = [1, 2, 3]
b = a
a[0] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b تأثیر نمی‌پذیرد
```

```zymbol
// آرایه‌های تودرتو
ماتریس = [[۱,۲,۳],[۴,۵,۶],[۷,۸,۹]]
>> ماتریس[۱][۲] ¶    // → ۶
```

## آرایه‌گذاری (Destructuring)

```zymbol
// آرایه
arr = [۱۰, ۲۰, ۳۰, ۴۰, ۵۰]
[الف, ب, ج] = arr              // الف=۱۰  ب=۲۰  ج=۳۰
[اول, *بقیه] = arr              // اول=۱۰  بقیه=[۲۰,۳۰,۴۰,۵۰]
[م, _, ن] = [۱, ۲, ۳]           // _ نادیده گرفته می‌شود

// تاپل مکانی
نقطه = (۱۰۰, ۲۰۰)
(نم, نن) = نقطه                // نم=۱۰۰  نن=۲۰۰

// تاپل نامدار
شخص = (name: "آنا", age: ۲۵, city: "تهران")
(name: ن, age: س) = شخص        // ن="آنا"  س=۲۵
```

## تاپل‌ها

تاپل‌ها ظرف‌های مرتب **تغییرناپذیر** هستند که می‌توانند مقادیر با **نوع‌های مختلف** نگه دارند. برخلاف آرایه‌ها، عناصر پس از ایجاد قابل تغییر نیستند.

```zymbol
// مکانی
نقطه = (۱۰, ۲۰)
>> نقطه[۰] ¶    // → ۱۰

داده = (42, "hello", #1, 3.14)
>> داده[2] ¶     // → #1

// نامدار
شخص = (نام: "آلیس"، سن: ۲۵)
>> شخص.نام ¶    // → آلیس
>> شخص[۰] ¶    // → آلیس (ایندکس هم کار می‌کند)

// تودرتو
موضع = (م: ۱۰, ن: ۲۰)
ق = (موضع: موضع, برچسب: "مبدأ")
>> ق.موضع.م ¶   // → ۱۰
```

**تغییرناپذیری** — هرگونه تلاش برای تغییر عنصر یک تاپل خطای زمان اجرا است:

```zymbol
t = (10, 20, 30)
// t[0] = 99    // ❌ خطای زمان اجرا: تاپل‌ها تغییرناپذیرند
// t[0] += 5    // ❌ همان خطا
```

برای استخراج یک مقدار تغییریافته از `$~` (به‌روزرسانی تابعی) استفاده کنید — یک تاپل **جدید** برمی‌گرداند:

```zymbol
t = (10, 20, 30)
t2 = t[1]$~ 999
>> t ¶     // → (10, 20, 30)   ← اصل تغییر نکرد
>> t2 ¶    // → (10, 999, 30)

// تاپل نامدار — بازسازی صریح
شخص = (نام: "Alice", سن: 25)
بزرگ‌تر  = (نام: شخص.نام, سن: 26)
>> شخص.سن ¶    // → 25
>> بزرگ‌تر.سن ¶    // → 26
```

## توابع مرتبه بالا

> HOF نیاز به **لامبدا درون‌خطی** دارد — از متغیر لامبدا مستقیم استفاده نکنید.

```zymbol
اعداد = [۱, ۲, ۳, ۴, ۵, ۶, ۷, ۸, ۹, ۱۰]

// Map ($>) — تبدیل هر عنصر
مضاعف = اعداد$> (م -> م * ۲)
>> مضاعف ¶    // → [۲, ۴, ۶, ۸, ۱۰, ۱۲, ۱۴, ۱۶, ۱۸, ۲۰]

// Filter ($|) — فیلتر کردن عناصر
زوج = اعداد$| (م -> م % ۲ == ۰)
>> زوج ¶    // → [۲, ۴, ۶, ۸, ۱۰]

// Reduce ($<) — تجمیع (مقدار اولیه، (acc, elem) -> expr)
مجموع_کل = اعداد$< (۰, (جمع, م) -> جمع + م)
>> مجموع_کل ¶    // → ۵۵

// زنجیره از طریق میانی‌ها
گام۱ = اعداد$| (م -> م > ۳)
گام۲ = گام۱$> (م -> م * م)
>> گام۲ ¶    // → [۱۶, ۲۵, ۳۶, ۴۹, ۶۴, ۸۱, ۱۰۰]
```

## عملگر لوله

طرف راست همیشه نیاز به `_` به عنوان جانشین مقدار دارد:

```zymbol
دوبرابر = م -> م * ۲
جمع = (الف، ب) -> الف + ب
یک‌اضافه = م -> م + ۱

۵ |> دوبرابر(_)         // → ۱۰
۱۰ |> جمع(_, ۵)         // → ۱۵
۵ |> جمع(۲, _)          // → ۷

// زنجیره
ن = ۵ |> دوبرابر(_) |> یک‌اضافه(_) |> دوبرابر(_)
>> ن ¶    // → ۲۲  (۵→۱۰→۱۱→۲۲)
```

## سیستم‌های عددی

Zymbol می‌تواند اعداد را در **69 بلوک رقمی Unicode** نمایش دهد — فارسی، عربی-هندی، تایلندی، کلینگون pIqaD، ریاضی پررنگ، ارقام LCD و بیشتر. حالت فعال فقط خروجی `>>` را تحت تأثیر قرار می‌دهد؛ محاسبات داخلی همیشه دودویی هستند.

### فعال‌سازی سیستم

رقم `۰` و رقم `۹` سیستم مورد نظر را بین `#…#` بنویسید:

```zymbol
#۰۹#    // فارسی (Extended Arabic-Indic) (U+06F0–U+06F9)
```

### خروجی و مقادیر بولی

```zymbol
x = ۴۲
>> x ¶          // → ۴۲   (پیش‌فرض ASCII)

#۰۹#
>> x ¶          // → ۴۲
>> ۳.۱۴ ¶       // → ۳.۱۴   (نقطه اعشار همیشه ASCII)
>> ۱ + ۲ ¶      // → ۳

// بولی: پیشوند # همیشه ASCII، رقم طبق حالت
>> #۱ ¶         // → #۱   (درست در فارسی)
>> #۰ ¶         // → #۰   (غلط — متمایز از ۰ صفر)

x = ۲۸ > ۴
>> x ¶          // → #۱   (نتیجه مقایسه از حالت فعال پیروی می‌کند)
```

### لیترال‌های بومی در کد منبع

ارقام هر سیستم پشتیبانی‌شده لیترال‌های معتبر در بازه‌ها، مودولو و مقایسه‌ها هستند:

```zymbol
#۰۹#

@ ن:۱..۱۵ {
    ? ن % ۱۵ == ۰ { >> "FizzBuzz" ¶ }
    _? ن % ۳  == ۰ { >> "Fizz" ¶ }
    _? ن % ۵  == ۰ { >> "Buzz" ¶ }
    _ { >> ن ¶ }
}
```

### لیترال‌های بولی در هر سیستم

`#` + رقم `۰` یا `۱` از هر بلوک پشتیبانی‌شده یک لیترال بولی معتبر است:

```zymbol
#٠٩#
نشط = #١        // مثل #۱
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **همیشه ASCII** است. `#۰` (غلط) در هر سیستمی از نظر بصری با `۰` (صفر صحیح) متمایز است.
