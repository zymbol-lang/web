`°` (សញ្ញាដឺក្រេ, U+00B0) ចាប់ផ្តើមអថេរដោយស្វ័យប្រវត្តិទៅតម្លៃអព្យាក្រឹតរបស់វានៅពេលប្រើប្រាស់ដំបូង:

```zymbol

@ n:លេខ {
    °សរុប += n    // ចាប់ផ្តើមដោយស្វ័យប្រវត្តិទៅ ០ នៅពីលើរង្វិលជុំ; រស់រានមានជីវិតបន្ទាប់ពី @
```

> `°x` (បុព្វបទ) បោះយុថ្កានៅពីលើរង្វិលជុំ — លទ្ធផលអាចចូលដំណើរការបានបន្ទាប់ពី `@`។

## បឋម TUI

ប្រមាណវិធីចំណុចប្រទាក់អ្នកប្រើប្រាស់ស្ថានីយសម្រាប់កម្មវិធីអន្តរកម្ម។ ភាគច្រើនត្រូវការប្លុក `>>| { }` (អេក្រង់ជំនួស + របៀបឆៅ)។

```zymbol
>>| {
    >>!                             // សម្អាតអេក្រង់ជំនួស
    >>~ (១, ១, ០, ១០) > "កំពុងដំណើរការ"   // ជួរដេក ១, ជួរឈរ ១, fg=១០ (បៃតង)
    @~ ១០០០                         // ផ្អាក ១ វិនាទី (១០០០ ms)
    >>~ (២, ១) > "បានបញ្ចប់។"
}
// ស្ថានីយត្រូវបានស្ដារឡើងវិញដោយស្វ័យប្រវត្តិនៅពេលចាកចេញ
```

```zymbol
// ការចុចគ្រាប់ចុច និងទំហំស្ថានីយ
>>| {
    [ជួរដេក, ជួរឈរ] = >>?              // សួរវិមាត្រស្ថានីយ
    >>~ (១, ១) > "ស្ថានីយ: " ជួរដេក " x " ជួរឈរ
    <<| គ្រាប់ចុច                         // អានការចុចគ្រាប់ចុចរារាំង
    >>~ (២, ១) > "បានចុច: " គ្រាប់ចុច
}
```

> `>>!` សម្អាតអេក្រង់។ `>>?` ត្រឡប់ `[ជួរដេក, ជួរឈរ]`។ `@~ N` ដេក N មិល្លីវិនាទី។
> `<<|` អានការចុចគ្រាប់ចុចមួយ (រារាំង); `<<|?` ស្ទង់មតិដោយមិនរារាំង (ត្រឡប់ `'\0'` ប្រសិនបើគ្មាន)។
> តាប់ពលទិន្នផលតាមទីតាំង: `(ជួរដេក, ជួរឈរ, BKS, fg, bg)` — រន្ធណាមួយអាចត្រូវបានលុបចោលដោយប្រើក្បៀស (`>>~ (,,, 196) > "ក្រហម"`)។
> BKS ប៊ីតម៉ាស្ក៍: `1`=ដិត, `2`=ទ្រេត, `4`=គូសបន្ទាត់ពីក្រោម។ ក្ដារលាយពណ៌ ANSI 256 (`0`=លំនាំដើមស្ថានីយ)។
> មានតែ tree-walker (លើកលែងតែ `>>!`, `>>?`, `@~`, `>>~` ដែលដំណើរការផងដែរក្នុង `--vm`)។

```zymbol
@ ផ:ផ្លែឈើ { >> ផ ¶ }         // សម្រាប់ធាតុនីមួយៗក្នុងអារេ

@ ត:"សួស្តី" { >> ត "-" }
>> ¶                          // → ស-ួ-ស-្ត-ី-  (សម្រាប់តួអក្សរនីមួយៗក្នុងខ្សែអក្សរ)

@ i:១..១០ {
    ? i % ២ == ០ { @> }       // @> បន្ត
    ? i > ៧ { @! }             // @! បំបែក
    >> i " "
}
>> ¶                          // → ១ ៣ ៥ ៧

// រង្វិលជុំគ្មានកំណត់
i = ០
@ {
    i++
    ? i >= ៥ { @! }
    >> i " "
}
>> ¶                          // → ១ ២ ៣ ៤

// រង្វិលជុំដែលមានស្លាក (ការបំបែកជាប់គ្នា)
រាប់ = ០
@:ខាងក្រៅ {
    រាប់++
    ? រាប់ >= ៣ { @:ខាងក្រៅ! }
}
>> រាប់ ¶                    // → ៣
```

## មុខងារ

```zymbol
បូក(a, b) { <~ a + b }
>> បូក(៣, ៤) ¶    // → ៧

ហ្វាក់តូរីយ៉ាល់(n) {
    ? n <= ១ { <~ ១ }
    <~ n * ហ្វាក់តូរីយ៉ាល់(n - ១)
}
>> ហ្វាក់តូរីយ៉ាល់(៥) ¶    // → ១២០
```

មុខងារមាន **វិសាលភាពដាច់ដោយឡែក** — ពួកវាមិនអាចអានអថេរខាងក្រៅបានទេ។ ប្រើប៉ារ៉ាម៉ែត្រទិន្នផល `<~>` ដើម្បីកែប្រែអថេររបស់អ្នកហៅ:

```zymbol
ប្តូរ(a<~, b<~) {
    បណ្តោះអាសន្ន = a
    a = b
    b = បណ្តោះអាសន្ន
}
x = ១០
y = ២០
ប្តូរ(x, y)
>> "x=" x " y=" y ¶    // → x=២០ y=១០
```

> មុខងារដែលមានឈ្មោះគឺ **តម្លៃថ្នាក់ទីមួយ** — បញ្ជូនដោយផ្ទាល់: `លេខ$> ទ្វេដង`។ សម្រាប់រុំព័ទ្ធ: `x -> fn(x)` ក៏ត្រឹមត្រូវដែរ។

## ឡាមប្ដា និងការបិទ

```zymbol
ទ្វេដង = x -> x * ២
បូក = (a, b) -> a + b
>> ទ្វេដង(៥) ¶    // → ១០
>> បូក(៣, ៧) ¶  // → ១០

// ឡាមប្ដាប្លុក
ចាត់ថ្នាក់ = x -> {
    ? x > ០ { <~ "វិជ្ជមាន" }
    _? x < ០ { <~ "អវិជ្ជមាន" }
    <~ "សូន្យ"
}

// ការបិទ — ចាប់យកវិសាលភាពខាងក្រៅ
កត្តា = ៣
បីដង = x -> x * កត្តា
>> បីដង(៧) ¶    // → ២១

// រោងចក្រ
អ្នកបង្កើត_បូក(n) { <~ x -> x + n }
បូកដប់ = អ្នកបង្កើត_បូក(១០)
>> បូកដប់(៥) ¶    // → ១៥

// ក្នុងអារេ
ប្រមាណវិធី = [x -> x+១, x -> x*២, x -> x*x]
>> ប្រមាណវិធី[៣](៥) ¶    // → ២៥
```

## អារេ

អារេ **អាចផ្លាស់ប្តូរបាន** និងមានធាតុ **នៃប្រភេទដូចគ្នា**។

```zymbol
arr = [១, ២, ៣, ៤, ៥]

x = arr[១]      // ១ — ចូលដំណើរការ (១-គោល: ធាតុទីមួយ)
x = arr[-១]     // ៥ — សន្ទស្សន៍អវិជ្ជមាន (ធាតុចុងក្រោយ)
x = arr$#       // ៥ — ប្រវែង (ប្រើ (arr$#) ក្នុង >>)

arr = arr$+ ៦            // បន្ថែម → [១,២,៣,៤,៥,៦]
arr២ = arr$+[២] ៩៩       // បញ្ចូលនៅទីតាំង ២ (១-គោល)
arr៣ = arr$- ៣           // លុបការកើតឡើងដំបូងនៃតម្លៃ
arr៤ = arr$-- ៣          // លុបការកើតឡើងទាំងអស់
arr៥ = arr$-[១]          // លុបនៅសន្ទស្សន៍ ១ (ធាតុទីមួយ)
arr៦ = arr$-[២..៣]       // លុបជួរ (១-គោល, បញ្ចូលទាំងចុង)

មាន = arr$? ៣            // #១ — មាន
ទីតាំង = arr$?? ៣           // [៣] — សន្ទស្សន៍ទាំងអស់នៃតម្លៃ (១-គោល)
ចំណិត = arr$[១..៣]          // [១,២,៣] — ចំណិត (១-គោល, បញ្ចូលទាំងចុង)
ចំណិត២ = arr$[១:៣]          // [១,២,៣] — ដូចគ្នា, វាក្យសម្ព័ន្ធផ្អែកលើចំនួន

ឡើង = arr$^+             // តម្រៀបឡើង (មានតែបុព្វកាល)
ចុះ = arr$^-            // តម្រៀបចុះ (មានតែបុព្វកាល)

// អារេតាប់ពលដែលមានឈ្មោះ/តាមទីតាំង — ប្រើ $^ ជាមួយឡាមប្ដាប្រៀបធៀប
មូលដ្ឋានទិន្នន័យ = [(ឈ្មោះ: "ខាឡា", អាយុ: ២៨), (ឈ្មោះ: "អាណា", អាយុ: ២៥), (ឈ្មោះ: "បប", អាយុ: ៣០)]
តាមអាយុ  = មូលដ្ឋានទិន្នន័យ$^ (a, b -> a.អាយុ < b.អាយុ)    // តាមអាយុឡើង (<)
តាមឈ្មោះ = មូលដ្ឋានទិន្នន័យ$^ (a, b -> a.ឈ្មោះ > b.ឈ្មោះ)   // តាមឈ្មោះចុះ (>)
>> តាមអាយុ[១].ឈ្មោះ ¶     // → អាណា
>> តាមឈ្មោះ[១].ឈ្មោះ ¶    // → ខាឡា

// ការធ្វើបច្ចុប្បន្នភាពធាតុដោយផ្ទាល់ (មានតែអារេ)
arr[១] = ៩៩              // ចាត់តាំង
arr[២] += ៥              // សមាសធាតុ: +=  -=  *=  /=  %=  ^=

// ការធ្វើបច្ចុប្បន្នភាពមុខងារ — ត្រឡប់អារេថ្មី; ដើមមិនផ្លាស់ប្តូរទេ
arr២ = arr[២]$~ ៩៩
```

> ប្រមាណវិធីប្រមូលផ្តុំទាំងអស់ត្រឡប់ **អារេថ្មី**។ ចាត់តាំងត្រឡប់: `arr = arr$+ 4`។
> `$+` អាចត្រូវបានភ្ជាប់ជាខ្សែ: `arr = arr$+ 5$+ 6$+ 7`។ ប្រមាណវិធីផ្សេងទៀតប្រើការចាត់តាំងកម្រិតមធ្យម។
> **សន្ទស្សន៍គឺ 1-គោល**: `arr[1]` គឺជាធាតុទីមួយ; `arr[0]` គឺជាកំហុសពេលដំណើរការ។
> `$^+` / `$^-` តម្រៀប **អារេបុព្វកាល** (លេខ, ខ្សែអក្សរ)។ សម្រាប់អារេតាប់ពល ប្រើ `$^` ជាមួយឡាមប្ដាប្រៀបធៀប — ទិសដៅត្រូវបានអ៊ិនកូដក្នុងឡាមប្ដា (`<` = ឡើង, `>` = ចុះ)។

**ន័យវិទ្យាតម្លៃ** — ការចាត់តាំងអារេទៅអថេរផ្សេងទៀតបង្កើតច្បាប់ចម្លងឯករាជ្យ:

```zymbol
a = [១, ២, ៣]
b = a
a[១] = ៩៩
>> a ¶    // → [៩៩, ២, ៣]
>> b ¶    // → [១, ២, ៣]   ← b មិនត្រូវបានប៉ះពាល់ទេ
```

```zymbol
// អារេជាប់គ្នា (សន្ទស្សន៍ ១-គោល)
ម៉ាទ្រីស = [[១,២,៣],[៤,៥,៦],[៧,៨,៩]]
>> ម៉ាទ្រីស[២][៣] ¶    // → ៦  (ជួរដេក ២, ជួរឈរ ៣)
```

## ការបំបែករចនាសម្ព័ន្ធ

```zymbol
// អារេ
arr = [១០, ២០, ៣០, ៤០, ៥០]
[a, b, c] = arr              // a=១០  b=២០  c=៣០
[ទីមួយ, *សល់] = arr         // ទីមួយ=១០  សល់=[២០,៣០,៤០,៥០]
[x, _, z] = [១, ២, ៣]        // _ បោះបង់

// តាប់ពលតាមទីតាំង
ចំណុច = (១០០, ២០០)
(px, py) = ចំណុច             // px=១០០  py=២០០

// តាប់ពលដែលមានឈ្មោះ
មនុស្ស = (ឈ្មោះ: "អាណា", អាយុ: ២៥, ទីក្រុង: "ម៉ាឌ្រីដ")
(ឈ្មោះ: n, អាយុ: អ) = មនុស្ស   // n="អាណា"  អ=២៥
```

## តាប់ពល

តាប់ពលគឺជាកុងតឺន័រតាមលំដាប់ **ដែលមិនអាចផ្លាស់ប្តូរបាន** ដែលអាចផ្ទុកតម្លៃ **នៃប្រភេទផ្សេងគ្នា**។
មិនដូចអារេទេ ធាតុមិនអាចផ្លាស់ប្តូរបានបន្ទាប់ពីការបង្កើត។

```zymbol
// តាមទីតាំង — ប្រភេទលាយគ្នាត្រូវបានអនុញ្ញាត
ចំណុច = (១០, ២០)
>> ចំណុច[១] ¶    // → ១០

ទិន្នន័យ = (៤២, "សួស្តី", #១, ៣.១៤)
>> ទិន្នន័យ[៣] ¶     // → #១

// មានឈ្មោះ
មនុស្ស = (ឈ្មោះ: "អាលីស", អាយុ: ២៥)
>> មនុស្ស.ឈ្មោះ ¶    // → អាលីស
>> មនុស្ស[១] ¶      // → អាលីស  (សន្ទស្សន៍ក៏ដំណើរការដែរ, ១-គោល)

// ជាប់គ្នា
ទីតាំង = (x: ១០, y: ២០)
p = (ទីតាំង: ទីតាំង, ស្លាក: "ប្រភពដើម")
>> p.ទីតាំង.x ¶        // → ១០
```

**ភាពមិនអាចផ្លាស់ប្តូរបាន** — ការប៉ុនប៉ងណាមួយដើម្បីកែប្រែធាតុតាប់ពលគឺជាកំហុសពេលដំណើរការ:

```zymbol
t = (១០, ២០, ៣០)
// t[១] = ៩៩    // ❌ កំហុសពេលដំណើរការ: តាប់ពលមិនអាចផ្លាស់ប្តូរបានទេ
// t[១] += ៥    // ❌ កំហុសដូចគ្នា
```

ដើម្បីទទួលបានតម្លៃដែលបានកែប្រែ សូមប្រើ `$~` (ការធ្វើបច្ចុប្បន្នភាពមុខងារ) — ត្រឡប់ **តាប់ពលថ្មី**:

```zymbol
t = (១០, ២០, ៣០)
t២ = t[២]$~ ៩៩៩
>> t ¶     // → (១០, ២០, ៣០)   ← ដើមមិនផ្លាស់ប្តូរទេ
>> t២ ¶    // → (១០, ៩៩៩, ៣០)

// តាប់ពលដែលមានឈ្មោះ — កសាងឡើងវិញយ៉ាងច្បាស់លាស់
មនុស្ស = (ឈ្មោះ: "អាលីស", អាយុ: ២៥)
ចាស់ជាង  = (ឈ្មោះ: មនុស្ស.ឈ្មោះ, អាយុ: ២៦)
>> មនុស្ស.អាយុ ¶    // → ២៥
>> ចាស់ជាង.អាយុ ¶     // → ២៦
```

## មុខងារលំដាប់ខ្ពស់

```zymbol
លេខ = [១, ២, ៣, ៤, ៥, ៦, ៧, ៨, ៩, ១០]

ទ្វេដង  = លេខ$> (x -> x * ២)                  // map  → [២,៤,៦…២០]
គូ    = លេខ$| (x -> x % ២ == ០)           // filter → [២,៤,៦,៨,១០]
សរុប    = លេខ$< (០, (អ្នកប្រមូល, x) -> អ្នកប្រមូល + x)     // reduce → ៥៥

// ភ្ជាប់ខ្សែតាមរយៈកម្រិតមធ្យម
ជំហាន១ = លេខ$| (x -> x > ៣)
ជំហាន២ = ជំហាន១$> (x -> x * x)
>> ជំហាន២ ¶    // → [១៦, ២៥, ៣៦, ៤៩, ៦៤, ៨១, ១០០]

// មុខងារដែលមានឈ្មោះអាចត្រូវបានបញ្ជូនដោយផ្ទាល់ទៅ HOF
ទ្វេដង(x) { <~ x * ២ }
ធំ(x) { <~ x > ៥ }
r = លេខ$> ទ្វេដង       // ✅ ឯកសារយោងផ្ទាល់
r = លេខ$| ធំ       // ✅ ឯកសារយោងផ្ទាល់
```

## ប្រមាណវិធីបំពង់

ផ្នែកខាងស្តាំតែងតែត្រូវការ `_` ជាអ្នកកាន់កន្លែងសម្រាប់តម្លៃដែលត្រូវបានបំពង់:

```zymbol
ទ្វេដង = x -> x * ២
បូក = (a, b) -> a + b
បង្កើន = x -> x + ១

r១ = ៥ |> ទ្វេដង(_)        // → ១០
r២ = ១០ |> បូក(_, ៥)       // → ១៥
r៣ = ៥ |> បូក(២, _)        // → ៧

// ភ្ជាប់ខ្សែ
r = ៥ |> ទ្វេដង(_) |> បង្កើន(_) |> ទ្វេដង(_)
>> r ¶    // → ២២  (៥→១០→១១→២២)
```

## ការគ្រប់គ្រងកំហុស

```zymbol
!? {

} :! ##Div {
    >> "ចែកនឹងសូន្យ" ¶
} :! {
```

## របៀបលេខ

Zymbol អាចបង្ហាញលេខក្នុង **69 អក្សរក្រមលេខយូនីកូដ** — ទេវៈនាគរី, អារ៉ាប់-ឥណ្ឌា, ថៃ, Klingon pIqaD, គណិតវិទ្យាដិត, ផ្នែក LCD, និងច្រើនទៀត។ របៀបសកម្មប៉ះពាល់តែទិន្នផល `>>` ប៉ុណ្ណោះ; នព្វន្ធផ្ទៃក្នុងគឺតែងតែជាគោលពីរ។

### ការធ្វើឱ្យសកម្មស្គ្រីប

សរសេរខ្ទង់ `0` និង `9` នៃស្គ្រីបគោលដៅក្នុង `#…#`:

```zymbol
#०९#    // ទេវៈនាគរី   (U+0966–U+096F)
#٠٩#    // អារ៉ាប់-ឥណ្ឌា (U+0660–U+0669)
#๐๙#    // ថៃ         (U+0E50–U+0E59)
#09#    // កំណត់ឡើងវិញទៅ ASCII
```

### ទិន្នផល និងប៊ូលីន

```zymbol
x = ៤២
>> x ¶          // → ៤២   (លំនាំដើម ASCII)

#०९#
>> x ¶          // → ४२
>> ៣.១៤ ¶       // → ३.१४   (ចំណុចទសភាគតែងតែជា ASCII)
>> ១ + ២ ¶      // → ३

// ប៊ូលីន: បុព្វបទ # តែងតែជា ASCII, ខ្ទង់សម្របខ្លួន
>> #១ ¶         // → #१   (ពិតក្នុងទេវៈនាគរី)
>> #០ ¶         // → #०   (មិនពិត — ខុសគ្នាពី ០ ចំនួនគត់សូន្យ)

x = ២៨ > ៤
>> x ¶          // → #१   (លទ្ធផលប្រៀបធៀបធ្វើតាមរបៀបសកម្ម)
```

### ខ្ទង់តាមព្យញ្ជនៈដើមក្នុងកូដប្រភព

ខ្ទង់នៃស្គ្រីបដែលគាំទ្រណាមួយគឺជាតម្លៃតាមព្យញ្ជនៈត្រឹមត្រូវ — ក្នុងជួរ, ម៉ូឌុយឡូ, ការប្រៀបធៀប:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### តម្លៃប៊ូលីនតាមព្យញ្ជនៈក្នុងស្គ្រីបណាមួយ

`#` + ខ្ទង់ `0` ឬ `1` ពីប្លុកណាមួយគឺជាតម្លៃប៊ូលីនតាមព្យញ្ជនៈត្រឹមត្រូវ:

```zymbol
#०९#
សកម្ម = #१        // ដូចគ្នានឹង #១
>> សកម្ម ¶        // → #१
>> (#१ && #०) ¶ // → #०
```

> `#` **តែងតែជា ASCII**។ `#0` (មិនពិត) តែងតែខុសគ្នាដោយមើលឃើញពី `0` (ចំនួនគត់សូន្យ) ក្នុងគ្រប់ស្គ្រីបទាំងអស់។

## ប្រមាណវិធីទិន្នន័យ

```zymbol
// ការបំលែងប្រភេទ
f = ##.៤២         // → ៤២.០  (ទៅជាចំនួនទសភាគ)
i = ###៣.៧        // → ៤     (ទៅជាចំនួនគត់, បង្គត់)
t = ##!៣.៧        // → ៣     (ទៅជាចំនួនគត់, កាត់)

// វិភាគខ្សែអក្សរទៅជាលេខ
v១ = #|"៤២"|      // → ៤២  (ចំនួនគត់)
v២ = #|"៣.១៤"|    // → ៣.១៤  (ចំនួនទសភាគ)
v៣ = #|"abc"|     // → "abc"  (មានសុវត្ថិភាព, គ្មានកំហុស)

// បង្គត់ / កាត់
π = ៣.១៤១៥៩២៦៥
បង្គត់២ = #.២|π|      // → ៣.១៤  (បង្គត់ទៅ ២ ខ្ទង់ទសភាគ)
បង្គត់៤ = #.៤|π|      // → ៣.១៤១៦
កាត់២ = #!២|π|      // → ៣.១៤  (កាត់)

// ការធ្វើទ្រង់ទ្រាយលេខ
ទ្រង់ទ្រាយ = #,|១២៣៤៥៦៧|  // → ១,២៣៤,៥៦៧  (បំបែកដោយសញ្ញាក្បៀស)
វិទ្យាសាស្ត្រ = #^|១២៣៤៥.៦៧៨|    // → ១.២៣៤៥៦៧៨e៤  (វិទ្យាសាស្ត្រ)

// តម្លៃតាមព្យញ្ជនៈគោល
a = 0x41         // → 'A'  (គោលដប់ប្រាំមួយ)
b = 0b01000001   // → 'A'  (គោលពីរ)
c = 0o101        // → 'A'  (គោលប្រាំបី)

// ទិន្នផលបំលែងគោល
គោលដប់ប្រាំមួយ = 0x|២៥៥|    // → "0x00FF"
គោលពីរ = 0b|៦៥|     // → "0b1000001"
គោលប្រាំបី = 0o|៨|      // → "0o10"
គោលដប់ = 0d|២៥៥|    // → "0d0255"
```

| `###` | បំលែងទៅជាចំនួនគត់ (បង្គត់) | `##!` | បំលែងទៅជាចំនួនគត់ (កាត់) |

| `#,\|..\|` | ទ្រង់ទ្រាយក្បៀស | `#^\|..\|` | វិទ្យាសាស្ត្រ |

| `<<\|` | ការចុចគ្រាប់ចុចរារាំង | `<<\|?` | ស្ទង់មតិការចុចគ្រាប់ចុចមិនរារាំង |
| `@~ N` | ដេក N មិល្លីវិនាទី | `$*` | ធ្វើខ្សែអក្សរឡើងវិញ N ដង |

### v0.0.5 — បឋម TUI, និយមន័យក្តៅ & ការធ្វើខ្សែអក្សរឡើងវិញ _(ឧសភា 2026)_

- **ការបំបែក** សញ្ញាបំបែកដៃផ្គូផ្គង: `លំនាំ : លទ្ធផល` → `លំនាំ => លទ្ធផល`
- **ការបំបែក** ឈ្មោះក្លែងក្លាយនាំចូល: `<# ផ្លូវ <= ឈ្មោះក្លែងក្លាយ` → `<# ផ្លូវ => ឈ្មោះក្លែងក្លាយ`
- **ការបំបែក** ប្តូរឈ្មោះនាំចេញ: `#> { fn <= សាធារណៈ }` → `#> { fn => សាធារណៈ }`
- **បានបន្ថែម** ប្លុក TUI `>>| { }` — អេក្រង់ជំនួស + របៀបឆៅ; សម្អាតនៅពេលចាកចេញ
- **បានបន្ថែម** ទិន្នផលតាមទីតាំង `>>~ (ជួរដេក, ជួរឈរ, BKS, fg, bg) > ធាតុ` — រន្ធដោតដាច់ៗ, ពណ៌ ANSI 256
- **បានបន្ថែម** ការបញ្ចូលគ្រាប់ចុច `<<| អថេរ` (រារាំង) និង `<<|? អថេរ` (ស្ទង់មតិមិនរារាំង)
- **បានបន្ថែម** `>>!` សម្អាតអេក្រង់, `>>?` សួរទំហំស្ថានីយ, `@~ N` ដេក N មិល្លីវិនាទី
- **បានបន្ថែម** និយមន័យក្តៅ `°x` / `x°` — ចាប់ផ្តើមអថេរដោយស្វ័យប្រវត្តិនៅពេលប្រើប្រាស់ដំបូងក្នុងរង្វិលជុំ
- **បានបន្ថែម** ការធ្វើខ្សែអក្សរឡើងវិញ `ខ្សែអក្សរ $* N` — ធ្វើខ្សែអក្សរឡើងវិញ N ដង
- **VM** ភាពស្មើ: 436/436 តេស្តបានប្រលងជាប់

### v0.0.4 — សន្ទស្សន៍ 1-គោល, មុខងារថ្នាក់ទីមួយ & ម៉ូឌុលប្លុក _(មេសា 2026)_

- **ការបំបែក** សន្ទស្សន៍ទាំងអស់ត្រូវបានផ្លាស់ប្តូរទៅ **1-គោល** — `arr[1]` គឺជាធាតុទីមួយ; `arr[0]` គឺជាកំហុសពេលដំណើរការ
- **បានបន្ថែម** មុខងារដែលមានឈ្មោះគឺ **តម្លៃថ្នាក់ទីមួយ** — បញ្ជូនដោយផ្ទាល់ទៅ HOF: `លេខ$> ទ្វេដង`
- **បានបន្ថែម** **វាក្យសម្ព័ន្ធប្លុកចាំបាច់** សម្រាប់ម៉ូឌុល: `# ឈ្មោះ { ... }` — វាក្យសម្ព័ន្ធសំប៉ែតត្រូវបានដកចេញ
- **បានបន្ថែម** សន្ទស្សន៍ពហុវិមាត្រ: `arr[i>j>k]` (រុករក), `arr[p ; q]` (ការទាញយកសំប៉ែត)
- **បានបន្ថែម** ការបំលែងប្រភេទ: `##.កន្សោម` (ចំនួនទសភាគ), `###កន្សោម` (ចំនួនគត់បង្គត់), `##!កន្សោម` (ចំនួនគត់កាត់)
- **បានបន្ថែម** ការបំបែកខ្សែអក្សរ: `ខ្សែអក្សរ$/ សញ្ញាកំណត់` — ត្រឡប់ `Array(ខ្សែអក្សរ)`
- **បានបន្ថែម** ការបង្កើតការភ្ជាប់: `មូលដ្ឋាន$++ a b c` — បន្ថែមធាតុច្រើន
- **បានបន្ថែម** រង្វិលជុំដង: `@ N { }` — ធ្វើម្តងទៀតយ៉ាងពិតប្រាកដ N ដង
- **បានបន្ថែម** វាក្យសម្ព័ន្ធរង្វិលជុំដែលមានស្លាក: `@:ឈ្មោះ { }`, `@:ឈ្មោះ!`, `@:ឈ្មោះ>` — ជំនួស `@ @ឈ្មោះ` / `@! ឈ្មោះ`
- **បានបន្ថែម** ច្បាប់វិសាលភាពអថេរ: អថេរ `_ឈ្មោះ` មានវិសាលភាពប្លុកពិតប្រាកដ; `\ អថេរ` បំផ្លាញមុនពេល
- **បានបន្ថែម** លំនាំប្រៀបធៀបផ្គូផ្គង: `< 0 =>`, `> 5 =>`, `== 42 =>`, ល
- **បានបន្ថែម** កំហុសម៉ូឌុល E013: សេចក្តីថ្លែងការណ៍ដែលអាចប្រតិបត្តិបាននៅក្នុងតួម៉ូឌុលត្រូវបានហាមឃាត់
- **បានកែតម្រូវ** `alias.CONST` ឥឡូវដោះស្រាយបានត្រឹមត្រូវ; `#>` អាចលេចឡើងបន្ទាប់ពីនិយមន័យមុខងារ
- **VM** ភាពស្មើពេញលេញ: 393/393 តេស្តបានប្រលងជាប់
