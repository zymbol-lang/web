# Zymbol-Lang को संक्षिप्त मार्गदर्शिका

**Zymbol-Lang** एक प्रतीकात्मक प्रोग्रामिङ भाषा हो। यसमा कुनै कीवर्ड छैन — सबै कुरा प्रतीक हो। यो कुनै पनि मानव भाषामा एउटै तरिकाले काम गर्छ।

- कुनै कीवर्ड छैन (`if`, `while`, `return` अवस्थित छैनन् — केवल प्रतीकहरू `?`, `@`, `<~`)
- पूर्ण Unicode — कुनै पनि भाषा वा इमोजीमा पहिचानकर्ता 👋
- मानव-भाषा तटस्थ — कोड हरेक भाषामा समान छ

## चर र स्थिरांक (Variables and Constants)

```zymbol
च = १०           // चर (परिवर्तनयोग्य)
पाई := ३.१४१५९   // स्थिरांक (अपरिवर्तनीय — पुनः असाइन गर्दा त्रुटि)
नाम = "अनु"
सक्रिय = #१     // boolean true
```

```zymbol
च = १०
च += ५    // १५
च -= ३    // १२
च *= २    // २४
च /= ३    // ८
च %= ३    // २
च ^= २    // ४
च++       // ५
च--       // ४
```

## डाटा प्रकार तालिका (Data Types)

| प्रकार       | उदाहरण              | `#?` प्रतीक | नोट                                   |
|--------------|---------------------|-------------|---------------------------------------|
| पूर्णाङ्क    | `४२`, `-७`          | `###`       | 64-bit signed                         |
| दशमलव        | `३.१४`, `१.५e१०`    | `##.`       | वैज्ञानिक संकेत मान्य                |
| स्ट्रिङ      | `"नमस्ते"`          | `##"`       | इन्टरपोलेसन: `"नमस्ते {नाम}"`        |
| क्यारेक्टर   | `'क'`               | `##'`       | एक Unicode अक्षर                     |
| बुलियन       | `#१`, `#०`          | `##?`       | संख्या 1 र 0 होइन                    |
| सरणी         | `[१, २, ३]`         | `##]`       | सबै तत्वहरू एउटै प्रकारका हुनुपर्छ   |
| टुपल         | `(क, ख)`            | `##)`       | स्थितिगत                             |
| नामित टुपल  | `(x: १, y: २)`      | `##)`       | नाम वा अनुक्रमणिकाद्वारा पहुँच      |

```zymbol
// प्रकार जाँच — (type, digits, value) फर्काउँछ
meta = ४२#?
>> meta ¶         // → (###, २, ४२)
t = meta[०]
>> t ¶            // → ###
```

## आउटपुट र इनपुट (Output and Input)

```zymbol
// आउटपुट — स्वचालित रूपमा नयाँ लाइन थप्दैन
>> "नमस्ते" ¶                    // ¶ वा \\ ले स्पष्ट नयाँ लाइन दिन्छ
>> "क=" क " ख=" ख ¶             // juxtaposition द्वारा धेरै मानहरू
>> "योगफल=" जोड्नु(२, ३) ¶      // कुनै पनि स्थितिमा प्रकार्य कल
>> (सूची$#) ¶                   // postfix अपरेटरलाई कोष्ठक चाहिन्छ

// इनपुट
<< नाम                          // prompt बिना — चरमा पढ्छ
<< "तपाईंको नाम? " नाम          // prompt सहित
```

> `¶` र `\\` नयाँ लाइनको रूपमा समकक्ष छन्।

## स्ट्रिङ

```zymbol
// तीन जोड्ने तरिकाहरू
नाम = "अनु"
उमेर = २५

सन्देश = "नमस्ते ", नाम, "!"         // अल्पविराम — असाइनमेन्टमा
>> "नमस्ते " नाम " तपाईं " उमेर " वर्षको हुनुहुन्छ" ¶    // juxtaposition — >> मा
विवरण = "नमस्ते {नाम}, तपाईं {उमेर} वर्षको हुनुहुन्छ"   // interpolation — कतै पनि
```

```zymbol
स = "नमस्ते दुनिया"
ल = स$#                  // length
उ = स$[०..५]             // "नमस्त"  (अन्त बाहेक)
छ = स$? "दुनि"           // #१
भ = "क,ख,ग,घ" / ','      // [क, ख, ग, घ]
ब = स$~~["त":"ट"]         // replace
ब१ = स$~~["त":"ट":१]      // first N
```

> `+` केवल संख्याहरूका लागि। Strings को लागि `,`, juxtaposition वा interpolation प्रयोग गर्नुहोस्।

## प्रवाह नियन्त्रण (Control Flow)

```zymbol
च = ७

// सरल if
? च > ० { >> "धनात्मक" ¶ }

// if / else-if / else
? च > १०० {
    >> "ठूलो" ¶
} _? च > ० {
    >> "धनात्मक" ¶
} _? च == ० {
    >> "शून्य" ¶
} _ {
    >> "ऋणात्मक" ¶
}
```

`{ }` ब्लकहरू एकल लाइनका लागि पनि **अनिवार्य** छन्।

## मिलान (Match)

```zymbol
// दायरासहित Match
अंक = ८५
श्रेणी = ?? अंक {
    ९०..१०० : 'अ'
    ८०..८९  : 'ब'
    ७०..७९  : 'स'
    _       : 'फ'
}
>> श्रेणी ¶    // → ब

// स्ट्रिङसहित Match
रंग = "रातो"
कोड = ?? रंग {
    "रातो"   : "#FF००००"
    "हरियो"  : "#००FF००"
    _        : "#००००००"
}

// गार्डसहित Match (मनमाने सर्तहरू)
तापमान = -५
अवस्था = ?? तापमान {
    _? तापमान < ०  : "हिउँ"
    _? तापमान < २० : "चिसो"
    _? तापमान < ३५ : "न्यानो"
    _              : "तातो"
}
>> अवस्था ¶    // → हिउँ

// Statement form (block arms)
?? न {
    ०       : { >> "शून्य" ¶ }
    _? न < ०: { >> "ऋणात्मक" ¶ }
    _       : { >> "धनात्मक" ¶ }
}
```

## लूप (Loops)

```zymbol
@ न:०..४  { >> न " " }        // दायरा समावेशी:  ० १ २ ३ ४
@ न:१..९:२ { >> न " " }       // चरणसहित:         १ ३ ५ ७ ९
@ न:५..०:१ { >> न " " }       // उल्टो:            ५ ४ ३ २ १ ०

न = १
@ न <= ६४ { न *= २ }
>> न ¶                        // → १२८  (while)

फलहरू = ["स्याउ", "नाशपाती", "अंगूर"]
@ फ:फलहरू { >> फ ¶ }         // for-each array

@ च:"नमस्ते" { >> च "-" }
>> ¶                          // → न-म-स्-त-े-  (for-each string)

@ न:१..१० {
    ? न % २ == ० { @> }       // @> continue
    ? न > ७ { @! }             // @! break
    >> न " "
}
>> ¶                          // → १ ३ ५ ७

// अनन्त लूप
i = ०
@ {
    i++
    ? i >= ५ { @! }
    >> i " "
}
>> ¶                          // → १ २ ३ ४

// Labeled loop (nested break)
count = ०
@ @outer {
    count++
    ? count >= ३ { @! outer }
}
>> count ¶                    // → ३
```

## प्रकार्यहरू (Functions)

```zymbol
// घोषणा र कल
जोड्नु(क, ख) { <~ क + ख }
>> जोड्नु(३, ४) ¶    // → ७

// पुनरावृत्ति (Recursion)
गुणनखण्ड(न) {
    ? न <= १ { <~ १ }
    <~ न * गुणनखण्ड(न - १)
}
>> गुणनखण्ड(५) ¶    // → १२०
```

प्रकार्यहरूको **पृथक scope** छ — बाहिरी चरहरूमा पहुँच छैन। Caller चरहरू परिमार्जन गर्न output parameters `<~` प्रयोग गर्नुहोस्:

```zymbol
साट्नु(क<~, ख<~) {
    अस्थायी = क
    क = ख
    ख = अस्थायी
}
च = १०
ज = २०
साट्नु(च, ज)
>> "च=" च " ज=" ज ¶    // → च=२० ज=१०
```

> नामित प्रकार्यहरू first-class मान होइनन्। तर्कको रूपमा पास गर्न, लपेट्नुहोस्: `x -> जोड्नु(x)`.

## ल्याम्बडा र क्लोजर

```zymbol
// सरल ल्याम्बडा (अन्तर्निहित return)
दोब्बर = च -> च * २
जोड्नु = (क, ख) -> क + ख
>> दोब्बर(५) ¶    // → १०
>> जोड्नु(३, ७) ¶  // → १०

// ब्लक ल्याम्बडा (स्पष्ट return)
वर्गीकृत_मान = च -> {
    ? च > ० { <~ "धनात्मक" }
    _? च < ० { <~ "ऋणात्मक" }
    <~ "शून्य"
}

// क्लोजर — ल्याम्बडाहरूले बाहिरी scope चरहरू capture गर्छन्
गुणांक = ३
तेब्बर = च -> च * गुणांक
>> तेब्बर(७) ¶    // → २१

// प्रकार्य फ्याक्ट्री
जोड्ने_बनाउनु(न) { <~ च -> च + न }
दस_जोड = जोड्ने_बनाउनु(१०)
>> दस_जोड(५) ¶    // → १५

// ल्याम्बडा मानको रूपमा: सरणीमा भण्डारण
सञ्चालन = [च -> च+१, च -> च*२, च -> च*च]
>> सञ्चालन[२](५) ¶    // → २५
```

## सूची (Arrays)

सूचीहरू **परिवर्तनयोग्य** छन् र **एउटै प्रकारका** तत्वहरू राख्छन्।

```zymbol
सूची = [१, २, ३, ४, ५]

सूची[०]          // १ — पहुँच (०-आधारित)
सूची[-१]         // ५ — नकारात्मक सूचकाङ्क (अन्तिम)
सूची$#           // ५ — लम्बाई

सूची = सूची$+ ६            // append → [१,२,३,४,५,६]
सूची२ = सूची$+[२] ९९       // सूचकाङ्क २ मा राख्नु
सूची३ = सूची$- ३           // पहिलो घटना हटाउनु
सूची४ = सूची$-- ३          // सबै घटना हटाउनु
सूची५ = सूची$-[०]          // सूचकाङ्कले हटाउनु
सूची६ = सूची$-[१..३]       // दायरा हटाउनु (अन्त बाहेक)

छ = सूची$? ३            // #१
स्थ = सूची$?? ३          // [२] — सबै सूचकाङ्क
टुक्रा = सूची$[०..३]     // [१,२,३]
टुक्रा२ = सूची$[०:३]     // [१,२,३] — गणना वाक्यरचना

बढ्दो = सूची$^+          // बढ्दो क्रम (primitives)
घट्दो = सूची$^-          // घट्दो क्रम (primitives)

// Named tuple arrays
डेटा = [(नाम: "Carla", उ: २८), (नाम: "Ana", उ: २५), (नाम: "Bob", उ: ३०)]
उ_क्रम  = डेटा$^ (क, ख -> क.उ < ख.उ)
न_क्रम = डेटा$^ (क, ख -> क.नाम > ख.नाम)
>> उ_क्रम[०].नाम ¶     // → Ana
>> न_क्रम[०].नाम ¶    // → Carla

// जगाने तत्व अद्यावधिक (सूचीमात्र)
सूची[१] = ९९              // असाइन गर्नुहोस्
सूची[०] += ५              // संयुक्त: +=  -=  *=  /=  %=  ^=

// Functional अद्यावधिक — नयाँ सूची फर्काउँछ; मूल अपरिवर्तित रहन्छ
सूची२ = सूची[१]$~ ९९
```

> सबै collection operators ले **नयाँ सरणी** फर्काउँछन्। फिर्ता असाइन गर्नुहोस्: `सूची = सूची$+ ४`.
> Chaining छैन: छुट्टाछुट्टै असाइनमेन्ट प्रयोग गर्नुहोस्।
> `$^+` / `$^-` ले **primitive arrays** sort गर्छन्। Tuple arrays को लागि comparator lambda सहित `$^` प्रयोग गर्नुहोस्।

**मूल्य शब्दार्थ** — सूची अर्को चरमा असाइन गर्दा एउटा स्वतन्त्र प्रति सिर्जना हुन्छ:

```zymbol
a = [1, 2, 3]
b = a
a[0] = 99
>> a ¶    // → [99, 2, 3]
>> b ¶    // → [1, 2, 3]   ← b अप्रभावित रहन्छ
```

```zymbol
// Nested arrays
matrix = [[१,२,३],[४,५,६],[७,८,९]]
>> matrix[१][२] ¶    // → ६
```

## डिस्ट्रक्चरिङ

```zymbol
// सरणी
सूची = [१०, २०, ३०, ४०, ५०]
[क, ख, ग] = सूची              // क=१०  ख=२०  ग=३०

बिन्दु = (१००, २००)
(bx, by) = बिन्दु             // bx=१००  by=२००

// Named tuple
व्यक्ति = (नाम: "Ana", उ: २५, सह: "Kathmandu")
(नाम: न, उ: उ_व) = व्यक्ति   // न="Ana"  उ_व=२५
```

## टुपल (Tuples)

टुपलहरू **अपरिवर्तनीय** क्रमबद्ध कन्टेनरहरू हुन् जसले **विभिन्न प्रकारका** मानहरू राख्न सक्छन्। सूचीको विपरीत, सिर्जना पछि तत्वहरू परिवर्तन गर्न सकिँदैन।

```zymbol
// स्थितिगत
बिन्दु = (१०, २०)
>> बिन्दु[०] ¶    // → १०

डेटा = (४२, "नमस्ते", #१, ३.१४)
>> डेटा[२] ¶     // → #१

// नामित
व्यक्ति = (नाम: "एलिस", उमेर: २५)
>> व्यक्ति.नाम ¶    // → एलिस
>> व्यक्ति[०] ¶     // → एलिस (अनुक्रमणिका पनि काम गर्छ)

// नेस्टेड
pos = (x: १०, y: २०)
p = (pos: pos, label: "origin")
>> p.pos.x ¶        // → १०
```

**अपरिवर्तनीयता** — टुपल तत्व परिवर्तन गर्ने कुनै पनि प्रयास रनटाइम त्रुटि हो:

```zymbol
t = (१०, २०, ३०)
// t[०] = ९९    // ❌ रनटाइम त्रुटि: टुपलहरू अपरिवर्तनीय छन्
// t[०] += ५    // ❌ समान त्रुटि
```

परिवर्तित मान प्राप्त गर्न `$~` (functional अद्यावधिक) प्रयोग गर्नुहोस् — **नयाँ** टुपल फर्काउँछ:

```zymbol
t = (१०, २०, ३०)
t२ = t[१]$~ ९९९
>> t ¶     // → (१०, २०, ३०)   ← मूल अपरिवर्तित
>> t२ ¶    // → (१०, ९९९, ३०)

// नामित टुपल — स्पष्ट रूपमा पुनर्निर्माण गर्नुहोस्
व्यक्ति = (नाम: "एलिस", उमेर: २५)
जेठो  = (नाम: व्यक्ति.नाम, उमेर: २६)
>> व्यक्ति.उमेर ¶    // → २५
>> जेठो.उमेर ¶       // → २६
```

## उच्च-क्रम प्रकार्यहरू (Higher-Order Functions)

HOF अपरेटरहरूलाई **इनलाइन ल्याम्बडा** चाहिन्छ — प्रत्यक्ष ल्याम्बडा चर होइन।

```zymbol
संख्याहरू = [१, २, ३, ४, ५, ६, ७, ८, ९, १०]

दोब्बरहरू = संख्याहरू$> (च -> च * २)                // map → [२,४,६…२०]
सम       = संख्याहरू$| (च -> च % २ == ०)            // filter → [२,४,६,८,१०]
```

## अंक प्रणालीहरू

Zymbol **69 Unicode अंक ब्लकहरू**मा संख्याहरू प्रदर्शित गर्न सक्छ — देवनागरी, अरबी-भारतीय, थाई, क्लिंगन pIqaD, बोल्ड गणितीय, LCD अंक, र अरू धेरै। सक्रिय मोडले केवल `>>` आउटपुटलाई असर गर्छ; आन्तरिक अंकगणित सधैँ बाइनरी हुन्छ।

### प्रणाली सक्रिय गर्नुहोस्

लक्ष्य लिपिका `०` र `९` अंकहरू `#…#` बीच लेख्नुहोस्:

```zymbol
#०९#    // देवनागरी    (U+0966–U+096F)
#٠٩#    // अरबी-भारतीय (U+0660–U+0669)
#๐๙#    // थाई         (U+0E50–U+0E59)
#09#    // ASCII रिसेट
```

### आउटपुट र बूलियन

```zymbol
x = ४२
>> x ¶          // → ४२   (पूर्वनिर्धारित ASCII)

#०९#
>> x ¶          // → ४२
>> ३.१४ ¶       // → ३.१४   (दशमलव बिन्दु सधैँ ASCII)
>> १ + २ ¶      // → ३

// बूलियन: # उपसर्ग सधैँ ASCII, अंक मोड अनुसार
>> #१ ¶         // → #१   (देवनागरीमा सत्य)
>> #० ¶         // → #०   (असत्य — ० शून्यभन्दा भिन्न)

x = २८ > ४
>> x ¶          // → #१   (तुलना परिणामले सक्रिय मोड पछ्याउँछ)
```

### स्रोतमा नेटिभ लिटरलहरू

कुनै पनि समर्थित प्रणालीका अंकहरू दायरा, मड्युलो, तुलनामा वैध लिटरल हुन्:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### कुनै पनि प्रणालीमा बूलियन लिटरल

कुनै पनि समर्थित ब्लकको `#` + `०` वा `१` अंक एक वैध बूलियन लिटरल हो:

```zymbol
#٠٩#
نشط = #١        // #१ सरह
>> نشط ¶        // → #١
>> (#١ && #٠) ¶ // → #٠
```

> `#` **सधैँ ASCII** हो। `#०` (असत्य) कुनै पनि प्रणालीमा `०` (शून्य पूर्णांक) भन्दा दृश्यात्मक रूपमा भिन्न हुन्छ।
