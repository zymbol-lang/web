`°` (डिग्री चिन्ह, U+00B0) पहिलो प्रयोगमा चरलाई यसको तटस्थ मानमा स्वचालित रूपमा प्रारम्भ गर्दछ:

```zymbol

@ n:सङ्ख्याहरू {
    °जम्मा += n    // लुपको माथि स्वचालित रूपमा ० मा प्रारम्भ; @ पछि बाँचिरहन्छ
}
>> जम्मा ¶         // → १४
```

> `°x` (उपसर्ग) लुपको माथि एङ्कर गर्दछ — परिणाम `@` पछि पहुँचयोग्य छ।
> `x°` (प्रत्यय) लुपको भित्र एङ्कर गर्दछ — जब लुप समाप्त हुन्छ मर्छ।

## टीयूआई प्रिमिटिभहरू

अन्तरक्रियात्मक प्रोग्रामहरूको लागि टर्मिनल प्रयोगकर्ता इन्टरफेस अपरेटरहरू। धेरैजसोलाई `>>| { }` ब्लक आवश्यक छ (वैकल्पिक स्क्रिन + कच्चा मोड)।

```zymbol
>>| {
    >>!                             // वैकल्पिक स्क्रिन सफा गर्नुहोस्
    >>~ (१, १, ०, १०) > "चलिरहेको छ"   // पङ्क्ति १, स्तम्भ १, fg=१० (हरियो)
    @~ १०००                         // १ सेकेन्ड रोकिनुहोस् (१००० ms)
    >>~ (२, १) > "सम्पन्न भयो।"
}
// बाहिर निस्कँदा टर्मिनल स्वचालित रूपमा पुनर्स्थापित हुन्छ
```

```zymbol
// कुञ्जी प्रेस र टर्मिनल आकार
>>| {
    [पङ्क्तिहरू, स्तम्भहरू] = >>?              // टर्मिनलको आयामहरू सोध्नुहोस्
    >>~ (१, १) > "टर्मिनल: " पङ्क्तिहरू " x " स्तम्भहरू
    <<| कुञ्जी                         // ब्लकिङ कुञ्जी प्रेस पढ्नुहोस्
    >>~ (२, १) > "थिच्नुभयो: " कुञ्जी
}
```

> `>>!` स्क्रिन सफा गर्दछ। `>>?` `[पङ्क्तिहरू, स्तम्भहरू]` फर्काउँछ। `@~ N` N मिलिसेकेन्ड सुत्छ।
> `<<|` एउटा कुञ्जी प्रेस पढ्छ (ब्लकिङ); `<<|?` ब्लक नगरी पोल गर्दछ (कुनै नभए `'\0'` फर्काउँछ)।
> स्थानीय आउटपुट टपल: `(पङ्क्ति, स्तम्भ, BKS, fg, bg)` — कुनै पनि स्लट अल्पविरामले छोड्न सकिन्छ (`>>~ (,,, 196) > "रातो"`)।
> BKS बिटमास्क: `1`=बोल्ड, `2`=इटालिक, `4`=रेखाङ्कन। ANSI 256 रङ प्यालेट (`0`=टर्मिनल पूर्वनिर्धारित)।
> केवल tree-walker (`>>!`, `>>?`, `@~`, `>>~` बाहेक जुन `--vm` मा पनि चल्छन्)।

## एरे

एरेहरू **परिवर्तनशील** हुन्छन् र **समान प्रकारका** तत्वहरू धारण गर्छन्।

```zymbol
arr = [१, २, ३, ४, ५]

x = arr[१]      // १ — पहुँच (१-आधारित: पहिलो तत्व)
x = arr[-१]     // ५ — ऋणात्मक सूचकाङ्क (अन्तिम तत्व)
x = arr$#       // ५ — लम्बाइ (>> मा (arr$#) प्रयोग गर्नुहोस्)

arr = arr$+ ६            // थप्नुहोस् → [१,२,३,४,५,६]
arr२ = arr$+[२] ९९       // स्थिति २ मा घुसाउनुहोस् (१-आधारित)
arr३ = arr$- ३           // मानको पहिलो घटना हटाउनुहोस्
arr४ = arr$-- ३          // सबै घटनाहरू हटाउनुहोस्
arr५ = arr$-[१]          // सूचकाङ्क १ मा हटाउनुहोस् (पहिलो तत्व)
arr६ = arr$-[२..३]       // दायरा हटाउनुहोस् (१-आधारित, अन्त समावेशी)

छ = arr$? ३            // #१ — समावेश गर्दछ
स्थानहरू = arr$?? ३           // [३] — मानको सबै सूचकाङ्कहरू (१-आधारित)
स्लाइस = arr$[१..३]          // [१,२,३] — स्लाइस (१-आधारित, अन्त समावेशी)
स्लाइस२ = arr$[१:३]          // [१,२,३] — समान, गणना-आधारित सिन्ट्याक्स

आरोही = arr$^+             // आरोही क्रममा मिलाउनुहोस् (केवल आदिम)
अवरोही = arr$^-            // अवरोही क्रममा मिलाउनुहोस् (केवल आदिम)

// नामाङ्कित/स्थानीय टपल एरे — तुलनाकारी लाम्ब्डा सहित $^ प्रयोग गर्नुहोस्
डाटाबेस = [(नाम: "कार्ला", उमेर: २८), (नाम: "अना", उमेर: २५), (नाम: "बब", उमेर: ३०)]
उमेर_अनुसार  = डाटाबेस$^ (a, b -> a.उमेर < b.उमेर)    // उमेर अनुसार आरोही (<)
नाम_अनुसार = डाटाबेस$^ (a, b -> a.नाम > b.नाम)   // नाम अनुसार अवरोही (>)
>> उमेर_अनुसार[१].नाम ¶     // → अना
>> नाम_अनुसार[१].नाम ¶    // → कार्ला

// सीधा तत्व अद्यावधिक (केवल एरे)
arr[१] = ९९              // असाइन गर्नुहोस्
arr[२] += ५              // संयुक्त: +=  -=  *=  /=  %=  ^=

// कार्यात्मक अद्यावधिक — नयाँ एरे फर्काउँछ; मूल अपरिवर्तित
arr२ = arr[२]$~ ९९
```

> सबै सङ्ग्रह अपरेटरहरूले **नयाँ एरे** फर्काउँछन्। फिर्ता असाइन गर्नुहोस्: `arr = arr$+ 4`।
> `$+` शृङ्खलाबद्ध गर्न सकिन्छ: `arr = arr$+ 5$+ 6$+ 7`। अन्य अपरेटरहरूले मध्यवर्ती असाइनमेन्टहरू प्रयोग गर्छन्।
> **सूचकाङ्कन 1-आधारित**: `arr[1]` पहिलो तत्व हो; `arr[0]` रनटाइम त्रुटि हो।
> `$^+` / `$^-` **आदिम एरेहरू** (सङ्ख्याहरू, स्ट्रिङहरू) मिलाउँछन्। टपल एरेहरूको लागि तुलनाकारी लाम्ब्डा सहित `$^` प्रयोग गर्नुहोस् — दिशा लाम्ब्डामा इन्कोड गरिएको छ (`<` = आरोही, `>` = अवरोही)।

**मान शब्दार्थ** — एरेलाई अर्को चरमा असाइन गर्दा स्वतन्त्र प्रति सिर्जना हुन्छ:

```zymbol
a = [१, २, ३]
b = a
a[१] = ९९
>> a ¶    // → [९९, २, ३]
>> b ¶    // → [१, २, ३]   ← b अप्रभावित
```

```zymbol
// नेस्टेड एरेहरू (१-आधारित सूचकाङ्कन)
म्याट्रिक्स = [[१,२,३],[४,५,६],[७,८,९]]
>> म्याट्रिक्स[२][३] ¶    // → ६  (पङ्क्ति २, स्तम्भ ३)
```

```zymbol
// एरे
arr = [१०, २०, ३०, ४०, ५०]
[a, b, c] = arr              // a=१०  b=२०  c=३०

(नाम: n, उमेर: u) = व्यक्ति   // n="अना"  u=२५
```

## टपल

टपलहरू **अपरिवर्तनीय** क्रमबद्ध कन्टेनरहरू हुन् जसले **विभिन्न प्रकारका** मानहरू धारण गर्न सक्छन्।
एरेहरूको विपरीत, सिर्जना पछि तत्वहरू परिवर्तन गर्न सकिँदैन।

```zymbol
// स्थानीय — मिश्रित प्रकारहरू अनुमत
बिन्दु = (१०, २०)
>> बिन्दु[१] ¶    // → १०

डाटा = (४२, "हेल्लो", #१, ३.१४)
>> डाटा[३] ¶     // → #१

// नामाङ्कित
व्यक्ति = (नाम: "एलिस", उमेर: २५)
>> व्यक्ति.नाम ¶    // → एलिस
>> व्यक्ति[१] ¶      // → एलिस  (सूचकाङ्क पनि काम गर्दछ, १-आधारित)

// नेस्टेड
स्थान = (x: १०, y: २०)
p = (स्थान: स्थान, लेबल: "मूल")
>> p.स्थान.x ¶        // → १०
```

**अपरिवर्तनीयता** — टपल तत्व परिवर्तन गर्ने कुनै पनि प्रयास रनटाइम त्रुटि हो:

```zymbol
t = (१०, २०, ३०)
// t[१] = ९९    // ❌ रनटाइम त्रुटि: टपलहरू अपरिवर्तनीय छन्
// t[१] += ५    // ❌ समान त्रुटि
```

परिमार्जित मान प्राप्त गर्न `$~` (कार्यात्मक अद्यावधिक) प्रयोग गर्नुहोस् — **नयाँ** टपल फर्काउँछ:

```zymbol
t = (१०, २०, ३०)
t२ = t[२]$~ ९९९
>> t ¶     // → (१०, २०, ३०)   ← मूल अपरिवर्तित
>> t२ ¶    // → (१०, ९९९, ३०)

// नामाङ्कित टपल — स्पष्ट रूपमा पुनर्निर्माण गर्नुहोस्
व्यक्ति = (नाम: "एलिस", उमेर: २५)
ठुलो  = (नाम: व्यक्ति.नाम, उमेर: २६)
>> व्यक्ति.उमेर ¶    // → २५
>> ठुलो.उमेर ¶     // → २६
```

## उच्च-क्रम फङ्क्सनहरू

```zymbol
सङ्ख्याहरू = [१, २, ३, ४, ५, ६, ७, ८, ९, १०]

दोब्बर_गरिएको  = सङ्ख्याहरू$> (x -> x * २)                  // map  → [२,४,६…२०]
जोर    = सङ्ख्याहरू$| (x -> x % २ == ०)           // filter → [२,४,६,८,१०]
जम्मा    = सङ्ख्याहरू$< (०, (सञ्चयक, x) -> सञ्चयक + x)     // reduce → ५५

// मध्यवर्तीहरू मार्फत शृङ्खला
चरण१ = सङ्ख्याहरू$| (x -> x > ३)
चरण२ = चरण१$> (x -> x * x)
>> चरण२ ¶    // → [१६, २५, ३६, ४९, ६४, ८१, १००]

// नामाङ्कित फङ्क्सनहरू सीधै HOF मा पास गर्न सकिन्छ
दोब्बर(x) { <~ x * २ }
ठुलो_छ(x) { <~ x > ५ }
r = सङ्ख्याहरू$> दोब्बर       // ✅ सीधा सन्दर्भ
r = सङ्ख्याहरू$| ठुलो_छ       // ✅ सीधा सन्दर्भ
```

## पाइप अपरेटर

दायाँ पक्षलाई सधैं पाइप गरिएको मानको लागि `_` प्लेसहोल्डर आवश्यक हुन्छ:

```zymbol
दोब्बर = x -> x * २
योग = (a, b) -> a + b
वृद्धि = x -> x + १

प१ = ५ |> दोब्बर(_)        // → १०
प२ = १० |> योग(_, ५)       // → १५
प३ = ५ |> योग(२, _)        // → ७

// शृङ्खलाबद्ध
प = ५ |> दोब्बर(_) |> वृद्धि(_) |> दोब्बर(_)
>> प ¶    // → २२  (५→१०→११→२२)
```

## साङ्ख्यिक मोडहरू

Zymbol ले सङ्ख्याहरू **69 युनिकोड अङ्क लिपिहरू** मा प्रदर्शन गर्न सक्छ — देवनागरी, अरबी-भारतीय, थाई, क्लिङ्गन pIqaD, गणितीय बोल्ड, एलसीडी सेगमेन्टहरू, र थप। सक्रिय मोडले केवल `>>` आउटपुटलाई मात्र असर गर्दछ; आन्तरिक अङ्कगणित सधैं बाइनरी हुन्छ।

### लिपि सक्रिय गर्ने

लक्ष्य लिपिको `0` र `9` अङ्कहरू `#…#` भित्र लेख्नुहोस्:

```zymbol
#०९#    // देवनागरी   (U+0966–U+096F)
#٠٩#    // अरबी-भारतीय (U+0660–U+0669)
#๐๙#    // थाई         (U+0E50–U+0E59)
#09#    // ASCII मा रिसेट गर्नुहोस्
```

### आउटपुट र बुलियनहरू

```zymbol
x = ४२
>> x ¶          // → ४२   (ASCII पूर्वनिर्धारित)

#०९#
>> x ¶          // → ४२
>> ३.१४ ¶       // → ३.१४   (दशमलव बिन्दु सधैं ASCII)
>> १ + २ ¶      // → ३

// बुलियनहरू: # उपसर्ग सधैं ASCII, अङ्क अनुकूल हुन्छ
>> #१ ¶         // → #१   (देवनागरीमा सत्य)
>> #० ¶         // → #०   (असत्य — ० पूर्णाङ्क शून्यबाट भिन्न)

x = २८ > ४
>> x ¶          // → #१   (तुलना परिणाम सक्रिय मोड पछ्याउँदछ)
```

### स्रोतमा मूल अङ्क लिटरलहरू

कुनै पनि समर्थित लिपिका अङ्कहरू मान्य लिटरलहरू हुन् — दायराहरू, मोडुलो, तुलनाहरूमा:

```zymbol
#०९#

@ i:१..१५ {
    ? i % १५ == ० { >> "FizzBuzz" ¶ }
    _? i % ३  == ० { >> "Fizz" ¶ }
    _? i % ५  == ० { >> "Buzz" ¶ }
    _ { >> i ¶ }
}
```

### कुनै पनि लिपिमा बुलियन लिटरलहरू

`#` + कुनै पनि ब्लकबाट अङ्क `0` वा `1` मान्य बुलियन लिटरल हो:

```zymbol
#٠٩#
सक्रिय = #١        // #१ सरह
>> सक्रिय ¶        // → #१
>> (#१ && #०) ¶ // → #०
```

> `#` **सधैं ASCII** हो। `#0` (असत्य) सधैं प्रत्येक लिपिमा `0` (पूर्णाङ्क शून्य) भन्दा दृश्यात्मक रूपमा भिन्न हुन्छ।

## डाटा अपरेटरहरू

```zymbol
// प्रकार रूपान्तरण कास्टहरू
f = ##.४२         // → ४२.०  (फ्लोटिङमा)
i = ###३.७        // → ४     (पूर्णाङ्कमा, गोल)
t = ##!३.७        // → ३     (पूर्णाङ्कमा, ट्रङ्केट)

// स्ट्रिङलाई सङ्ख्यामा पार्स गर्नुहोस्
v१ = #|"४२"|      // → ४२  (पूर्णाङ्क)
v२ = #|"३.१४"|    // → ३.१४  (फ्लोटिङ)
v३ = #|"abc"|     // → "abc"  (सुरक्षित, त्रुटि छैन)

// गोल / ट्रङ्केट
π = ३.१४१५९२६५
गोल२ = #.२|π|      // → ३.१४  (२ दशमलव स्थानहरूमा गोल)
गोल४ = #.४|π|      // → ३.१४१६
ट्रङ्क२ = #!२|π|      // → ३.१४  (ट्रङ्केट)

// सङ्ख्या ढाँचा
ढाँचा = #,|१२३४५६७|  // → १,२३४,५६७  (अल्पविराम-पृथक)
वैज्ञानिक = #^|१२३४५.६७८|    // → १.२३४५६७८e४  (वैज्ञानिक)

// आधार लिटरलहरू
a = 0x41         // → 'A'  (हेक्स)
b = 0b01000001   // → 'A'  (बाइनरी)
c = 0o101        // → 'A'  (अक्टल)

// आधार रूपान्तरण आउटपुट
हेक्स = 0x|२५५|    // → "0x00FF"
बाइनरी = 0b|६५|     // → "0b1000001"
अक्टल = 0o|८|      // → "0o10"
दशमलव = 0d|२५५|    // → "0d0255"
```

## प्रतीक सन्दर्भ

| प्रतीक | सञ्चालन | प्रतीक | सञ्चालन |
|--------|-----------|--------|-----------|
| `=` | चर | `$#` | लम्बाइ |
| `:=` | स्थिराङ्क | `$+` | थप्नुहोस् (शृङ्खलाबद्ध गर्न मिल्ने) |
| `>>` | आउटपुट | `$+[i]` | सूचकाङ्कमा घुसाउनुहोस् (1-आधारित) |
| `<<` | इनपुट | `$-` | मान अनुसार पहिलो हटाउनुहोस् |
| `¶` / `\\` | नयाँ लाइन | `$--` | मान अनुसार सबै हटाउनुहोस् |
| `?` | यदि | `$-[i]` | सूचकाङ्कमा हटाउनुहोस् (1-आधारित) |
| `_?` | अन्यथा-यदि | `$-[i..j]` | दायरा हटाउनुहोस् (1-आधारित) |
| `_` | अन्यथा / वाइल्डकार्ड | `$?` | समावेश गर्दछ |
| `??` | म्याच | `$??` | सबै सूचकाङ्कहरू पत्ता लगाउनुहोस् (1-आधारित) |
| `@` | लुप | `$[s..e]` | स्लाइस (1-आधारित) |
| `@ N { }` | N पटक लुप | `$>` | map |
| `@!` | तोड्नुहोस् | `$\|` | filter |
| `@>` | जारी राख्नुहोस् | `$<` | reduce |
| `@:नाम { }` | लेबलयुक्त लुप | `$/ सीमाङ्क` | स्ट्रिङ विभाजन गर्नुहोस् |
| `@:नाम!` | लेबल तोड्नुहोस् | `$++ a b c` | कन्क्याट बिल्ड |
| `@:नाम>` | लेबल जारी राख्नुहोस् | `arr[i>j>k]` | नेभिगेसन सूचकाङ्क |
| `->` | लाम्ब्डा | `arr[i] = मान` | तत्व अद्यावधिक गर्नुहोस् (केवल एरे) |

| `$^` | तुलनाकारी सहित मिलाउनुहोस् (टपल) | `<~` | फर्काउनुहोस् |
| `\|>` | पाइप | `!?` | प्रयास गर्नुहोस् |
| `:!` | पक्रनुहोस् | `:>` | अन्त्यमा |
| `#1` | सत्य | `#0` | असत्य |
| `$!` | त्रुटि हो | `$!!` | त्रुटि प्रसार गर्नुहोस् |
| `<#` | आयात गर्नुहोस् | `#>` | निर्यात गर्नुहोस् |

| `.` | फिल्ड पहुँच | `#?` | प्रकार मेटाडाटा |
| `#\|..\|` | सङ्ख्या पार्स गर्नुहोस् | `##.` | फ्लोटिङमा रूपान्तरण गर्नुहोस् |
| `###` | पूर्णाङ्कमा रूपान्तरण गर्नुहोस् (गोल) | `##!` | पूर्णाङ्कमा रूपान्तरण गर्नुहोस् (ट्रङ्केट) |

## रिलिज चेन्जलग

### v0.0.5 — टीयूआय प्रिमिटिभहरू, हट परिभाषा र स्ट्रिङ दोहोर्याउने _(मे २०२६)_

- **ब्रेकिङ** म्याच आर्म सेपरेटर: `प्याटर्न : परिणाम` → `प्याटर्न => परिणाम`
- **ब्रेकिङ** आयात उपनाम: `<# पाथ <= उपनाम` → `<# पाथ => उपनाम`
- **ब्रेकिङ** निर्यात पुनर्नामकरण: `#> { fn <= पब }` → `#> { fn => पब }`
- **थपियो** टीयूआय ब्लक `>>| { }` — वैकल्पिक स्क्रिन + कच्चा मोड; बाहिर निस्कँदा सफा गर्दछ
- **थपियो** स्थानीय आउटपुट `>>~ (पङ्क्ति, स्तम्भ, BKS, fg, bg) > आइटमहरू` — विरल स्लटहरू, ANSI 256 रङ
- **थपियो** कुञ्जी इनपुट `<<| चर` (ब्लकिङ) र `<<|? चर` (नन-ब्लकिङ पोल)
- **थपियो** `>>!` स्क्रिन सफा गर्नुहोस्, `>>?` टर्मिनल आकार सोध्नुहोस्, `@~ N` N मिलिसेकेन्ड सुत्नुहोस्
- **थपियो** हट परिभाषा `°x` / `x°` — लुपहरूमा पहिलो प्रयोगमा चर स्वचालित प्रारम्भ
- **थपियो** स्ट्रिङ दोहोर्याउने `स्ट्रिङ $* N` — स्ट्रिङलाई N पटक दोहोर्याउनुहोस्
- **भिएम** समानता: 436/436 परीक्षणहरू पास

### v0.0.4 — 1-आधारित सूचकाङ्कन, प्रथम-श्रेणी फङ्क्सनहरू र ब्लक मोड्युलहरू _(अप्रिल २०२६)_

- **ब्रेकिङ** सबै सूचकाङ्कन **1-आधारित** मा परिवर्तन गरियो — `arr[1]` पहिलो तत्व हो; `arr[0]` रनटाइम त्रुटि हो
- **थपियो** नामाङ्कित फङ्क्सनहरू **प्रथम-श्रेणी मानहरू** हुन् — सीधै HOF मा पास गर्नुहोस्: `सङ्ख्याहरू$> दोब्बर`
- **थपियो** मोड्युलहरूको लागि **ब्लक सिन्ट्याक्स आवश्यक** — `# नाम { ... }` — फ्ल्याट सिन्ट्याक्स हटाइयो
- **थपियो** बहु-आयामी सूचकाङ्कन: `arr[i>j>k]` (नेभिगेसन), `arr[p ; q]` (फ्ल्याट निकासी)
- **थपियो** प्रकार रूपान्तरण कास्टहरू: `##.एक्सप्रेसन` (फ्लोटिङ), `###एक्सप्रेसन` (पूर्णाङ्क गोल), `##!एक्सप्रेसन` (पूर्णाङ्क ट्रङ्केट)
- **थपियो** स्ट्रिङ विभाजन: `स्ट्रिङ$/ सीमाङ्क` — `Array(स्ट्रिङ)` फर्काउँछ
- **थपियो** कन्क्याट बिल्ड: `आधार$++ a b c` — बहुविध आइटमहरू थप्छ
- **थपियो** टाइम्स लुप: `@ N { }` — ठीक N पटक दोहोर्याउनुहोस्
- **थपियो** लेबलयुक्त लुप सिन्ट्याक्स: `@:नाम { }`, `@:नाम!`, `@:नाम>` — `@ @नाम` / `@! नाम` को सट्टा
- **थपियो** चर स्कोप नियमहरू: `_नाम` चरहरूको सटीक ब्लक स्कोप हुन्छ; `\ चर` चाँडै नष्ट गर्दछ
- **थपियो** म्याच तुलना ढाँचाहरू: `< 0 =>`, `> 5 =>`, `== 42 =>` आदि
- **थपियो** मोड्युल E013 त्रुटि: मोड्युल बडीमा कार्यान्वित योग्य विवरणहरू निषेधित छन्
- **सुधारियो** `alias.CONST` अब सही रूपमा समाधान गर्दछ; `#>` फङ्क्सन परिभाषाहरू पछि आउन सक्छ
- **भिएम** पूर्ण समानता: 393/393 परीक्षणहरू पास
